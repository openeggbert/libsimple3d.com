Background: triggers vs. solid colliders

In previous physics parts you added AddBoxCollider, AddCapsuleCollider, and AddSphereCollider to create surfaces that rigid bodies physically bounce off or rest on. Trigger volumes are the opposite: they have no physical response whatsoever. An entity walking through a trigger is not blocked, pushed, or otherwise affected by the collision. Instead the engine fires a callback so you can react in code.

Solid collider

Generates a collision manifold. Bodies separate, friction is applied, impulses are exchanged. Use for floors, walls, crates — anything the player should not pass through.

Trigger volume

No manifold, no separation force. The overlapping body passes straight through. Instead, SetOnTriggerEnter fires on the first overlapping frame and SetOnTriggerExit fires when the overlap ends. Use for checkpoints, collectibles, area effects, death zones, and door openers.

Triggers are placed on CollisionLayer::Trigger (bit 4) automatically when you call any of the AddTrigger* helpers. You only need to touch the layer/mask settings explicitly when you want to filter which bodies can activate the volume.

Complete working example

The example below builds a level containing a moving player, a checkpoint gate (box trigger), a death-zone pit (sphere trigger), and a field of ten collectible coins (sphere triggers, destroyed on pickup). Every new API from this part is used at least once. 

#include <Simple3D/Simple3D.h>

using namespace Simple3D;

// ── Forward declarations ─────────────────────────────────────────────
static constexpr int COIN_COUNT = 10;

class TriggerDemo : public Game
{
public:
    // ── Scene objects ─────────────────────────────────────────────────
    Entity*  player_        = nullptr;
    Camera*  cam_           = nullptr;

    // ── UI ────────────────────────────────────────────────────────────
    Label*   scoreLabel_    = nullptr;
    Label*   statusLabel_   = nullptr;

    // ── State ─────────────────────────────────────────────────────────
    int      coinsCollected_ = 0;

    // Entities queued for deferred destruction (cannot destroy inside callback)
    std::vector<Entity*> pendingDestroy_;

    // ─────────────────────────────────────────────────────────────────
    void Start() override
    {
        SetWindowTitle("Trigger Demo");
        SetClearColor(Color(0.18f, 0.18f, 0.22f, 1.0f));

        // ── Lighting ──────────────────────────────────────────────────
        Entity* sun = CreateEntity("Sun");
        sun->AddDirectionalLight(Color(1.0f, 0.95f, 0.8f), 1.2f);
        sun->SetRotation(Quaternion(50.0f, 30.0f, 0.0f));

        // ── Ground ────────────────────────────────────────────────────
        Entity* ground = CreateEntity("Ground");
        ground->AddModel("Models/Plane.mdl");
        ground->SetScale(30.0f);
        ground->SetMaterialColor(Color(0.4f, 0.55f, 0.4f));
        ground->AddRigidBody(0.0f);
        ground->AddBoxCollider(Vector3(30.0f, 0.1f, 30.0f));
        ground->SetCollisionLayer(CollisionLayer::StaticGeometry);

        // ── Player ────────────────────────────────────────────────────
        player_ = CreateEntity("Player");
        player_->AddModel("Models/Box.mdl");
        player_->SetMaterialColor(Color(0.2f, 0.5f, 1.0f));
        player_->SetPosition(0.0f, 1.0f, 0.0f);
        player_->AddTag("player");   // used by trigger callbacks to identify the player

        CharacterController* cc = player_->AddCharacterController(0.35f, 1.8f);
        cc->SetGravity(22.0f);

        // Player lives on the Actor layer and collides with StaticGeometry + Trigger.
        player_->SetCollisionLayer(CollisionLayer::Actor);
        player_->SetCollisionMask(MakeCollisionMask({
            CollisionLayer::StaticGeometry,
            CollisionLayer::Trigger
        }));

        // ── Camera ────────────────────────────────────────────────────
        cam_ = CreateCamera();
        cam_->Follow(player_, 9.0f, 4.0f);
        cam_->SetSmoothFollow(true, 5.0f);

        // ── HUD ───────────────────────────────────────────────────────
        scoreLabel_ = CreateLabel("Coins: 0 / 10");
        scoreLabel_->SetPosition(10, 10);
        scoreLabel_->SetFontSize(20);
        scoreLabel_->SetColor(Color(1.0f, 0.9f, 0.2f));

        statusLabel_ = CreateLabel("Walk into the coins and the gate!");
        statusLabel_->SetPosition(10, 38);
        statusLabel_->SetFontSize(15);
        statusLabel_->SetColor(Color::WHITE);

        // ── Triggers ──────────────────────────────────────────────────
        SpawnCoins();
        SpawnCheckpointGate();
        SpawnDeathZone();
    }

    // ─────────────────────────────────────────────────────────────────
    // SpawnCoins — creates 10 collectible coin entities arranged in a ring.
    // Each coin uses a sphere trigger volume and destroys itself on contact
    // with the player.  Destruction is deferred to avoid modifying the
    // physics world while inside a callback.
    // ─────────────────────────────────────────────────────────────────
    void SpawnCoins()
    {
        for (int i = 0; i < COIN_COUNT; ++i)
        {
            float angle = (float)i * (360.0f / COIN_COUNT) * 3.14159f / 180.0f;
            float rx    = Cos(angle) * 7.0f;
            float rz    = Sin(angle) * 7.0f;

            Entity* coin = CreateEntity("Coin_" + String(i));
            coin->AddModel("Models/Sphere.mdl");
            coin->SetScale(0.4f);
            coin->SetMaterialColor(Color(1.0f, 0.85f, 0.1f));
            coin->SetPosition(rx, 0.5f, rz);
            coin->AddTag("coin");

            // Attach a sphere trigger volume (radius 0.6).
            // AddTriggerSphere automatically sets CollisionLayer::Trigger on this entity.
            coin->AddTriggerSphere(0.6f);

            // Only react to bodies on the Actor layer — ignore static geometry,
            // other triggers, etc.  MakeCollisionMask converts the enum list to
            // the correct bitmask integer.
            coin->SetCollisionMask(MakeCollisionMask({ CollisionLayer::Actor }));

            // Capture 'coin' by value so the lambda stays valid even after
            // other coins have been collected.  Use 'this' to reach game state.
            coin->SetOnTriggerEnter([this, coin](Entity* other)
            {
                // Guard: only the player should collect coins.
                if (!other->HasTag("player")) return;

                // Queue the coin for destruction next frame.
                // Calling DestroyEntity directly here would modify the physics
                // broadphase while the engine is still iterating contacts —
                // undefined behaviour.  The pendingDestroy_ list is processed
                // safely at the top of Update() instead.
                pendingDestroy_.push_back(coin);

                ++coinsCollected_;
                scoreLabel_->SetText("Coins: " + String(coinsCollected_) + " / 10");
                PlaySound("Sounds/coin.wav", 0.9f);

                if (coinsCollected_ == COIN_COUNT)
                    statusLabel_->SetText("All coins collected! Head for the gate.");
            });
        }
    }

    // ─────────────────────────────────────────────────────────────────
    // SpawnCheckpointGate — a tall box trigger that only reacts to the player.
    // On enter: display a message and play a sound.
    // On exit:  update the HUD to show the player has left the zone.
    // ─────────────────────────────────────────────────────────────────
    void SpawnCheckpointGate()
    {
        Entity* gate = CreateEntity("CheckpointGate");
        gate->AddModel("Models/Box.mdl");
        gate->SetScale(Vector3(3.0f, 4.0f, 0.3f));
        gate->SetMaterialColor(Color(0.1f, 0.8f, 0.4f, 0.35f));
        gate->SetPosition(0.0f, 2.0f, -12.0f);

        // AddTriggerBox takes the FULL extents (not half-extents), matching the
        // convention used by AddBoxCollider.
        gate->AddTriggerBox(Vector3(3.0f, 4.0f, 0.6f));
        gate->SetCollisionMask(MakeCollisionMask({ CollisionLayer::Actor }));

        gate->SetOnTriggerEnter([this](Entity* other)
        {
            if (!other->HasTag("player")) return;
            statusLabel_->SetText("Checkpoint reached!");
            PlaySound("Sounds/checkpoint.wav", 1.0f);
        });

        gate->SetOnTriggerExit([this](Entity* other)
        {
            if (!other->HasTag("player")) return;
            statusLabel_->SetText("Left the checkpoint zone.");
        });
    }

    // ─────────────────────────────────────────────────────────────────
    // SpawnDeathZone — a large sphere trigger in a pit.  Walking in kills
    // the player and resets their position.  Demonstrates that triggers
    // can be used without visible geometry.
    // ─────────────────────────────────────────────────────────────────
    void SpawnDeathZone()
    {
        Entity* pit = CreateEntity("DeathZone");
        // No model — the trigger volume is invisible.  This is intentional;
        // in a real game you would show a hazard texture or a particle effect.
        pit->SetPosition(12.0f, 0.5f, 0.0f);

        pit->AddTriggerSphere(3.5f);
        pit->SetCollisionMask(MakeCollisionMask({ CollisionLayer::Actor }));

        pit->SetOnTriggerEnter([this](Entity* other)
        {
            if (!other->HasTag("player")) return;

            // Teleport the player back to the start without destroying anything.
            other->SetPosition(0.0f, 1.0f, 0.0f);
            other->SetLinearVelocity(Vector3::ZERO);
            statusLabel_->SetText("You fell into the pit! Respawned.");
            PlaySound("Sounds/hurt.wav", 0.8f);
        });
    }

    // ─────────────────────────────────────────────────────────────────
    void Update(float dt) override
    {
        // ── Flush deferred destruction ───────────────────────────────
        // Destroy coin entities that were collected during the previous physics
        // step.  This must happen at the start of Update(), before any new
        // physics contacts are processed, so the broadphase is in a clean state.
        for (Entity* e : pendingDestroy_)
            DestroyEntity(e);
        pendingDestroy_.clear();

        // ── Player movement ──────────────────────────────────────────
        CharacterController* cc = player_->GetComponent<CharacterController>();
        if (!cc) return;

        float mx = 0.0f, mz = 0.0f;
        if (IsKeyDown(Key::Left)  || IsKeyDown(Key::A)) mx -= 1.0f;
        if (IsKeyDown(Key::Right) || IsKeyDown(Key::D)) mx += 1.0f;
        if (IsKeyDown(Key::Up)    || IsKeyDown(Key::W)) mz -= 1.0f;
        if (IsKeyDown(Key::Down)  || IsKeyDown(Key::S)) mz += 1.0f;

        cc->Move(Vector3(mx, 0.0f, mz) * 5.0f, dt);

        if (IsKeyPressed(Key::Space) && cc->IsOnGround())
            cc->Jump(10.0f);

        if (IsKeyPressed(Key::Escape))
            Quit();
    }
};

int main()
{
    return Simple3D::CreateGame<TriggerDemo>()->Run();
}

Step-by-step explanation

AddTriggerBox, AddTriggerSphere, AddTriggerCapsule

These three helpers attach a trigger-only collision shape to an entity. Unlike their AddBoxCollider / AddSphereCollider counterparts, they generate no physics response when another body overlaps them. Internally each call:

  1. Creates a Urho3D CollisionShape with the ghost / trigger flag set.
  2. Automatically places the entity on CollisionLayer::Trigger (bit 4) so the physics broadphase can find it efficiently.

The size argument matches the convention used by the corresponding solid collider: AddTriggerBox takes full extents (not half-extents), AddTriggerSphere takes a radius, and AddTriggerCapsule takes radius then height. 

// Box trigger — full extents 4 m wide, 2 m tall, 4 m deep
zone->AddTriggerBox(Vector3(4.0f, 2.0f, 4.0f));

// Sphere trigger — radius 1.5 m
pickup->AddTriggerSphere(1.5f);

// Capsule trigger — radius 0.5 m, height 2.0 m
// Useful for character-shaped detection areas (e.g. a proximity sensor on an enemy).
sensor->AddTriggerCapsule(0.5f, 2.0f);

SetOnTriggerEnter(callback)

Registers a function to be called the first frame that a qualifying physics body overlaps the trigger volume. The callback receives a single Entity* other pointer — the entity whose collider entered the volume. Setting a new callback with a second call to SetOnTriggerEnter replaces the previous one; passing nullptr clears it.

The callback is fired from inside the physics update, so avoid heavyweight work such as loading assets, calling DestroyEntity, or any function that modifies the scene graph. See the deferred destruction pattern below for how to safely destroy entities collected during callbacks. 

// Minimal enter callback
door->SetOnTriggerEnter([this](Entity* other)
{
    if (other->HasTag("player"))
        statusLabel_->SetText("Door opening...");
});

// Clear an existing callback
door->SetOnTriggerEnter(nullptr);

SetOnTriggerExit(callback)

Mirrors SetOnTriggerEnter but fires when the overlapping body stops touching the trigger volume. This happens either because the body physically moved out of the volume, or because the body was destroyed (in that case, the exit callback may or may not fire depending on timing — do not rely on it to clean up destroyed entities). As with enter callbacks, the same Entity* other pointer is passed. 

// Show different messages on enter and exit
aura->SetOnTriggerEnter([this](Entity* other)
{
    if (other->HasTag("player")) buffActive_ = true;
});

aura->SetOnTriggerExit([this](Entity* other)
{
    if (other->HasTag("player")) buffActive_ = false;
});

CollisionLayer::Trigger

The CollisionLayer enum (from <Simple3D/Core/Collision.h>, also included by Simple3D.h) defines named bit positions for the Bullet physics broadphase filter: 

enum class CollisionLayer : unsigned {
    None           = 0,
    Actor          = 1,     // player / NPCs / dynamic bodies
    StaticGeometry = 2,     // ground, walls, solid platforms
    Trigger        = 4,     // auto-set by AddTrigger*() helpers
    All            = 0xFFFF
};

You rarely need to set CollisionLayer::Trigger manually because AddTriggerBox, AddTriggerSphere, and AddTriggerCapsule all set it automatically. You do need to reference it when configuring other entities' collision masks so they can interact with trigger volumes: 

// Player must include Trigger in its mask to activate trigger volumes
player_->SetCollisionMask(MakeCollisionMask({
    CollisionLayer::StaticGeometry,
    CollisionLayer::Trigger          // without this, the player ignores all triggers
}));

MakeCollisionMask({...})

MakeCollisionMask accepts an initializer list of CollisionLayer values and returns a single unsigned int bitmask with those bits set. Using it instead of raw bit arithmetic keeps code readable and type-safe. 

// Static geometry only
wall->SetCollisionMask(MakeCollisionMask({ CollisionLayer::Actor }));

// Actors and static geometry but NOT other triggers
bullet->SetCollisionMask(MakeCollisionMask({
    CollisionLayer::Actor,
    CollisionLayer::StaticGeometry
}));

// Everything (the default)
box->SetCollisionMask(MakeCollisionMask({ CollisionLayer::All }));

The mask is checked bidirectionally: a trigger fires its callback only if the entering body's layer is set in the trigger's mask and the trigger's layer (Trigger) is set in the entering body's mask. That is why you must add CollisionLayer::Trigger to the player's mask and set the trigger's own mask to include CollisionLayer::Actor.

other->HasTag("player") inside a callback

When multiple entity types pass through the same trigger, you need to distinguish them. HasTag is the canonical way to do this. Assign tags to entities at creation time (in Start() or when spawning) and check them in the callback. Multiple tags on one entity are fully supported. 

// Assign tags during spawn
player_->AddTag("player");
enemy->AddTag("enemy");
enemy->AddTag("ai");

// In a trigger that reacts to different entity types differently
warpGate->SetOnTriggerEnter([this](Entity* other)
{
    if (other->HasTag("player"))
    {
        // Transport the player to the next level
        LoadNextLevel();
    }
    else if (other->HasTag("enemy"))
    {
        // Enemies are repelled by the gate instead
        other->ApplyImpulse(Vector3(0.0f, 5.0f, 5.0f));
    }
});

Deferred DestroyEntity inside callbacks

The trigger callbacks are fired from within Bullet's contact processing loop. At that exact moment the engine is iterating over collision pairs, so any call that removes a physics body — such as DestroyEntity — will corrupt the broadphase data structure and cause a crash or silent heap corruption.

The safe pattern is a pending destroy list: push the entity pointer into a std::vector inside the callback, then drain the vector at the beginning of Update(), before any physics contacts are processed for the new frame. 

// Member variable
std::vector<Entity*> pendingDestroy_;

// Inside a trigger callback — NEVER call DestroyEntity directly here
item->SetOnTriggerEnter([this, item](Entity* other)
{
    if (!other->HasTag("player")) return;
    pendingDestroy_.push_back(item);  // safe: no scene-graph modification yet
    ++score_;
});

// In Update() — flush the list before any other logic
void Update(float dt) override
{
    for (Entity* e : pendingDestroy_)
        DestroyEntity(e);
    pendingDestroy_.clear();

    // ... rest of Update
}

The same rule applies to SetActive(false) when you want to hide rather than destroy an entity after collection. If your game respawns collectibles (e.g. health packs that reappear after 30 seconds), deactivate and re-activate from Update() rather than from inside the callback.

Collectible coin pattern

The combination of trigger sphere + tag check + deferred destruction is so common for pickups that it is worth spelling out as a reusable recipe. Here is the minimal version, stripped of the demo's ring layout, suitable for copy-pasting into any project: 

// ── Spawn a single collectible at a given world position ─────────────
void SpawnPickup(Vector3 pos, const std::string& name, int value)
{
    Entity* pickup = CreateEntity(name);
    pickup->AddModel("Models/Sphere.mdl");
    pickup->SetScale(0.35f);
    pickup->SetMaterialColor(Color(1.0f, 0.85f, 0.1f));
    pickup->SetPosition(pos);
    pickup->AddTag("pickup");

    // 1. Attach the trigger volume (sets CollisionLayer::Trigger automatically).
    pickup->AddTriggerSphere(0.55f);

    // 2. Only fire when the Actor layer touches the trigger.
    pickup->SetCollisionMask(MakeCollisionMask({ CollisionLayer::Actor }));

    // 3. Capture pickup by value so the lambda is self-contained.
    pickup->SetOnTriggerEnter([this, pickup, value](Entity* other)
    {
        // 4. Tag check — only the player collects pickups.
        if (!other->HasTag("player")) return;

        // 5. Defer destruction — never call DestroyEntity inside a callback.
        pendingDestroy_.push_back(pickup);

        // 6. Update game state — safe to do here.
        score_ += value;
        scoreLabel_->SetText("Score: " + String(score_));
        PlaySound("Sounds/coin.wav", 0.9f);
    });
}

// Ensure the player mask includes Trigger so these volumes activate
void SetupPlayer(Entity* p)
{
    p->AddTag("player");
    p->AddCharacterController(0.35f, 1.8f);
    p->SetCollisionLayer(CollisionLayer::Actor);
    p->SetCollisionMask(MakeCollisionMask({
        CollisionLayer::StaticGeometry,
        CollisionLayer::Trigger
    }));
}

Lua equivalent

Trigger volumes and their callbacks are fully accessible from Lua. The callbacks receive an entity userdata as their first (and only) argument, equivalent to the Entity* other pointer in C++. 

-- Scripts/coin_pickup.lua
-- Attach this script to a coin entity to make it collectible.
-- The game object must have a pendingDestroy table on its game global
-- (or use a simpler SetActive approach shown below).

-- References set from the outside (or in Start if you have them):
local VALUE = 10

function Start()
    -- self is the entity this script is attached to.
    self:AddTriggerSphere(0.55)
    self:SetCollisionMask(MakeCollisionMask({ CollisionLayer.Actor }))

    self:SetOnTriggerEnter(function(other)
        if not other:HasTag("player") then return end

        -- Deactivate immediately (safe inside callback)
        -- This hides the coin; DestroyEntity is called from Update or after
        -- the level ends.
        self:SetActive(false)

        -- Update shared score (stored on a global table for simplicity)
        GameState.score = GameState.score + VALUE
        game:PlaySound("Sounds/coin.wav", 0.9)
    end)
end
-- Scripts/checkpoint.lua
-- Attach to any box-trigger entity acting as a checkpoint.

function Start()
    self:AddTriggerBox(Vector3(3.0, 4.0, 0.6))
    self:SetCollisionMask(MakeCollisionMask({ CollisionLayer.Actor }))

    self:SetOnTriggerEnter(function(other)
        if other:HasTag("player") then
            GameState.lastCheckpoint = other:GetPosition()
            game:PlaySound("Sounds/checkpoint.wav", 1.0)
        end
    end)

    self:SetOnTriggerExit(function(other)
        if other:HasTag("player") then
            -- Could log or react to the player leaving the safe zone
        end
    end)
end

In Lua, SetActive(false) is safe to call inside trigger callbacks because it does not destroy the physics body immediately — it only disables the scene node. Call DestroyEntity from Lua's Update function (or leave cleanup to level unloading) exactly as you would in C++.

Patterns and tips

Invisible vs. visible trigger volumes

Trigger volumes do not require a visible model. For game-play elements where the boundary should be hidden from the player (death zones, ambient audio triggers, NPC awareness radii), simply skip AddModel. For visible zones such as checkpoints or interactable doors, add a semi-transparent model to the same entity and set its alpha below 1: 

// Visible checkpoint arch — translucent box
arch->AddModel("Models/Box.mdl");
arch->SetMaterialColor(Color(0.1f, 0.9f, 0.4f, 0.3f));  // alpha = 0.3
arch->AddTriggerBox(Vector3(4.0f, 5.0f, 0.8f));

Proximity sensor for enemies

A trigger capsule on an enemy is an efficient way to implement a proximity sensor without ray-casting every frame. Put it on a child entity so it can be resized independently of the enemy mesh: 

Entity* sensor = enemy_->CreateChild("ProxSensor");
sensor->SetLocalPosition(0.0f, 0.0f, 0.0f);
sensor->AddTriggerCapsule(6.0f, 4.0f);  // large detection radius
sensor->SetCollisionMask(MakeCollisionMask({ CollisionLayer::Actor }));

sensor->SetOnTriggerEnter([this](Entity* other)
{
    if (other->HasTag("player"))
        fsm_->SetState("chase");   // switch AI state (see Part 25)
});

sensor->SetOnTriggerExit([this](Entity* other)
{
    if (other->HasTag("player"))
        fsm_->SetState("patrol");
});

One-time triggers

To fire a trigger only once (e.g. a story cutscene), clear the callback inside the callback itself by setting it to nullptr after queuing any work: 

cutscene->SetOnTriggerEnter([this, cutscene](Entity* other)
{
    if (!other->HasTag("player")) return;

    // Remove callback immediately so it fires exactly once.
    cutscene->SetOnTriggerEnter(nullptr);

    // Queue scene transition for next frame
    pendingScene_ = "cutscene_intro.xml";
});

Common mistakes

  • Calling DestroyEntity directly inside a trigger callback. This is the most dangerous mistake with triggers. Bullet is iterating its contact list when the callback fires; removing a body mid-iteration corrupts the broadphase and typically causes a crash on the next physics step. Always use the deferred destroy pattern: push to a list, drain in Update().
  • Forgetting to include CollisionLayer::Trigger in the player's mask. The player's SetCollisionMask must list CollisionLayer::Trigger or the player will never activate any trigger volume, no matter how many AddTrigger* calls you make. Both sides of the pair must opt in.
  • Setting the trigger's mask to CollisionLayer::All unintentionally. If a trigger responds to all layers, it will fire when static geometry (walls, floors) overlaps it during level load. Always narrow the mask to CollisionLayer::Actor for player/enemy-only triggers.
  • Capturing an Entity* by reference in a lambda when the entity may be destroyed. Lambdas that capture coin by reference (using [&]) will dereference a dangling pointer after the coin is destroyed. Capture by value ([this, coin]) so the pointer snapshot taken at registration time is preserved safely in the closure.
  • Mixing AddBoxCollider and AddTriggerBox on the same entity. An entity can have multiple collision shapes, but mixing a solid collider with a trigger volume on the same body produces undefined behaviour in most physics engines. If you need both a solid surface and a detection area, create a child entity for the trigger and leave the parent with only the solid collider.
  • Relying on SetOnTriggerExit firing after DestroyEntity. When you destroy an entity that is currently overlapping a trigger, the exit callback may not fire. Do not use the exit callback to clean up state that depends on the removed entity's continued existence.
  • Adding a trigger volume but forgetting AddRigidBody on the entering entity. The broadphase only tracks bodies that have a RigidBody component. CharacterController adds its own internal body, so players work. Pure visual entities with no rigid body will never activate a trigger regardless of mask settings.

What you learned

In this part you mastered the trigger system in simple-3d:

  • Trigger shapes: AddTriggerBox, AddTriggerSphere, and AddTriggerCapsule attach non-solid detection volumes to any entity and automatically assign CollisionLayer::Trigger.
  • Callbacks: SetOnTriggerEnter fires on the first overlapping frame; SetOnTriggerExit fires when the body leaves. Both receive an Entity* other pointer.
  • Filtering: CollisionLayer::Trigger and MakeCollisionMask control exactly which physics bodies can activate a volume. The check is bidirectional — both the entering body's mask and the trigger's mask must include the other's layer.
  • Identity: other->HasTag("player") inside a callback is the clean way to distinguish the player from other physics bodies.
  • Deferred destruction: Never call DestroyEntity inside a trigger callback. Queue pointers in a pendingDestroy_ vector and drain it at the top of Update().
  • Collectible coin pattern: Trigger sphere + tag check + deferred destroy + score update is the standard recipe for any pickup in simple-3d.
