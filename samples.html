Samples

simple-3d ships six samples that demonstrate the framework from different angles: a minimal "Hello World", a 3D physics reference, a 2D game, a full vertical-slice platformer, an automated Lua API smoke test, and the complete unit-test suite. All six targets build from the same CMake configure step.

Every sample is located under samples/ in the repository and builds into build-u3d/. See Build All Samples at the bottom of this page for the one-command configure-and-build.

HelloWorld

The minimal entry point into simple-3d. Creates a spinning box in an empty scene with a HUD timer in the corner. Press ESC to quit. The entire game fits in roughly 30 lines of C++.

Features demonstrated

  • CreateEntity — scene object creation
  • AddModel — attaching a mesh to an entity
  • CreateCamera — setting up the scene camera
  • CreateLabel — on-screen HUD text
  • SetClearColor — background colour

Build

cmake --build build-u3d --target HelloWorld

Run

./build-u3d/bin/HelloWorld

Platformer3D

A low-level 3D physics reference. Demonstrates a player capsule on a ground plane with full jump mechanics and a third-person follow camera. Intended as a clean starting point for any 3D game that needs character physics.

Features demonstrated

  • AddRigidBody — dynamic and static rigid bodies
  • AddCapsuleCollider — capsule collision shape for the player
  • CharacterController — slope-filtered grounding, gravity, jump impulse
  • Third-person follow camera with smooth lerp and collision avoidance

Build

cmake --build build-u3d --target Platformer3D

Run

./build-u3d/bin/Platformer3D

Game2D

A 2D orthographic game scene. Loads a Tiled tile map, places a static sprite, and enables Box2D physics. Demonstrates that 3D and 2D can coexist in the same scene using the same unified API.

Features demonstrated

  • SetOrthographic — switching the camera to 2D orthographic projection
  • AddSprite2D — attaching a static 2D sprite to an entity
  • LoadTileMap — loading a Tiled .tmx tile map into the scene
  • AddRigidBody2D — Box2D physics bodies alongside 3D Bullet physics

Build

cmake --build build-u3d --target Game2D

Run

./build-u3d/bin/Game2D

BlupiProto

The acceptance gate for the entire framework — a vertical slice of a 3D platformer. Includes a playable character with animations, collectible coins, AI enemies, a pause menu, and save/load support. BlupiProto must build cleanly before any new subsystem is merged.

Features demonstrated

  • CharacterController — player movement, jump, ground detection
  • AnimStateMachine — condition-driven animation state transitions
  • BehaviorFSM — AI enemy patrol and chase states
  • CollisionLayer::Trigger — trigger volumes for coin pickup and hazard zones
  • SaveData — multi-slot JSON save/load (coins collected, level progress)
  • UI Panel and Button — pause menu with resume and restart actions
  • FadeOut / FadeIn — level transition screen fades

Build

cmake --build build-u3d --target BlupiProto

Run

./build-u3d/bin/BlupiProto

LuaSmoke

An automated Lua API smoke test that runs headlessly and prints PASS or FAIL for each assertion. Covers entity hierarchy, local transforms, gamepad no-ops, and the full breadth of the Lua bindings surface. No manual interaction required — check the console output.

Features demonstrated

  • Full Lua API coverage — CharacterController, AnimStateMachine, BehaviorFSM, AudioSource, ParticleEmitter, InputActions, all UI widgets
  • Entity child hierarchy — CreateChild, GetParent, GetChildren
  • Local transforms — SetLocalPosition, SetLocalRotation, TranslateLocal
  • Per-assertion PASS / FAIL output to the console

Build

cmake --build build-u3d --target LuaSmoke

Run

./build-u3d/bin/LuaSmoke

Simple3DTests

The complete unit-test suite. 78 tests, all 78 must pass. Tests run without a graphics window and complete in seconds. Run this after every change to the framework core to catch regressions.

Features demonstrated

  • CollisionLayer enum — 11 tests covering layer masks and flag combinations
  • AnimStateMachine — 13 tests: bool/float/trigger conditions, auto transitions
  • SaveData — 14 tests: multi-slot JSON, int/string/bool/float round-trips
  • InputActions — 17 tests: bind/axis/dead-zone clamping, persistence round-trip
  • BehaviorFSM — 14 tests: state transitions, patrol/chase built-ins
  • Entity tagging — 9 tests: AddTag, HasTag, FindEntitiesByTag

Build

cmake -S . -B build-u3d -DSIMPLE3D_BUILD_TESTS=ON
cmake --build build-u3d --target Simple3DTests

Run

./build-u3d/Simple3DTests

Build All Samples

A single configure-and-build step compiles every sample and the unit-test suite together. Run these commands from the root of the simple-3d repository. 

cmake -S . -B build-u3d -DSIMPLE3D_BUILD_SAMPLES=ON -DSIMPLE3D_BUILD_TESTS=ON
cmake --build build-u3d

After the build completes, binaries are available at:
