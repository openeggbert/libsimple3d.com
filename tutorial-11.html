  • How to safely retrieve an existing AudioSource from an entity without creating a duplicate using GetAudioSource()

Overview

Part 10 introduced flat, non-positional audio: PlaySound() plays a fire-and-forget effect at constant volume everywhere in the scene, and PlayMusic() streams a background track. These calls are fine for UI sounds and background music, but they cannot represent the sound of a fire pit that grows louder as you walk toward it or an enemy whose growl fades as you sneak away.

That is what spatial audio solves. In simple-3d, every Entity can carry an AudioSource component. The source's position tracks its parent entity in world space. The engine compares that position against the listener — a designated entity whose position represents the player's ears — and attenuates the volume according to the distance between the two. Stereo panning is applied automatically as well: sounds to the left of the listener appear in the left speaker and vice versa.

This tutorial builds a small interactive scene with four audio sources, each demonstrating a different aspect of the system: a continuously looping enemy hum, a crackling fire with a shorter audible range, a bell that can be silenced and re-struck at different pitches, and a constant ambient wind that uses the non-positional PlayAmbient() path. Walk the player toward and away from each source to hear the attenuation change in real time.

Complete source

The listing below is the complete, self-contained program. Place all required OGG/WAV files under your project's Data/Sounds/ directory. The step-by-step explanation that follows covers every new API call in detail. 

// main.cpp — Part 11: Spatial Audio
#include <Simple3D/Simple3D.h>

using namespace Simple3D;

class SpatialAudioDemo : public Game {
public:

    void Start() override {
        SetWindowTitle("Part 11 — Spatial Audio");
        SetClearColor(Color(0.05f, 0.05f, 0.10f, 1.0f));

        // ── Directional light ─────────────────────────────────────────
        Entity* sun = CreateEntity("Sun");
        sun->AddDirectionalLight(Color(1.0f, 0.92f, 0.78f), 1.1f);
        sun->SetRotation(Quaternion(50.0f, 35.0f, 0.0f));

        // ── Ground ────────────────────────────────────────────────────
        Entity* ground = CreateEntity("Ground");
        ground->AddModel("Models/Plane.mdl");
        ground->SetScale(30.0f);
        ground->SetPosition(0.0f, -1.0f, 0.0f);
        ground->AddBoxCollider(Vector3(60.0f, 0.4f, 60.0f));
        ground->SetCollisionLayer(CollisionLayer::StaticGeometry);

        // ── Player ────────────────────────────────────────────────────
        player_ = CreateEntity("Player");
        player_->AddModel("Models/Box.mdl");
        player_->SetScale(0.8f);
        player_->SetPosition(0.0f, 0.0f, 0.0f);
        cc_ = player_->AddCharacterController(0.35f, 1.8f);
        cc_->SetGravity(20.0f);
        player_->SetCollisionLayer(CollisionLayer::Actor);
        player_->SetCollisionMask(MakeCollisionMask({
            CollisionLayer::StaticGeometry
        }));

        // ════════════════════════════════════════════════════════════
        // STEP 1 — SetAudioListener
        // Designate the player as the 3D audio listener.  All AudioSources
        // will attenuate in volume relative to this entity's world position.
        // Call this once in Start() before any AudioSource begins playing.
        // ════════════════════════════════════════════════════════════
        SetAudioListener(player_);

        // ── Enemy — continuous positional hum ─────────────────────────
        enemy_ = CreateEntity("Enemy");
        enemy_->AddModel("Models/Box.mdl");
        enemy_->SetScale(1.3f);
        enemy_->SetMaterialColor(Color(0.75f, 0.12f, 0.12f));
        enemy_->SetPosition(10.0f, 0.0f, 0.0f);

        // ════════════════════════════════════════════════════════════
        // STEP 2 — AddAudioSource + SetRange + SetGain + SetLooped + Play
        // Attach a positional audio source to the enemy entity.
        // The source position tracks the entity automatically.
        // ════════════════════════════════════════════════════════════
        AudioSource* enemySrc = enemy_->AddAudioSource();
        enemySrc->SetRange(2.0f, 20.0f);  // full vol ≤ 2 m, silent ≥ 20 m
        enemySrc->SetGain(0.75f);           // 75 % volume ceiling
        enemySrc->SetLooped(true);          // repeat indefinitely
        enemySrc->Play("Sounds/enemy_hum.ogg");

        // ── Fire pit — crackling sound with a shorter range ────────────
        fire_ = CreateEntity("Fire");
        fire_->SetPosition(-5.0f, 0.0f, 6.0f);

        AudioSource* fireSrc = fire_->AddAudioSource();
        fireSrc->SetRange(1.5f, 12.0f);   // smaller audible radius than enemy
        fireSrc->SetGain(0.85f);
        fireSrc->SetLooped(true);
        fireSrc->Play("Sounds/fire_crackle.ogg");

        // ── Bell — one-shot with Stop/restart and pitch variation ──────
        // The AudioSource is created lazily inside RingBell() the first
        // time E is pressed, using GetAudioSource() + AddAudioSource().
        bell_ = CreateEntity("Bell");
        bell_->AddModel("Models/Sphere.mdl");
        bell_->SetScale(0.5f);
        bell_->SetMaterialColor(Color(0.9f, 0.78f, 0.2f));
        bell_->SetPosition(0.0f, 2.0f, -4.0f);

        // ════════════════════════════════════════════════════════════
        // STEP 3 — PlayAmbient (non-positional background loop)
        // PlayAmbient() routes through the Music gain channel and plays
        // at constant volume regardless of the listener's position.
        // Perfect for wind, rain, and wide-area atmospheric sounds.
        // ════════════════════════════════════════════════════════════
        Entity* sky = CreateEntity("Sky");
        AudioSource* windSrc = sky->AddAudioSource();
        windSrc->PlayAmbient("Sounds/wind.ogg");

        // ── Camera ────────────────────────────────────────────────────
        cam_ = CreateCamera();
        cam_->Follow(player_, 7.0f, 3.0f);
        cam_->SetSmoothFollow(true, 6.0f);

        // ── HUD ───────────────────────────────────────────────────────
        hud_ = CreateLabel("WASD - Move   E - Ring bell   Esc - Quit");
        hud_->SetPosition(14, 12);
        hud_->SetFontSize(13);
        hud_->SetColor(Color(0.6f, 0.6f, 0.6f, 1.0f));

        bellLabel_ = CreateLabel("Bell rings: 0");
        bellLabel_->SetPosition(14, 32);
        bellLabel_->SetFontSize(15);
        bellLabel_->SetColor(Color(0.9f, 0.78f, 0.2f, 1.0f));

        rings_        = 0;
        bellCooldown_ = 0.0f;
    }

    void Update(float dt) override {
        if (IsKeyPressed(Key::Escape)) Quit();

        // ── Player movement — listener moves with the player ───────────
        float mx = 0.0f, mz = 0.0f;
        if (IsKeyDown(Key::W)) mz =  1.0f;
        if (IsKeyDown(Key::S)) mz = -1.0f;
        if (IsKeyDown(Key::A)) mx = -1.0f;
        if (IsKeyDown(Key::D)) mx =  1.0f;
        cc_->Move(Vector3(mx, 0.0f, mz) * 5.0f, dt);

        // ── Bell: Stop old ring, vary pitch, replay ────────────────────
        bellCooldown_ -= dt;
        if (IsKeyPressed(Key::E) && bellCooldown_ <= 0.0f) {
            RingBell();
            bellCooldown_ = 0.8f;
        }
    }

private:
    void RingBell() {
        // ════════════════════════════════════════════════════════════
        // STEP 4 — GetAudioSource + Stop + SetPitch + SetLooped + Play
        // Retrieve the existing AudioSource or create one on the first call.
        // Stop any ring that is still decaying before launching the new one.
        // ════════════════════════════════════════════════════════════
        AudioSource* src = bell_->GetAudioSource();   // nullptr on first call
        if (!src) src = bell_->AddAudioSource();

        // Interrupt any ongoing ring before starting the next one
        if (src->IsPlaying()) src->Stop();

        // Cycle through five pitches: 0.80, 0.90, 1.00, 1.10, 1.20
        float pitch = 0.80f + (rings_ % 5) * 0.10f;
        src->SetPitch(pitch);          // 1.0 = normal; 0.5 = octave down; 2.0 = octave up

        src->SetRange(3.0f, 30.0f);
        src->SetGain(0.90f);
        src->SetLooped(false);        // ring exactly once — do not repeat
        src->Play("Sounds/bell.wav");

        ++rings_;
        bellLabel_->SetText("Bell rings: " + std::to_string(rings_));
    }

    Entity*              player_       = nullptr;
    Entity*              enemy_        = nullptr;
    Entity*              fire_         = nullptr;
    Entity*              bell_         = nullptr;
    Camera*              cam_          = nullptr;
    CharacterController* cc_           = nullptr;
    Label*               hud_          = nullptr;
    Label*               bellLabel_    = nullptr;

    int   rings_        = 0;
    float bellCooldown_ = 0.0f;
};

int main(int, char**) {
    auto game = Simple3D::CreateGame<SpatialAudioDemo>();
    game->SetResourcePrefixPaths("/path/to/u3d/bin");
    return game->Run();
}

Step-by-step explanation

1. SetAudioListener(Entity*) — designating the ears

SetAudioListener(player_);

Every 3D audio system needs two participants: a source (where the sound comes from) and a listener (where the ears are). In real life the listener is always the player's head. In simple-3d you tell the engine which entity plays that role by calling SetAudioListener() once in Start().

The engine reads the listener's world position and orientation every frame. When an AudioSource is far from the listener its volume is reduced; when it is close the volume is louder, up to the ceiling set by SetGain(). Stereo panning is computed automatically: a source to the listener's left appears predominantly in the left speaker.

The entity does not need any special component — it just needs to exist and move. In most games you pass the player entity. For first-person games you might pass the camera anchor. If SetAudioListener() is never called, the listener defaults to the scene origin; all positional sounds still attenuate, but they attenuate relative to (0, 0, 0) rather than the player.

2. entity->AddAudioSource() — attaching a positional source

AudioSource* enemySrc = enemy_->AddAudioSource();

AddAudioSource() attaches a SoundSource3D component to the entity and returns a pointer to the AudioSource wrapper. The source's world position is synchronized with the entity's world position automatically every frame — if the entity moves, the sound moves with it. You do not need to update it manually.

The call is idempotent: if you call it more than once on the same entity it returns the existing source rather than creating a second one. This makes it safe to call from trigger callbacks or other code paths where you are not sure whether a source has been added yet. That said, the preferred pattern when you only need to retrieve an existing source — without potentially creating one — is GetAudioSource() (see step 7 below).

After calling AddAudioSource() you should configure the source's parameters (SetRange, SetGain, SetLooped, SetPitch) before calling Play(). Parameters set after Play() take effect immediately except for SetPitch(), which takes effect on the next Play() call.

3. src->SetRange(near, far) — distance attenuation

enemySrc->SetRange(2.0f, 20.0f);   // full vol ≤ 2 m, silent ≥ 20 m
fireSrc->SetRange(1.5f, 12.0f);    // smaller audible radius for fire

SetRange(near, far) defines the distance attenuation envelope. Both parameters are in world units (metres in the default simple-3d scale):

  • near — at distances equal to or less than this value the source plays at its full SetGain volume. Think of it as the "loud zone" immediately around the source.
  • far — at distances equal to or greater than this value the source is completely inaudible. Think of it as the outer radius of perception.
  • Between near and far the volume falls off linearly — halfway between the two values the sound plays at roughly 50 % of its SetGain ceiling.

Choosing good range values requires knowing the physical scale of your scene. In this demo the enemy is 10 m from the player's start position, so a far distance of 20 m means the hum is audible from the start. The fire sits 7.8 m away and has a far distance of 12 m, so the player must walk a little closer before hearing the crackling.

For sounds that should be heard throughout a large level — a distant waterfall, a clue sound — set a large far value such as 80 m or more. For intimate sounds — a whisper, a ticking clock — keep far at 5 m or less to ensure they are only audible up close.

4. src->SetGain(float) — volume ceiling

enemySrc->SetGain(0.75f);   // 75 % max volume for the enemy hum
fireSrc->SetGain(0.85f);   // fire is slightly louder

SetGain() sets the maximum volume for this source. The value ranges from 0.0 (completely silent) to 1.0 (full volume). When the listener is within the near distance, the source plays at exactly this gain. Beyond near the effective volume is SetGain * attenuation_factor, where the factor falls from 1.0 to 0.0 between the near and far distances.

SetGain() does not override the master volume. The final output level is: 

// Effective volume = SetGain × attenuation × SetMasterVolume × channel gain

Use SetGain() to balance relative loudness between different sources in the scene. A howling wind at gain 0.3 and a gunshot at gain 1.0 will produce a realistic mix where the environmental sound sits clearly behind the action sound.

5. src->SetLooped(bool) — looping vs. one-shot

enemySrc->SetLooped(true);    // enemy hum: repeat forever
src->SetLooped(false);        // bell ring: play exactly once

SetLooped(true) makes the sound restart automatically from the beginning as soon as it reaches the end of the audio file. Use this for continuous sounds: engine hums, footstep loops, ambient fire, running water. The sound plays uninterrupted until Stop() is called.

SetLooped(false) (the default) plays the sound exactly once. When the last sample plays, the source becomes idle and IsPlaying() returns false. Call Play(path) again to replay from the beginning.

SetLooped() must be called before Play() to take effect on the next playback. Changing it while a sound is already playing will apply from the next loop point or next Play() call, not immediately.

6. src->Play(path) — loading and starting playback

enemySrc->Play("Sounds/enemy_hum.ogg");
fireSrc->Play("Sounds/fire_crackle.ogg");
src->Play("Sounds/bell.wav");

Play(path) loads the audio file from the resource cache and starts playback immediately. The path is relative to a Data/ directory, the same as model and texture paths. Supported formats are OGG Vorbis (.ogg) and uncompressed PCM WAV (.wav). OGG is recommended for longer looped sounds because it compresses significantly; WAV is preferred for very short one-shot effects where the decode overhead would add latency.

The call applies all parameters that were set before it: SetLooped, SetGain, SetPitch, and SetRange all take effect on this Play() call. If you call Play() on a source that is already playing, it restarts from the beginning.

In Start() you can call Play() on continuous looped sources straight away — the engine will start the audio subsystem before Start() returns, so there is no risk of calling Play() too early.

7. src->Stop() and src->IsPlaying()

if (src->IsPlaying()) src->Stop();

Stop() halts playback immediately. The playback cursor resets to the beginning of the file. Once stopped, IsPlaying() returns false and the source is idle until Play() is called again.

IsPlaying() returns true from the moment Play() is called until either the file ends (for non-looped sounds) or Stop() is called. For looped sounds it remains true until explicitly stopped. A typical pattern in the bell code above is to check IsPlaying() before calling Stop() — although Stop() on an already-idle source is a safe no-op, the guard makes intent explicit.

Common uses for Stop():

  • Interrupting a looped sound when an enemy is destroyed
  • Cutting off a previous ring or explosion sound before starting a new one
  • Muting all sounds when pausing the game

Retrieving an existing AudioSource

The complementary retrieval method entity->GetAudioSource() returns the attached AudioSource* if one exists, or nullptr if AddAudioSource() has not been called yet. Use it when you need to control a source that was set up in Start() from a callback or helper function, without risking the creation of a duplicate source: 

AudioSource* src = bell_->GetAudioSource();
if (!src) src = bell_->AddAudioSource();

8. src->SetPitch(float) — frequency scaling

float pitch = 0.80f + (rings_ % 5) * 0.10f;
src->SetPitch(pitch);   // cycles: 0.80, 0.90, 1.00, 1.10, 1.20

SetPitch() scales the playback frequency of the sound. The relationship to perceived pitch is logarithmic and musically defined:

SetPitch valuePerceived effect
1.0Normal pitch — exactly as the audio file was recorded
0.5One octave lower — half the frequency
2.0One octave higher — double the frequency
0.9Slightly lower — subtle downward shift, natural for slower/heavier actions
1.2Slightly higher — subtle upward shift, natural for smaller/faster actions

Pitch scaling is implemented internally as a sample rate adjustment. It also changes the perceived speed of the sound: a sound at pitch 2.0 plays twice as fast and finishes in half the time. For looped sounds this shortens each loop cycle; for one-shot sounds it shortens the total playback duration.

In the bell example above, SetPitch() is called before each Play() call so successive presses produce a rising and falling sequence of bell tones from the same audio file. This technique — pitch-shifting a single recorded sample — is widely used in games to add variety to footsteps, impacts, and pickups without recording many separate files.

9. src->PlayAmbient(path) — non-positional background loops

AudioSource* windSrc = sky->AddAudioSource();
windSrc->PlayAmbient("Sounds/wind.ogg");

PlayAmbient() is a special-purpose play call that bypasses distance attenuation entirely. The sound plays at constant volume everywhere in the scene, looped automatically, on the Music gain channel. Walking toward or away from the entity that owns the source makes no difference to the perceived volume.

Use PlayAmbient() for sounds that are conceptually everywhere at once: wind, rain, ocean waves, cave dripping, the hum of a spaceship interior. These sounds set the mood of the level as a whole rather than pointing the player toward a specific location. Using a regular looped Play() for these would make their volume fluctuate as the player moves, which would feel wrong for a wind that is supposed to blow everywhere equally.

The Music channel is separate from the Sound channel used by Play(). You can control the Music channel gain independently (in a future API version) and it is kept separate from the master volume's effects on sound effects. For background music that should play non-spatially, you can also use PlayMusic() from the flat API covered in Part 10 — the difference is that PlayAmbient() is entity-bound (and could be extended with spatial behaviour in future releases) while PlayMusic() is always a pure stream with no entity association.

Lua equivalent

The same spatial audio setup in Lua mirrors the C++ API almost one-to-one. Methods on AudioSource objects are called with the Lua colon syntax. SetAudioListener is called on the game global. Key names are passed as strings to Input:IsKeyDown() and Input:IsKeyPressed(). 

-- Part 11: Spatial Audio — Lua equivalent

local bellCooldown = 0

    game:SetClearColor(Color(0.05, 0.05, 0.10, 1.0))

    -- Bell entity (AudioSource created lazily in Update)
    bell = game:CreateEntity("Bell")
    bell:AddModel("Models/Sphere.mdl")
    bell:SetScale(0.5)
    bell:SetPosition(0, 2, -4)

    -- Ambient wind — non-positional, constant volume
    local sky = game:CreateEntity("Sky")
    local windSrc = sky:AddAudioSource()
    windSrc:PlayAmbient("Sounds/wind.ogg")

    -- Camera
    local cam = game:CreateCamera()
    cam:Follow(player, 7.0, 3.0)
    cam:SetSmoothFollow(true, 6.0)

function Update(dt)
    if Input:IsKeyPressed("Escape") then game:Quit() end

    -- Player movement (listener moves automatically with the entity)
    local mx, mz = 0, 0
    if Input:IsKeyDown("W") then mz =  1 end
    if Input:IsKeyDown("S") then mz = -1 end
    if Input:IsKeyDown("A") then mx = -1 end
    if Input:IsKeyDown("D") then mx =  1 end
    cc:Move(Vector3(mx, 0, mz) * 5, dt)

    -- Bell: lazy-create source, stop old ring, vary pitch, play
    bellCooldown = bellCooldown - dt
    if Input:IsKeyPressed("E") and bellCooldown <= 0 then
        local src = bell:GetAudioSource()
        if not src then src = bell:AddAudioSource() end

        if src:IsPlaying() then src:Stop() end

        local pitch = 0.80 + (rings % 5) * 0.10
        src:SetPitch(pitch)
        src:SetRange(3.0, 30.0)
        src:SetGain(0.90)
        src:SetLooped(false)
        src:Play("Sounds/bell.wav")

        rings = rings + 1
        bellCooldown = 0.8
    end
end

entity->GetAudioSource() AudioSource* Returns the attached AudioSource, or nullptr if one has not been added yet.

Common mistakes

Common mistakes — Part 11

  • Forgetting to call SetAudioListener(). Without it the listener stays at the world origin (0, 0, 0). Sounds attenuate relative to that fixed point rather than the player. The result is that all sounds seem to vary in volume when you stand still, and volume does not change as you walk toward or away from a source. Always call SetAudioListener(player_) in Start().
  • Calling Play() before SetLooped(). SetLooped() must be configured before Play() is called. If you call Play() first and SetLooped(true) second, the sound will play once and stop — the loop flag is read at the moment playback begins. Always set all parameters first, then call Play() last.
  • Swapping near and far in SetRange(). SetRange(20.0f, 2.0f) (near > far) produces undefined or silent output. near must always be smaller than far. A common mistake is passing the values in the wrong order when reading documentation that lists parameters as "min distance, max distance". Double-check: first argument = close distance (loud), second argument = far distance (silent).
  • Calling AddAudioSource() repeatedly to get the source in callbacks. Although AddAudioSource() is idempotent, the intent is clearer and slightly more efficient when you use GetAudioSource() to retrieve an existing source and only call AddAudioSource() as a fallback when the result is nullptr. The pattern auto* s = e->GetAudioSource(); if (!s) s = e->AddAudioSource(); makes it obvious that you do not want duplicate sources.
  • Not calling Stop() before re-playing a one-shot sound. If a non-looped sound is still playing and you call Play() again, it restarts from the beginning abruptly. For sounds where overlap sounds bad (a bell, a single gun click), call Stop() first. For sounds where rapid re-triggering with overlap is acceptable (rapid footsteps, gunfire), you can skip the stop.
  • Using PlayAmbient() for sounds that should attenuate with distance. PlayAmbient() permanently bypasses distance attenuation. If you want a sound to be loud near its source and quiet far away, use Play() with SetLooped(true) and an appropriate SetRange() instead. Reserve PlayAmbient() for truly global atmospheric sounds that should be equally loud everywhere.
  • Setting SetPitch() after Play() and expecting it to apply immediately. Pitch takes effect at the start of the next playback. If you call src->SetPitch(1.5f) while a sound is already playing, the change will be heard on the next Play() call, not mid-stream. To hear an immediate pitch change, call Stop() first, then SetPitch(), then Play() again.
  • Passing the near distance as the total audible radius. Beginners sometimes write SetRange(0.0f, 10.0f) thinking the sound is full volume at the source and fades to nothing at 10 m — which is actually correct. But then they write SetRange(10.0f, 10.0f) thinking "full at 10 m, fade after 10 m" — which makes the sound inaudible everywhere because near equals far. Remember: near is the loudest zone (0 m to near distance = full volume), not the fade-out start. If you want the sound to start fading immediately from the source, set near to 0 or a very small value like 0.5.

Try it yourself

Run the example and experiment with these changes to deepen your understanding of the spatial audio system:

  1. Change the enemy's SetRange(2.0f, 20.0f) to SetRange(0.5f, 5.0f). Now the enemy hum only reaches 5 metres. Start the game, walk toward the enemy and listen to the abrupt cutoff at 5 m. Then restore the range to something more natural.
  2. Mute the enemy hum when its health reaches zero. After the enemy is "destroyed" (just enemy_->SetActive(false) for the demo), call enemy_->GetAudioSource()->Stop() to silence it. Verify that the source stays silent after reactivation until you call Play() again.
  3. Add a second fire entity at a different position with SetRange(1.0f, 8.0f) and the same sound file. Walk between the two fires and listen to how each contributes independently to the soundscape — the engine mixes both sources simultaneously.
  4. Swap the bell's cycling pitch for a random pitch each press: pick a random value between 0.7 and 1.3 using 0.7f + (rand() % 60) / 100.0f. Compare the natural randomness against the deterministic cycling. For footsteps, random pitch variation is usually preferred.
  5. Replace the ambient windSrc->PlayAmbient("Sounds/wind.ogg") with a regular windSrc->SetLooped(true); windSrc->Play("Sounds/wind.ogg") and set a range of SetRange(0.0f, 8.0f). Walk toward and away from the sky entity's position. Observe how the wind now gets louder and quieter — often undesirable for atmospheric sounds, which is exactly why PlayAmbient() exists.
  6. Integrate with the trigger system. Place a trigger box in the scene. When the player enters, call fire_->GetAudioSource()->SetGain(0.0f) to silence the fire source without stopping it (the fire keeps playing internally; it is just muted). When the player exits, restore SetGain(0.85f). This pattern is useful for room-based audio transitions.

What you learned

This part covered the entire spatial audio API that simple-3d wraps around Urho3D's SoundSource3D system:

  • SetAudioListener(entity) designates the scene entity that acts as the player's ears. Every AudioSource in the scene attenuates relative to this entity's world position, updated automatically each frame. Call it once in Start().
  • entity->AddAudioSource() attaches a positional audio component. The component tracks the entity's world position. Use GetAudioSource() to retrieve it later without risk of creating a duplicate.
  • SetRange(near, far) defines the distance attenuation envelope. The sound is at full SetGain within near metres, linearly falls off between near and far, and is completely silent beyond far.
  • SetGain(float) sets the volume ceiling (0–1) at the near distance. It combines with distance attenuation and the master volume slider.
  • SetLooped(bool) controls whether the sound repeats automatically. Set it before Play(). Use true for ambient continuous sounds and false for one-shot effects.
  • Play(path) loads and starts the sound. Respects all previously configured parameters. Restarting a playing source calls it again. Stop() halts playback immediately. IsPlaying() lets you query the current state before deciding whether to stop or restart.
  • SetPitch(float) scales the playback frequency. Values above 1.0 produce higher pitch and shorter duration; values below 1.0 produce lower pitch and longer duration. Use it to add natural variety to repeated sounds from a single audio file.
  • PlayAmbient(path) plays a looped, non-positional sound on the Music gain channel. Distance from the listener has no effect. Use this for global atmospheric sounds that should be equally audible everywhere in the scene.

With spatial audio in place your game world will feel significantly more alive. As the player explores, the soundscape changes dynamically — distant enemies become audible before they are visible, fire sounds grow louder as the player approaches, and a subtle constant wind ties the whole scene together without demanding the player's attention.
