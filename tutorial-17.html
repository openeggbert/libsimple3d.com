Overview

Every game eventually needs a HUD — a score counter, a health bar, a minimap icon, a pause menu. Up to this point you have already used CreateLabel() (introduced in Part 01) to display raw text. Part 17 completes the picture by introducing the three remaining screen-space widget types: Image, Panel, and Button.

All four types live in namespace Simple3D::UI. Image, Panel, and Button inherit a common Element base class that provides positioning, sizing, visibility, depth ordering, and anchor alignment. Label predates the Element hierarchy and has its own interface, but it exposes the same conceptual operations and can be reparented into an Element group using an overloaded AddChild.

This tutorial builds a complete scene with a live HUD (score counter, icon strip, status bar) and a pause menu (semi-transparent dark panel containing a title label, a resume button, and a quit button). Pressing Escape toggles the pause menu. When the menu is visible a single SetVisible(false) call on the root panel hides everything inside it automatically.

Widget types at a glance

TypeFactory methodUnderlying rendererTypical use
Label CreateLabel(text) Urho3D::Text Score counter, hint text, damage numbers
UI::Image CreateImage(texturePath="") Urho3D::BorderImage Icons, health bar fill, atlas sprites, coloured rects
UI::Panel CreatePanel() Urho3D::UIElement Opaque or semi-transparent containers for grouping widgets
UI::Button CreateButton(text="") Urho3D::Button Menu items, clickable HUD elements

Complete source

The program below builds a small interactive demo that exercises every API introduced in this part. When you run it you will see a spinning box in the centre of a dark scene, a top-left HUD panel containing a life icon, a health-bar image, and a live score label, and a centred pause menu that toggles with Escape. Read through the full listing first, then work through the step-by-step explanation below. 

// main.cpp — Part 17: UI System
#include <Simple3D/Simple3D.h>

using namespace Simple3D;

// ─────────────────────────────────────────────────────────────────────────────
// UIDemo — demonstrates Label, Image, Panel, and Button together.
// ─────────────────────────────────────────────────────────────────────────────
class UIDemo : public Game {
public:

    void Start() override {
        SetWindowTitle("Part 17 — UI System");
        SetClearColor(Color(0.06f, 0.06f, 0.10f, 1.0f));

        // ── 3D scene (background) ────────────────────────────────────────────
        Entity* sun = CreateEntity("Sun");
        sun->AddDirectionalLight(Color(1.0f, 0.9f, 0.8f), 1.2f);
        sun->SetRotation(Quaternion(45.0f, 45.0f, 0.0f));

        box_ = CreateEntity("Box");
        box_->AddModel("Models/Box.mdl");
        box_->SetScale(1.8f);

        cam_ = CreateCamera();
        cam_->SetPosition(0.0f, 2.0f, -7.0f);
        cam_->LookAt(Vector3::ZERO);

        // ── Build the HUD ────────────────────────────────────────────────────
        // A dark panel in the top-left corner groups the HUD elements so that
        // a single SetVisible call shows or hides the whole block.
        BuildHUD();

        // ── Build the pause menu ─────────────────────────────────────────────
        // A semi-transparent panel centred on screen; hidden until Escape is pressed.
        BuildPauseMenu();
    }

    void Update(float dt) override {
        // Spin the background box
        yaw_ += 35.0f * dt;
        box_->SetRotation(Quaternion(0.0f, yaw_, 15.0f));

        // Toggle pause on Escape (only when the menu is not blocking input)
        if (IsKeyPressed(Key::Escape)) {
            paused_ = !paused_;
            pauseMenu_->SetVisible(paused_);
            // Dim the HUD while paused so the menu stands out
            hudPanel_->SetVisible(!paused_);
        }

        if (paused_) return;   // freeze game logic while paused

        // Accumulate score over time and update the label
        score_ += 10 * dt;
        scoreLbl_->SetText("Score: " + std::to_string(static_cast<int>(score_)));
    }

private:
    // ── 3D objects ────────────────────────────────────────────────────────
    Entity* box_ = nullptr;
    Camera* cam_ = nullptr;
    float   yaw_ = 0.0f;

    // ── UI state ──────────────────────────────────────────────────────────
    UI::Panel*  hudPanel_  = nullptr;
    Label*      scoreLbl_  = nullptr;
    UI::Panel*  pauseMenu_ = nullptr;
    bool        paused_    = false;
    float       score_     = 0.0f;

    // ─────────────────────────────────────────────────────────────────────
    // BuildHUD — top-left overlay: life icon, health bar, score counter
    // ─────────────────────────────────────────────────────────────────────
    void BuildHUD() {
        // Outer panel: anchored to TopLeft, 8 px margin from edge.
        // SetAnchor(TopLeft) is the default, but we state it explicitly for clarity.
        hudPanel_ = CreatePanel();
        hudPanel_->SetSize(230, 88);
        hudPanel_->SetAnchor(UI::Anchor::TopLeft);
        hudPanel_->SetPosition(8, 8);
        hudPanel_->SetColor(Color(0.0f, 0.0f, 0.0f, 0.55f));
        hudPanel_->SetZOrder(1);   // HUD sits above the 3D scene

        // ── Life icon — textured Image sampled from a 64×64 atlas ────────────
        // SetImageRect(x, y, w, h) selects a sub-rectangle of the texture in pixels.
        // Here we take the first 32×32 tile at the top-left of the atlas.
        UI::Image* lifeIcon = CreateImage("Textures/hud_icons.png");
        lifeIcon->SetSize(32, 32);
        lifeIcon->SetPosition(8, 8);    // relative to panel after AddChild
        lifeIcon->SetImageRect(0, 0, 32, 32);          // top-left tile
        lifeIcon->SetColor(Color(1.0f, 0.5f, 0.5f, 1.0f));  // warm tint

        // ── Health bar background — untextured coloured rect ─────────────────
        UI::Image* hpBg = CreateImage();   // no texture = solid colour
        hpBg->SetSize(172, 14);
        hpBg->SetPosition(48, 14);
        hpBg->SetColor(Color(0.15f, 0.05f, 0.05f, 1.0f));  // dark red trough

        // ── Health bar fill ───────────────────────────────────────────────────
        UI::Image* hpFill = CreateImage();
        hpFill->SetSize(138, 14);         // ~80% full
        hpFill->SetPosition(48, 14);
        hpFill->SetColor(Color(0.15f, 0.80f, 0.20f, 1.0f));  // green fill
        hpFill->SetZOrder(1);             // draw on top of trough

        // ── Score label ───────────────────────────────────────────────────────
        scoreLbl_ = CreateLabel("Score: 0");
        scoreLbl_->SetPosition(8, 46);
        scoreLbl_->SetFontSize(16);
        scoreLbl_->SetColor(Color(0.9f, 0.9f, 0.7f, 1.0f));  // warm white

        // Reparent everything into hudPanel_.
        // From this point positions are relative to hudPanel_'s top-left corner.
        // Hiding hudPanel_ automatically hides all children.
        hudPanel_->AddChild(lifeIcon);
        hudPanel_->AddChild(hpBg);
        hudPanel_->AddChild(hpFill);
        hudPanel_->AddChild(scoreLbl_);
    }

    // ─────────────────────────────────────────────────────────────────────
    // BuildPauseMenu — centred overlay: dark panel, title, two buttons
    // ─────────────────────────────────────────────────────────────────────
    void BuildPauseMenu() {
        // Root panel: SetAnchor(Center) makes SetPosition an offset from the middle
        // of the screen.  (0, 0) centres the panel's top-left corner on screen.
        // To truly centre the 300x180 panel we offset by half its size.
        pauseMenu_ = CreatePanel();
        pauseMenu_->SetSize(300, 180);
        pauseMenu_->SetAnchor(UI::Anchor::Center);
        pauseMenu_->SetPosition(-150, -90);   // offset by half size to fully centre
        pauseMenu_->SetColor(Color(0.0f, 0.0f, 0.0f, 0.80f));
        pauseMenu_->SetZOrder(10);   // well above the HUD
        pauseMenu_->SetVisible(false);   // start hidden

        // ── Title label ───────────────────────────────────────────────────────
        Label* title = CreateLabel("PAUSED");
        title->SetPosition(85, 16);
        title->SetFontSize(26);
        title->SetColor(Color::WHITE);

        // ── Resume button ─────────────────────────────────────────────────────
        // SetOnClick takes any callable that accepts no arguments and returns void.
        // Lambda captures [this] so it can call ResumeGame().
        UI::Button* resumeBtn = CreateButton("Resume");
        resumeBtn->SetSize(200, 44);
        resumeBtn->SetPosition(50, 68);
        resumeBtn->SetColor(Color(0.10f, 0.40f, 0.12f, 0.95f));
        resumeBtn->SetOnClick([this] {
            paused_ = false;
            pauseMenu_->SetVisible(false);
            hudPanel_->SetVisible(true);
        });

        // ── Quit button ───────────────────────────────────────────────────────
        UI::Button* quitBtn = CreateButton("Quit");
        quitBtn->SetSize(200, 44);
        quitBtn->SetPosition(50, 122);
        quitBtn->SetColor(Color(0.40f, 0.10f, 0.10f, 0.95f));
        quitBtn->SetOnClick([this] { Quit(); });

        // Reparent into the menu panel (positions above are now panel-relative)
        pauseMenu_->AddChild(title);
        pauseMenu_->AddChild(resumeBtn);
        pauseMenu_->AddChild(quitBtn);
    }
};

int main(int, char**) {
    auto game = Simple3D::CreateGame<UIDemo>();
    game->SetResourcePrefixPaths("/path/to/u3d/bin");
    return game->Run();
}

Step-by-step explanation

1. The Simple3D::UI namespace

All UI widget types except Label live under Simple3D::UI. When you include <Simple3D/Simple3D.h> (the single-header umbrella) all UI types are available without any extra includes. With a using namespace Simple3D; directive at file scope you can write UI::Image, UI::Panel, UI::Button, and UI::Anchor directly.

Label is the oldest widget and lives directly in Simple3D::, so it is just Label* after the using directive. For the rest of this tutorial, unqualified Image, Panel, and Button always mean the Simple3D::UI:: variants, not anything from the standard library.

2. The shared Element base class

UI::Image, UI::Panel, and UI::Button all inherit from UI::Element. This base class supplies the five methods that every widget shares:

MethodReturnsNotes
SetPosition(int x, int y)—Pixel offset from the anchor point
SetSize(int w, int h)—Pixel dimensions of the widget
SetVisible(bool)—Hides the widget and all its children
SetZOrder(int z)—Higher = rendered on top; default 0
SetAnchor(UI::Anchor a)—Aligns the widget to a screen region
AddChild(Element* child)—Reparents a child widget
AddChild(Label* label)—Overload for Label (not an Element subclass)
GetPosition()IntVector2Current pixel position
GetSize()IntVector2Current pixel size
IsVisible()boolCurrent visibility state

Because all three widget types inherit these methods, you can store an array of UI::Element* and operate on heterogeneous widget groups with a single loop — useful for enabling or disabling an entire UI screen.

3. CreatePanel()

UI::Panel* hudPanel_ = CreatePanel();
hudPanel_->SetSize(230, 88);
hudPanel_->SetAnchor(UI::Anchor::TopLeft);
hudPanel_->SetPosition(8, 8);
hudPanel_->SetColor(Color(0.0f, 0.0f, 0.0f, 0.55f));

CreatePanel() creates a plain filled rectangle. Its only widget-specific method is SetColor(Color) which sets the background fill colour, and SetOpacity(float) which multiplies the overall alpha of the element and all its children. A Panel with alpha less than 1 in its colour produces a semi-transparent overlay that lets the 3D scene show through.

A panel with no size or no colour is invisible, so always call both SetSize and SetColor during setup.

4. CreateImage() — textures and coloured rectangles

// Textured image auto-sized from the PNG:
UI::Image* icon = CreateImage("Textures/hud_icons.png");
icon->SetSize(32, 32);
icon->SetColor(Color(1.0f, 0.5f, 0.5f, 1.0f));   // tint: warm pink
icon->SetImageRect(0, 0, 32, 32);                  // top-left 32x32 tile

// Untextured coloured rectangle:
UI::Image* bar = CreateImage();   // no path = solid colour only
bar->SetSize(138, 14);
bar->SetColor(Color(0.15f, 0.80f, 0.20f, 1.0f));  // green

CreateImage(path) loads a PNG, JPG, or other texture and creates a screen-space image widget. Passing no argument (or an empty string) creates an untextured widget that renders as a solid coloured rectangle — perfect for health bar fills, border decorations, or debug overlays.

SetColor(Color) — tinting. When a texture is loaded, SetColor modulates the texture colour per-vertex. Pure white Color(1,1,1,1) leaves the texture unchanged. Any other colour multiplies the texture RGB values by the tint RGB, allowing you to tint a greyscale icon sheet any colour you want at zero texture cost.

SetImageRect(x, y, w, h) — atlas sampling. Many games pack dozens of icons into a single texture atlas to reduce draw calls. SetImageRect selects a sub-rectangle of the bound texture measured in pixels. For example, if hud_icons.png is a 128×64 sheet of 32×32 tiles arranged in a 4×2 grid, you can pick the third tile in the first row with SetImageRect(64, 0, 32, 32). The widget then displays only that tile.

5. SetAnchor() and the UI::Anchor enum

By default every widget is positioned with its top-left corner at (0, 0) — the top-left of the window. SetAnchor() changes this reference point. After calling it, SetPosition(x, y) specifies a pixel offset from that anchor point, not from the absolute top-left of the screen.

The nine available anchor values cover every corner, edge midpoint, and the screen centre:

Anchor valueReference point on screenTypical use
UI::Anchor::TopLeftTop-left corner (default)HUD panels, score displays
UI::Anchor::TopCenterTop edge, horizontally centredCentered title bar, wave counter
UI::Anchor::TopRightTop-right cornerTimer, ammo count
UI::Anchor::CenterLeftLeft edge, vertically centredVertical menu strip
UI::Anchor::CenterExact screen centrePause menus, splash screens, crosshairs
UI::Anchor::CenterRightRight edge, vertically centredSkill bar, minimap
UI::Anchor::BottomLeftBottom-left cornerSubtitles, context hints
UI::Anchor::BottomCenterBottom edge, horizontally centredDialogue text, action prompts
UI::Anchor::BottomRightBottom-right cornerMinimap, inventory shortcut bar

Anchors are especially powerful for resolution independence. A button with SetAnchor(BottomRight) and SetPosition(-170, -60) always sits 170 pixels from the right edge and 60 pixels from the bottom, regardless of whether the window is 1280×720 or 3840×2160. 

// Timer in the top-right corner — stays there at any resolution
Label* timerLbl = CreateLabel("01:30");
timerLbl->SetPosition(-80, 12);
timerLbl->SetFontSize(20);
timerLbl->SetColor(Color::WHITE);
// Note: Label itself uses SetPosition pixel-from-top-left, not SetAnchor.
// To anchor a Label, add it as a child of an anchored Panel or Image.

// Action prompt at screen bottom-centre
UI::Panel* promptPanel = CreatePanel();
promptPanel->SetSize(280, 36);
promptPanel->SetAnchor(UI::Anchor::BottomCenter);
promptPanel->SetPosition(-140, -48);   // centred horizontally, 48 px up from bottom
promptPanel->SetColor(Color(0.0f, 0.0f, 0.0f, 0.7f));

// Minimap container anchored to bottom-right
UI::Panel* mapPanel = CreatePanel();
mapPanel->SetSize(128, 128);
mapPanel->SetAnchor(UI::Anchor::BottomRight);
mapPanel->SetPosition(-136, -136);   // 8 px margin from each edge
mapPanel->SetColor(Color(0.1f, 0.1f, 0.2f, 0.8f));

Tip — Label anchoring

Label does not have a SetAnchor method of its own. To anchor a label (for example, to keep it glued to the bottom-centre of the screen), create an anchored Panel and call panel->AddChild(label). The label position then becomes relative to the panel's top-left corner, and the panel handles the screen alignment.

6. SetZOrder() — depth ordering

hudPanel_->SetZOrder(1);    // HUD layer
pauseMenu_->SetZOrder(10);  // pause menu always above HUD
hpFill->SetZOrder(1);       // fill renders on top of trough background

SetZOrder(int z) controls which widget renders in front when two widgets overlap. Higher values appear on top. All widgets start at Z-order 0. The Z-order of a parent does not automatically override its children — each element's absolute Z-order is evaluated independently, so a child with Z-order 2 will appear above a sibling with Z-order 1 even when both are reparented into the same panel.

A common layering strategy is: background panels at 0, HUD elements at 1–5, notifications at 6–8, and modal menus at 10+. This makes it easy to reason about depth without needing to adjust values later.

7. SetVisible() and parent-child visibility

pauseMenu_->SetVisible(false);   // hides the panel AND all its children at once
pauseMenu_->SetVisible(true);    // shows them all again

Hiding a parent element via SetVisible(false) automatically hides the entire subtree of children reparented into it. You do not need to iterate children and hide each one manually. When you call SetVisible(true) on the parent, all children reappear in their previous state.

This is the main reason to group related widgets under a panel: one boolean toggle controls the whole UI screen. The pause menu example exploits this to hide the title label and both buttons by calling pauseMenu_->SetVisible(false) on the root panel alone.

8. Label methods: SetText(), SetFontSize(), SetColor()

scoreLbl_ = CreateLabel("Score: 0");
scoreLbl_->SetPosition(8, 46);
scoreLbl_->SetFontSize(16);
scoreLbl_->SetColor(Color(0.9f, 0.9f, 0.7f, 1.0f));

// In Update():
scoreLbl_->SetText("Score: " + std::to_string(static_cast<int>(score_)));

Although CreateLabel was introduced back in Part 01, this part is the first to use its methods in the context of a properly structured UI system, so it is worth reviewing them:

  • SetText(string) replaces the displayed string. Call it from Update() to create live readouts such as score counters, timers, or debug values. The string is UTF-8, so accented characters and basic Unicode symbols work provided the loaded font contains the required glyphs.
  • SetFontSize(int pts) sets the text point size using the current font. The default font is Fonts/Anonymous Pro.ttf (a monospaced programmer font that ships with Urho3D). Call label->SetFont("Fonts/MyFont.ttf") before SetFontSize to switch fonts.
  • SetColor(Color) sets the text colour. The alpha component controls transparency, which is useful for fade-in and fade-out effects. Color::WHITE is the most common value for legible text on dark backgrounds.

9. CreateButton() and SetOnClick()

UI::Button* resumeBtn = CreateButton("Resume");
resumeBtn->SetSize(200, 44);
resumeBtn->SetPosition(50, 68);
resumeBtn->SetColor(Color(0.10f, 0.40f, 0.12f, 0.95f));
resumeBtn->SetOnClick([this] {
    paused_ = false;
    pauseMenu_->SetVisible(false);
    hudPanel_->SetVisible(true);
});

CreateButton(text) creates a labelled clickable button. The text argument initialises the button label; you can change it later with btn->SetText("New Label"). Like Panel, the button also has SetColor for its background fill and SetOpacity for overall transparency.

SetOnClick(std::function<void()>) attaches a callback that fires when the user releases the mouse button while the cursor is inside the button's rectangle. The callback is a standard C++ callable — a free function pointer, a member function wrapped in std::bind, or most commonly a lambda. Using a lambda with [this] capture is the simplest pattern because it lets the callback access member variables directly.

Calling SetOnClick a second time completely replaces the previous callback. This is useful for rebinding buttons dynamically (for example, assigning different actions to the same button depending on the current game state).

Click is fired on mouse-button release, not on press. This matches platform HIG conventions and lets the user drag out of the button area to cancel an accidental click.

10. AddChild() — building UI groups

UI::Panel* panel = CreatePanel();
panel->SetSize(300, 180);
panel->SetPosition(100, 100);

Label*      title  = CreateLabel("PAUSED");
UI::Button* btn    = CreateButton("Resume");

title->SetPosition(85, 16);   // will be relative to panel after AddChild
btn->SetPosition(50, 68);

panel->AddChild(title);   // overload for Label* (Label is not an Element subclass)
panel->AddChild(btn);

panel->SetVisible(false);   // hides panel + title + btn in one call

Calling parent->AddChild(child) detaches child from the UI root and makes its position relative to the parent's top-left corner. This is the fundamental way to group widgets so they move together.

There are two AddChild overloads. The normal overload accepts UI::Element*, which covers Image*, Panel*, and Button*. A separate overload accepts Label* because Label predates the Element hierarchy and does not inherit from it. Both overloads behave identically from a visibility and layout perspective.

Set child positions before calling AddChild. The positions you supply in SetPosition are always panel-relative once the child has been reparented. If you call SetPosition after AddChild the result is the same — the engine interprets the coordinates relative to the current parent either way.

11. Ownership and lifetime

All UI objects returned by CreateLabel, CreateImage, CreatePanel, and CreateButton are owned by the Game instance. You must never call delete on any of these pointers. They remain alive for the entire application session — there is no DestroyPanel or DestroyButton in the current API version. To make a widget disappear from the screen, call SetVisible(false).

Lua equivalent

The same HUD and pause menu can be written in Lua. The factory methods are called on the global game object and all methods use the colon syntax. The lambda callback for SetOnClick becomes a plain Lua function passed as an argument.

end

function Update(dt)
    yaw = yaw + 35 * dt
    box:SetRotation(Quaternion(0, yaw, 15))

    if Input:IsKeyPressed("Escape") then
        paused = not paused
        pauseMenu:SetVisible(paused)
        hudPanel:SetVisible(not paused)
    end

    if paused then return end

    score = score + 10 * dt
    scoreLbl:SetText("Score: " .. math.floor(score))
end

The key Lua differences are: anchor names are passed as plain strings ("TopLeft", "Center", etc.) rather than enum values, SetOnClick accepts an anonymous function() ... end block, and string concatenation uses .. instead of +.

Anchor demo — all nine positions

The snippet below creates nine small coloured panels, one at each anchor position, to make the alignment model completely visible. Drop this into a Start() method as a quick visual test during development. 

void PlaceAnchorMarkers() {
    // Helper lambda so we don't repeat ourselves nine times.
    auto place = [this](UI::Anchor anchor, int ox, int oy, Color c) {

Common mistakes

Common mistakes — Part 17

  • Setting child positions after AddChild without understanding they are now parent-relative. If you call SetPosition(400, 300) on a button and then add it to a panel at (100, 100), the button ends up at screen position (500, 400), not (400, 300). Set all child positions with panel-relative coordinates in mind before or after calling AddChild — the result is the same, but thinking in panel-local space from the start avoids confusion.
  • Forgetting that SetAnchor(Center) does not centre the widget's origin on screen. SetAnchor(Center) aligns the widget's top-left corner to the screen centre. SetPosition(0, 0) with Anchor::Center places the top-left corner exactly at the centre, making the visible widget appear to the right of and below centre. To truly centre a 300×180 panel, use SetPosition(-150, -90) — negative half the width and half the height.
  • Not calling SetSize() on Panel or Image. Newly created panels and images have zero width and zero height by default. A zero-size element is invisible even if its colour is fully opaque and visible is true. Always call SetSize during setup. For Image with a texture, you still need to explicitly call SetSize because auto-sizing to the texture is not the default behaviour.
  • Calling delete on a widget pointer. All UI objects are owned by the Game instance. Calling delete pauseMenu_ causes a double-free crash. Use SetVisible(false) to hide widgets you no longer need on screen.
  • Using SetZOrder to fix overlap without understanding parent scope. Z-orders within a parent panel are evaluated relative to siblings. A child with SetZOrder(5) inside a panel at SetZOrder(1) does not automatically render above a standalone element at SetZOrder(3) — the root Z-order of the panel determines its position relative to the standalone element, and child Z-orders only matter within the panel. If menus clip under HUD elements, raise the root panel's Z-order, not the children's.
  • Capturing widget pointers by value in lambdas before the widgets are created. If you write a lambda that references pauseMenu_ before pauseMenu_ has been assigned, the captured pointer is null. Capture [this] instead of the raw pointer so the lambda dereferences the member variable at the time it executes, not at the time it is created.
  • Expecting SetOnClick to fire on mouse-down. The callback fires on mouse-button release while the cursor is still inside the button area. This is intentional — it matches platform conventions and lets users cancel by dragging out of the button. Do not try to work around this by polling IsMouseButtonDown over the button area; use the callback as designed.
  • Using SetImageRect with incorrect pixel coordinates. SetImageRect(x, y, w, h) takes pixel coordinates, not normalised UV coordinates. If your atlas is 128×64 and you want the second 32×32 tile, the call is SetImageRect(32, 0, 32, 32), not SetImageRect(0.25f, 0, 0.25f, 0.5f). Using UV fractions produces no visible error at compile time but selects the wrong region at runtime.

Try it yourself

Once the example is running, try these modifications to deepen your understanding:

  1. Add a third button to the pause menu labelled "Settings" at position (50, 168) and have its SetOnClick print a debug message to the console using URHO3D_LOGINFO("Settings clicked"). Increase the panel height to 220 to make room.
  2. Create a small crosshair using two UI::Image widgets (one horizontal, one vertical, each 2 pixels thick) anchored to UI::Anchor::Center with offsets that centre them at exactly the screen midpoint.
  3. Make the health bar dynamically shrink. Store the hpFill pointer as a member, then in Update() reduce its width by 4 pixels per second: hpFill_->SetSize(hpFill_->GetSize().x_ - 4*dt, 14). Clamp to zero using std::max.
  4. Replace the hard-coded pause key check with a named action. Call BindAction("Pause").Key(Key::Escape) in Start() and use IsActionPressed("Pause") in Update() (as covered in Part 16).
  5. Add a transparent image as a background for the score — use CreateImage() with no texture argument, SetSize(120, 24), and SetColor(Color(0,0,0,0.4f)), positioned just behind the score label inside hudPanel_. Adjust the Z-orders so the panel renders before the label.

What you learned

This part introduced the full Simple3D::UI widget toolkit:

  • CreateImage(path) creates a screen-space image from a texture file. Calling it without a path produces an untextured coloured rectangle. SetColor() applies a vertex tint to textured images and sets the fill colour for untextured ones. SetImageRect(x, y, w, h) samples a pixel-coordinate sub-rectangle from a texture atlas.
  • CreatePanel() creates a plain filled rectangle ideal for grouping widgets. SetColor(Color) with an alpha less than 1 makes it semi-transparent. SetOpacity(float) multiplies the overall alpha of the panel and all its children.
  • CreateButton(text) creates a clickable labelled button. SetOnClick(callback) attaches a std::function<void()> — typically a C++ lambda — that fires when the mouse is released over the button area.
  • The shared UI::Element base class gives every widget five fundamental methods: SetPosition, SetSize, SetVisible, SetZOrder, and SetAnchor.
  • SetAnchor(UI::Anchor) aligns a widget to one of nine screen regions. After calling it, SetPosition is a pixel offset from that region rather than from the absolute top-left corner. The nine values are TopLeft, TopCenter, TopRight, CenterLeft, Center, CenterRight, BottomLeft, BottomCenter, BottomRight.
  • SetZOrder(int) controls render depth. Widgets with higher Z-orders appear in front of those with lower values. Set modal menus to a Z-order well above HUD elements.
  • SetVisible(false) on a parent element hides the entire subtree of children simultaneously. This is the key reason to group related widgets under a panel.
  • AddChild(Element* / Label*) reparents a widget so that its position becomes relative to the parent's top-left corner and its visibility is controlled by the parent.
  • Ownership: all UI objects are owned by the Game instance. Never call delete on widget pointers. Use SetVisible(false) to hide widgets you no longer need on screen.
