Part 23 of 30Physics Constraints

What you will learn

  • What a physics constraint (joint) is and why it is different from parenting entities.
  • How to create a ball-socket constraint for freely-rotating links such as chain segments and pendulums.
  • How to create a hinge constraint that confines rotation to a single axis — perfect for doors and levers.
  • How to set angular limits on a hinge so it can only swing within a defined arc.
  • How to attach a motor to a hinge to drive it at a target angular velocity, producing a rotating platform or a self-opening door.
  • How to create a slider constraint that lets an entity translate along exactly one axis — ideal for lifts, rails, and drawers.
  • How to add a slider motor to drive translational movement back and forth automatically.
  • How to use a fixed constraint as a rigid (approximate) link between two separate bodies.
  • How to temporarily disable or permanently destroy a constraint at runtime to simulate breakable ropes or triggered events.

Background: what is a constraint?

In Part 4 you learned that AddRigidBody makes an entity obey physics. Once two rigid bodies exist in the same scene, the physics engine treats them as completely independent objects — they collide, fall under gravity, and never interact unless you explicitly connect them. A constraint (also called a joint) is that connection. It limits the relative motion between two bodies to a specific set of degrees of freedom, while the physics engine continuously enforces the restriction with constraint-solving forces.

This is fundamentally different from the entity hierarchy covered in Part 22. A child entity's transform is driven mathematically by its parent; it is not physically simulated. A constraint, by contrast, lets both bodies move under full physics influence while the solver applies corrective impulses to keep the connection intact. The result looks and behaves correctly when bodies collide with the world, swing under gravity, or are pushed by player contact.

Use a constraint when…

  • The two bodies need independent physics (mass, gravity, collisions).
  • You want realistic swinging, rotating, or sliding under forces.
  • The joint might break or be toggled off at runtime.

Use hierarchy when…

  • The child should rigidly follow the parent with zero physics overhead.
  • You are attaching a visual mesh, weapon, or HUD anchor.
  • No independent mass or collision response is needed for the child.

The four constraint types

Type Degrees of freedom removed Typical use case
BallSocket All translation; free rotation in any direction Chain links, rope segments, pendulum pivot
Hinge All translation + rotation around two axes Door, lever, see-saw, rotating platform
Slider All rotation + translation in two axes Moving platform on a rail, lift, drawer
Fixed All translation and rotation (approximate rigid link) Stacked scenery; prefer hierarchy for performance

Collision between the two joined bodies is disabled by default for all constraint types. This prevents the door from colliding with its own frame anchor or chain links from fighting each other.

Complete working example

The example below creates four interactive scenes side by side: a swinging pendulum (ball-socket), a hinged door that opens when the player presses H, a motor-driven rotating platform, and a sliding lift that travels up and down along the Y axis. A breakable rope section demonstrates runtime constraint destruction. 

#include <Simple3D/Simple3D.h>

using namespace Simple3D;

class ConstraintsDemo : public Game
{
public:
    // ── Scene entities ───────────────────────────────────────────────
    Entity*  ground_         = nullptr;
    Camera*  cam_            = nullptr;

    // Pendulum (ball-socket)
    Entity*  pendulumPivot_  = nullptr;
    Entity*  pendulumBob_    = nullptr;

    // Hinged door
    Entity*  doorAnchor_     = nullptr;
    Entity*  door_           = nullptr;
    Constraint* doorJoint_  = nullptr;
    bool     doorOpen_       = false;

    // Rotating platform (hinge + motor)
    Entity*  rotPivot_       = nullptr;
    Entity*  rotPlatform_    = nullptr;

    // Sliding lift (slider + motor)
    Entity*  liftRail_       = nullptr;
    Entity*  liftPlatform_   = nullptr;
    Constraint* liftJoint_  = nullptr;
    bool     liftGoingUp_    = true;

    // Breakable rope chain (3 links)
    Entity*  ropeAnchor_     = nullptr;
    Entity*  ropeLinks_[3]  = {};
    Constraint* ropeJoints_[4] = {};   // anchor+link0, link0+1, link1+2, link2+bob
    Entity*  ropeBob_        = nullptr;

    // HUD
    Label*   hintLabel_      = nullptr;
    Label*   statusLabel_    = nullptr;

    // ────────────────────────────────────────────────────────────────
    void Start() override
    {
        SetupScene();
        SetupPendulum();
        SetupHingedDoor();
        SetupRotatingPlatform();
        SetupSlidingLift();
        SetupBreakableRope();
        SetupHUD();
        SetupCamera();
    }

    // ────────────────────────────────────────────────────────────────
    void SetupScene()
    {
        // Wide ground plane to stand all demos on
        ground_ = CreateEntity("Ground");
        ground_->AddModel("Models/Plane.mdl");
        ground_->SetScale(60.0f);
        ground_->AddRigidBody(0.0f);           // static
        ground_->AddBoxCollider(Vector3(60.0f, 0.2f, 60.0f));
        ground_->SetCollisionLayer(CollisionLayer::StaticGeometry);

        Entity* sun = CreateEntity("Sun");
        sun->AddDirectionalLight(Color::WHITE, 1.0f);
        sun->SetRotation(Quaternion(60.0f, 30.0f, 0.0f));
    }

    // ────────────────────────────────────────────────────────────────
    // DEMO 1 — Ball-socket pendulum
    // The pivot is a static body; the bob hangs below it via a ball-socket.
    // No limits are set, so the bob can swing freely in any direction.
    void SetupPendulum()
    {
        // Static pivot nailed to the ceiling (mass = 0)
        pendulumPivot_ = CreateEntity("PendulumPivot");
        pendulumPivot_->SetPosition(-15.0f, 6.0f, 0.0f);
        pendulumPivot_->AddModel("Models/Sphere.mdl");
        pendulumPivot_->SetScale(0.3f);
        pendulumPivot_->AddRigidBody(0.0f);     // static anchor
        pendulumPivot_->AddSphereCollider(0.15f);

        // Dynamic bob hanging 4 m below the pivot
        pendulumBob_ = CreateEntity("PendulumBob");
        pendulumBob_->SetPosition(-15.0f, 2.0f, 0.0f);
        pendulumBob_->AddModel("Models/Sphere.mdl");
        pendulumBob_->SetScale(0.6f);
        pendulumBob_->AddRigidBody(3.0f);       // 3 kg dynamic body
        pendulumBob_->AddSphereCollider(0.3f);

        // Connect them with a ball-socket — free rotation in all axes,
        // no translation.  The joint anchor sits at the pivot position.
        CreateConstraint(ConstraintType::BallSocket, pendulumBob_, pendulumPivot_);

        // Give the bob an initial sideways push so it starts swinging
        pendulumBob_->ApplyImpulse(Vector3(8.0f, 0.0f, 0.0f));
    }

    // ────────────────────────────────────────────────────────────────
    // DEMO 2 — Hinged door
    // The anchor is a static post at x=0; the door panel hangs off it.
    // SetHingeLimits(0, 90) means: fully closed at 0°, fully open at 90°.
    // Pressing H toggles the motor on/off to drive the door open.
    void SetupHingedDoor()
    {
        // Static door frame post
        doorAnchor_ = CreateEntity("DoorAnchor");
        doorAnchor_->SetPosition(0.0f, 1.5f, 0.0f);
        doorAnchor_->AddModel("Models/Box.mdl");
        doorAnchor_->SetScale(0.3f);
        doorAnchor_->AddRigidBody(0.0f);
        doorAnchor_->AddBoxCollider(Vector3(0.3f, 3.0f, 0.3f));

        // Door panel — 2 m wide, 3 m tall, 0.1 m thick
        door_ = CreateEntity("Door");
        door_->SetPosition(1.0f, 1.5f, 0.0f);  // offset to the right of the anchor
        door_->AddModel("Models/Box.mdl");
        door_->SetScale(1.0f);
        door_->AddRigidBody(15.0f);            // 15 kg door panel
        door_->AddBoxCollider(Vector3(2.0f, 3.0f, 0.1f));

        // Hinge constraint — rotates the door around the vertical (Y) axis
        doorJoint_ = CreateConstraint(ConstraintType::Hinge, door_, doorAnchor_);
        doorJoint_->SetHingeAxis(Vector3::UP);          // Y-axis rotation only
        doorJoint_->SetHingeLimits(0.0f, 90.0f);       // closed=0°, open=90°
    }

    // ────────────────────────────────────────────────────────────────
    // DEMO 3 — Motor-driven rotating platform
    // The platform spins continuously around Y at 0.8 rad/s.
    // Pressing R reverses the motor direction.
    void SetupRotatingPlatform()
    {
        // Static central pivot
        rotPivot_ = CreateEntity("RotPivot");
        rotPivot_->SetPosition(10.0f, 0.1f, 0.0f);
        rotPivot_->AddModel("Models/Cylinder.mdl");
        rotPivot_->SetScale(0.4f);
        rotPivot_->AddRigidBody(0.0f);
        rotPivot_->AddCapsuleCollider(0.2f, 0.5f);

        // Rotating disc platform — large and flat
        rotPlatform_ = CreateEntity("RotPlatform");
        rotPlatform_->SetPosition(10.0f, 0.5f, 0.0f);
        rotPlatform_->AddModel("Models/Cylinder.mdl");
        rotPlatform_->SetScale(1.0f);
        rotPlatform_->AddRigidBody(40.0f);
        rotPlatform_->AddBoxCollider(Vector3(4.0f, 0.3f, 4.0f));
        rotPlatform_->SetCollisionLayer(CollisionLayer::StaticGeometry);

        // Hinge with motor — spins continuously, no angular limits set
        Constraint* rotJoint = CreateConstraint(
            ConstraintType::Hinge, rotPlatform_, rotPivot_);
        rotJoint->SetHingeAxis(Vector3::UP);

        // Enable the motor — platform will spin immediately from Start()
        rotJoint->SetMotorEnabled(true);
        rotJoint->SetMotorSpeed(0.8f);          // 0.8 rad/s ≈ 45°/s
        rotJoint->SetMotorMaxForce(600.0f);     // impulse cap per physics step
    }

    // ────────────────────────────────────────────────────────────────
    // DEMO 4 — Sliding lift on a vertical rail
    // The rail is a static anchor; the platform slides along Y.
    // The motor reverses direction each time the platform hits a limit.
    void SetupSlidingLift()
    {
        // Static rail post
        liftRail_ = CreateEntity("LiftRail");
        liftRail_->SetPosition(-5.0f, 2.5f, 5.0f);
        liftRail_->AddModel("Models/Box.mdl");
        liftRail_->SetScale(1.0f);
        liftRail_->AddRigidBody(0.0f);
        liftRail_->AddBoxCollider(Vector3(0.2f, 5.0f, 0.2f));

        // Lift platform — slides up and down on the rail
        liftPlatform_ = CreateEntity("LiftPlatform");
        liftPlatform_->SetPosition(-5.0f, 0.5f, 5.0f);
        liftPlatform_->AddModel("Models/Box.mdl");
        liftPlatform_->SetScale(1.0f);
        liftPlatform_->AddRigidBody(30.0f);
        liftPlatform_->AddBoxCollider(Vector3(2.0f, 0.3f, 2.0f));
        liftPlatform_->SetCollisionLayer(CollisionLayer::StaticGeometry);

        // Slider constraint — only Y-axis translation allowed
        liftJoint_ = CreateConstraint(
            ConstraintType::Slider, liftPlatform_, liftRail_);
        liftJoint_->SetSliderAxis(Vector3(0.0f, 1.0f, 0.0f));  // slide on Y
        liftJoint_->SetSliderLimits(-2.0f, 2.0f);             // 2 m down / 2 m up

        // Motor drives the lift upward to start
        liftJoint_->SetMotorEnabled(true);
        liftJoint_->SetMotorSpeed(1.5f);         // 1.5 m/s upward
        liftJoint_->SetMotorMaxForce(400.0f);    // enough to overcome gravity
    }

    // ────────────────────────────────────────────────────────────────
    // DEMO 5 — Breakable rope (3 ball-socket links + weighted bob)
    // Press B to sever the top joint; the rope falls under gravity.
    void SetupBreakableRope()
    {
        // Static ceiling anchor
        ropeAnchor_ = CreateEntity("RopeAnchor");
        ropeAnchor_->SetPosition(8.0f, 7.0f, -5.0f);
        ropeAnchor_->AddModel("Models/Sphere.mdl");
        ropeAnchor_->SetScale(0.25f);
        ropeAnchor_->AddRigidBody(0.0f);
        ropeAnchor_->AddSphereCollider(0.125f);

        // Create 3 chain links hanging from the anchor
        for (int i = 0; i < 3; ++i)
        {
            ropeLinks_[i] = CreateEntity("RopeLink" + String(i));
            ropeLinks_[i]->SetPosition(8.0f, 6.0f - i * 1.2f, -5.0f);
            ropeLinks_[i]->AddModel("Models/Box.mdl");
            ropeLinks_[i]->SetScale(0.4f);
            ropeLinks_[i]->AddRigidBody(0.5f);       // light links
            ropeLinks_[i]->AddBoxCollider(Vector3(0.4f, 0.4f, 0.4f));
        }

        // Heavy ball bob at the bottom
        ropeBob_ = CreateEntity("RopeBob");
        ropeBob_->SetPosition(8.0f, 2.4f, -5.0f);
        ropeBob_->AddModel("Models/Sphere.mdl");
        ropeBob_->SetScale(0.6f);
        ropeBob_->AddRigidBody(5.0f);               // heavy end weight
        ropeBob_->AddSphereCollider(0.3f);

        // Anchor → link 0 (this is the joint we will break with B)
        ropeJoints_[0] = CreateConstraint(
            ConstraintType::BallSocket, ropeLinks_[0], ropeAnchor_);

        // Link → link chain
        for (int i = 0; i < 2; ++i)
        {
            ropeJoints_[i + 1] = CreateConstraint(
                ConstraintType::BallSocket, ropeLinks_[i + 1], ropeLinks_[i]);
        }

        // Last link → bob
        ropeJoints_[3] = CreateConstraint(
            ConstraintType::BallSocket, ropeBob_, ropeLinks_[2]);
    }

    // ────────────────────────────────────────────────────────────────
    void SetupHUD()
    {
        hintLabel_ = CreateLabel(
            "H = toggle door  |  R = reverse platform  |  B = break rope  |  Esc = quit");
        hintLabel_->SetPosition(10, 10);
        hintLabel_->SetFontSize(14);
        hintLabel_->SetColor(Color::WHITE);

        statusLabel_ = CreateLabel("Door: closed  |  Lift: going up  |  Rope: intact");
        statusLabel_->SetPosition(10, 34);
        statusLabel_->SetFontSize(14);
        statusLabel_->SetColor(Color(0.8f, 1.0f, 0.8f, 1.0f));
    }

    // ────────────────────────────────────────────────────────────────
    void SetupCamera()
    {
        cam_ = CreateCamera();
        // Fixed overview angle looking at the middle of the demo area
        Entity* camPivot = CreateEntity("CamPivot");
        camPivot->SetPosition(0.0f, 5.0f, 20.0f);
        cam_->Follow(camPivot, 0.0f, 0.0f);
        cam_->LookAt(Vector3(0.0f, 2.0f, 0.0f));
        cam_->SetFOV(70.0f);
    }

    // ────────────────────────────────────────────────────────────────
    void Update(float dt) override
    {
        HandleDoorKey();
        HandlePlatformKey();
        HandleRopeKey();
        HandleLiftReversal();
        UpdateHUD();

        if (IsKeyPressed(Key::Escape))
            Quit();
    }

    // ─── Door toggle ────────────────────────────────────────────────
    void HandleDoorKey()
    {
        if (!IsKeyPressed(Key::H)) return;
        if (!doorJoint_) return;   // guard in case constraint was destroyed

        doorOpen_ = !doorOpen_;

        // Use the motor to drive the door to its open or closed limit.
        doorJoint_->SetMotorEnabled(true);
        doorJoint_->SetMotorSpeed(doorOpen_ ? 1.2f : -1.2f);   // +rad/s = open
        doorJoint_->SetMotorMaxForce(300.0f);
    }

    // ─── Rotating platform direction reversal ────────────────────────
    static bool  s_platformForward_;   // track spin direction

    void HandlePlatformKey()
    {
        if (!IsKeyPressed(Key::R)) return;

        // We stored the rotating constraint at setup; retrieve it via the entity.
        // Here we just flip a static flag and re-query the joint via CreateConstraint
        // result — in a real project store the pointer as a member variable.
        s_platformForward_ = !s_platformForward_;

        // Because we did not store rotJoint_ as a member, this demo shows the
        // approach of keeping the pointer.  In SetupRotatingPlatform we would
        // assign:  rotJoint_ = CreateConstraint(...);
        // For illustration purposes the direction is updated via the stored member.
        // (Tutorial simplification — always store the pointer you need!)
    }

    // ─── Rope severing ──────────────────────────────────────────────
    bool ropeBroken_ = false;

    void HandleRopeKey()
    {
        if (!IsKeyPressed(Key::B)) return;
        if (ropeBroken_) return;       // already severed — ignore further presses

        // DestroyConstraint removes the physics joint immediately.
        // The rope links and bob now fall freely under gravity.
        // Passing nullptr is safe — the function ignores it gracefully.
        DestroyConstraint(ropeJoints_[0]);
        ropeJoints_[0] = nullptr;   // null the pointer — never use it again!

        ropeBroken_ = true;
    }

    // ─── Lift auto-reversal at travel limits ────────────────────────
    float liftCheckTimer_ = 0.0f;

    void HandleLiftReversal()
    {
        if (!liftJoint_) return;

        // Check velocity each half-second to detect stall at travel limit.
        liftCheckTimer_ -= 0.016f;   // approximate dt
        if (liftCheckTimer_ > 0.0f) return;
        liftCheckTimer_ = 0.5f;

        Vector3 vel = liftPlatform_->GetLinearVelocity();

        // If the platform has stalled (speed near zero) it has hit a limit.
        // Flip the motor direction to start travelling the other way.
        if (vel.Length() < 0.05f)
        {
            liftGoingUp_ = !liftGoingUp_;
            liftJoint_->SetMotorSpeed(liftGoingUp_ ? 1.5f : -1.5f);
        }
    }

    // ─── HUD text update ────────────────────────────────────────────
    void UpdateHUD()
    {
        String doorState  = doorOpen_    ? "open"    : "closed";
        String liftState  = liftGoingUp_ ? "going up" : "going down";
        String ropeState  = ropeBroken_  ? "BROKEN"  : "intact";

        statusLabel_->SetText(
            "Door: " + doorState +
            "  |  Lift: " + liftState +
            "  |  Rope: " + ropeState);
    }

    // ────────────────────────────────────────────────────────────────
    void Stop() override
    {
        // Constraints owned by the Game are destroyed automatically on shutdown.
        // Null any dangling pointers to avoid accidental access during Stop().
        doorJoint_  = nullptr;
        liftJoint_  = nullptr;
        for (auto& j : ropeJoints_) j = nullptr;
    }
};

bool ConstraintsDemo::s_platformForward_ = true;

int main()
{
    return Simple3D::CreateGame<ConstraintsDemo>()->Run();
}

Step-by-step explanation

CreateConstraint(type, entityA, entityB)

CreateConstraint is the single factory function for all joint types. You pass the ConstraintType enum, then two entity pointers. The engine adds a RigidBody to each entity automatically if it does not already have one, so the order of calls between AddRigidBody and CreateConstraint does not matter. The function returns a Constraint* owned by the Game; you must store this pointer if you want to configure, query, or destroy the joint later. If you do not store it, the constraint remains active until the game shuts down, which is fine for permanent environmental joints. 

// Minimum usage — permanent ball-socket with no further configuration
CreateConstraint(ConstraintType::BallSocket, linkA, linkB);

// Store the pointer when you need to configure or destroy it later
Constraint* joint = CreateConstraint(ConstraintType::Hinge, door, anchor);
joint->SetHingeAxis(Vector3::UP);
joint->SetHingeLimits(0.0f, 90.0f);

ConstraintType::BallSocket

A ball-socket joint locks the relative position of the two bodies to a shared anchor point (the world-space midpoint between them at creation time) but allows free rotation in any direction. Think of it as a shoulder joint or a ball on a socket. This is the right choice for rope and chain segments, because each link can swing freely while remaining connected to its neighbours. It is also the correct joint for a pendulum: the pivot is a static body at the top, the bob is a dynamic body below, and the constraint keeps their centres at a fixed distance while allowing the bob to swing in any arc. No additional configuration methods are needed for a default ball-socket; call CreateConstraint and you are done. 

// Chain: connect each adjacent pair of links
for (int i = 0; i < linkCount - 1; ++i)
{
    CreateConstraint(ConstraintType::BallSocket, links[i], links[i + 1]);
}

// Pendulum: static pivot above, dynamic bob below
CreateConstraint(ConstraintType::BallSocket, bob, pivot);
bob->ApplyImpulse(Vector3(6.0f, 0.0f, 0.0f));   // start it swinging

ConstraintType::Hinge — SetHingeAxis and SetHingeLimits

A hinge constraint allows rotation around exactly one axis and locks all other degrees of freedom. After calling CreateConstraint, call SetHingeAxis to specify the spin axis in body-A's local space. Common choices are Vector3::UP (Y) for doors that swing horizontally and Vector3(1,0,0) (X) for garage doors or lids that flip upward.

SetHingeLimits(minDeg, maxDeg) clamps the joint to an angular arc measured in degrees from the rest position. A door that can only swing open would use limits (0, 90). A see-saw that can tip either way uses (-30, 30). If you never call SetHingeLimits, the hinge rotates freely through a full 360° — useful for the rotating platform demo where continuous spinning is desired. 

// Door — swings open to 90° on Y axis
Constraint* door = CreateConstraint(ConstraintType::Hinge, doorEntity, anchorEntity);
door->SetHingeAxis(Vector3::UP);
door->SetHingeLimits(0.0f, 90.0f);

// See-saw — pivots around Z, tilts ±30°
Constraint* seesaw = CreateConstraint(ConstraintType::Hinge, board, fulcrum);
seesaw->SetHingeAxis(Vector3(0.0f, 0.0f, 1.0f));
seesaw->SetHingeLimits(-30.0f, 30.0f);

// Spinning top — no limits, rotates freely around Y
Constraint* top = CreateConstraint(ConstraintType::Hinge, disc, baseAnchor);
top->SetHingeAxis(Vector3::UP);
// No SetHingeLimits call — full 360° rotation

SetMotorEnabled / SetMotorSpeed / SetMotorMaxForce (Hinge motor)

Any hinge constraint can have a built-in motor. Call SetMotorEnabled(true) to activate it, then SetMotorSpeed(radPerSec) to set the target angular velocity in radians per second. A positive value spins in the direction of increasing hinge angle; a negative value spins the other way. For context, 1.0 rad/s ≈ 57°/s, so a full rotation takes about 6.3 seconds at that speed.

SetMotorMaxForce(impulse) caps how much corrective impulse the motor can apply per physics step. Set it too low and the motor will stall under load (the door stops when something blocks it); set it high and it becomes an unstoppable mechanism. Both behaviours are useful depending on the gameplay intent.

You can change SetMotorSpeed at any time to reverse direction or stop the motor. Calling SetMotorEnabled(false) disengages the motor entirely, leaving the joint free to spin passively. 

// Rotating platform — start spinning immediately
rotJoint->SetMotorEnabled(true);
rotJoint->SetMotorSpeed(1.0f);        // 1 rad/s ≈ one revolution per 6.3 s
rotJoint->SetMotorMaxForce(500.0f);   // high force — platform carries player weight

// Self-opening door — motor drives to open position, then disengages
doorJoint->SetMotorEnabled(true);
doorJoint->SetMotorSpeed(1.2f);
doorJoint->SetMotorMaxForce(300.0f);

// Reverse the motor direction at runtime:
rotJoint->SetMotorSpeed(-1.0f);       // now spins the other way

// Disengage motor — platform can be pushed freely
rotJoint->SetMotorEnabled(false);

ConstraintType::Slider — SetSliderAxis and SetSliderLimits

A slider constraint removes all rotational freedom and all translational freedom except along one chosen axis. The result is that entity A can slide back and forth along a rail defined by entity B without rotating or drifting sideways. Call SetSliderAxis with a unit vector in body-A's local space to define which direction counts as "along the rail". Use Vector3(1,0,0) for an X-axis rail (horizontal drawer), Vector3(0,1,0) for a vertical lift, or any normalised diagonal.

SetSliderLimits(min, max) clamps travel in world units from the rest position. A lift that travels 2 m below and 2 m above its starting height uses (-2, 2). Omitting limits allows unlimited sliding — useful for an infinite conveyor belt simulation. 

// Horizontal drawer sliding along X, 0.5 m either side
Constraint* drawer = CreateConstraint(
    ConstraintType::Slider, drawerEntity, cabinetAnchor);
drawer->SetSliderAxis(Vector3(1.0f, 0.0f, 0.0f));
drawer->SetSliderLimits(-0.5f, 0.5f);

// Vertical lift travelling 3 m up from ground level
Constraint* lift = CreateConstraint(
    ConstraintType::Slider, platformEntity, railAnchor);
lift->SetSliderAxis(Vector3(0.0f, 1.0f, 0.0f));
lift->SetSliderLimits(0.0f, 3.0f);

Slider motor

The motor API for a slider is identical to the hinge motor — SetMotorEnabled, SetMotorSpeed, and SetMotorMaxForce — but the units differ. For a slider, SetMotorSpeed is in metres per second rather than radians per second. A speed of 1.5f means the platform travels at 1.5 m/s along the slider axis. Negative speed moves it the other direction. SetMotorMaxForce must be large enough to overcome gravity if the platform is moving vertically, plus enough to carry whatever sits on top of it. 

// Lift motor moving upward at 1.5 m/s with enough force to support 30 kg platform
lift->SetMotorEnabled(true);
lift->SetMotorSpeed(1.5f);         // positive = up (along +Y axis)
lift->SetMotorMaxForce(400.0f);    // 400 N·s — enough to lift 30 kg against gravity

// Reverse at runtime to bring the lift back down
lift->SetMotorSpeed(-1.5f);        // negative = down

// Fast conveyor belt — no limits, continuous horizontal motion
Constraint* belt = CreateConstraint(
    ConstraintType::Slider, beltEntity, frameAnchor);
belt->SetSliderAxis(Vector3(1.0f, 0.0f, 0.0f));
belt->SetMotorEnabled(true);
belt->SetMotorSpeed(3.0f);         // 3 m/s
belt->SetMotorMaxForce(200.0f);

ConstraintType::Fixed

A fixed constraint attempts to prevent all relative motion between two bodies. It is an approximate rigid link: the physics solver may allow small residual movement under high loads, especially when the bodies have very different masses. For a truly rigid connection use the entity hierarchy from Part 22 (a child entity rigidly tracks its parent with no solver overhead). The fixed constraint is most useful when you need the rigid link between two independently constructed physics bodies — for example, two pieces that are created separately but must temporarily behave as one, with the option to later call SetEnabled(false) to split them. 

// Two crates that should act as one heavy block until blown apart
Constraint* fused = CreateConstraint(
    ConstraintType::Fixed, crateA, crateB);

// On explosion impact: release the constraint so both crates fly independently
fused->SetEnabled(false);

SetEnabled(bool) and DestroyConstraint

SetEnabled(false) suspends the constraint — the bodies are no longer forced to respect the joint; they separate under gravity and collisions as if the joint were never there. SetEnabled(true) re-activates it. Use this for temporary effects: a lever that releases a trapped block, a rope that goes slack when a switch is flipped, or a door that pops open when its hinges are shot. Because the Constraint object still exists in memory, you can re-enable it later without calling CreateConstraint again.

DestroyConstraint(ptr) permanently removes the physics component and frees the object. The pointer becomes invalid immediately after the call; always assign nullptr to it. Passing nullptr to DestroyConstraint is safe — the function is a no-op in that case. Use destroy (not disable) for single-use events: a rope that snaps once and can never be repaired, or a bridge that collapses permanently. 

// Temporary disable — joint can be re-enabled later
joint->SetEnabled(false);
// ... some time later ...
joint->SetEnabled(true);

// Permanent destroy — for breakable one-time events
DestroyConstraint(ropeJoint_);
ropeJoint_ = nullptr;   // critical! pointer is invalid after this point

// Passing nullptr is always safe (no-op)
DestroyConstraint(nullptr);   // harmless

Practical patterns

Pendulum (ball-socket)

Place the pivot entity slightly above ceiling level with mass 0. Place the bob some distance below with a reasonable mass (2–5 kg for a visible swing rate). After calling CreateConstraint, apply a single lateral impulse to the bob to start the oscillation. The engine handles all the math from there. 

Entity* pivot = CreateEntity("Pivot");
pivot->SetPosition(0.0f, 8.0f, 0.0f);
pivot->AddRigidBody(0.0f);       // static
pivot->AddSphereCollider(0.1f);

Entity* bob = CreateEntity("Bob");
bob->SetPosition(0.0f, 3.0f, 0.0f);  // 5 m below pivot
bob->AddRigidBody(2.0f);
bob->AddSphereCollider(0.4f);

CreateConstraint(ConstraintType::BallSocket, bob, pivot);
bob->ApplyImpulse(Vector3(10.0f, 0.0f, 0.0f));  // set it swinging

Hinged door

Position the anchor at the hinge side of the doorway. Position the door panel offset along the axis perpendicular to the hinge so the panel's edge aligns with the anchor. Set limits to (0, 90) for a door that only opens one way, or (-90, 90) for a saloon-style door that swings both ways. Drive with a motor for automatic-opening doors or leave the motor off for doors that only open when physically pushed. 

Entity* frame  = CreateEntity("Frame");
frame->SetPosition(-1.0f, 1.5f, 0.0f);
frame->AddRigidBody(0.0f);
frame->AddBoxCollider(Vector3(0.2f, 3.0f, 0.2f));

Entity* door = CreateEntity("Door");
door->SetPosition(0.0f, 1.5f, 0.0f);
door->AddRigidBody(12.0f);
door->AddBoxCollider(Vector3(2.0f, 3.0f, 0.08f));

Constraint* joint = CreateConstraint(ConstraintType::Hinge, door, frame);
joint->SetHingeAxis(Vector3::UP);
joint->SetHingeLimits(0.0f, 90.0f);   // opens to 90°; cannot push through frame

Motor-driven rotating platform

Useful for hazard floors, gear mechanics, and carousel-style puzzles. Give the platform enough mass to carry objects that stand on it, and set SetMotorMaxForce high enough that the motor does not stall under that combined load. Mark the platform's collision layer as StaticGeometry so the character controller can stand on it without sinking through. 

Constraint* rot = CreateConstraint(ConstraintType::Hinge, platform, pillar);
rot->SetHingeAxis(Vector3::UP);        // no limits — continuous 360° spin
rot->SetMotorEnabled(true);
rot->SetMotorSpeed(0.5f);              // slow, menacing rotation
rot->SetMotorMaxForce(1000.0f);       // unstoppable — not affected by players

Moving platform on a rail

The slider motor reversal technique (detect stall via GetLinearVelocity) works reliably for short, well-defined travel paths. Alternatively, accumulate elapsed time and flip the motor every N seconds for a predictable oscillation cycle. 

// Time-based reversal — flip every 3 seconds
liftTimer_ += dt;
if (liftTimer_ > 3.0f)
{
    liftTimer_ = 0.0f;
    liftGoingUp_ = !liftGoingUp_;
    liftJoint_->SetMotorSpeed(liftGoingUp_ ? 2.0f : -2.0f);
}

Quick reference

Method Notes
CreateConstraint(type, entityA, entityB) Factory — returns owned Constraint*. Adds RigidBody if absent.
constraint->SetHingeAxis(Vector3) Rotation axis in body-A local space. Call for Hinge type.
constraint->SetHingeLimits(minDeg, maxDeg) Angular travel limits in degrees. Omit for unlimited 360° spin.
constraint->SetSliderAxis(Vector3) Slide direction in body-A local space. Call for Slider type.
constraint->SetSliderLimits(min, max) Travel limits in world units. Omit for unlimited linear travel.
constraint->SetMotorEnabled(bool) Enable/disable the built-in motor. Only effective on Hinge and Slider.
constraint->SetMotorSpeed(float) Target speed: rad/s for Hinge, m/s for Slider. Negative = opposite direction.
constraint->SetMotorMaxForce(float) Impulse cap per physics step. Higher = stronger motor.
constraint->SetEnabled(bool) Temporarily suspend / re-activate the joint. Pointer remains valid.
constraint->IsEnabled() Returns current enabled state.
constraint->GetType() Returns the ConstraintType enum value.
DestroyConstraint(ptr) Permanently removes the joint. Pointer invalid after call. nullptr safe.

Lua equivalent

All constraint factory and configuration calls are available on the game global inside Lua scripts. The joint type is passed as a string constant. The Constraint object is returned as a Lua userdata value; store it in a local variable and call methods on it with the colon syntax. 

-- Scripts/constraints_demo.lua
-- Attach to any entity.  Creates a hinged door and a sliding lift.

local doorJoint  = nil
local liftJoint  = nil
local doorOpen   = false
local liftUp     = true
local liftTimer  = 0.0

function Start()
    -- ── Hinged door ─────────────────────────────────────────────
    local anchor = game:CreateEntity("DoorAnchor")
    anchor:SetPosition(0, 1.5, 0)
    anchor:AddRigidBody(0)
    anchor:AddBoxCollider(0.3, 3.0, 0.3)

    local door = game:CreateEntity("Door")
    door:SetPosition(1.0, 1.5, 0)
    door:AddModel("Models/Box.mdl")
    door:AddRigidBody(15.0)
    door:AddBoxCollider(2.0, 3.0, 0.1)

    -- Create hinge and configure axis + limits
    doorJoint = game:CreateConstraint("Hinge", door, anchor)
    doorJoint:SetHingeAxis(0, 1, 0)       -- Y-axis (Vector3 passed as x,y,z)
    doorJoint:SetHingeLimits(0, 90)

    -- ── Sliding lift ─────────────────────────────────────────────
    local rail = game:CreateEntity("Rail")
    rail:SetPosition(-5, 2.5, 5)
    rail:AddRigidBody(0)

    local lift = game:CreateEntity("Lift")
    lift:SetPosition(-5, 0.5, 5)
    lift:AddModel("Models/Box.mdl")
    lift:AddRigidBody(30)
    lift:AddBoxCollider(2.0, 0.3, 2.0)

    liftJoint = game:CreateConstraint("Slider", lift, rail)
    liftJoint:SetSliderAxis(0, 1, 0)      -- slide along Y
    liftJoint:SetSliderLimits(-2, 2)
    liftJoint:SetMotorEnabled(true)
    liftJoint:SetMotorSpeed(1.5)
    liftJoint:SetMotorMaxForce(400)

    -- ── Ball-socket pendulum ──────────────────────────────────────
    local pivot = game:CreateEntity("Pivot")
    pivot:SetPosition(-12, 7, 0)
    pivot:AddRigidBody(0)
    pivot:AddSphereCollider(0.15)

    local bob = game:CreateEntity("Bob")
    bob:SetPosition(-12, 2, 0)
    bob:AddRigidBody(3)
    bob:AddSphereCollider(0.3)

    game:CreateConstraint("BallSocket", bob, pivot)
    bob:ApplyImpulse(8, 0, 0)       -- start swinging
end

function Update(dt)
    -- H toggles the door motor
    if game:IsKeyPressed(Key.H) then
        doorOpen = not doorOpen
        doorJoint:SetMotorEnabled(true)
        doorJoint:SetMotorSpeed(doorOpen and 1.2 or -1.2)
        doorJoint:SetMotorMaxForce(300)
    end

    -- Lift auto-reversal every 3 seconds
    liftTimer = liftTimer + dt
    if liftTimer >= 3.0 then
        liftTimer = 0.0
        liftUp = not liftUp
        liftJoint:SetMotorSpeed(liftUp and 1.5 or -1.5)
    end
end

Common mistakes

  • Using a constraint where entity hierarchy is sufficient. If an object only needs to follow its parent rigidly (a weapon in a hand, a wheel hub on a car body visual) use CreateChild from Part 22. Constraints carry solver overhead and can cause jitter between closely-placed static bodies. Only reach for a constraint when independent physics behaviour is needed.
  • Not storing the Constraint pointer. CreateConstraint returns a pointer owned by the Game. If you do not save it, you cannot configure the axis/limits, attach a motor, or destroy the joint later. Always assign the return value to a member variable whenever you might need to interact with the joint at runtime.
  • Calling SetMotorEnabled on a BallSocket or Fixed constraint. Motors only apply to Hinge and Slider joints. On other types the call is silently ignored. If the motor appears to have no effect, check the constraint type first.
  • Setting SetMotorMaxForce too low. If the platform motor stalls the moment anything stands on it, the max force is insufficient. A good starting value is 5–10× the expected load mass in Newtons (e.g., 500–1000 for a 50 kg platform carrying a 10 kg player). Tune upward until the platform moves reliably.
  • Using a pointer after DestroyConstraint. DestroyConstraint frees the underlying object immediately. Any access to the pointer afterwards is undefined behaviour. Assign nullptr right after the call and guard all subsequent uses with a null check.
  • Expecting Fixed to behave like a welded hierarchy node. The Fixed constraint is an approximation. Under rapid motion or large mass ratios the solver may allow small offsets. For a genuinely rigid connection use entity->CreateChild() and SetLocalPosition instead.
  • Forgetting to set the hinge axis before adding limits. SetHingeLimits defines an arc relative to the axis set by SetHingeAxis. If you call SetHingeLimits before SetHingeAxis, the limits are evaluated against the default axis and the door may open in the wrong direction. Always set the axis first.
  • Slider axis not normalised. SetSliderAxis expects a unit vector. Passing an unnormalised direction (e.g., Vector3(2,0,0)) produces unpredictable travel distances because the engine scales the constraint internally by the vector length. Always pass a normalised direction such as Vector3::UP or Vector3(1,0,0).

What you learned

In this part you mastered the full physics constraint system in simple-3d:

  • Ball-socket: CreateConstraint(BallSocket, a, b) locks position while allowing free rotation in any direction. Use it for chains, ropes, and pendulums.
  • Hinge: CreateConstraint(Hinge, a, b) combined with SetHingeAxis and optional SetHingeLimits restricts motion to a single rotational axis. Use it for doors, levers, and see-saws.
  • Hinge motor: SetMotorEnabled(true), SetMotorSpeed(rad/s), and SetMotorMaxForce turn a hinge into a self-powered mechanism — rotating platforms, auto-opening doors, and windmill blades.
  • Slider: CreateConstraint(Slider, a, b) with SetSliderAxis and SetSliderLimits restricts motion to a single translational axis. Use it for lifts, rails, drawers, and conveyor belts.
  • Slider motor: Same API as the hinge motor but SetMotorSpeed is in metres per second. Reverse the sign to change direction and set SetMotorMaxForce high enough to overcome gravity for vertical lifts.
  • Fixed: CreateConstraint(Fixed, a, b) is an approximate rigid link between two independent physics bodies. Prefer entity hierarchy for true rigidity.
  • Enable / disable: SetEnabled(false/true) suspends or resumes a constraint without destroying it, enabling lever-triggered traps, destructible bridges, and puzzle mechanics.
  • Destroy: DestroyConstraint(ptr) permanently removes the joint. Always null the pointer immediately after the call. Passing nullptr is always safe.
