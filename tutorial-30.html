Background: networking in simple-3d

simple-3d provides client/server multiplayer through ENet, a thin UDP library that adds reliable and sequenced delivery on top of raw datagrams. The NetworkManager class wraps ENet's connection primitives and gives you callbacks for connection events and incoming messages.

One crucial point distinguishes simple-3d from engines that have built-in network replication: entity replication is not automatic. No entity position, velocity, model, or component state is ever sent to another machine on your behalf. Every piece of game state that clients need to know about must be serialised into a byte buffer and sent explicitly using the messaging API. This gives you complete control over bandwidth and update frequency, at the cost of writing the serialisation code yourself.

Server role

The server is authoritative. It runs the physics simulation, validates player input, and broadcasts authoritative state to all connected clients every frame (or at a fixed network tick rate). One instance of your game binary calls StartServer().

Client role

Each client connects with Connect(), sends local player input to the server each frame, and receives authoritative state updates to render. Remote entities on the client are moved by applying the received positions rather than by local physics.

A single binary can decide its role at startup based on a command-line argument, a configuration file, or a menu choice. The example in this part uses a simple boolean flag isServer_.

Message ID ranges

Every message carries an integer ID so the receiver knows what the payload contains. IDs 0–99 are reserved internally by Urho3D's network layer. Your game must use IDs in the range 100–32767. A good practice is to define an enum class so the IDs are named and self-documenting: 

// Define all game message types in one place.
enum class MsgId : int
{
    PlayerInput  = 100,  // client -> server: local movement input
    PlayerState  = 101,  // server -> all:    authoritative position
    PlayerSpawn  = 102,  // server -> all:    a new peer joined
    PlayerLeave  = 103,  // server -> all:    a peer disconnected
    ChatMessage  = 104,  // either direction:  text string
};

Complete working example

The example below builds a minimal networked session where up to eight players move coloured boxes around a flat arena. The server runs the simulation and broadcasts each player's position every frame. Clients send their WASD input and render the positions they receive. Both roles run in the same binary, selected by the boolean isServer_ member. 

#include <Simple3D/Simple3D.h>
#include <cstring>   // memcpy
#include <map>

using namespace Simple3D;

// ─────────────────────────────────────────────────────────────────────────────
// Message IDs — must be in range [100, 32767].
// ─────────────────────────────────────────────────────────────────────────────
enum class MsgId : int
{
    PlayerInput  = 100,   // client -> server: 2 floats (dx, dz)
    PlayerState  = 101,   // server -> all:    int id + 3 floats (x,y,z) per player
    PlayerSpawn  = 102,   // server -> all:    int clientId joined
    PlayerLeave  = 103,   // server -> all:    int clientId left
};

static constexpr int   NET_PORT     = 27015;
static constexpr int   MAX_CLIENTS  = 8;
static constexpr float MOVE_SPEED   = 5.0f;

// ─────────────────────────────────────────────────────────────────────────────
// Helper: pack a float into a byte vector at a given offset.
// ─────────────────────────────────────────────────────────────────────────────
static void PackFloat(std::vector<uint8_t>& buf, size_t offset, float v)
{
    memcpy(buf.data() + offset, &v, sizeof(float));
}

static float UnpackFloat(const std::vector<uint8_t>& buf, size_t offset)
{
    float v;
    memcpy(&v, buf.data() + offset, sizeof(float));
    return v;
}

static void PackInt(std::vector<uint8_t>& buf, size_t offset, int v)
{
    memcpy(buf.data() + offset, &v, sizeof(int));
}

static int UnpackInt(const std::vector<uint8_t>& buf, size_t offset)
{
    int v;
    memcpy(&v, buf.data() + offset, sizeof(int));
    return v;
}

// ─────────────────────────────────────────────────────────────────────────────
// NetArenaGame — demonstrates the full NetworkManager API.
//
// Run with isServer_ = true on one machine and false on clients.
// In a real game you would read this from argv[] or a start screen.
// ─────────────────────────────────────────────────────────────────────────────
class NetArenaGame : public Game
{
public:
    // ── Role ──────────────────────────────────────────────────────────────
    bool             isServer_  = true;   // set false for client instances

    // ── Networking ────────────────────────────────────────────────────────
    NetworkManager*  net_       = nullptr;

    // ── Scene ─────────────────────────────────────────────────────────────
    // Server: one entity per connected client, keyed by client id.
    // Client: one entity for the local player + one per remote player.
    Entity*                         localPlayer_  = nullptr;
    std::map<int, Entity*>          remotePlayers_;   // clientId -> entity

    // Server only: track positions received from clients this frame.
    std::map<int, Vector3>          serverPositions_;

    // ── UI ────────────────────────────────────────────────────────────────
    Label*           statusLabel_ = nullptr;
    Label*           countLabel_  = nullptr;
    Label*           hintLabel_   = nullptr;

    // ── Camera ────────────────────────────────────────────────────────────
    Camera*          cam_         = nullptr;

    // ─────────────────────────────────────────────────────────────────────
    // Start() — initialise networking and scene.
    // ─────────────────────────────────────────────────────────────────────
    void Start() override
    {
        SetAppName("NetArena");
        SetWindowTitle(isServer_ ? "NetArena — Server" : "NetArena — Client");

        // ── Ground ────────────────────────────────────────────────────
        Entity* ground = CreateEntity("Ground");
        ground->AddModel("Models/Plane.mdl");
        ground->SetScale(20.0f);
        ground->AddRigidBody(0.0f);
        ground->AddBoxCollider(Vector3(20.0f, 0.5f, 20.0f));
        ground->SetCollisionLayer(CollisionLayer::StaticGeometry);

        // ── Light ─────────────────────────────────────────────────────
        Entity* light = CreateEntity("Sun");
        light->AddDirectionalLight(Color::WHITE, 1.0f);
        light->SetRotation(Quaternion(45.0f, 45.0f, 0.0f));

        // ── Camera ────────────────────────────────────────────────────
        cam_ = CreateCamera();

        // ── Local player entity ───────────────────────────────────────
        // On the server, "local player" is a dummy observer.
        // On a client, it is the player the user controls.
        localPlayer_ = CreateEntity("LocalPlayer");
        localPlayer_->AddModel("Models/Box.mdl");
        localPlayer_->SetScale(0.8f);
        localPlayer_->SetMaterialColor(Color(0.2f, 0.6f, 1.0f, 1.0f));  // blue = local
        localPlayer_->SetPosition(0.0f, 0.5f, 0.0f);

        if (!isServer_)
        {
            CharacterController* cc = localPlayer_->AddCharacterController(0.35f, 1.6f);
            cc->SetGravity(20.0f);
            localPlayer_->SetCollisionLayer(CollisionLayer::Actor);
            localPlayer_->SetCollisionMask(MakeCollisionMask({
                CollisionLayer::StaticGeometry
            }));

            cam_->Follow(localPlayer_, 8.0f, 5.0f);
            cam_->SetSmoothFollow(true, 6.0f);
        }
        else
        {
            // Server observer camera — fixed overhead view.
            localPlayer_->SetActive(false);
            cam_->LookAt(Vector3::ZERO);
        }

        // ── UI ────────────────────────────────────────────────────────
        statusLabel_ = CreateLabel(isServer_ ? "Server starting..."
                                               : "Connecting...");
        statusLabel_->SetPosition(10, 10);
        statusLabel_->SetFontSize(18);
        statusLabel_->SetColor(Color::WHITE);

        countLabel_ = CreateLabel("Players: 0");
        countLabel_->SetPosition(10, 38);
        countLabel_->SetFontSize(14);
        countLabel_->SetColor(Color(0.8f, 0.8f, 0.8f, 1.0f));

        hintLabel_ = CreateLabel(isServer_ ? "Waiting for clients on port 27015"
                                             : "WASD / Arrow keys to move");
        hintLabel_->SetPosition(10, 60);
        hintLabel_->SetFontSize(13);
        hintLabel_->SetColor(Color(0.65f, 0.65f, 0.65f, 1.0f));

        // ── NetworkManager ────────────────────────────────────────────
        // Call CreateNetworkManager() once in Start().
        // Retrieve it later with GetNetworkManager() from any method.
        net_ = CreateNetworkManager();

        // ── Wire callbacks (both sides register the message handler) ──
        net_->SetOnMessage([this](int sender, int msgId,
                                   const std::vector<uint8_t>& data)
        {
            OnNetMessage(sender, msgId, data);
        });

        if (isServer_)
        {
            // Server-only: client lifecycle callbacks.
            net_->SetOnClientConnected([this](int id)
            {
                OnClientJoined(id);
            });

            net_->SetOnClientDisconnected([this](int id)
            {
                OnClientLeft(id);
            });

            // Start listening.  Simple3D assigns each connecting client a
            // stable integer id that persists until they disconnect.
            net_->StartServer(NET_PORT, MAX_CLIENTS);
            statusLabel_->SetText("Server listening on port 27015");
        }
        else
        {
            // Client-only: server lifecycle callbacks.
            net_->SetOnServerConnected([this]()
            {
                statusLabel_->SetText("Connected to server");
                hintLabel_->SetColor(Color(0.4f, 1.0f, 0.4f, 1.0f));
            });

            net_->SetOnServerDisconnected([this]()
            {
                statusLabel_->SetText("Disconnected from server");
                hintLabel_->SetColor(Color(1.0f, 0.4f, 0.4f, 1.0f));

                // Remove all remote player entities on disconnect.
                RemoveAllRemotePlayers();
            });

            // Initiate connection.  The OnServerConnected callback fires
            // asynchronously once the handshake succeeds.
            net_->Connect("127.0.0.1", NET_PORT);
        }
    }

    // ─────────────────────────────────────────────────────────────────────
    // Update() — per-frame logic.
    // ─────────────────────────────────────────────────────────────────────
    void Update(float dt) override
    {
        if (IsKeyPressed(Key::Escape))
        {
            // Graceful shutdown: disconnect before quitting.
            if (net_->IsServer())
                net_->StopServer();
            else if (net_->IsConnected())
                net_->Disconnect();
            Quit();
        }

        if (isServer_)
        {
            ServerUpdate(dt);
        }
        else
        {
            ClientUpdate(dt);
        }
    }

    // ─────────────────────────────────────────────────────────────────────
    // ServerUpdate() — runs on the hosting machine each frame.
    // Moves player entities based on received input, then broadcasts
    // all positions back to every connected client.
    // ─────────────────────────────────────────────────────────────────────
    void ServerUpdate(float dt)
    {
        // Move each server-side player entity according to the most recently
        // received input deltas (set inside OnNetMessage for MsgId::PlayerInput).
        for (auto& [clientId, entity] : remotePlayers_)
        {
            Vector3 vel = entity->GetLinearVelocity();
            // XZ velocity was set when we received PlayerInput; let physics handle Y.
            entity->SetLinearVelocity(Vector3(vel.x_, vel.y_, vel.z_));
        }

        // Broadcast authoritative positions to all clients.
        // Packet layout: [4 bytes: count] then per-player: [4 bytes id][12 bytes pos]
        int count = static_cast<int>(remotePlayers_.size());
        std::vector<uint8_t> buf(4 + count * (4 + 12));

        PackInt(buf, 0, count);
        size_t offset = 4;
        for (auto& [clientId, entity] : remotePlayers_)
        {
            Vector3 pos = entity->GetPosition();
            PackInt  (buf, offset,      clientId);
            PackFloat(buf, offset + 4,  pos.x_);
            PackFloat(buf, offset + 8,  pos.y_);
            PackFloat(buf, offset + 12, pos.z_);
            offset += 16;
        }

        // Broadcast sends to every currently connected client.
        net_->Broadcast(static_cast<int>(MsgId::PlayerState), buf);

        // Update player count HUD.
        countLabel_->SetText("Players: " + std::string(std::to_string(count)));
    }

    // ─────────────────────────────────────────────────────────────────────
    // ClientUpdate() — runs on each client machine each frame.
    // Reads local input and sends it to the server.
    // ─────────────────────────────────────────────────────────────────────
    void ClientUpdate(float dt)
    {
        if (!net_->IsConnected())
            return;   // not yet connected — do nothing

        // Read WASD / arrow key input into a movement direction.
        float dx = 0.0f, dz = 0.0f;
        if (IsKeyDown(Key::Left)  || IsKeyDown(Key::A))  dx -= 1.0f;
        if (IsKeyDown(Key::Right) || IsKeyDown(Key::D))  dx += 1.0f;
        if (IsKeyDown(Key::Up)    || IsKeyDown(Key::W))  dz -= 1.0f;
        if (IsKeyDown(Key::Down)  || IsKeyDown(Key::S))  dz += 1.0f;

        // Apply movement locally for immediate responsiveness
        // (client-side prediction).
        auto* cc = localPlayer_->GetComponent<CharacterController>();
        if (cc)
            cc->Move(Vector3(dx, 0.0f, dz) * MOVE_SPEED, dt);

        // Send our input to the server.
        // Packet: [4 bytes float dx][4 bytes float dz]
        std::vector<uint8_t> buf(8);
        PackFloat(buf, 0, dx * MOVE_SPEED);
        PackFloat(buf, 4, dz * MOVE_SPEED);

        // SendToServer delivers the message to the server from this client.
        net_->SendToServer(static_cast<int>(MsgId::PlayerInput), buf);
    }

    // ─────────────────────────────────────────────────────────────────────
    // OnNetMessage() — fired on both server and client whenever a
    // message arrives.  Dispatch by msgId.
    //
    //   On the server side:  sender = the client's integer id
    //   On the client side:  sender = 0  (the server)
    // ─────────────────────────────────────────────────────────────────────
    void OnNetMessage(int sender, int msgId,
                       const std::vector<uint8_t>& data)
    {
        if (isServer_ && msgId == static_cast<int>(MsgId::PlayerInput))
        {
            // Server received movement input from a client.
            // Apply the velocity to that client's server-side entity.
            auto it = remotePlayers_.find(sender);
            if (it == remotePlayers_.end()) return;

            float vx = UnpackFloat(data, 0);
            float vz = UnpackFloat(data, 4);

            Entity* e   = it->second;
            Vector3 vel = e->GetLinearVelocity();
            e->SetLinearVelocity(Vector3(vx, vel.y_, vz));
        }
        else if (!isServer_ && msgId == static_cast<int>(MsgId::PlayerState))
        {
            // Client received authoritative positions from the server.
            // Move all remote player entities to match.
            int    count  = UnpackInt(data, 0);
            size_t offset = 4;
            for (int i = 0; i < count; ++i)
            {
                int   id = UnpackInt  (data, offset);
                float  x = UnpackFloat(data, offset + 4);
                float  y = UnpackFloat(data, offset + 8);
                float  z = UnpackFloat(data, offset + 12);
                offset += 16;

                auto it = remotePlayers_.find(id);
                if (it != remotePlayers_.end())
                    it->second->SetPosition(x, y, z);
            }
        }
        else if (!isServer_ && msgId == static_cast<int>(MsgId::PlayerSpawn))
        {
            // Server announced a new peer — create a remote player entity.
            int newId = UnpackInt(data, 0);
            SpawnRemotePlayer(newId);
            countLabel_->SetText("Players: " +
                std::string(std::to_string((int)remotePlayers_.size() + 1)));
        }
        else if (!isServer_ && msgId == static_cast<int>(MsgId::PlayerLeave))
        {
            // Server announced a departing peer — remove their entity.
            int goneId = UnpackInt(data, 0);
            DespawnRemotePlayer(goneId);
            countLabel_->SetText("Players: " +
                std::string(std::to_string((int)remotePlayers_.size() + 1)));
        }
    }

    // ─────────────────────────────────────────────────────────────────────
    // Server-side connection event handlers.
    // ─────────────────────────────────────────────────────────────────────
    void OnClientJoined(int clientId)
    {
        // Create a server-side physics entity for the new client.
        Entity* e = CreateEntity("Remote_" + std::to_string(clientId));
        e->AddModel("Models/Box.mdl");
        e->SetScale(0.8f);
        e->SetMaterialColor(Color(1.0f, 0.45f, 0.1f, 1.0f));   // orange = remote
        e->AddRigidBody(1.0f);
        e->AddBoxCollider(Vector3(0.8f, 0.8f, 0.8f));
        e->SetCollisionLayer(CollisionLayer::Actor);
        e->SetCollisionMask(MakeCollisionMask({CollisionLayer::StaticGeometry}));
        e->SetPosition(0.0f, 1.0f, 0.0f);
        remotePlayers_[clientId] = e;

        // Tell all clients that a new player spawned.
        // Packet: [4 bytes: the new client id]
        std::vector<uint8_t> buf(4);
        PackInt(buf, 0, clientId);
        net_->Broadcast(static_cast<int>(MsgId::PlayerSpawn), buf);

        countLabel_->SetText("Players: " +
            std::string(std::to_string((int)remotePlayers_.size())));
        statusLabel_->SetText("Client " + std::to_string(clientId) + " joined");
    }

    void OnClientLeft(int clientId)
    {
        // Destroy the server-side entity for this client.
        auto it = remotePlayers_.find(clientId);
        if (it != remotePlayers_.end())
        {
            DestroyEntity(it->second);
            remotePlayers_.erase(it);
        }

        // Tell remaining clients to remove their copy of this player.
        std::vector<uint8_t> buf(4);
        PackInt(buf, 0, clientId);
        net_->Broadcast(static_cast<int>(MsgId::PlayerLeave), buf);

        countLabel_->SetText("Players: " +
            std::string(std::to_string((int)remotePlayers_.size())));
        statusLabel_->SetText("Client " + std::to_string(clientId) + " left");
    }

    // ─────────────────────────────────────────────────────────────────────
    // Client-side helpers: create / destroy visual entities for peers.
    // ─────────────────────────────────────────────────────────────────────
    void SpawnRemotePlayer(int id)
    {
        if (remotePlayers_.count(id)) return;  // already exists
        Entity* e = CreateEntity("Remote_" + std::to_string(id));
        e->AddModel("Models/Box.mdl");
        e->SetScale(0.8f);
        e->SetMaterialColor(Color(1.0f, 0.45f, 0.1f, 1.0f));  // orange = remote
        e->SetPosition(0.0f, 0.5f, 0.0f);
        remotePlayers_[id] = e;
    }

    void DespawnRemotePlayer(int id)
    {
        auto it = remotePlayers_.find(id);
        if (it != remotePlayers_.end())
        {
            DestroyEntity(it->second);
            remotePlayers_.erase(it);
        }
    }

    void RemoveAllRemotePlayers()
    {
        for (auto& [id, e] : remotePlayers_)
            DestroyEntity(e);
        remotePlayers_.clear();
    }

    // ─────────────────────────────────────────────────────────────────────
    void Stop() override
    {
        // Always clean up the networking layer on exit.
        if (net_ && net_->IsServer())
            net_->StopServer();
        else if (net_ && net_->IsConnected())
            net_->Disconnect();
    }
};

int main()
{
    return Simple3D::CreateGame<NetArenaGame>()->Run();
}

Step-by-step explanation

CreateNetworkManager() and GetNetworkManager()

CreateNetworkManager() initialises ENet on the current machine and returns a pointer to the NetworkManager instance. Call it exactly once in Start(). From any other method — including callbacks registered later — you can retrieve the same object with GetNetworkManager() without needing to store the pointer in a member variable: 

// In Start():
NetworkManager* net = CreateNetworkManager();

// From any other method, once created:
NetworkManager* net = GetNetworkManager();   // returns nullptr if not yet created

Storing the returned pointer in a member variable (as in the example, net_) is the most convenient pattern, since GetNetworkManager() has a small look-up cost compared to a member access. Either approach is correct.

net->StartServer(port, maxClients)

StartServer opens a UDP socket on the given port and begins listening for incoming ENet connections. The second argument caps how many clients can be connected simultaneously; defaults to 128 if omitted. The call returns immediately; the server begins accepting connections on the next engine tick. 

// Listen on port 27015, allow up to 8 simultaneous clients.
net_->StartServer(27015, 8);

// Default max clients (128):
net_->StartServer(7777);

// Shut down the server (closes the socket, disconnects all clients):
net_->StopServer();

// Query whether this instance is currently running as a server:
if (net_->IsServer()) { /* ... */ }

After calling StartServer, IsServer() returns true. IsConnected() and IsClient() both return false on the hosting machine.

net->Connect(host, port)

Connect initiates an ENet handshake with the specified host address (IPv4 string or hostname) and port. The connection attempt is asynchronous; the SetOnServerConnected callback fires once the handshake succeeds, and SetOnServerDisconnected fires if the host drops the connection later. IsConnected() becomes true only after the handshake completes. 

// Connect to a local server:
net_->Connect("127.0.0.1", 27015);

// Connect to a LAN host:
net_->Connect("192.168.1.10", 27015);

// Poll state before sending — IsConnected() is false until handshake:
if (net_->IsConnected())
    net_->SendToServer(100, payload);

net->Disconnect()

Disconnect sends ENet's graceful disconnect packet and closes the client connection. The server receives the corresponding SetOnClientDisconnected event. Call Disconnect() before quitting or before presenting the main menu so the server can clean up the slot immediately rather than timing out. 

// Graceful client shutdown:
if (net_->IsConnected())
    net_->Disconnect();
Quit();

net->IsServer(), net->IsClient(), net->IsConnected()

These three predicates let you branch on the current networking role at any point in your code: 

if (net_->IsServer())
{
    // This instance is hosting — may or may not have clients yet.
}
if (net_->IsClient())
{
    // This instance is a client — call Connect() was made.
    // IsConnected() may still be false if the handshake is in progress.
}
if (net_->IsConnected())
{
    // Client: handshake is complete and the connection is alive.
    // Safe to call SendToServer().
}

IsClient() returns true as soon as Connect() is called, even before the handshake completes. Use IsConnected() when you need to confirm that messages can actually be sent.

Sending messages: Broadcast, SendToClient, SendToServer

simple-3d does not replicate entity state automatically. Every piece of game state that needs to reach other machines must be packed into a std::vector<uint8_t> and sent with one of three methods: 

// Server -> all connected clients:
net_->Broadcast(static_cast<int>(MsgId::PlayerState), positionBuffer);

// Server -> one specific client (by id assigned at connect time):
net_->SendToClient(clientId, static_cast<int>(MsgId::PlayerSpawn), buf);

// Client -> server:
net_->SendToServer(static_cast<int>(MsgId::PlayerInput), inputBuffer);

All three methods deliver messages reliably and in sequence using ENet's reliable delivery channel by default. Message IDs 0–99 are reserved by Urho3D internals; always use IDs in the range 100–32767 for your own messages.

net->SetOnMessage(callback)

SetOnMessage registers a single callback that fires on both the server and the client whenever any user message arrives. The callback receives three arguments:

  • sender — on the server side, the integer id of the client that sent the message; on the client side, always 0 (representing the server).
  • msgId — the integer ID passed to Broadcast, SendToClient, or SendToServer.
  • data — the raw byte payload as a const std::vector<uint8_t>&.
net_->SetOnMessage([this](int sender, int msgId,
                           const std::vector<uint8_t>& data)
{
    // Dispatch by message id:
    if (msgId == static_cast<int>(MsgId::PlayerInput))
    {
        float vx = UnpackFloat(data, 0);
        float vz = UnpackFloat(data, 4);
        // Apply velocity to client's server-side entity (sender = clientId here).
        remotePlayers_[sender]->SetLinearVelocity(Vector3(vx, 0.0f, vz));
    }
    else if (msgId == static_cast<int>(MsgId::PlayerState))
    {
        // Apply authoritative positions (sender = 0 = server on client side).
        ApplyPositionUpdate(data);
    }
});

SetOnClientConnected and SetOnClientDisconnected

These server-only callbacks fire when a client completes or terminates its connection. The argument is the integer client id — a stable value assigned by simple-3d on connection that remains unique until the client disconnects. The same id is used as the first argument of SendToClient and appears as the sender in SetOnMessage callbacks. 

// Called once when a new client finishes its handshake:
net_->SetOnClientConnected([this](int id)
{
    // Spawn a server-side entity for this client, then notify all clients.
    OnClientJoined(id);
});

// Called once when a client sends a disconnect or times out:
net_->SetOnClientDisconnected([this](int id)
{
    // Destroy server-side entity and tell remaining clients to remove it.
    OnClientLeft(id);
});

SetOnServerConnected and SetOnServerDisconnected

These client-only callbacks fire when the connection to the server changes state. SetOnServerConnected fires after a successful handshake; SetOnServerDisconnected fires when the server closes the connection or when a network timeout is detected. 

// Wire these before calling Connect():
net_->SetOnServerConnected([this]()
{
    // Connection confirmed — safe to send messages and show the game UI.
    statusLabel_->SetText("Connected!");
});

net_->SetOnServerDisconnected([this]()
{
    // Clean up remote player visuals; present a reconnect UI.
    RemoveAllRemotePlayers();
    statusLabel_->SetText("Lost connection to server");
});

Manual game-state synchronisation

Because simple-3d does not replicate entities automatically, the server is responsible for gathering the state of all player objects each frame, packing the data into a byte buffer, and broadcasting it. Clients receive the buffer, unpack it, and set the positions of their local "remote player" display entities. The pattern used in this tutorial packs data with raw memcpy: 

// Pack: write a float at a byte offset inside a buffer.
void PackFloat(std::vector<uint8_t>& buf, size_t offset, float v)
{
    memcpy(buf.data() + offset, &v, sizeof(float));
}

// Broadcast all player positions (server, called every Update frame):
int count = static_cast<int>(remotePlayers_.size());
std::vector<uint8_t> buf(4 + count * 16);  // 4-byte count + 16 bytes per player
PackInt(buf, 0, count);
size_t off = 4;
for (auto& [id, e] : remotePlayers_)
{
    Vector3 pos = e->GetPosition();
    PackInt  (buf, off,      id);
    PackFloat(buf, off + 4,  pos.x_);
    PackFloat(buf, off + 8,  pos.y_);
    PackFloat(buf, off + 12, pos.z_);
    off += 16;
}
net_->Broadcast(static_cast<int>(MsgId::PlayerState), buf);

simple-3d does not ship a higher-level serialiser. For larger projects you can integrate any byte-packing library (such as flatbuffers, cereal, or a hand-written bit-packing scheme) and pass the resulting std::vector<uint8_t> directly to the send methods.

Lua equivalent

All networking methods are accessible through the game global inside any Lua script. The Lua API mirrors the C++ signatures exactly. The example below is a minimal client-only script that connects, sends WASD input, and updates a remote-player label when messages arrive. Attach it to any persistent entity. 

-- Scripts/net_client.lua
-- Attach to a persistent entity (marked in C++ with SetPersistentEntity).
-- Assumes the server is running on 127.0.0.1:27015.

local NET_PORT    = 27015
local MSG_INPUT   = 100
local MSG_STATE   = 101
local MSG_SPAWN   = 102
local MSG_LEAVE   = 103
local MOVE_SPEED  = 5.0

local net         = nil
local statusLabel = nil

function Start()
    -- Create the network manager once in Start().
    net = game:CreateNetworkManager()

    statusLabel = game:CreateLabel("Connecting...")
    statusLabel:SetPosition(10, 10)
    statusLabel:SetFontSize(18)
    statusLabel:SetColor(Color(1, 1, 1, 1))

    -- Register the unified message handler.
    net:SetOnMessage(function(sender, msgId, data)
        if msgId == MSG_STATE then
            -- Apply authoritative positions broadcast by the server.
            local count  = data:UnpackInt(0)
            local offset = 4
            for i = 1, count do
                local id = data:UnpackInt(offset)
                local x  = data:UnpackFloat(offset + 4)
                local y  = data:UnpackFloat(offset + 8)
                local z  = data:UnpackFloat(offset + 12)
                offset = offset + 16
                local e = game:FindEntity("Remote_" .. id)
                if e then e:SetPosition(x, y, z) end
            end
        elseif msgId == MSG_SPAWN then
            local newId = data:UnpackInt(0)
            local e = game:CreateEntity("Remote_" .. newId)
            e:AddModel("Models/Box.mdl")
            e:SetScale(0.8)
            e:SetPosition(0, 0.5, 0)
        elseif msgId == MSG_LEAVE then
            local goneId = data:UnpackInt(0)
            local e = game:FindEntity("Remote_" .. goneId)
            if e then game:DestroyEntity(e) end
        end
    end)

    -- Connection state callbacks.
    net:SetOnServerConnected(function()
        statusLabel:SetText("Connected to server")
    end)

    net:SetOnServerDisconnected(function()
        statusLabel:SetText("Disconnected from server")
    end)

    -- Connect.  IsConnected() is false until the handshake succeeds.
    net:Connect("127.0.0.1", NET_PORT)
end

function Update(dt)
    if not net:IsConnected() then return end

    -- Sample movement keys and send input to the server.
    local dx = 0.0
    local dz = 0.0
    if game:IsKeyDown(Key.A)    or game:IsKeyDown(Key.Left)  then dx = dx - MOVE_SPEED end
    if game:IsKeyDown(Key.D)    or game:IsKeyDown(Key.Right) then dx = dx + MOVE_SPEED end
    if game:IsKeyDown(Key.W)    or game:IsKeyDown(Key.Up)    then dz = dz - MOVE_SPEED end
    if game:IsKeyDown(Key.S)    or game:IsKeyDown(Key.Down)  then dz = dz + MOVE_SPEED end

    -- Build a small 8-byte buffer: float dx, float dz.
    local buf = ByteBuffer.New(8)
    buf:PackFloat(0, dx)
    buf:PackFloat(4, dz)

    net:SendToServer(MSG_INPUT, buf)

    if game:IsKeyPressed(Key.Escape) then
        net:Disconnect()
        game:Quit()
    end
end

Design patterns and tips

Authoritative server model

The pattern in this tutorial is the most common approach for competitive or cheat-resistant games. The server owns all simulation state: it moves player entities based on received inputs, runs physics, and sends the results back to clients. Clients predict their own movement locally to hide latency, but accept server corrections when the authoritative position differs significantly.

Choosing a network tick rate

Broadcasting full game state every frame can waste bandwidth for games with many players or slow connections. A common strategy is to accumulate input every frame but send state at a fixed network tick rate (e.g. 20–30 Hz): 

float netTimer_ = 0.0f;
static constexpr float NET_TICK = 1.0f / 20.0f;   // 20 Hz

void ServerUpdate(float dt)
{
    netTimer_ += dt;
    if (netTimer_ >= NET_TICK)
    {
        netTimer_ -= NET_TICK;
        BroadcastPositions();   // only send 20 times per second
    }
}

Sending to a single client versus broadcasting

Use SendToClient(clientId, ...) for messages that only one peer needs — for example, sending the full world state to a newly joined client (other clients already have it) or sending a private notification such as "you were voted off". Use Broadcast(...) for state that every client must know, such as position updates or chat messages. 

// Server: welcome packet sent only to the newly joined client.
void OnClientJoined(int id)
{
    std::vector<uint8_t> worldSnapshot = SerialiseFullWorldState();
    net_->SendToClient(id, static_cast<int>(MsgId::PlayerState), worldSnapshot);

    // Then announce the newcomer to everyone (including themselves).
    std::vector<uint8_t> spawnMsg(4);
    PackInt(spawnMsg, 0, id);
    net_->Broadcast(static_cast<int>(MsgId::PlayerSpawn), spawnMsg);
}

Using the same binary for server and client

For convenience during development, keep server and client code in one binary and select the role via a command-line argument or a startup menu. In a shipped game you might separate them into a dedicated server build and a client build using preprocessor flags. 

int main(int argc, char* argv[])
{
    auto* game = Simple3D::CreateGame<NetArenaGame>();

    // Pass "--server" on the command line to run as the host.
    auto* g = static_cast<NetArenaGame*>(game);
    for (int i = 1; i < argc; ++i)
        if (std::string(argv[i]) == "--server")
            g->isServer_ = true;

    return game->Run();
}

Common mistakes

  • Calling SendToServer() before IsConnected() is true. Connect() starts an async handshake. Sending a message before the handshake completes is a no-op and the data is silently dropped. Always check IsConnected() or send inside the SetOnServerConnected callback.
  • Using message IDs below 100. IDs 0–99 are reserved by Urho3D's network layer. Using them for your own messages will conflict with internal packets and produce erratic behaviour. Always start your enum at 100 or higher, and keep all IDs below 32768.
  • Forgetting that entity replication is not automatic. Creating an entity on the server does not create it on any client. You must explicitly send a spawn message to every client and have each client create the entity when it receives that message. Likewise, destroying a server-side entity requires sending a despawn message so clients can remove their copies.
  • Reading data beyond the end of the payload buffer. If you try to UnpackFloat at an offset past the end of data, you get undefined behaviour. Always design your packets so the receiver can compute exactly how many bytes to read before iterating, and add a size guard if the sender might send variable-length payloads.
  • Not calling StopServer() or Disconnect() before Quit(). Simply quitting without cleaning up leaves the ENet socket open briefly. On the server side, clients will time out rather than receiving a clean disconnect event. Always call StopServer() or Disconnect() — ideally in Stop() and also when Escape is pressed — for a graceful shutdown.
  • Calling CreateNetworkManager() more than once. Call it exactly once in Start(). Calling it a second time has undefined results. Use GetNetworkManager() to retrieve the already-created instance from other parts of your code.
  • Storing stale client ids after a disconnect. Client ids are assigned at connect time and become invalid the moment the client disconnects. Do not cache a client id in a data structure and attempt to call SendToClient with it after receiving an OnClientDisconnected event for that id.
  • Broadcasting at full frame rate on a slow connection. Sending a full position snapshot every frame at 60 Hz for many players can saturate a consumer broadband link. Use a fixed network tick rate (20–30 Hz is typical) and only send deltas or prioritised updates for entities that moved significantly.

What you learned

In this final part of the tutorial series you learned how to add multiplayer networking to a simple-3d game using the NetworkManager API:

  • Creating the manager: call CreateNetworkManager() once in Start(); retrieve it later with GetNetworkManager().
  • Hosting: StartServer(port, maxClients) opens a UDP port and begins accepting connections. StopServer() closes it cleanly. IsServer() confirms the hosting role.
  • Connecting: Connect(host, port) initiates an async handshake. IsConnected() becomes true only after success. Disconnect() sends a graceful goodbye before quitting.
  • Querying state: IsServer(), IsClient(), and IsConnected() let you branch logic correctly regardless of which role the current instance is playing.
  • Sending messages: Broadcast reaches all clients; SendToClient reaches one; SendToServer sends from a client to the host. Message IDs must be in the range 100–32767.
  • Receiving messages: SetOnMessage registers a unified callback on both sides; the sender argument is the client id on the server or 0 (the server) on the client.
  • Connection events: SetOnClientConnected and SetOnClientDisconnected give the server per-peer lifecycle hooks; SetOnServerConnected and SetOnServerDisconnected give the client equivalent notifications.
  • Manual replication: entity state is never sent automatically; pack positions and events into byte buffers with memcpy and send them on a fixed network tick rate for efficient use of bandwidth.
