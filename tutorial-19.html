Background: what is SaveData?

Most games need to remember something between sessions: the player's score, the current level, audio settings, or whether a tutorial was already shown. simple-3d provides Simple3D::SaveData, a lightweight key-value store that serialises to and from a JSON file on disk. There are no databases or binary formats to worry about — the file is human-readable, which makes debugging simple.

In-memory store

A SaveData object holds an in-memory map of typed values. You can read and write keys freely without touching the filesystem. The data is only persisted when you explicitly call Save(slot).

JSON on disk

Save(slot) serialises the in-memory map to save_N.json in the application's OS-specific directory. Load(slot) reads it back. If the file is missing, Load returns false and the object stays empty — your default values take over automatically.

Where save files are written

Before calling Save or Load for the first time, call SetAppName to fix the directory. The default name is "Simple3DGame" if you do not call it. 

// In Start() or main() — call once before any Save/Load
SetAppName("PlatformerAdventure");

The resulting paths for slot N are:

PlatformPath
Linux ~/.local/share/<appName>/save_N.json
Windows %APPDATA%\<appName>\save_N.json

For example, with SetAppName("PlatformerAdventure") and Save(0) on Linux, the file is written to ~/.local/share/PlatformerAdventure/save_0.json. The directory is created automatically if it does not exist.

JSON format on disk

Each type lives in its own section so the parser can distinguish a bool from an int without ambiguity. Here is an example file for slot 0: 

{
  "ints": {
    "score": 4200,
    "level": 3,
    "lives": 5
  },
  "floats": {
    "masterVolume": 0.800000012
  },
  "bools": {
    "musicEnabled": true,
    "tutorialShown": true
  },
  "strings": {
    "playerName": "Alice"
  }
}

Because the file is plain JSON, you can open it in any text editor to inspect or reset values during development. Never rely on this format being stable across engine versions in production; use it for debugging only.

Complete working example

The example below builds a small RPG-style demo. It demonstrates every save-and-load API call covered in this part: setting the app name, writing and reading all four value types, the first-run default pattern, autosave on quit, manual save slots, checking whether a slot exists, and deleting a slot from the save-selection UI. 

#include <Simple3D/Simple3D.h>

using namespace Simple3D;

// ── Save slot conventions ─────────────────────────────────────────────
// Slot 0  = autosave  (written automatically on quit and level transition)
// Slots 1-3 = manual saves (player-controlled from the pause menu)
static constexpr int SLOT_AUTO   = 0;
static constexpr int SLOT_SAVE1  = 1;
static constexpr int SLOT_SAVE2  = 2;
static constexpr int SLOT_SAVE3  = 3;

class SaveLoadDemo : public Game
{
public:
    // ── Game state ───────────────────────────────────────────────────
    int         score_          = 0;
    int         currentLevel_   = 1;
    int         lives_          = 3;
    float       masterVolume_   = 0.8f;
    bool        musicEnabled_   = true;
    bool        tutorialShown_  = false;
    std::string playerName_     = "Player";

    // ── UI ───────────────────────────────────────────────────────────
    Entity*  player_      = nullptr;
    Entity*  ground_      = nullptr;
    Camera*  cam_         = nullptr;
    Label*   scoreLabel_  = nullptr;
    Label*   levelLabel_  = nullptr;
    Label*   infoLabel_   = nullptr;
    Label*   slotLabel_   = nullptr;

    // ────────────────────────────────────────────────────────────────
    void Start() override
    {
        // STEP 1 — Set the app name BEFORE any Save/Load call.
        // The name becomes part of the OS save directory path:
        //   Linux:   ~/.local/share/PlatformerAdventure/save_N.json
        //   Windows: %APPDATA%\PlatformerAdventure\save_N.json
        SetAppName("PlatformerAdventure");

        // STEP 2 — Load the autosave (slot 0) on startup.
        // GetSaveData(slot) returns a SaveData& owned by the Game;
        // no manual lifetime management needed.
        LoadFromSlot(SLOT_AUTO);

        // ── Scene setup ─────────────────────────────────────────────
        ground_ = CreateEntity("Ground");
        ground_->AddModel("Models/Plane.mdl");
        ground_->SetScale(20.0f);
        ground_->AddRigidBody(0.0f);
        ground_->AddBoxCollider(Vector3(20.0f, 0.1f, 20.0f));

        player_ = CreateEntity("Player");
        player_->AddModel("Models/Box.mdl");
        player_->SetPosition(0.0f, 1.0f, 0.0f);
        CharacterController* cc = player_->AddCharacterController(0.35f, 1.8f);
        cc->SetGravity(20.0f);

        Entity* sun = CreateEntity("Sun");
        sun->AddDirectionalLight(Color::WHITE, 1.0f);
        sun->SetRotation(Quaternion(45.0f, 30.0f, 0.0f));

        cam_ = CreateCamera();
        cam_->Follow(player_, 7.0f, 3.0f);
        cam_->SetSmoothFollow(true, 5.0f);

        // ── HUD ─────────────────────────────────────────────────────
        scoreLabel_ = CreateLabel("");
        scoreLabel_->SetPosition(10, 10);
        scoreLabel_->SetFontSize(18);
        scoreLabel_->SetColor(Color::WHITE);

        levelLabel_ = CreateLabel("");
        levelLabel_->SetPosition(10, 36);
        levelLabel_->SetFontSize(16);
        levelLabel_->SetColor(Color(0.8f, 1.0f, 0.8f, 1.0f));

        infoLabel_ = CreateLabel(
            "F5=autosave  F6=save1  F7=save2  F8=save3 | "
            "F9=load1  F10=load2  F11=load3 | DEL=delete slot1");
        infoLabel_->SetPosition(10, 62);
        infoLabel_->SetFontSize(12);
        infoLabel_->SetColor(Color(0.7f, 0.7f, 0.7f, 1.0f));

        slotLabel_ = CreateLabel("");
        slotLabel_->SetPosition(10, 82);
        slotLabel_->SetFontSize(14);
        slotLabel_->SetColor(Color(1.0f, 0.9f, 0.4f, 1.0f));

        // Apply audio setting loaded from save
        SetMasterVolume(masterVolume_);
        if (musicEnabled_)
            PlayMusic("Music/theme.ogg");

        // Show tutorial message on first run only
        if (!tutorialShown_)
        {
            slotLabel_->SetText("Welcome! Use WASD to move. Collect coins to earn points.");
            tutorialShown_ = true;   // persisted at next save
        }

        UpdateHUD();
    }

    // ────────────────────────────────────────────────────────────────
    void Update(float dt) override
    {
        HandleMovement(dt);
        HandleCoinPickup();
        HandleSaveKeys();
        HandleLoadKeys();
        HandleDeleteKey();
        HandleQuit();
    }

    // ────────────────────────────────────────────────────────────────
    void Stop() override
    {
        // Autosave on quit so progress is never lost.
        SaveToSlot(SLOT_AUTO);
        StopMusic();
    }

    // ═══════════════════════════════════════════════════════════════
    // SAVE / LOAD HELPERS
    // ═══════════════════════════════════════════════════════════════

    // SaveToSlot — pack all game state into the SaveData object and write to disk.
    void SaveToSlot(int slot)
    {
        // GetSaveData(slot) returns a SaveData& owned by the Game.
        // The same reference is returned every time for the same slot.
        SaveData& save = GetSaveData(slot);

        // Set accepts int, float, bool, or std::string — all via the same
        // overloaded name.  If the key already exists its value is replaced.
        save.Set("score",         score_);
        save.Set("level",         currentLevel_);
        save.Set("lives",         lives_);
        save.Set("masterVolume",  masterVolume_);
        save.Set("musicEnabled",  musicEnabled_);
        save.Set("tutorialShown", tutorialShown_);
        save.Set("playerName",    playerName_);

        // Save(slot) serialises in-memory data to save_N.json on disk.
        // Returns false on I/O error (e.g., no write permission).
        if (!save.Save(slot))
            slotLabel_->SetText("ERROR: save failed for slot " + String(slot));
        else
            slotLabel_->SetText("Saved to slot " + String(slot));
    }

    // LoadFromSlot — read data back from disk and apply defaults for missing keys.
    void LoadFromSlot(int slot)
    {
        SaveData& save = GetSaveData(slot);

        // Load(slot) reads save_N.json and populates the in-memory store.
        // Returns false if the file is missing or cannot be parsed —
        // which is the normal case on first launch.
        if (!save.Load(slot))
        {
            // First run (or corrupted file): all member variables keep
            // their in-class default values.  Nothing to do here.
            if (slotLabel_)
                slotLabel_->SetText("No save in slot " + String(slot) + " — using defaults");
            return;
        }

        // GetInt(key, default) — returns the stored int, or 'default' if the
        // key is absent (first run after adding a new feature, etc.).
        score_        = save.GetInt   ("score",         0);
        currentLevel_ = save.GetInt   ("level",         1);
        lives_        = save.GetInt   ("lives",         3);

        // GetFloat(key, default) — returns stored float or default.
        masterVolume_ = save.GetFloat ("masterVolume",  0.8f);

        // GetBool(key, default) — returns stored bool or default.
        musicEnabled_   = save.GetBool  ("musicEnabled",  true);
        tutorialShown_  = save.GetBool  ("tutorialShown", false);

        // GetString(key, default) — returns stored string or default.
        playerName_   = save.GetString("playerName",    "Player");

        if (slotLabel_)
            slotLabel_->SetText(
                "Loaded slot " + String(slot) + " — welcome back, " + String(playerName_.c_str()) + "!");

        UpdateHUD();
    }

    // ────────────────────────────────────────────────────────────────
    void UpdateHUD()
    {
        if (scoreLabel_)
            scoreLabel_->SetText(
                String(playerName_.c_str()) + "  Score: " + String(score_) +
                "  Lives: " + String(lives_));
        if (levelLabel_)
            levelLabel_->SetText("Level: " + String(currentLevel_));
    }

    // ────────────────────────────────────────────────────────────────
    void HandleMovement(float dt)
    {
        CharacterController* cc = player_->GetComponent<CharacterController>();
        if (!cc) return;

        float mx = 0.0f, mz = 0.0f;
        if (IsKeyDown(Key::Left)  || IsKeyDown(Key::A)) mx -= 1.0f;
        if (IsKeyDown(Key::Right) || IsKeyDown(Key::D)) mx += 1.0f;
        if (IsKeyDown(Key::Up)    || IsKeyDown(Key::W)) mz -= 1.0f;
        if (IsKeyDown(Key::Down)  || IsKeyDown(Key::S)) mz += 1.0f;

        cc->Move(Vector3(mx, 0.0f, mz) * 5.0f, dt);
        if (IsKeyPressed(Key::Space) && cc->IsOnGround())
            cc->Jump(10.0f);
    }

    void HandleCoinPickup()
    {
        // Pressing C simulates picking up a coin
        if (IsKeyPressed(Key::C))
        {
            score_ += 100;
            if (score_ % 1000 == 0)
            {
                currentLevel_++;
                slotLabel_->SetText("Level up! Autosaving...");
                SaveToSlot(SLOT_AUTO);   // autosave at level transition
            }
            UpdateHUD();
        }
    }

    // ────────────────────────────────────────────────────────────────
    void HandleSaveKeys()
    {
        if (IsKeyPressed(Key::F5))  SaveToSlot(SLOT_AUTO);
        if (IsKeyPressed(Key::F6))  SaveToSlot(SLOT_SAVE1);
        if (IsKeyPressed(Key::F7))  SaveToSlot(SLOT_SAVE2);
        if (IsKeyPressed(Key::F8))  SaveToSlot(SLOT_SAVE3);
    }

    void HandleLoadKeys()
    {
        if (IsKeyPressed(Key::F9))
        {
            // Check existence before loading to give a friendly message.
            if (SaveData::Exists(SLOT_SAVE1))
                LoadFromSlot(SLOT_SAVE1);
            else
                slotLabel_->SetText("Slot 1 is empty.");
        }
        if (IsKeyPressed(Key::F10))
        {
            if (SaveData::Exists(SLOT_SAVE2))
                LoadFromSlot(SLOT_SAVE2);
            else
                slotLabel_->SetText("Slot 2 is empty.");
        }
        if (IsKeyPressed(Key::F11))
        {
            if (SaveData::Exists(SLOT_SAVE3))
                LoadFromSlot(SLOT_SAVE3);
            else
                slotLabel_->SetText("Slot 3 is empty.");
        }
    }

    void HandleDeleteKey()
    {
        // Delete key erases slot 1 from disk permanently.
        if (IsKeyPressed(Key::Delete))
        {
            if (SaveData::Delete(SLOT_SAVE1))
            {
                // Also clear the cached in-memory data for this slot.
                GetSaveData(SLOT_SAVE1).Clear();
                slotLabel_->SetText("Slot 1 deleted.");
            }
            else
            {
                slotLabel_->SetText("Slot 1 was already empty.");
            }
        }
    }

    void HandleQuit()
    {
        if (IsKeyPressed(Key::Escape))
            Quit();   // Stop() is called automatically — autosave fires there
    }
};

int main()
{
    return Simple3D::CreateGame<SaveLoadDemo>()->Run();
}

Step-by-step explanation

SetAppName(name)

Call SetAppName once, before any Save or Load call, to set the subdirectory that holds all save files for your game. Internally it calls the static SaveData::SetGlobalAppName(name), which fixes the base directory for all subsequent disk operations. The default name — used if you never call this — is "Simple3DGame". Choose something unique so that two games installed on the same machine do not share each other's save files. 

// Call once in Start() before any Save/Load operation
SetAppName("MyRpgGame");

// From that point, Save(0) writes to:
//   Linux:   ~/.local/share/MyRpgGame/save_0.json
//   Windows: %APPDATA%\MyRpgGame\save_0.json

Simple3D::SaveData — standalone vs. Game::GetSaveData(slot)

You can create a Simple3D::SaveData object as a plain stack variable whenever you need one. This is useful for temporary or one-off operations. The more common approach inside a Game subclass is GetSaveData(slot), which returns a SaveData& owned by the engine. The engine creates the object on first access and keeps it alive as long as the game runs, so you never have to manage its lifetime. Calling GetSaveData(0) ten times always returns the same object. 

// Approach A — standalone (own the object yourself)
Simple3D::SaveData data;
data.Set("score", score_);
data.Save(0);

// Approach B — Game-owned (recommended inside a Game subclass)
SaveData& save = GetSaveData(0);  // lazy-created, cached
save.Set("score", score_);
save.Save(0);

data.Set(key, value)

Set is a single overloaded method that accepts an int, float, bool, or std::string as its second argument. The compiler selects the right overload automatically. If the key already exists — even with a different type from a previous call — the old value and type are replaced by the new ones. Keys are case-sensitive strings; prefer lowercase camel-case names to match the JSON output style. 

save.Set("score",        score_);         // int overload
save.Set("masterVolume", masterVolume_);  // float overload
save.Set("musicEnabled", musicEnabled_);  // bool overload
save.Set("playerName",   playerName_);    // string overload

GetInt / GetFloat / GetBool / GetString — the default-value pattern

Each typed getter takes the key and a default value. If the key is present in the loaded data, the stored value is returned. If the key is absent — because the file is brand new, the key was added in a newer version of the game, or Load returned false — the default is returned instead. This is the correct way to handle the first-run case: you write the default in the getter call and it works automatically, with no explicit if (!fileExists) branching required. 

// First run: Load() returns false, GetInt returns the default 0.
// Subsequent runs: the stored value is returned.
score_  = save.GetInt   ("score",        0);
level_  = save.GetInt   ("level",        1);  // default = level 1
vol_    = save.GetFloat ("masterVolume", 0.8f);
music_  = save.GetBool  ("musicEnabled", true);
name_   = save.GetString("playerName",   "Player");

// Template shorthand — same result, fewer characters when T is clear
score_  = save.Get<int>  ("score",        0);
vol_    = save.Get<float>("masterVolume", 0.8f);

data.Save(slot) and data.Load(slot)

Save(slot) serialises the in-memory key-value map to save_N.json on disk. It returns true on success and false if a filesystem error occurs — for example if the directory is read-only or the disk is full. Always check the return value in production code so you can show the player a meaningful error message instead of silently losing their progress.

Load(slot) reads the corresponding JSON file into the in-memory store and returns true. If the file does not exist or cannot be parsed, it returns false and leaves the object empty. A false return is not an error that needs logging — it is the expected state on the very first launch. The typical pattern is: 

if (!save.Load(0))
{
    // No save file yet (first run) — member defaults are already in place.
}
else
{
    score_  = save.GetInt("score", 0);
    level_  = save.GetInt("level", 1);
    // ... read remaining keys
}

You can omit the else branch entirely if you call Get* unconditionally, because the defaults in the getter calls cover the missing-key case automatically.

data.HasKey(key) and data.Clear()

HasKey(key) returns true if the key exists in the in-memory store with any type. Use it when the presence of a key has semantic meaning — for example, to tell apart "player typed an empty string" from "player never visited the name screen". 

if (!save.HasKey("playerName"))
{
    // Key was never written — route to the name-entry screen
    ShowNameEntryScreen();
}

Clear() wipes all keys from the in-memory map without touching the file on disk. Call it before rebuilding a save from scratch — for instance, when starting a new game and the player opts to overwrite an existing slot. After Clear(), the next Set / Save calls produce a fresh file with only the keys you explicitly write. 

// "New Game" flow — erase in-memory state, write clean defaults
save.Clear();
save.Set("score",       0);
save.Set("level",       1);
save.Set("playerName",  playerName_);
save.Save(SLOT_SAVE1);

SaveData::Exists(slot) and SaveData::Delete(slot) — static helpers

SaveData::Exists(slot) is a static method that checks whether the JSON file for the given slot is present on disk without loading its contents. Use it to decide whether to enable a "Continue" button or show a slot as occupied in a save-selection menu. SaveData::SlotExists(slot) is an alias for the same function.

SaveData::Delete(slot) removes the file from disk and returns true if the file existed and was successfully deleted, or false if the file was absent. After deletion, also call GetSaveData(slot).Clear() to purge the cached in-memory data so the stale values do not linger. SaveData::DeleteSlot(slot) is an alias. 

// Populate a save-selection UI
for (int s = 1; s <= 3; ++s)
{
    if (SaveData::Exists(s))
        slotButtons_[s]->SetText("Slot " + String(s) + " — occupied");
    else
        slotButtons_[s]->SetText("Slot " + String(s) + " — empty");
}

// Delete a slot on player request
SaveData::Delete(2);
GetSaveData(2).Clear();   // clear the engine's cached copy too

Multi-slot pattern: slot 0 = autosave, slots 1–3 = manual

The convention used throughout simple-3d projects reserves slot 0 for the automatic save that runs without player interaction — on level transition, on pause, and on quit. Slots 1, 2, and 3 are player-controlled saves that the player writes from a pause-menu button and loads from a title-screen selection. Using separate slot indices ensures that an autosave never silently overwrites a manual save the player created at a difficult-to-reach checkpoint. 

// On level cleared — autosave fires without player involvement
void OnLevelCleared()
{
    currentLevel_++;
    SaveToSlot(SLOT_AUTO);   // slot 0
}

// Pause-menu "Save" button — writes player's chosen manual slot
saveBtn_->SetOnClick([this] {
    SaveToSlot(activeManualSlot_);  // 1, 2, or 3 chosen in the UI
    pausePanel_->SetVisible(false);
});

// Title-screen slot picker — load whichever slot the player selects
for (int s = 1; s <= 3; ++s)
{
    slotBtn_[s]->SetOnClick([this, s] {
        LoadFromSlot(s);
    });
    // Grey out empty slots
    slotBtn_[s]->SetColor(SaveData::Exists(s)
        ? Color(0.2f, 0.6f, 0.2f, 0.9f)
        : Color(0.3f, 0.3f, 0.3f, 0.6f));
}

Lua equivalent

SaveData is exposed to Lua through the game global. The method names and signatures are identical to C++. Lua scripts attached with entity:AddScript() can read and write save data the same way as C++. 

-- Scripts/save_manager.lua
-- Attach this script to a persistent entity to manage saving and loading.

local SLOT_AUTO  = 0
local SLOT_SAVE1 = 1
local SLOT_SAVE2 = 2
local SLOT_SAVE3 = 3

-- Game state (would normally be shared via a module or global)
local score         = 0
local currentLevel  = 1
local masterVolume  = 0.8
local musicEnabled  = true
local playerName    = "Player"

function saveToSlot(slot)
    local save = game:GetSaveData(slot)
    save:Set("score",        score)
    save:Set("level",        currentLevel)
    save:Set("masterVolume", masterVolume)
    save:Set("musicEnabled", musicEnabled)
    save:Set("playerName",   playerName)
    if not save:Save(slot) then
        print("ERROR: save failed for slot " .. slot)
    end
end

function loadFromSlot(slot)
    local save = game:GetSaveData(slot)
    if not save:Load(slot) then
        -- First run or missing file — keep defaults
        print("No save in slot " .. slot .. ", using defaults")
        return
    end

    -- Each getter returns the stored value, or the second argument as default
    score        = save:GetInt   ("score",        0)
    currentLevel = save:GetInt   ("level",        1)
    masterVolume = save:GetFloat ("masterVolume", 0.8)
    musicEnabled = save:GetBool  ("musicEnabled", true)
    playerName   = save:GetString("playerName",   "Player")

    print("Loaded slot " .. slot .. " — welcome back, " .. playerName)
end

function Start()
    -- SetAppName is called from C++ main() before scripts run, or
    -- you can call it here if this is a pure-Lua project.
    game:SetAppName("PlatformerAdventure")

    -- Load autosave on startup
    loadFromSlot(SLOT_AUTO)

    game:SetMasterVolume(masterVolume)
    if musicEnabled then
        game:PlayMusic("Music/theme.ogg")
    end
end

function Update(dt)
    -- F5 = autosave
    if game:IsKeyPressed(Key.F5) then
        saveToSlot(SLOT_AUTO)
    end

    -- F6/F7/F8 = manual slots
    if game:IsKeyPressed(Key.F6) then saveToSlot(SLOT_SAVE1) end
    if game:IsKeyPressed(Key.F7) then saveToSlot(SLOT_SAVE2) end
    if game:IsKeyPressed(Key.F8) then saveToSlot(SLOT_SAVE3) end

    -- F9/F10/F11 = load manual slots (check existence first)
    if game:IsKeyPressed(Key.F9) then
        if SaveData.Exists(SLOT_SAVE1) then
            loadFromSlot(SLOT_SAVE1)
        else
            print("Slot 1 is empty")
        end
    end
    if game:IsKeyPressed(Key.F10) then
        if SaveData.Exists(SLOT_SAVE2) then
            loadFromSlot(SLOT_SAVE2)
        else
            print("Slot 2 is empty")
        end
    end

    -- Delete key = erase slot 1
    if game:IsKeyPressed(Key.Delete) then
        SaveData.Delete(SLOT_SAVE1)
        game:GetSaveData(SLOT_SAVE1):Clear()
        print("Slot 1 deleted")
    end
end

function Stop()
    -- Autosave on quit
    saveToSlot(SLOT_AUTO)
    game:StopMusic()
end

Patterns and tips

Saving settings vs. saving game progress

It is good practice to keep global settings (volume, resolution, keybindings) in slot 0 alongside the autosave, but tagged with a distinct key prefix. Alternatively, use a dedicated slot — for instance slot 99 — for settings only, so that "New Game" never accidentally wipes the player's audio preferences when overwriting slot 0. 

// In Stop() — write both progress and settings to slot 0
SaveData& save = GetSaveData(SLOT_AUTO);
save.Set("score",        score_);
save.Set("level",        currentLevel_);
save.Set("cfg_volume",   masterVolume_);   // prefix distinguishes settings
save.Set("cfg_music",    musicEnabled_);
save.Save(SLOT_AUTO);

Versioning your save data

When you add new keys in a later version of the game, older save files will not contain them. Always use the default parameter in every getter call — that default is the "upgrade path" for saves created before the key existed. You can also store an explicit version number and migrate data programmatically: 

int saveVersion = save.GetInt("saveVersion", 1);
if (saveVersion < 2)
{
    // Key "lives" did not exist before version 2 — backfill the default
    lives_ = 3;
    save.Set("saveVersion", 2);
}
else
{
    lives_ = save.GetInt("lives", 3);
}

Persisting audio settings (from Part 10)

Part 10 showed a pattern where SaveData stores the master volume. Here is the complete round-trip for convenience: 

void Start() override
{
    SetAppName("MyGame");
    SaveData& save = GetSaveData(0);
    save.Load(0);
    masterVol_ = save.GetFloat("masterVol", 0.8f);
    SetMasterVolume(masterVol_);
    PlayMusic("Music/theme.ogg");
}

void Stop() override
{
    SaveData& save = GetSaveData(0);
    save.Set("masterVol", masterVol_);
    save.Save(0);
    StopMusic();
}

Common mistakes

  • Forgetting to call SetAppName before the first Save/Load. If you skip it, files land in a directory named "Simple3DGame" — the library default. Two different games with the same oversight will silently share save files and corrupt each other's data. Always call SetAppName at the very start of Start().
  • Treating a false return from Load as an error. On the very first launch there is no save file, so Load correctly returns false. Do not log this as an error or show a warning dialog — it is the normal first-run state. Reserve error handling for cases where Save returns false (write failure) or where Load fails on a slot that SaveData::Exists confirmed was present (possible file corruption).
  • Calling Save every frame. Disk writes have latency and can cause frame stutter. Save at meaningful game events only: level clear, checkpoint reached, player-initiated save, and application quit. If you need frequent persistence, accumulate changes in-memory with Set and save once at the end.
  • Using the same slot for autosave and manual saves. If autosave fires while the player is mid-level and overwrites their carefully placed manual checkpoint save, they lose that checkpoint permanently. Reserve slot 0 exclusively for autosave; never let player-driven manual saves write to slot 0.
  • Not calling Clear() before overwriting an existing slot with "New Game" data. If the slot previously stored keys that no longer exist in the new save (because a feature was removed), those orphaned keys will persist in the JSON file after the next Save. Call save.Clear() first to guarantee the file contains only what you explicitly write.
  • Forgetting to call GetSaveData(slot).Clear() after SaveData::Delete(slot). Delete removes the file from disk but leaves the Game's cached SaveData object intact. If you then call SaveData::Exists it correctly returns false, but if some code path calls GetSaveData(slot).GetInt(...) without loading first, it will see the stale in-memory values. Always pair a Delete with a Clear on the same slot.
  • Relying on the JSON file format across engine versions. The JSON layout (the "ints", "floats", etc. sections) is an implementation detail. Never write code that reads or edits the file directly at runtime; always go through the SaveData API. The format may change in future engine releases.
  • Using hardcoded slot numbers instead of named constants. Scattering the number 0 through your codebase to mean "autosave" makes the intent unclear and turns a slot renumber into a grep-and-replace exercise. Define static constexpr int SLOT_AUTO = 0; at the top of your class and reference that constant everywhere.

What you learned

In this part you built a complete save-and-load system using Simple3D::SaveData:

  • App name: SetAppName(name) fixes the OS directory where all save_N.json files are written. Call it once before any disk operation.
  • Writing data: data.Set(key, value) stores any of the four supported types — int, float, bool, std::string — under a string key in memory.
  • Reading data: GetInt, GetFloat, GetBool, and GetString each accept a default value that is returned when the key is absent. This default is the only code you need to handle the first-run case — no separate branch required. The template Get<T> is a shorthand for the same four methods.
  • Persistence: data.Save(slot) writes the in-memory map to save_N.json; data.Load(slot) reads it back. Both return bool to indicate success.
  • In-memory utilities: HasKey(key) checks whether a key was loaded; Clear() wipes the in-memory map without touching the file on disk.
  • Static helpers: SaveData::Exists(slot) checks whether a file is present before attempting a load; SaveData::Delete(slot) removes a file and returns whether it existed.
  • Multi-slot pattern: slot 0 is the autosave (fired on level transition and quit); slots 1–3 are player-controlled manual saves written from the pause menu. This separation ensures autosaves never silently clobber checkpoints the player intentionally created.
  • Game-owned caching: GetSaveData(slot) returns a SaveData& managed by the engine, eliminating the need to juggle object lifetimes yourself.
