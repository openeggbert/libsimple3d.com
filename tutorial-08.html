Background: Skeletal animation in simple-3d

A skeletal mesh is a 3D model that contains a hierarchy of bones. Each animation clip records how those bones move over time. simple-3d stores clips as .ani files that live alongside the model in your Data/ folder. When you load a skinned model with AddModel("Models/Hero.fbx") the engine imports both the mesh and any embedded clips; you then drive them at runtime through an AnimationController component.

The controller supports blend layers: numbered slots that each play one clip at a time and whose outputs are combined before the final pose is computed. Layer 0 is the base layer and typically carries locomotion animations (idle, run, jump). Layer 1 sits on top and is ideal for upper-body actions (wave, aim, attack) that must play at the same time as the character runs. Clips on the same layer cross-fade automatically when you call Play(); clips on different layers blend additively.

Complete working example

The following program creates a third-person character that uses the CharacterController from Part 07 for movement and an AnimationController to drive a four-clip animation setup: idle, run, jump (all on layer 0) and a wave gesture (layer 1). Sprint speed is reflected in the run cycle with SetSpeed(), and the wave can be blended at a custom strength with SetWeight(). 

#include <Simple3D/Simple3D.h>
using namespace Simple3D;

class AnimDemo : public Game {

    Entity*              player_      = nullptr;
    AnimationController* anim_        = nullptr;
    CharacterController* cc_          = nullptr;
    Camera*              cam_         = nullptr;

    // Name of the clip currently active on layer 0
    std::string          currentClip_ = "";

public:
    void Start() override {
        SetWindowTitle("Skeletal Animation Demo");
        SetClearColor(Color(0.13f, 0.18f, 0.28f));
        SetAmbientColor(Color(0.22f, 0.25f, 0.32f));

        // ── Ground ──────────────────────────────────────────────
        Entity* ground = CreateEntity("Ground");
        ground->AddModel("Models/Plane.mdl");
        ground->SetScale(30.0f);
        ground->SetMaterialColor(Color(0.26f, 0.34f, 0.24f));
        ground->AddBoxCollider(Vector3(30.0f, 0.2f, 30.0f));
        ground->AddRigidBody(0.0f);   // static
        ground->SetCollisionLayer(CollisionLayer::StaticGeometry);

        // ── Directional light ────────────────────────────────────
        Entity* sun = CreateEntity("Sun");
        sun->AddDirectionalLight(Color(1.0f, 0.95f, 0.85f), 1.2f);
        sun->SetRotation(Quaternion(50.0f, 30.0f, 0.0f));
        sun->SetCastShadows(true);

        // ── Player ───────────────────────────────────────────────
        player_ = CreateEntity("Player");
        player_->SetPosition(0.0f, 0.0f, 0.0f);

        // Skinned mesh — must be loaded BEFORE AddAnimationController
        player_->AddModel("Models/Hero.fbx");

        // ── Animation setup ──────────────────────────────────────
        // 1. Attach the AnimationController component to the entity.
        //    This must be called before GetAnimationController().
        player_->AddAnimationController();

        // 2. Retrieve the pointer for later use.
        anim_ = player_->GetAnimationController();

        // 3. Start the idle animation on layer 0 immediately (no fade-in).
        anim_->Play("Anims/Idle.ani", 0, true, 0.0f);
        currentClip_ = "idle";

        // ── Physics / controller ─────────────────────────────────
        cc_ = player_->AddCharacterController(0.35f, 1.8f);
        cc_->SetGravity(20.0f);
        player_->SetCollisionLayer(CollisionLayer::Actor);
        player_->SetCollisionMask(MakeCollisionMask({
            CollisionLayer::StaticGeometry
        }));

        // ── Camera ───────────────────────────────────────────────
        cam_ = CreateCamera();
        cam_->Follow(player_, 6.0f, 2.0f);
        cam_->LookAt(player_);
        cam_->SetSmoothFollow(true, 5.0f);
    }

    void Update(float dt) override {
        // ── Movement input ───────────────────────────────────────
        float mx = 0.0f, mz = 0.0f;
        if (IsKeyDown(Key::A) || IsKeyDown(Key::Left))  mx -= 1.0f;
        if (IsKeyDown(Key::D) || IsKeyDown(Key::Right)) mx += 1.0f;
        if (IsKeyDown(Key::W) || IsKeyDown(Key::Up))    mz -= 1.0f;
        if (IsKeyDown(Key::S) || IsKeyDown(Key::Down))  mz += 1.0f;

        bool  sprinting = IsKeyDown(Key::LShift);
        float speed     = sprinting ? 7.0f : 4.0f;
        cc_->Move(Vector3(mx, 0.0f, mz) * speed, dt);

        if (IsKeyPressed(Key::Space))
            cc_->TryJump(10.0f);

        // ── Layer 0 : locomotion (idle / run / jump) ─────────────
        bool grounded = cc_->IsOnGround();
        bool moving   = (mx != 0.0f || mz != 0.0f);

        if (!grounded && currentClip_ != "jump") {
            // Became airborne — cross-fade into looping jump cycle
            anim_->Play("Anims/Jump.ani", 0, true, 0.15f);
            currentClip_ = "jump";

        } else if (grounded && moving && currentClip_ != "run") {
            // Landed or started moving — cross-fade to run
            anim_->Play("Anims/Run.ani", 0, true, 0.25f);
            currentClip_ = "run";

        } else if (grounded && !moving && currentClip_ != "idle") {
            // Came to a stop — cross-fade back to idle
            anim_->Play("Anims/Idle.ani", 0, true, 0.25f);
            currentClip_ = "idle";
        }

        // Adjust run-cycle speed to match sprint vs walk movement speed
        if (currentClip_ == "run") {
            anim_->SetSpeed("Anims/Run.ani", sprinting ? 1.5f : 1.0f);
        }

        // ── Layer 1 : upper-body wave overlay ────────────────────
        // Press E to begin or stop the wave; it plays independently
        // of whatever is running on the base layer (layer 0).
        if (IsKeyPressed(Key::E)) {
            if (anim_->IsPlaying("Anims/Wave.ani")) {
                // Already waving — fade it out over 0.2 s
                anim_->Stop("Anims/Wave.ani", 0.2f);
            } else {
                // Start wave on layer 1, non-looped, 0.15 s fade-in
                anim_->Play("Anims/Wave.ani", 1, false, 0.15f);
                // Apply 80 % weight — lets a hint of body motion show through
                anim_->SetWeight("Anims/Wave.ani", 0.8f);
            }
        }

        // Optional: inspect current playback position for sound cues
        // float t = anim_->GetTime("Anims/Run.ani"); // seconds into clip
    }

    void Stop() override {
        // Gracefully stop every active animation before the scene tears down
        if (anim_) anim_->StopAll();
    }
};

int main() {
    return Simple3D::CreateGame<AnimDemo>()->Run();
}

Controls: WASD / arrow keys to move, LShift to sprint, Space to jump, E to toggle the wave gesture on the upper body.

Step-by-step: every new API call explained

entity->AddAnimationController()

AddAnimationController() attaches the engine's animation component to an entity. You must call it after loading the skinned mesh with AddModel() so the component can discover the skeleton. The call is idempotent: calling it twice on the same entity has no effect and does not reset existing animation state. 

player_->AddModel("Models/Hero.fbx");   // load skinned mesh first
player_->AddAnimationController();       // then attach the controller

entity->GetAnimationController()

GetAnimationController() returns a pointer to the AnimationController that was attached by AddAnimationController(). If you call it before attaching the component, or on an entity that never had one, the return value is nullptr. Store the pointer in a member variable during Start() so you can call animation methods every frame without the overhead of a lookup. 

AnimationController* anim_ = player_->GetAnimationController();
if (!anim_) {
    // AddAnimationController() was not called — nothing will work
    return;
}

anim->Play(clipName, layer, loop, fadeTime)

Play() is the primary way to start an animation clip. It works as an exclusive play on the specified layer: any clip already on that layer is faded out over fadeTime seconds while the new clip fades in over the same duration. This gives you smooth, automatic cross-fading between states without any manual blending math.

ParameterTypeDefaultDescription
clipNamestringrequiredPath to the .ani clip file (relative to Data/)
layerint0Blend layer index. 0 = base, 1 = upper-body overlay, etc.
loopbooltrueWhether the clip loops continuously or plays once and freezes on the last frame
fadeTimefloat0.2fCross-fade duration in seconds. Use 0.0f for an instant cut.
 
// Shortest form — defaults: layer 0, looped, 0.2 s fade
anim_->Play("Anims/Idle.ani");

// Explicit: layer 0, looped, 0.25 s cross-fade
anim_->Play("Anims/Run.ani", 0, true, 0.25f);

// Jump is looped so it hovers during a long arc; instant cut on take-off
anim_->Play("Anims/Jump.ani", 0, true, 0.15f);

// Wave on layer 1 — plays non-looped alongside whatever is on layer 0
anim_->Play("Anims/Wave.ani", 1, false, 0.15f);

Tip

When loop is false the clip plays exactly once and then freezes on its last frame. If you want to return the character to idle after a one-shot wave, check IsPlaying() and call Stop() or issue a new Play() on that layer. Part 09 covers the Animation State Machine which automates this with AddAutoTransition().

anim->Stop(clipName, fadeTime)

Stop() ends a specific clip with a fade-out. During the fade window the clip's influence is smoothly reduced to zero before the component stops ticking it. Pass 0.0f for an immediate hard stop. 

anim_->Stop("Anims/Wave.ani", 0.2f);   // fade out over 200 ms
anim_->Stop("Anims/Run.ani",  0.0f);   // cut immediately

anim->StopAll()

StopAll() immediately halts every active clip on every layer. It is best called from Game::Stop() or just before destroying the entity, so the engine does not continue ticking dead animation state. 

void Stop() override {
    if (anim_) anim_->StopAll();
}

anim->IsPlaying(clipName)

Returns true while the named clip is active on any layer, including during its fade-out window. Use it to guard transitions, trigger sound effects at the right moment, or decide whether a one-shot action has finished. 

if (anim_->IsPlaying("Anims/Wave.ani")) {
    anim_->Stop("Anims/Wave.ani", 0.2f);
} else {
    anim_->Play("Anims/Wave.ani", 1, false, 0.15f);
}

anim->SetWeight(clipName, 0–1)

SetWeight() controls how strongly a clip contributes to the final pose, expressed as a normalised float in the range [0, 1]. A weight of 1.0 (the default) means the clip has full influence over the bones it animates. A weight of 0.5 blends halfway between the clip's target pose and whatever the lower layer is providing. Setting weight to 0.0 effectively mutes the clip without stopping it.

The most common use case is partial upper-body overlays. For example, an aim pose on layer 1 at weight 0.6 lets 40 % of the run-cycle's arm motion remain visible, making the blend look natural rather than robotic. 

// Full-strength wave — arms follow the animation entirely
anim_->Play("Anims/Wave.ani", 1, false, 0.15f);
anim_->SetWeight("Anims/Wave.ani", 1.0f);

// Partial aim overlay — 60 % influence lets run-cycle arms bleed through
anim_->Play("Anims/Aim.ani", 1, true, 0.2f);
anim_->SetWeight("Anims/Aim.ani", 0.6f);

// Fade influence down dynamically each frame
float aimStrength = 1.0f - (distanceToTarget / maxAimDistance);
anim_->SetWeight("Anims/Aim.ani", aimStrength);

anim->SetSpeed(clipName, multiplier)

SetSpeed() changes the playback rate of a clip without re-issuing a Play() call. A multiplier of 1.0 is normal speed, 2.0 plays the clip twice as fast, and 0.5 plays it in slow motion. Negative values are not supported.

The most practical application is matching the visual run cycle to the character's actual movement speed. When the player sprints you increase both the physics movement speed and the animation playback rate so the feet do not slide against the ground. 

// Normal-speed run cycle
anim_->SetSpeed("Anims/Run.ani", 1.0f);

// Sprint: movement speed is 1.75× faster, so animation is too
anim_->SetSpeed("Anims/Run.ani", 1.75f);

// Slow-motion power-up effect
anim_->SetSpeed("Anims/Idle.ani", 0.4f);

// Reset to default after the power-up expires
anim_->SetSpeed("Anims/Idle.ani", 1.0f);

anim->GetTime(clipName)

Returns the current playback position of a clip in seconds from the start of that clip. This is useful for synchronising sound effects with specific moments in an animation — for example, playing a footstep sound when the run cycle passes the foot-strike frame, or triggering a sword-clash effect at the peak of an attack animation. 

float t = anim_->GetTime("Anims/Run.ani");

// Run cycle is 0.8 s long; foot strike at 0.2 s and 0.6 s
if ((t > 0.18f && t < 0.22f) || (t > 0.58f && t < 0.62f)) {
    PlaySound("Sounds/footstep.wav", 0.6f);
}

Blend layers: combining animations simultaneously

The animation system uses numbered blend layers to let you play multiple clips at the same time without one wiping out the other. The two most important layers are:

LayerConventional useExample clips
0 — base layer Full-body locomotion: controls the whole skeleton Idle, Walk, Run, Jump, Fall, Land
1 — upper-body overlay Upper-body actions that run on top of locomotion Wave, Aim, Attack, Throw, Gesture

Each layer is independent. Calling Play("Anims/Wave.ani", 1, ...) starts the wave on layer 1 while layer 0 keeps running whatever locomotion clip was already active. The engine blends the two outputs bone by bone, with layer 1 winning on any bone it influences at the current weight.

There is no fixed limit to the number of layers. Layers 0 and 1 are a convention, not a rule. You could use layer 2 for a facial animation and layer 3 for procedural IK corrections if needed. 

// Layer 0 — base locomotion
anim_->Play("Anims/Run.ani",  0, true,  0.25f);

// Layer 1 — upper-body wave plays simultaneously
anim_->Play("Anims/Wave.ani", 1, false, 0.15f);
anim_->SetWeight("Anims/Wave.ani", 0.85f);

// The character now runs AND waves at the same time.
// When the wave finishes (non-looped), only the run clip remains visible.
// Pressing E again calls Stop() to fade the wave out early.

The idle / run / jump pattern

Almost every action game uses the same three-clip pattern on layer 0 to cover a character's basic locomotion needs. The idea is simple: track which clip is currently active in a string variable, then compare it with the desired state each frame and issue a Play() call only when the state changes. Calling Play() on a clip that is already playing on that layer is harmless but unnecessary; guarding with currentClip_ avoids restarting the fade-in counter.

Input state currentClip_ Action ─────────────────────────────────────────────────────────────────── grounded, not moving → "idle" Play Idle.ani (loop) grounded, moving → "run" Play Run.ani (loop) airborne → "jump" Play Jump.ani (loop) Transition example: was "idle", player pressed W → Play("Anims/Run.ani", 0, true, 0.25f) ← cross-fade 250 ms currentClip_ = "run" player releases W (still grounded) → Play("Anims/Idle.ani", 0, true, 0.25f) ← cross-fade 250 ms currentClip_ = "idle"

The guard pattern — if (currentClip_ != "run") — prevents Play() from being called every single frame while the character is already running, which would restart the cross-fade every frame and cause a visual stutter. 

void updateLocomotion(bool grounded, bool moving) {
    if (!grounded && currentClip_ != "jump") {
        anim_->Play("Anims/Jump.ani", 0, true, 0.15f);
        currentClip_ = "jump";

    } else if (grounded && moving && currentClip_ != "run") {
        anim_->Play("Anims/Run.ani", 0, true, 0.25f);
        currentClip_ = "run";

    } else if (grounded && !moving && currentClip_ != "idle") {
        anim_->Play("Anims/Idle.ani", 0, true, 0.25f);
        currentClip_ = "idle";
    }
}

Going further

The idle/run/jump pattern implemented here by hand is exactly the problem that Part 09 — Animation State Machine solves declaratively with AddAnimStateMachine(), AddTransition(), and AddAutoTransition(). The state machine approach is less error-prone for complex graphs (walk/run/sprint/crouch/swim) and is recommended for production projects. This manual pattern is still useful when you want fine control over exactly when and how each transition fires.

Lua equivalent

The same animation system is accessible from a Lua script attached to the player entity with entity:AddScript("Scripts/player_anim.lua"). The Lua API mirrors the C++ API one-to-one: entity and game are the two implicit globals available in every script. 

-- Scripts/player_anim.lua
-- Attach to the Player entity with entity:AddScript()
-- Requires that CharacterController was already added in C++ (or a prior script).

local anim        = nil
local cc          = nil
local currentClip = ""

function Start()
    -- Attach the animation controller to this entity
    entity:AddAnimationController()
    anim = entity:GetAnimationController()

    -- Start idle immediately (no fade)
    anim:Play("Anims/Idle.ani", 0, true, 0.0)
    currentClip = "idle"

    -- Retrieve the character controller set up in C++ or another script
    cc = entity:GetCharacterController()
end

function Update(dt)
    -- Movement
    local mx, mz = 0, 0
    if game:IsKeyDown("A") then mx = mx - 1 end
    if game:IsKeyDown("D") then mx = mx + 1 end
    if game:IsKeyDown("W") then mz = mz - 1 end
    if game:IsKeyDown("S") then mz = mz + 1 end

    local sprinting = game:IsKeyDown("LShift")
    local speed = sprinting and 7.0 or 4.0
    cc:Move(Vector3(mx, 0, mz) * speed, dt)

    if game:IsKeyPressed("Space") then
        cc:TryJump(10.0)
    end

    -- Layer 0 locomotion
    local grounded = cc:IsOnGround()
    local moving   = (mx ~= 0 or mz ~= 0)

    if not grounded and currentClip ~= "jump" then
        anim:Play("Anims/Jump.ani", 0, true, 0.15)
        currentClip = "jump"

    elseif grounded and moving and currentClip ~= "run" then
        anim:Play("Anims/Run.ani", 0, true, 0.25)
        currentClip = "run"

    elseif grounded and not moving and currentClip ~= "idle" then
        anim:Play("Anims/Idle.ani", 0, true, 0.25)
        currentClip = "idle"
    end

    -- Keep run-cycle speed in sync with sprint
    if currentClip == "run" then
        anim:SetSpeed("Anims/Run.ani", sprinting and 1.5 or 1.0)
    end

    -- Layer 1 wave toggle (E key)
    if game:IsKeyPressed("E") then
        if anim:IsPlaying("Anims/Wave.ani") then
            anim:Stop("Anims/Wave.ani", 0.2)
        else
            anim:Play("Anims/Wave.ani", 1, false, 0.15)
            anim:SetWeight("Anims/Wave.ani", 0.8)
        end
    end
end

Lua note

The Lua equivalent above gives you hot-reloadable animation logic without recompiling. For prototyping new gestures or adjusting fade times, editing the .lua file and reloading the scene is much faster than a full C++ rebuild. Once you are happy with the feel, you can port the final values back to C++ or keep the script as the production implementation.

Common mistakes

Watch out for these pitfalls

Calling GetAnimationController() before AddAnimationController().
GetAnimationController() returns nullptr if the component has not been attached. Storing that null and calling methods on it later causes an immediate crash. Always call AddAnimationController() first, then retrieve the pointer.

Adding the animation controller before the model.
The component needs to bind to the skeleton that lives inside the skinned mesh. If you call AddAnimationController() before AddModel(), there is no skeleton to bind to and every subsequent Play() call will silently do nothing. Order matters: model first, controller second.

Calling Play() every frame without guarding.
Issuing Play("Anims/Run.ani", ...) inside Update() without checking currentClip_ restarts the cross-fade on every single frame, causing an endless micro-fade that never lets the clip settle. The character will appear to flicker or stutter. Always guard with a current-state variable or use IsPlaying().

Forgetting that SetWeight() applies immediately but is not persistent.
When you call Play() again on the same clip the weight resets to 1.0 internally as part of the fade-in. If you need a non-default weight after each transition, always call SetWeight() immediately after Play().

Using layer 0 for everything.
Placing a wave animation on layer 0 will cross-fade away from the run clip, leaving the character standing still while waving. Upper-body overlays must go on layer 1 (or higher) so they do not displace the locomotion clip.

Non-looped clips that freeze on the last frame.
A one-shot wave animation (loop = false) plays once and then holds its final pose indefinitely. If you do not call Stop() afterwards the character will remain in the wave-end pose forever. Either check IsPlaying() and call Stop() when it finishes, or use the Animation State Machine (Part 09) which handles this automatically via AddAutoTransition().

SetSpeed() with a value of 0 or negative.
A speed of 0.0f freezes the clip on its current frame, which looks like a bug at runtime. Negative values are not supported and produce undefined behaviour. Clamp speed to a small positive minimum such as 0.05f if you need an extreme slow-motion effect.

Putting it all together: animation lifecycle diagram

Entity "Player" │ ├── AddModel("Models/Hero.fbx") ← imports skeleton + clip data │ └── AddAnimationController() ← binds to skeleton │ ├── Layer 0 (base locomotion) │ ├── Idle.ani loop=true weight=1.0 speed=1.0 │ ├── Run.ani loop=true weight=1.0 speed=1.0 → 1.5 (sprint) │ └── Jump.ani loop=true weight=1.0 speed=1.0 │ │ │ └── Play() cross-fades between these automatically │ (only one active at a time per layer) │ └── Layer 1 (upper-body overlay) └── Wave.ani loop=false weight=0.8 speed=1.0 │ └── Plays simultaneously with layer 0 Stopped via Stop() or when clip ends

What you learned

  • AddAnimationController / GetAnimationController — always call AddModel() first, then attach the controller and store the returned pointer for use in Update().
  • Play(clipName, layer, loop, fadeTime) — plays a clip exclusively on the given layer with an automatic cross-fade, smoothly blending out the previous clip on that same layer.
  • Stop(clipName, fadeTime) — fades out one specific clip; StopAll() clears every layer instantly.
  • IsPlaying(clipName) — lets you test whether a clip is still running so you can guard transitions and avoid redundant Play() calls.
  • SetWeight(clipName, 0–1) — controls how strongly a clip blends into the final pose; essential for partial upper-body overlays that should not completely override locomotion arm motion.
  • SetSpeed(clipName, multiplier) — adjusts playback rate at runtime to match visual cycle speed with physical movement speed.
  • GetTime(clipName) — reads the current playback position in seconds, useful for synchronising sound effects to specific animation frames.
  • Blend layers — layer 0 is the base (locomotion); layer 1 is an upper-body overlay. Different layers compose rather than compete, letting you run and wave at the same time.
  • Idle / run / jump pattern — guard every Play() call with a current-state variable to prevent restarting the fade-in every frame.

In Part 09 you will replace this hand-written state machine with the declarative AnimStateMachine API, adding condition-driven transitions, one-shot auto-returns, and float-threshold blending — all without a single if statement in Update().
