Background: how scenes work in simple-3d

In simple-3d, a scene is the set of all non-persistent entities currently alive in the world. When you call LoadScene or UnloadScene, every entity that was not explicitly marked persistent is destroyed and its memory is freed. Persistent entities — typically the player character and global managers — keep their components intact and simply find themselves in the new world geometry.

Scene files are .mc3.xml format, imported through the Mc3::Import bridge internally. You never call Mc3::Import directly; LoadScene does it for you.

Level list (recommended)

Register all your level file paths with SetLevelList once in Start(). Then use LoadLevel(index) or LoadNextLevel() to advance through them. GetCurrentLevelIndex() and GetLevelCount() let you guard against going past the last level.

Direct load (flexible)

For menus, cutscenes, or non-linear games, call LoadScene("path/to/scene.mc3.xml") directly. An optional lambda callback fires after the new scene is ready — useful for repositioning the player or triggering a fade-in from within the same call.

Recommended asset layout

Keep your level files under Data/ so the resource cache resolves paths consistently regardless of working directory: 

Data/
  levels/
    01.mc3.xml    // level 1 — grassy plains
    02.mc3.xml    // level 2 — underground caves
    03.mc3.xml    // level 3 — boss arena
  menus/
    main_menu.mc3.xml   // title screen geometry
  models/
    Player.mdl
  Textures/
    ...

Complete working example

The example below builds a three-level linear platformer. The player entity is created once, marked persistent, and carries through every level transition. A trigger volume in each level sets reachedExit_; the game then fades out, advances to the next level, and fades back in. After the final level, a win screen appears instead. 

#include <Simple3D/Simple3D.h>

using namespace Simple3D;

// ────────────────────────────────────────────────────────────────────
// LevelGame — demonstrates the full scene-management API.
// ────────────────────────────────────────────────────────────────────
class LevelGame : public Game
{
public:
    // ── Persistent objects (survive scene transitions) ───────────────
    Entity*  player_        = nullptr;
    Camera*  cam_           = nullptr;

    // ── Per-scene objects (recreated each level) ─────────────────────
    Entity*  exitTrigger_   = nullptr;

    // ── UI (always preserved) ────────────────────────────────────────
    Label*   levelLabel_    = nullptr;
    Label*   hintLabel_     = nullptr;
    UI::Panel* winPanel_    = nullptr;
    Label*   winLabel_      = nullptr;
    UI::Button* quitBtn_    = nullptr;

    // ── State ────────────────────────────────────────────────────────
    bool  reachedExit_   = false;
    bool  transitioning_ = false; // lock out re-entrance during fades

    // ── Fade duration used for every transition ───────────────────────
    static constexpr float FADE_TIME = 0.45f;

    // ────────────────────────────────────────────────────────────────
    // Start() — runs once before the first frame.
    // Create the persistent player, camera, and HUD here;
    // then register the level list and load level 0.
    // ────────────────────────────────────────────────────────────────
    void Start() override
    {
        SetAppName("LevelGame");

        // ── Persistent player ────────────────────────────────────────
        // Create once.  SetPersistentEntity ensures it is NOT
        // destroyed when UnloadScene / LoadScene is called.
        player_ = CreateEntity("Player");
        player_->AddModel("Models/Box.mdl");
        player_->SetScale(0.7f);

        CharacterController* cc = player_->AddCharacterController(0.35f, 1.6f);
        cc->SetGravity(22.0f);
        player_->SetCollisionLayer(CollisionLayer::Actor);
        player_->SetCollisionMask(MakeCollisionMask({
            CollisionLayer::StaticGeometry,
            CollisionLayer::Trigger
        }));

        // Mark player as persistent BEFORE the first LoadLevel call.
        SetPersistentEntity(player_);

        // ── Camera ───────────────────────────────────────────────────
        // CreateCamera() is always preserved; no need to call
        // SetPersistentEntity on it.
        cam_ = CreateCamera();
        cam_->Follow(player_, 8.0f, 3.5f);
        cam_->SetSmoothFollow(true, 6.0f);
        cam_->SetCollisionEnabled(true);

        // ── HUD ──────────────────────────────────────────────────────
        // UI elements are always preserved across scene transitions.
        levelLabel_ = CreateLabel("Level 1");
        levelLabel_->SetPosition(10, 10);
        levelLabel_->SetFontSize(20);
        levelLabel_->SetColor(Color::WHITE);

        hintLabel_ = CreateLabel("Reach the glowing exit portal");
        hintLabel_->SetPosition(10, 38);
        hintLabel_->SetFontSize(14);
        hintLabel_->SetColor(Color(0.85f, 0.85f, 0.85f, 1.0f));

        // ── Win screen (hidden until all levels are complete) ─────────
        winPanel_ = CreatePanel();
        winPanel_->SetSize(420, 200);
        winPanel_->SetAnchor(UI::Anchor::Center);
        winPanel_->SetPosition(0, 0);
        winPanel_->SetColor(Color(0.0f, 0.0f, 0.0f, 0.75f));
        winPanel_->SetVisible(false);

        winLabel_ = CreateLabel("You Win!");
        winLabel_->SetPosition(135, 50);
        winLabel_->SetFontSize(36);
        winLabel_->SetColor(Color(1.0f, 0.9f, 0.2f, 1.0f));
        winPanel_->AddChild(winLabel_);

        quitBtn_ = CreateButton("Quit");
        quitBtn_->SetSize(140, 40);
        quitBtn_->SetPosition(140, 130);
        quitBtn_->SetColor(Color(0.15f, 0.4f, 0.15f, 0.9f));
        quitBtn_->SetOnClick([this] { Quit(); });
        winPanel_->AddChild(quitBtn_);

        // ── Level list ───────────────────────────────────────────────
        // Register all levels in play order.
        // LoadLevel(0) immediately loads the first scene and
        // sets GetCurrentLevelIndex() to 0.
        SetLevelList({
            "levels/01.mc3.xml",
            "levels/02.mc3.xml",
            "levels/03.mc3.xml"
        });

        // Fade in as the first level appears (no callback needed here).
        FadeIn(FADE_TIME);
        LoadLevel(0);

        // Set up per-scene objects after LoadLevel.
        SetupCurrentLevel();
    }

    // ────────────────────────────────────────────────────────────────
    // SetupCurrentLevel() — called once after every scene swap.
    // Spawns the player at the level's start point and hooks up
    // the exit trigger for this particular scene.
    // ────────────────────────────────────────────────────────────────
    void SetupCurrentLevel()
    {
        int idx = GetCurrentLevelIndex();

        // Update the HUD label.
        levelLabel_->SetText("Level " + String(idx + 1) +
                              " / " + String(GetLevelCount()));

        // Each level defines a spawn point entity named "Spawn".
        Entity* spawn = FindEntity("Spawn");
        if (spawn)
            player_->SetPosition(spawn->GetPosition() + Vector3(0.0f, 1.0f, 0.0f));
        else
            player_->SetPosition(0.0f, 2.0f, 0.0f); // fallback

        // The exit trigger volume is placed by the level artist.
        // We look it up by name and wire the callback here in code
        // so the game logic stays in C++ rather than the scene file.
        exitTrigger_ = FindEntity("Exit");
        if (exitTrigger_)
        {
            exitTrigger_->SetOnTriggerEnter([this](Entity* other) {
                if (other == player_)
                    reachedExit_ = true;
            });
        }
    }

    // ────────────────────────────────────────────────────────────────
    // Update() — player movement + exit detection.
    // ────────────────────────────────────────────────────────────────
    void Update(float dt) override
    {
        HandleMovement(dt);

        // When the player reaches the exit and no transition is
        // already in progress, start the fade-out / advance / fade-in chain.
        if (reachedExit_ && !transitioning_)
        {
            reachedExit_   = false;
            transitioning_ = true;

            FadeOut(FADE_TIME, [this] {
                // Are there more levels?
                if (GetCurrentLevelIndex() + 1 < GetLevelCount())
                {
                    // Advance to the next level in the list.
                    LoadNextLevel();
                    SetupCurrentLevel();

                    FadeIn(FADE_TIME);
                    transitioning_ = false;
                }
                else
                {
                    // All levels complete — show the win screen and fade in.
                    winPanel_->SetVisible(true);
                    hintLabel_->SetVisible(false);
                    FadeIn(FADE_TIME);
                    transitioning_ = false;
                }
            });
        }

        if (IsKeyPressed(Key::Escape))
            Quit();
    }

    // ────────────────────────────────────────────────────────────────
    void HandleMovement(float dt)
    {
        CharacterController* cc = player_->GetComponent<CharacterController>();
        if (!cc) return;

        float mx = 0.0f, mz = 0.0f;
        if (IsKeyDown(Key::Left)  || IsKeyDown(Key::A)) mx -= 1.0f;
        if (IsKeyDown(Key::Right) || IsKeyDown(Key::D)) mx += 1.0f;
        if (IsKeyDown(Key::Up)    || IsKeyDown(Key::W)) mz -= 1.0f;
        if (IsKeyDown(Key::Down)  || IsKeyDown(Key::S)) mz += 1.0f;

        cc->Move(Vector3(mx, 0.0f, mz) * 5.0f, dt);

        if (IsKeyPressed(Key::Space) && cc->IsOnGround())
            cc->TryJump(10.0f);
    }
};

int main()
{
    return Simple3D::CreateGame<LevelGame>()->Run();
}

Step-by-step explanation

LoadScene(path, callback={})

LoadScene is the fundamental scene-switching call. It performs the following sequence atomically: destroys all non-persistent entities, then imports the given .mc3.xml file via Mc3::Import internally. You do not need to include <Simple3D/Mc3/Mc3Bridge.h> or call Import yourself — LoadScene handles it.

The optional second argument is a std::function<void()> callback that fires once, synchronously, after the new scene has been imported but before the next Update frame. Use it to reposition the player, rebuild nav-mesh references, or trigger a FadeIn — all before the player ever sees the new scene. 

// Simplest form — load and forget
LoadScene("levels/02.mc3.xml");

// With callback — reposition player immediately after import
LoadScene("levels/02.mc3.xml", [this] {
    Entity* spawn = FindEntity("Spawn");
    if (spawn)
        player_->SetPosition(spawn->GetPosition());
    FadeIn(0.5f);
});

UnloadScene()

UnloadScene destroys all non-persistent entities and clears the physics world without loading anything new in their place. After the call, only persistent entities and UI elements remain. This is useful when you need a clean empty scene before procedurally generating content in C++, or before showing a menu that has no 3D geometry at all. 

// Clear geometry before building a procedural level
UnloadScene();
BuildProceduralDungeon();   // your own C++ level generator

// Unload and then let LoadScene fill the gap
UnloadScene();
LoadScene("menus/main_menu.mc3.xml");

Note that LoadScene calls UnloadScene first internally, so you rarely need to call both in sequence yourself.

FadeOut(duration, callback={}) and FadeIn(duration)

These two methods animate a full-screen black overlay. FadeOut interpolates the overlay opacity from 0 (transparent) to 1 (fully opaque) over duration seconds; once fully opaque, the optional callback fires. FadeIn runs the reverse animation: from fully opaque back to transparent.

The canonical transition pattern is: call FadeOut, do your scene swap inside its callback, then immediately call FadeIn. The player sees a smooth dip to black and back, covering any pop-in from level loading: 

FadeOut(0.4f, [this] {
    // Screen is fully black — safe to swap scene now.
    LoadNextLevel();
    SetupCurrentLevel();

    // Begin fading the new scene back in.
    FadeIn(0.4f);
});

If you call FadeOut or FadeIn while a previous fade is still running, the new call replaces it immediately from wherever the animation currently is. Both methods also accept an optional second callback to FadeIn if you need to know when the scene is fully visible again: 

FadeIn(0.5f); // no callback — fire and forget the reveal animation

// Or call FadeOut with no callback for a permanent blackout (cutscenes, etc.)
FadeOut(1.0f);

SetPersistentEntity(Entity*)

By default, every entity is destroyed when UnloadScene or LoadScene is called. Calling SetPersistentEntity on an entity exempts it from that destruction. Persistent entities keep all their components — CharacterController, AnimationController, AudioSource, attached scripts — and simply remain in the world when non-persistent entities are swept away.

Two important categories are always preserved without needing this call:

  • The camera — it lives outside the entity list entirely.
  • All UI elements (Label, Image, Panel, Button, ProgressBar, Slider) — they are attached to the UI subsystem, not the scene graph.

SetPersistentEntity can be called multiple times with different entities; each call adds one entity to the persistent set. Call it in Start() before the first LoadLevel or LoadScene so the entity is registered before any scene swap occurs. 

void Start() override {
    player_ = CreateEntity("Player");
    player_->AddCharacterController(0.35f, 1.6f);
    SetPersistentEntity(player_);   // survives all future LoadScene calls

    // You can mark multiple entities persistent if needed:
    musicManager_ = CreateEntity("MusicManager");
    musicManager_->AddScript("Scripts/music_manager.lua");
    SetPersistentEntity(musicManager_);

    // Camera is always persistent — no call needed.
    cam_ = CreateCamera();
    cam_->Follow(player_, 8.0f, 3.0f);
}

SetLevelList(vector<string> paths)

SetLevelList registers an ordered sequence of scene file paths as the game's level playlist. After calling it, LoadLevel(index) and LoadNextLevel() navigate within that list. Call it once in Start() with all level paths known at startup: 

SetLevelList({
    "levels/01.mc3.xml",   // index 0
    "levels/02.mc3.xml",   // index 1
    "levels/03.mc3.xml",   // index 2 — last level
});

You can call SetLevelList again later to replace the playlist entirely — useful if your game unlocks bonus levels at runtime, or if you have separate act structures with different level pools.

LoadLevel(int index)

Loads levelList[index] immediately. If index is out of bounds (negative or ≥ GetLevelCount()) the call is a silent no-op. After a successful load, GetCurrentLevelIndex() returns the new index. Use LoadLevel(0) in Start() to begin from the first level, or LoadLevel(savedIndex) to resume from a saved game. 

// Begin at the first level
LoadLevel(0);

// Resume from a saved position
int savedIdx = GetSaveData(0).GetInt("levelIndex", 0);
LoadLevel(savedIdx);

// Jump to the final level regardless of count
LoadLevel(GetLevelCount() - 1);

LoadNextLevel()

A convenience wrapper that loads levelList[GetCurrentLevelIndex() + 1]. If the current level is already the last one, the call is a no-op. Always guard with a bounds check before calling if you want to branch on the final level: 

if (GetCurrentLevelIndex() + 1 < GetLevelCount())
{
    LoadNextLevel();   // safe — there is a next level
}
else
{
    ShowCredits();     // custom end-game flow
}

GetCurrentLevelIndex()

Returns the zero-based index of the currently loaded level within the list registered by SetLevelList. Returns -1 if LoadLevel has not yet been called (i.e. no level from the list is active). This is the correct way to display "Level X of Y" HUD text and to determine which level to save in a checkpoint system. 

int idx   = GetCurrentLevelIndex();  // 0-based; -1 = no level loaded
int total = GetLevelCount();          // size of the level list

levelLabel_->SetText("Level " + String(idx + 1) +
                      " / "    + String(total));

// Save checkpoint
GetSaveData(0).Set("levelIndex", idx);
GetSaveData(0).Save(0);

GetLevelCount()

Returns the number of entries in the list passed to SetLevelList. Returns 0 if no list has been set. Use it to bound-check before advancing and to compute progress fractions (e.g. for a progress bar): 

float progress = (float)(GetCurrentLevelIndex() + 1) /
                 (float) GetLevelCount();
progressBar_->SetValue(progress);

Patterns and tips

Restart the current level

When the player falls off the map or runs out of lives, reload the same level using its index: 

void OnPlayerDied()
{
    FadeOut(0.3f, [this] {
        LoadLevel(GetCurrentLevelIndex());  // same index = restart
        SetupCurrentLevel();
        FadeIn(0.3f);
    });
}

Non-linear level selection (hub world)

For hub-style games where the player chooses which level to enter, call LoadLevel(index) directly with the chosen index rather than using LoadNextLevel: 

// Called when the player touches portal entity "Portal_2"
void OnPortalEntered(int levelIndex)
{
    FadeOut(0.5f, [this, levelIndex] {
        LoadLevel(levelIndex);
        SetupCurrentLevel();
        FadeIn(0.5f);
    });
}

Saving and restoring level progress

Integrate GetCurrentLevelIndex() with the SaveData system from Part 19 to persist progress across sessions: 

void Start() override {
    SetAppName("MyGame");

    SetLevelList({"levels/01.mc3.xml", "levels/02.mc3.xml", "levels/03.mc3.xml"});

    // Try to load the save; fall back to level 0.
    auto& save = GetSaveData(0);
    int startLevel = 0;
    if (save.Load(0))
        startLevel = save.GetInt("levelIndex", 0);

    FadeIn(0.5f);
    LoadLevel(startLevel);
    SetupCurrentLevel();
}

void OnLevelComplete()
{
    // Checkpoint: save progress before transitioning.
    auto& save = GetSaveData(0);
    save.Set("levelIndex", GetCurrentLevelIndex() + 1);
    save.Save(0);

    FadeOut(0.4f, [this] {
        LoadNextLevel();
        SetupCurrentLevel();
        FadeIn(0.4f);
    });
}

Using LoadScene directly (no level list)

For non-linear games or one-off scene loads (a main menu, a cutscene, a settings room), you can bypass the level list entirely and call LoadScene with a specific path: 

// Return to main menu from anywhere
void GoToMainMenu()
{
    FadeOut(0.5f, [this] {
        LoadScene("menus/main_menu.mc3.xml", [this] {
            // Position the menu camera once geometry is ready.
            cam_->SetPosition(Vector3(0.0f, 2.0f, -5.0f));
            cam_->LookAt(Vector3::ZERO);
            FadeIn(0.5f);
        });
    });
}

Quick reference

Method Returns Notes
LoadScene(path, callback={}) — Unloads current scene, imports .mc3.xml; callback fires after import.
UnloadScene() — Destroys all non-persistent entities; no new scene is loaded.
FadeOut(duration, callback={}) — Animates overlay opacity 0→1; callback fires when fully opaque.
FadeIn(duration) — Animates overlay opacity 1→0; reveals the scene.
SetPersistentEntity(Entity*) — Exempts entity from destruction on scene unload; call before first LoadScene.
SetLevelList(vector<string>) — Registers the ordered level path list; replaces any previous list.
LoadLevel(int index) — Loads levelList[index]; no-op if out of bounds.
LoadNextLevel() — Loads levelList[currentIndex + 1]; no-op if already last.
GetCurrentLevelIndex() int Zero-based index of the active level; -1 before first LoadLevel.
GetLevelCount() int Size of the registered level list; 0 if none set.

Lua equivalent

All scene management methods are available on the game global inside any Lua script attached with entity:AddScript(). The signatures mirror C++ exactly. Note that you cannot call SetPersistentEntity from Lua on a C++-created entity; mark entities persistent from your Start() override in C++ before handing scripts any entity references. 

-- Scripts/level_controller.lua
-- Attach this to a scene-management entity that is itself marked
-- persistent from C++.  Manages fade transitions and level advances.

local reachedExit   = false
local transitioning = false
local FADE_TIME     = 0.45

function Start()
    -- Register levels and load the first one.
    game:SetLevelList({
        "levels/01.mc3.xml",
        "levels/02.mc3.xml",
        "levels/03.mc3.xml"
    })

    game:FadeIn(FADE_TIME)
    game:LoadLevel(0)
end

function Update(dt)
    if reachedExit and not transitioning then
        reachedExit   = false
        transitioning = true

        game:FadeOut(FADE_TIME, function()
            local idx   = game:GetCurrentLevelIndex()
            local total = game:GetLevelCount()

            if idx + 1 < total then
                game:LoadNextLevel()
            else
                -- All levels done — load credits scene directly.
                game:LoadScene("menus/credits.mc3.xml")
            end

            game:FadeIn(FADE_TIME)
            transitioning = false
        end)
    end

    if game:IsKeyPressed(Key.Escape) then
        game:Quit()
    end
end

-- Called from a trigger entity's OnTriggerEnter callback
-- (wired in C++ with entity:SetOnTriggerEnter)
function OnExitReached()
    reachedExit = true
end

-- Convenience: reload the current level (respawn after death)
function RestartLevel()
    if not transitioning then
        transitioning = true
        game:FadeOut(0.3, function()
            game:LoadLevel(game:GetCurrentLevelIndex())
            game:FadeIn(0.3)
            transitioning = false
        end)
    end
end

Common mistakes

  • Calling SetPersistentEntity after the first LoadScene. Mark entities persistent in Start() before any LoadLevel or LoadScene call. If you call SetPersistentEntity on an entity that has already been destroyed by a scene transition, the pointer is dangling and the call has undefined behaviour.
  • Forgetting the transitioning guard flag. If the player is moving quickly, a trigger can fire multiple times per second. Without a bool transitioning_ guard, each fire starts a new FadeOut which replaces the in-progress fade, resulting in a flickering screen and possibly loading the same level twice. Set transitioning_ = true immediately when you start a fade and reset it only inside the fade-in completion.
  • Calling LoadLevel with no level list registered. If you call LoadLevel(0) before SetLevelList, the call is a no-op and GetCurrentLevelIndex() stays at -1. Always call SetLevelList first.
  • Expecting GetCurrentLevelIndex() to work before the first LoadLevel. It returns -1 until a level from the list has been loaded. Compute your HUD text or save-checkpoint data only after LoadLevel has been called at least once.
  • Calling LoadScene or LoadLevel from inside a trigger callback during Update. Setting a flag (like reachedExit_ = true) inside the trigger callback and acting on it at the top of the next Update frame is much safer than calling FadeOut or LoadLevel directly inside the callback. Direct calls inside trigger callbacks can fire mid-physics step and leave entity pointers in an inconsistent state.
  • Using FadeOut / FadeIn without nesting the load inside the FadeOut callback. If you call FadeOut, then immediately call LoadNextLevel on the very next line (outside the callback), the scene is swapped while the fade is still animating. The player will see the new scene pop in before the screen goes black. Always do the scene swap inside the callback.
  • Marking the camera as persistent via SetPersistentEntity. The camera is already always preserved; you do not need — and should not try — to pass it to SetPersistentEntity. CreateCamera() returns a Camera*, not an Entity*, and the function signature requires an Entity*.
  • Calling LoadScene and forgetting that it internally calls UnloadScene first. Any entity pointer you hold that was created in the previous scene (and is not persistent) becomes invalid immediately after LoadScene returns. Null-out all scene-local entity pointers after a scene swap and re-acquire them via FindEntity.

What you learned

In this part you gained full control over the scene lifecycle in simple-3d:

  • Loading scenes: LoadScene(path, callback) destroys all non-persistent entities and imports a .mc3.xml file. The optional callback lets you reposition entities and trigger a fade-in immediately after import.
  • Clearing scenes: UnloadScene() tears down all non-persistent entities without loading anything new, leaving a clean slate for procedural content or menus with no 3D geometry.
  • Fade transitions: FadeOut(duration, callback) and FadeIn(duration) animate a black overlay so scene swaps feel polished rather than abrupt. The canonical pattern is fade out, swap scene inside the callback, fade in.
  • Persistent entities: SetPersistentEntity(entity) keeps a specific entity and all its components alive through every scene transition. The camera and UI elements are always preserved without this call.
  • Level lists: SetLevelList(paths) registers an ordered playlist. LoadLevel(index) and LoadNextLevel() navigate it; GetCurrentLevelIndex() and GetLevelCount() let you display progress and guard against out-of-bounds advances.
  • Best practices: use a transitioning_ flag to prevent re-entrant fades, null out non-persistent entity pointers after a scene swap, and always call SetPersistentEntity before the first scene load.
