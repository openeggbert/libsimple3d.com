Part 16 of 30Input Actions

New in this part

What you will learn

  • How to clear all bindings with ClearInputBindings() when switching between scenes with different control schemes.
  • How to persist player-customized bindings to disk with SaveInputBindings() and restore them on the next launch with LoadInputBindings().
  • The equivalent Lua scripting approach for all of the above.

The problem with raw input checks

In Parts 03 and 15 you read input directly: IsKeyPressed(Key::Space) or IsGamepadButtonPressed(GamepadButton::A). For a small prototype that is acceptable, but it creates three problems as your game grows:

  1. Duplication across every input source. Every action that can come from either keyboard or gamepad requires two separate checks. Jump with LCtrl, Space, and gamepad A? That is three checks joined by ||, repeated everywhere jump is relevant.
  2. No single place to change. If you decide to rebind jump from Space to Z, you must hunt down every raw check in every file.
  3. No path to player remapping. Letting the player customize controls requires a rebinding UI that writes back to whatever data structure holds the key-to-action relationship. With scattered raw checks there is no such structure.

Input Actions solve all three problems. You declare each action once, attach as many input sources as you like, and then query by the abstract name. Adding or removing sources — or saving them to a JSON file — happens in exactly one place. 

// Before: raw checks — keyboard and gamepad duplicated in every location
if (IsKeyPressed(Key::LCtrl) || IsKeyPressed(Key::Space) ||
    IsGamepadButtonPressed(GamepadButton::A))
    cc_->TryJump(12.0f);

// After: one binding declared in Start(), one query in Update()
if (IsActionPressed("Jump"))
    cc_->TryJump(12.0f);

Concepts at a glance

TermMeaning
Action A named button event. Any number of keys and gamepad buttons can be sources. Query with IsActionDown / IsActionPressed.
Axis2D A named 2D directional input. Keyboard key-sets and analog sticks can both be sources. Query with GetAxis2D, which returns a Vector2.
Binding The association between an abstract name and its concrete input sources. Created via BindAction() or BindAxis2D().
IsActionDown True every frame any bound source is held down.
IsActionPressed True only on the very first frame any bound source transitions from released to pressed.
Normalized result GetAxis2D clamps the vector length to at most 1, so holding Up+Right on the keyboard is no faster than holding just Up.

Complete working example

The example below builds a third-person platformer with full keyboard and gamepad support, a jump action, a restart action, a quit action, and persistence of bindings across launches. Every new concept introduced in this part is used at least once. 

// InputActionsDemo.cpp
// Part 16 — Input Actions / Control Mapping
// Build: link against simple-3d

#include <Simple3D/Simple3D.h>
#include <cmath>      // std::atan2
using namespace Simple3D;

static constexpr float MOVE_SPEED  = 6.0f;
static constexpr float JUMP_VEL   = 12.0f;
static constexpr float RAD_TO_DEG = 57.29578f;

class InputActionsDemo : public Game {
public:
    void Start() override
    {
        SetAppName("InputActionsDemo");

        // ── Input bindings ────────────────────────────────────────────────
        //  Declared once in Start().  Any source triggers the same query.
        //
        //  Pattern: ResetInputBindings() first, then try LoadInputBindings().
        //  If the file does not exist (first launch), apply the built-in defaults.
        ResetInputBindings();
        if (!LoadInputBindings()) {
            ApplyDefaultBindings();
        }

        // ── Ground ────────────────────────────────────────────────────────
        Entity* ground = CreateEntity("Ground");
        ground->AddModel("Models/Plane.mdl");
        ground->AddRigidBody(0.0f);
        ground->AddBoxCollider(Vector3(30.0f, 0.2f, 30.0f));
        ground->SetCollisionLayer(CollisionLayer::StaticGeometry);

        // A platform in the air to practise jumping
        Entity* platform = CreateEntity("Platform");
        platform->SetPosition(5.0f, 2.0f, 0.0f);
        platform->AddModel("Models/Box.mdl");
        platform->AddRigidBody(0.0f);
        platform->AddBoxCollider(Vector3(4.0f, 0.4f, 4.0f));
        platform->SetCollisionLayer(CollisionLayer::StaticGeometry);

        // ── Player ────────────────────────────────────────────────────────
        player_ = CreateEntity("Player");
        player_->SetPosition(0.0f, 1.0f, 0.0f);
        player_->AddModel("Models/Hero.mdl");

        cc_ = player_->AddCharacterController(0.35f, 1.8f);
        cc_->SetGravity(25.0f);
        player_->SetCollisionLayer(CollisionLayer::Actor);
        player_->SetCollisionMask(MakeCollisionMask({
            CollisionLayer::StaticGeometry,
            CollisionLayer::Trigger
        }));

        // ── Camera ────────────────────────────────────────────────────────
        Camera* cam = CreateCamera();
        cam->Follow(player_, 7.0f, 2.5f);
        cam->LookAt(player_);
        cam->SetSmoothFollow(true, 5.0f);

        // ── Directional light ─────────────────────────────────────────────
        Entity* sun = CreateEntity("Sun");
        sun->SetRotation(Quaternion(45, 30, 0));
        sun->AddDirectionalLight(Color::WHITE, 1.2f);

        // ── HUD ───────────────────────────────────────────────────────────
        hintLabel_ = CreateLabel(
            "Arrows/Stick=move  Space/A=jump  R/Start=restart  Esc/B=quit");
        hintLabel_->SetPosition(20, 20);
        hintLabel_->SetFontSize(16);
        hintLabel_->SetColor(Color(0.9f, 0.9f, 0.9f, 1.0f));

        axisLabel_ = CreateLabel("Move axis: (0.00, 0.00)");
        axisLabel_->SetPosition(20, 46);
        axisLabel_->SetFontSize(16);
        axisLabel_->SetColor(Color(0.7f, 1.0f, 0.7f, 1.0f));

        jumpLabel_ = CreateLabel("Jump: idle");
        jumpLabel_->SetPosition(20, 72);
        jumpLabel_->SetFontSize(16);
        jumpLabel_->SetColor(Color(1.0f, 0.85f, 0.3f, 1.0f));
    }

    void Update(float dt) override
    {
        // IsActionPressed — true on the first frame the source is pressed.
        // Use for one-shot events: quit, restart, open menu, jump.
        if (IsActionPressed("Quit"))    { Quit(); return; }
        if (IsActionPressed("Restart")) { RestartLevel(); return; }

        // GetAxis2D — returns a Vector2 in [−1,1]×[−1,1], length ≤ 1.
        // Works identically whether the player is using keyboard or gamepad.
        Vector2 move = GetAxis2D("Move");

        // Rotate the player to face the direction of movement
        if (move.x_ != 0.0f || move.y_ != 0.0f) {
            float yaw = std::atan2(move.x_, move.y_) * RAD_TO_DEG;
            player_->SetRotation(Quaternion(0, yaw, 0));
        }

        // Drive the character controller with the normalized axis result
        cc_->Move(Vector3(move.x_, 0.0f, move.y_) * MOVE_SPEED, dt);

        // IsActionPressed for jump: one-shot per button press, never repeats
        // while the key is held.  TryJump() checks IsOnGround() internally.
        if (IsActionPressed("Jump")) {
            cc_->TryJump(JUMP_VEL);
            jumpLabel_->SetText("Jump: PRESSED");
        } else {
            // IsActionDown — true every frame the source remains held
            jumpLabel_->SetText(
                IsActionDown("Jump") ? "Jump: held" : "Jump: idle");
        }

        // Display live axis values on the HUD
        char buf[64];
        std::snprintf(buf, sizeof(buf),
            "Move axis: (%.2f, %.2f)", move.x_, move.y_);
        axisLabel_->SetText(buf);
    }

    void Stop() override
    {
        // Persist the current bindings so they survive the next launch
        SaveInputBindings();
    }

private:
    void ApplyDefaultBindings()
    {
        // BindAction creates (or retrieves) a named button action.
        // Chain .Key() and .GamepadButton() to add any number of sources.
        // Any one of them being active satisfies the action.

        BindAction("Jump")
            .Key(Key::LCtrl)
            .Key(Key::Space)
            .GamepadButton(GamepadButton::A);

        BindAction("Restart")
            .Key(Key::R)
            .GamepadButton(GamepadButton::Start);

        BindAction("Quit")
            .Key(Key::Escape)
            .GamepadButton(GamepadButton::B);

        // BindAxis2D creates (or retrieves) a named 2D axis.
        // .Keys(left, right, down, up) maps four keys:
        //   left  → x = −1   right → x = +1
        //   down  → y = −1   up    → y = +1
        // .GamepadStick(xAxis, yAxis) adds an analog stick pair.
        // .InvertY(true) negates the stick Y so that stick-forward (SDL = −1)
        //   produces +Y, consistent with the Up key producing +1.
        //   Keyboard axes are NOT inverted by this flag.
        BindAxis2D("Move")
            .Keys(Key::Left, Key::Right, Key::Down, Key::Up)
            .GamepadStick(GamepadAxis::LeftX, GamepadAxis::LeftY)
            .InvertY(true);
    }

    void RestartLevel()
    {
        // Reset player position as a simple restart
        player_->SetPosition(0.0f, 1.0f, 0.0f);
    }

    Entity*              player_    = nullptr;
    CharacterController* cc_        = nullptr;
    Label*               hintLabel_ = nullptr;
    Label*               axisLabel_ = nullptr;
    Label*               jumpLabel_ = nullptr;
};

int main()
{
    return Simple3D::CreateGame<InputActionsDemo>()->Run();
}

Step-by-step explanation

BindAction(name) — declaring a button action

BindAction("Jump") creates an ActionBinding object associated with the string "Jump" and returns a reference to it for method chaining. If a binding with that name already exists, the existing binding is returned so you can add more sources at any time. The name is entirely free-form — it just has to match what you pass to IsActionDown and IsActionPressed later.

After the call you chain one or more source methods. The most common are .Key(Key::Space) and .GamepadButton(GamepadButton::A). Each additional call registers one more source; the action is active when any of them is active: 

BindAction("Jump")
    .Key(Key::LCtrl)         // source 1: left control key
    .Key(Key::Space)          // source 2: space bar
    .GamepadButton(GamepadButton::A);  // source 3: gamepad A button

Declaring bindings once in Start() means you can change all three sources in one place without touching any of the game logic in Update().

IsActionPressed(name) vs IsActionDown(name)

These two query methods exist because "button pressed" and "button held" are different concepts that call for different responses in game code:

MethodReturns true when…Typical use
IsActionPressed(name) Any bound source transitions from released to pressed, on that single frame only Jump, restart, open menu, fire weapon (one shot per click)
IsActionDown(name) Any bound source is currently held down — true every frame until released Hold to run, hold to charge, hold to aim, continuous thrust

If you use IsActionDown for jump, holding the button causes the character to jump many times per second. Use IsActionPressed instead: it fires only on the frame the button goes down, not during the hold. Conversely, if you use IsActionPressed for a sprint modifier, the sprint activates for only one frame and then stops — use IsActionDown for anything that should persist while the key is held.

BindAxis2D(name) — declaring a 2D movement axis

BindAxis2D("Move") creates an Axis2DBinding and returns a chainable reference. This binding type aggregates multiple input sources into a single Vector2 that you read with GetAxis2D("Move"). The methods you chain onto it are:

  • .Keys(left, right, down, up) — maps four keyboard keys to the four cardinal directions. While left is held the X component is −1; while right is held it is +1. While down is held the Y component is −1; while up is held it is +1. Pressing two opposing keys (left and right) at the same time cancels to 0. You may call .Keys() more than once to register alternate key sets (e.g. WASD in addition to arrow keys).
  • .GamepadStick(xAxis, yAxis) — adds an analog stick pair as a source. GamepadAxis::LeftX maps to the result X, GamepadAxis::LeftY maps to the result Y. The global dead zone (set with SetGamepadDeadZone()) is applied automatically.
  • .InvertY(true) — negates the Y contribution from stick sources only. SDL and Urho3D report pushing the stick forward (away from you) as −1. The keyboard Up key produces +1. Without inversion the keyboard convention and stick convention conflict. InvertY(true) flips only the stick Y so that stick-forward becomes +1, matching the keyboard. Keyboard axes are unaffected by this flag.

GetAxis2D(name) — reading the 2D axis

Called in Update(), this method reads all active sources for the binding named name, combines them, and returns a Vector2. The result is normalized: its length is always at most 1. This means holding Up and Right simultaneously (45° diagonal) gives a vector of length 1, not √2. Without normalization keyboard players would move roughly 41% faster diagonally than straight ahead. 

Vector2 move = GetAxis2D("Move");
// move.x_ is in [−1, +1]: negative = left,  positive = right
// move.y_ is in [−1, +1]: negative = back,  positive = forward
// move.Length() is always ≤ 1

// Pass straight to the CharacterController:
cc_->Move(Vector3(move.x_, 0.0f, move.y_) * MOVE_SPEED, dt);

// Rotate the model to face the direction of movement:
if (move.x_ != 0.0f || move.y_ != 0.0f) {
    float yaw = std::atan2(move.x_, move.y_) * 57.29578f;
    player_->SetRotation(Quaternion(0, yaw, 0));
}

If the binding name does not exist (was never declared with BindAxis2D), GetAxis2D returns Vector2::ZERO safely, with no crash or error log. The same safe-default behavior applies to IsActionDown and IsActionPressed on missing action names (they return false).

ClearInputBindings() and ResetInputBindings()

ClearInputBindings() removes every action binding and every axis binding that was registered so far. ResetInputBindings() is an alias with the same effect; the two names exist because "reset" reads more naturally in the persistence pattern where you clear first and then reload.

The most common use case is switching between scenes with incompatible control schemes. A platformer level and a driving level might both use the left stick, but mean completely different things by it. Calling ClearInputBindings() before loading a new scene ensures there are no leftover bindings from the previous control scheme: 

// Switching from platformer to driving mini-game
ClearInputBindings();

BindAxis2D("Steer")
    .Keys(Key::Left, Key::Right, Key::Left, Key::Right)
    .GamepadStick(GamepadAxis::LeftX, GamepadAxis::LeftY);

BindAction("Accelerate")
    .Key(Key::Up)
    .GamepadButton(GamepadButton::A);

BindAction("Brake")
    .Key(Key::Down)
    .GamepadButton(GamepadButton::B);

SaveInputBindings() and LoadInputBindings()

These two methods persist the entire set of current bindings to a JSON file on disk and reload it. The file name defaults to controls.json and is written inside the application's save directory (~/.local/share/<AppName>/ on Linux, the appropriate user data folder on other platforms).

MethodReturnsNotes
SaveInputBindings() bool Writes <AppName>/controls.json. Returns false on I/O error.
SaveInputBindings("mycontrols.json") bool Custom filename within the same save directory.
LoadInputBindings() bool Loads controls.json. Returns false if absent or malformed.
ResetInputBindings() — Alias for ClearInputBindings(). Use before re-applying defaults.

The recommended persistence pattern is to always reset first, then attempt to load, and fall back to defaults if the file is not found (first launch) or is corrupt: 

void Start() override
{
    SetAppName("MyGame");

    // 1. Clear any previous bindings (safe to call on a fresh start).
    ResetInputBindings();

    // 2. Try to load the player's saved configuration from disk.
    //    Returns false on first run (file does not exist yet).
    if (!LoadInputBindings()) {

        // 3. Fall back to built-in defaults.
        BindAction("Jump")
            .Key(Key::Space)
            .GamepadButton(GamepadButton::A);

        BindAxis2D("Move")
            .Keys(Key::Left, Key::Right, Key::Down, Key::Up)
            .GamepadStick(GamepadAxis::LeftX, GamepadAxis::LeftY)
            .InvertY(true);
    }
}

void Stop() override
{
    // Persist the current bindings every time the game exits,
    // so any player-configured remapping survives the next launch.
    SaveInputBindings();
}

If your game includes a rebinding UI (not covered in this tutorial — there is no built-in one yet), call SaveInputBindings() after the player finishes customizing. The JSON file stores each action and axis name together with its complete list of sources, so the full configuration is restored on LoadInputBindings().

Why InvertY is necessary

This is one of the most confusing aspects of mixing keyboard and analog stick input, and it comes from a long-standing hardware convention.

SDL (the cross-platform input library used by simple-3d) reports the Y axis of every analog stick in screen space: pushing the stick away from you (forward) produces a negative value, and pulling the stick toward you (backward) produces a positive value. This matches the screen-Y axis convention where Y increases downward.

However, the keyboard axis from .Keys() follows the intuitive game-world convention: the Up key produces +1 (forward), and the Down key produces −1 (backward).

Without correction, a game built around the keyboard convention would have stick-forward move the character backward. .InvertY(true) negates only the stick Y component so that stick-forward also produces +1, making the two sources consistent. The keyboard contribution is never negated.

ActionRaw stick Y (SDL)After InvertY(true)Up key
Push stick forward−1+1—
Pull stick backward+1−1—
Press Up key——+1
Press Down key——−1

Missing bindings return safe defaults

Querying an action or axis name that was never registered is safe in simple-3d. The engine returns a predictable default value rather than crashing or logging an error. This is particularly useful during development when you add a query before you have finished setting up the binding: 

// These never crash, even if "Nonexistent" was never bound:
bool    b1 = IsActionDown("Nonexistent");     // → false
bool    b2 = IsActionPressed("Nonexistent");  // → false
Vector2 v  = GetAxis2D("Nonexistent");        // → Vector2::ZERO

Lua equivalent

The Lua scripting API mirrors C++ one-to-one. Game-level functions such as BindAction, BindAxis2D, IsActionPressed, GetAxis2D, ClearInputBindings, SaveInputBindings, and LoadInputBindings are available as globals. Method chaining uses Lua's colon syntax. 

-- Scripts/input_actions_demo.lua
-- Part 16 — Input Actions equivalent in Lua

local player -- Entity
local cc     -- CharacterController

local MOVE_SPEED  = 6.0
local JUMP_VEL    = 12.0
local RAD_TO_DEG  = 57.29578

local function applyDefaultBindings()
    BindAction("Jump")
        :Key(Key.LCtrl)
        :Key(Key.Space)
        :GamepadButton(GamepadButton.A)

    BindAction("Restart")
        :Key(Key.R)
        :GamepadButton(GamepadButton.Start)

    BindAction("Quit")
        :Key(Key.Escape)
        :GamepadButton(GamepadButton.B)

    -- BindAxis2D: chain Keys and GamepadStick, then InvertY
    BindAxis2D("Move")
        :Keys(Key.Left, Key.Right, Key.Down, Key.Up)
        :GamepadStick(GamepadAxis.LeftX, GamepadAxis.LeftY)
        :InvertY(true)
end

function Start()
    SetAppName("InputActionsDemo")

    -- Persistence: reset, try to load, fall back to defaults
    ResetInputBindings()
    if not LoadInputBindings() then
        applyDefaultBindings()
    end

    -- Ground
    local ground = CreateEntity("Ground")
    ground:AddModel("Models/Plane.mdl")
    ground:AddRigidBody(0.0)
    ground:AddBoxCollider(Vector3(30, 0.2, 30))
    ground:SetCollisionLayer(CollisionLayer.StaticGeometry)

    -- Player
    player = CreateEntity("Player")
    player:SetPosition(0, 1, 0)
    player:AddModel("Models/Hero.mdl")

    cc = player:AddCharacterController(0.35, 1.8)
    cc:SetGravity(25.0)
    player:SetCollisionLayer(CollisionLayer.Actor)
    player:SetCollisionMask(
        MakeCollisionMask({CollisionLayer.StaticGeometry, CollisionLayer.Trigger}))

    -- Camera
    local cam = CreateCamera()
    cam:Follow(player, 7.0, 2.5)
    cam:LookAt(player)
    cam:SetSmoothFollow(true, 5.0)

    -- Light
    local sun = CreateEntity("Sun")
    sun:SetRotation(Quaternion(45, 30, 0))
    sun:AddDirectionalLight(Color.WHITE, 1.2)
end

function Update(dt)
    -- IsActionPressed: one-shot, fires on the first frame the source goes down
    if IsActionPressed("Quit") then
        Quit()
        return
    end

    if IsActionPressed("Restart") then
        player:SetPosition(0, 1, 0)
        return
    end

    -- GetAxis2D: returns normalized Vector2 from keyboard or stick
    local move = GetAxis2D("Move")

    if move.x ~= 0 or move.y ~= 0 then
        local yaw = math.atan2(move.x, move.y) * RAD_TO_DEG
        player:SetRotation(Quaternion(0, yaw, 0))
    end

    cc:Move(Vector3(move.x, 0, move.y) * MOVE_SPEED, dt)

    -- IsActionPressed for jump (one-shot); IsActionDown for held states
    if IsActionPressed("Jump") then
        cc:TryJump(JUMP_VEL)
    end
end

function Stop()
    -- Persist the current bindings so they survive the next launch
    SaveInputBindings()
end

Lua tip: Vector2 member access

In Lua, Vector2 members are accessed as v.x and v.y (without trailing underscores). In C++ they are v.x_ and v.y_ to match Urho3D's naming convention.

Practical pattern: hold-to-sprint with action actions

The combination of IsActionDown (held modifier) and the axis query (direction) is the standard pattern for sprint-while-moving: 

// In Start()
BindAction("Sprint")
    .Key(Key::LShift)
    .GamepadButton(GamepadButton::LB);

// In Update(dt)
Vector2 move = GetAxis2D("Move");
float   top  = IsActionDown("Sprint") ? 10.0f : 5.0f;
cc_->Move(Vector3(move.x_, 0.0f, move.y_) * top, dt);

Notice that the sprint check uses IsActionDown because sprint is a persistent modifier that should stay active the entire time the key is held, while jump uses IsActionPressed because it should fire only once per button press, not every frame.

Common mistakes

Common mistakes to avoid

  • Using IsActionDown for jump. If you call cc_->TryJump() inside an IsActionDown check, holding the jump key keeps calling TryJump() many times per second. TryJump() checks IsOnGround() internally, so while the character is airborne it will not double-jump, but the moment the character lands it will immediately jump again. Use IsActionPressed for any action that should fire only once per button press.
  • Using IsActionPressed for sprint or charge. IsActionPressed returns true for exactly one frame. If you use it to toggle sprint, the sprint modifier is active for only a single frame and then stops. Use IsActionDown for any effect that should persist while the key is held.
  • Forgetting InvertY(true) on stick axes. Without inversion, pushing the analog stick forward makes the player character walk backward. The stick Y axis is reported as negative-forward by SDL, which is the opposite of the keyboard Up key which produces +1. Always add .InvertY(true) when combining a stick source with keyboard sources on the same axis.
  • Not calling SetAppName() before persistence. SaveInputBindings() and LoadInputBindings() write to and read from a directory named after the application. If you do not call SetAppName("MyGame") before the first save, the directory defaults to Simple3DGame, which may conflict with other projects.
  • Calling LoadInputBindings() without a preceding ResetInputBindings(). If existing bindings are already in memory and you load from file on top of them, you may end up with duplicate sources (both the hard-coded defaults and the file-based ones). Always call ResetInputBindings() first to start with a clean slate.
  • Querying an axis that was never bound. GetAxis2D("Nonexistent") returns Vector2::ZERO, which means no movement — this is safe but may be confusing during development if the binding name has a typo. Double-check that the name passed to BindAxis2D exactly matches the name passed to GetAxis2D.
  • Expecting .Keys(left, right, down, up) to accept four separate calls. The four keys must all be passed to a single .Keys() call in the order left, right, down, up. If you pass them in a different order, the directions will be swapped. To add a second key set (e.g. WASD), call .Keys() a second time on the same binding.

What is not yet implemented

simple-3d does not currently provide a built-in rebinding UI. There is no menu or dialog that lets the player reassign keys at runtime. You can build your own on top of the existing API: present your own UI, call ClearInputBindings() followed by BindAction() / BindAxis2D() with the new keys, and then call SaveInputBindings() to persist the choices. The JSON format and file path are stable, so a custom rebinding UI written today will continue to work with future engine versions.

What you learned

  • Raw key and button checks scattered across game code create a maintenance problem. Input Actions replace them with a single declaration per action and a single query per action, decoupling the input source from the game logic entirely.
  • BindAction(name) declares a named button action. Chaining .Key() and .GamepadButton() adds multiple sources; any one being active satisfies the action. The same name is passed to IsActionDown and IsActionPressed in Update().
  • IsActionPressed fires on the first frame a source goes down; IsActionDown fires every frame the source is held. Use IsActionPressed for one-shot events (jump, restart, fire) and IsActionDown for continuous modifiers (sprint, charge, aim).
  • BindAxis2D(name) declares a named 2D movement axis. .Keys(left, right, down, up) maps four keyboard keys; .GamepadStick(xAxis, yAxis) adds an analog stick pair. .InvertY(true) corrects the SDL stick Y convention so stick-forward produces the same +Y as the Up key.
  • GetAxis2D(name) returns a normalized Vector2 with length at most 1, so diagonal keyboard movement is never faster than straight movement. Pass the result directly to CharacterController::Move().
  • ClearInputBindings() (or its alias ResetInputBindings()) removes all registered bindings at once. Use it when switching between scenes that have incompatible control schemes.
  • SaveInputBindings() persists the current bindings to a JSON file in the application's save directory. LoadInputBindings() restores them. The recommended pattern is: reset, try to load, fall back to defaults if the file is absent (first launch). Call SaveInputBindings() in Stop() so customized bindings survive the next session.
  • The Lua API is identical: the same global functions exist, and method chaining uses Lua's colon syntax. Vector2 members are accessed as v.x and v.y (no trailing underscores).
