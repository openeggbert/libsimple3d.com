Part 29 of 30Sky and Atmosphere

New in this part

What you will learn

  • How to display a large sky sphere with SetSkyDome, and show or hide it at runtime with SetSkyDomeEnabled.
  • How to generate a smooth two-colour gradient sky without any texture file using SetSkyGradient.
  • How ambient light works and how to set its colour and brightness with SetAmbientColor.
  • How to enable linear fog and configure its colour and distance range with SetFogEnabled, SetFogColor, and SetFogRange.
  • How to apply a flat, unlit solid colour to a mesh entity using SetMaterialColor.
  • How to sample a single tile from a UV texture atlas with SetTileTexture.
  • How to capture the current framebuffer as a PNG with SaveScreenshot.
  • How to compose per-world atmosphere presets (grassland, cave, lava, space) that all these settings work together.

Background: how the atmosphere system works

simple-3d wraps Urho3D's Zone component to provide scene-wide atmosphere controls. A Zone is a bounding volume attached to an invisible node that covers the entire scene; simple-3d creates one automatically and exposes it through the SetAmbientColor, SetFogColor, and SetFogRange family of calls. You never touch the Zone directly — just call the Game methods from Start() or whenever you switch worlds.

The sky dome is a separate sphere of radius 450 that is rendered from the inside and follows the camera, so it never clips against the far plane. You can texture it from a file or let simple-3d generate a vertical gradient from two colours. Either approach requires a single call and takes effect on the very next frame.

Atmosphere (Zone)

Controls the minimum light on all surfaces (ambient) and the linear fog that blends geometry into a chosen colour as distance increases. All four atmosphere methods operate on the same underlying Zone; changing any one of them is instant and does not require recreating the scene.

Sky dome

A large inside-out sphere that provides the visible backdrop behind all geometry. Use SetSkyGradient for quick programmatic skies (no art assets needed) or SetSkyDome with a texture path for a hand-painted or photographic sky. Toggle visibility with SetSkyDomeEnabled.

Recommended asset layout

Texture-based sky domes and terrain atlases both live under Data/Textures/ so the resource cache resolves them by relative path regardless of working directory: 

Data/
  Textures/
    sky_day.png           // panoramic sky photograph or painted sky
    sky_stars.png         // star-field for a space scene
    terrain-atlas.png     // NxN grid of terrain tiles (e.g. 4x4 = 16 tiles)
  Models/
    Box.mdl
    Plane.mdl

The sky dome texture wraps cylindrically around the sphere interior. A 2:1 aspect-ratio equirectangular image works best, but simple square textures are also acceptable and will just repeat horizontally.

Complete working example

The program below builds a scene with a textured ground plane, two demonstration boxes (one with a solid unlit colour, one with a UV-atlas tile), and a directional sun light. Press 1–4 to switch between four atmosphere presets. Press F12 to capture a screenshot to disk. 

#include <Simple3D/Simple3D.h>
#include <string>

using namespace Simple3D;

// ─────────────────────────────────────────────────────────────────────
// AtmosphereDemo
//   Demonstrates sky dome, sky gradient, ambient light, linear fog,
//   per-entity material colour, UV-atlas texture crops and screenshot
//   capture.  Press 1–4 to switch world presets, F12 to screenshot.
// ─────────────────────────────────────────────────────────────────────
class AtmosphereDemo : public Game
{
public:
    // ── Scene entities ────────────────────────────────────────────────
    Entity* ground_    = nullptr;
    Entity* sun_       = nullptr;
    Entity* colorBox_  = nullptr; // flat unlit colour via SetMaterialColor
    Entity* atlasBox_  = nullptr; // UV-atlas tile via SetTileTexture

    // ── Camera ────────────────────────────────────────────────────────
    Camera* cam_ = nullptr;

    // ── UI ────────────────────────────────────────────────────────────
    Label* worldLabel_      = nullptr;
    Label* hintLabel_       = nullptr;
    Label* screenshotLabel_ = nullptr;

    // ── State ─────────────────────────────────────────────────────────
    int   currentWorld_    = 0;
    float screenshotTimer_ = 0.0f; // seconds to show the "saved" message

    // ─────────────────────────────────────────────────────────────────
    // Start() — called once before the first frame.
    // ─────────────────────────────────────────────────────────────────
    void Start() override
    {
        SetAppName("AtmosphereDemo");
        SetWindowTitle("Part 29 — Sky and Atmosphere");

        // ── Ground plane ─────────────────────────────────────────────
        // A large flat quad serves as the floor for all worlds.
        // We apply a tiled terrain-atlas texture so the ground shows
        // a repeating grass tile from a 4×4 UV atlas.
        ground_ = CreateEntity("Ground");
        ground_->AddModel("Models/Plane.mdl");
        ground_->SetScale(50.0f);
        ground_->AddRigidBody(0.0f);                              // static
        ground_->AddBoxCollider(Vector3(50.0f, 0.5f, 50.0f));
        ground_->SetCollisionLayer(CollisionLayer::StaticGeometry);

        // Atlas layout: 4 columns × 4 rows, each tile = 1/4 of UV space.
        // Tile (col=0, row=0) is the grass tile at the top-left corner.
        //   uOff   = col * tileSize = 0 * 0.25 = 0.0
        //   vOff   = row * tileSize = 0 * 0.25 = 0.0
        //   uScale = tileSize = 0.25
        //   vScale = tileSize = 0.25
        const float TILE = 1.0f / 4.0f; // 4×4 atlas
        ground_->SetTileTexture("Textures/terrain-atlas.png",
                                  0.0f,  0.0f,   // uOff, vOff — top-left tile
                                  TILE,  TILE);   // uScale, vScale — one tile

        // ── Flat unlit colour box (left) ─────────────────────────────
        // SetMaterialColor applies a fully unlit solid colour material.
        // No lighting calculation is done — the colour is constant
        // regardless of the scene's lights, ambient setting or fog.
        colorBox_ = CreateEntity("ColorBox");
        colorBox_->AddModel("Models/Box.mdl");
        colorBox_->SetPosition(-3.0f, 1.0f, 0.0f);
        colorBox_->SetMaterialColor(Color(0.2f, 0.8f, 0.3f)); // bright unlit green

        // ── UV-atlas textured box (right) ────────────────────────────
        // SetTileTexture samples the stone tile at column 1, row 0
        // from the same 4×4 terrain atlas used for the ground.
        // This box IS affected by directional lights and ambient light
        // (it uses a lit diffuse material), unlike the colour box above.
        atlasBox_ = CreateEntity("AtlasBox");
        atlasBox_->AddModel("Models/Box.mdl");
        atlasBox_->SetPosition(3.0f, 1.0f, 0.0f);
        // Column 1, row 0: uOff = 1*TILE = 0.25, vOff = 0
        atlasBox_->SetTileTexture("Textures/terrain-atlas.png",
                                    1 * TILE, 0.0f,    // stone tile
                                    TILE,     TILE);

        // ── Directional sun light ────────────────────────────────────
        sun_ = CreateEntity("Sun");
        sun_->AddDirectionalLight(Color(1.0f, 0.98f, 0.9f), 1.2f);
        sun_->SetRotation(Quaternion(45.0f, 40.0f, 0.0f));
        sun_->SetCastShadows(true);

        // ── Camera ───────────────────────────────────────────────────
        cam_ = CreateCamera();
        cam_->SetPosition(Vector3(0.0f, 4.0f, -12.0f));
        cam_->LookAt(Vector3(0.0f, 1.0f, 0.0f));
        cam_->SetFOV(65.0f);
        cam_->SetFarClip(500.0f);     // sky dome radius is 450; set far clip beyond it

        // ── HUD labels ───────────────────────────────────────────────
        worldLabel_ = CreateLabel("World: Grassland");
        worldLabel_->SetPosition(10, 10);
        worldLabel_->SetFontSize(20);
        worldLabel_->SetColor(Color::WHITE);

        hintLabel_ = CreateLabel(
            "1=Grassland  2=Cave  3=Volcanic  4=Space  F12=Screenshot  ESC=Quit");
        hintLabel_->SetPosition(10, 38);
        hintLabel_->SetFontSize(13);
        hintLabel_->SetColor(Color(0.85f, 0.85f, 0.85f, 1.0f));

        screenshotLabel_ = CreateLabel("");
        screenshotLabel_->SetPosition(10, 62);
        screenshotLabel_->SetFontSize(14);
        screenshotLabel_->SetColor(Color(0.4f, 1.0f, 0.4f, 1.0f));
        screenshotLabel_->SetVisible(false);

        // ── Load first world preset ───────────────────────────────────
        LoadWorld(0);
    }

    // ─────────────────────────────────────────────────────────────────
    // LoadWorld(id) — apply a complete atmosphere preset.
    // Call this from Start() and whenever the player presses 1–4.
    // All atmosphere API calls are safe to call at any time.
    // ─────────────────────────────────────────────────────────────────
    void LoadWorld(int id)
    {
        currentWorld_ = id;

        switch (id)
        {
        case 0: // ── Grassland ─────────────────────────────────────────
            // Programmatic gradient sky: deep blue at the zenith (top),
            // warm orange-peach at the horizon.
            // SetSkyGradient creates a 1×16 gradient texture internally —
            // no texture file needed.
            SetSkyGradient(Color(0.10f, 0.30f, 0.80f),   // zenith  — sky blue
                           Color(0.80f, 0.60f, 0.40f));  // horizon — warm peach
            SetSkyDomeEnabled(true);

            SetClearColor(Color(0.52f, 0.80f, 0.92f));
            SetAmbientColor(Color(0.25f, 0.30f, 0.20f));   // soft green-tinted ambient

            // Fog colour matches the clear / sky horizon colour so distant
            // objects fade smoothly into the sky rather than a hard edge.
            SetFogColor(Color(0.52f, 0.80f, 0.92f));
            SetFogRange(80.0f, 250.0f);  // starts at 80 m, fully opaque at 250 m
            SetFogEnabled(true);

            worldLabel_->SetText("World: Grassland");
            break;

        case 1: // ── Cave / night ──────────────────────────────────────
            // No sky visible underground — hide the dome entirely.
            // Very low ambient makes unlit surfaces nearly black,
            // giving an authentic underground feel.
            // Dense, short-range dark fog closes in the walls.
            SetSkyDomeEnabled(false);

            SetClearColor(Color(0.0f, 0.0f, 0.0f));
            SetAmbientColor(Color(0.05f, 0.05f, 0.10f)); // near-black blue-tint
            SetFogColor(Color(0.0f, 0.0f, 0.05f));
            SetFogRange(10.0f, 40.0f);   // dense fog: full opacity by 40 m
            SetFogEnabled(true);

            worldLabel_->SetText("World: Cave");
            break;

        case 2: // ── Volcanic / lava ────────────────────────────────────
            // Dark red gradient sky — murky at the zenith, glowing at
            // the horizon as if lit by distant lava flows.
            // High red-dominant ambient simulates reflected heat glow.
            SetSkyGradient(Color(0.10f, 0.02f, 0.00f),   // zenith  — near black-red
                           Color(0.70f, 0.20f, 0.00f));  // horizon — deep orange
            SetSkyDomeEnabled(true);

            SetClearColor(Color(0.15f, 0.05f, 0.00f));
            SetAmbientColor(Color(0.40f, 0.10f, 0.05f));  // hot red ambient
            SetFogColor(Color(0.30f, 0.05f, 0.00f));
            SetFogRange(20.0f, 80.0f);   // ash-choked air: visibility ~80 m
            SetFogEnabled(true);

            worldLabel_->SetText("World: Volcanic");
            break;

        case 3: // ── Space / void ───────────────────────────────────────
            // Texture-based sky dome — a hand-painted star-field image.
            // SetSkyDome("path") creates the dome sphere on the first call;
            // subsequent calls with a different path swap the texture.
            SetSkyDome("Textures/sky_stars.png");
            SetSkyDomeEnabled(true);

            SetClearColor(Color(0.00f, 0.00f, 0.02f));
            SetAmbientColor(Color(0.05f, 0.05f, 0.10f)); // dim starlight

            // No atmospheric fog in the vacuum of space.
            SetFogEnabled(false);

            worldLabel_->SetText("World: Space");
            break;
        }
    }

    // ─────────────────────────────────────────────────────────────────
    // Update() — input handling each frame.
    // ─────────────────────────────────────────────────────────────────
    void Update(float dt) override
    {
        // World preset selection
        if (IsKeyPressed(Key::Num1)) LoadWorld(0);
        if (IsKeyPressed(Key::Num2)) LoadWorld(1);
        if (IsKeyPressed(Key::Num3)) LoadWorld(2);
        if (IsKeyPressed(Key::Num4)) LoadWorld(3);

        // Screenshot capture with F12.
        // SaveScreenshot writes the current rendered framebuffer to a PNG
        // at the given path and returns true on success, false on I/O error.
        if (IsKeyPressed(Key::F12))
        {
            std::string path = "screenshot_world" + std::to_string(currentWorld_) + ".png";
            if (SaveScreenshot(path))
            {
                screenshotLabel_->SetText("Screenshot saved: " + path);
                screenshotLabel_->SetVisible(true);
                screenshotTimer_ = 3.0f; // display for 3 seconds
            }
        }

        // Auto-hide the screenshot confirmation message after 3 seconds
        if (screenshotTimer_ > 0.0f)
        {
            screenshotTimer_ -= dt;
            if (screenshotTimer_ <= 0.0f)
                screenshotLabel_->SetVisible(false);
        }

        if (IsKeyPressed(Key::Escape))
            Quit();
    }
};

int main()
{
    return Simple3D::CreateGame<AtmosphereDemo>()->Run();
}

Step-by-step explanation

SetSkyDome(texturePath)

SetSkyDome creates a large inside-out sphere of radius 450 world units and wraps the given texture around its interior surface. The sphere node is permanently parented to the camera, so it moves with it every frame and never clips against the far plane as long as cam->SetFarClip is set to at least 500. On the first call the sphere node and material are created; subsequent calls with a different path simply swap the texture on the existing material. Passing an empty string leaves the dome untextured (solid white). 

// File-based sky — a hand-painted equirectangular panorama
SetSkyDome("Textures/sky_day.png");

// Swap to a night sky when the player enters a dungeon exit
SetSkyDome("Textures/sky_night.png");

// Untextured dome (useful as a base before SetSkyGradient is called)
SetSkyDome("");

SetSkyDomeEnabled(bool)

Toggles the visibility of the sky dome without destroying it. When set to false, the sphere node is disabled; the clear colour you set with SetClearColor shows through the gaps in geometry instead. Underground scenes, indoor areas, or space sequences where you want a pure black background should call SetSkyDomeEnabled(false) after setting SetClearColor(Color(0,0,0)). 

// Show the sky (default after the first SetSkyDome or SetSkyGradient call)
SetSkyDomeEnabled(true);

// Hide for an underground level — black clear colour shows through
SetClearColor(Color(0.0f, 0.0f, 0.0f));
SetSkyDomeEnabled(false);

SetSkyGradient(zenithColor, horizonColor)

SetSkyGradient generates a tiny 1×16 gradient texture in memory — no image file required — and applies it to the sky dome sphere. The first argument is the colour at the very top of the dome (zenith) and the second is the colour at the equator (horizon). The gradient interpolates linearly between the two values. Calling SetSkyGradient implicitly enables the sky dome if it was previously hidden.

Use colour values in the 0–1 float range. Slightly oversaturated or high-value colours (e.g. Color(0.1f, 0.3f, 0.8f)) look more vibrant than clamped values because the sky is an emissive surface and is not affected by ambient light. 

// Daytime blue sky fading to warm horizon
SetSkyGradient(Color(0.10f, 0.30f, 0.80f),   // zenith  — sky blue
               Color(0.80f, 0.60f, 0.40f));  // horizon — warm peach

// Twilight: dark purple zenith, crimson horizon
SetSkyGradient(Color(0.10f, 0.05f, 0.20f),
               Color(0.70f, 0.15f, 0.05f));

// Overcast: flat grey from zenith to horizon
SetSkyGradient(Color(0.55f, 0.55f, 0.58f),
               Color(0.72f, 0.72f, 0.74f));

SetAmbientColor(color)

Ambient light is the constant minimum brightness applied to every surface in the scene regardless of whether any direct light reaches it. It is implemented via Urho3D's Zone component. The default is Color(0.3, 0.3, 0.3) (a moderate neutral grey). Setting it very low creates dramatic deep shadows (caves, night, space); setting it high produces the flat, shadowless look typical of overcast outdoor scenes.

Tinting the ambient colour gives the whole scene a mood: a slight green tint suggests forest or jungle, a blue tint reads as moonlight or underwater, and a red-orange tint conveys heat or volcanic illumination. 

// Dark cave or night scene — surfaces in shadow are nearly invisible
SetAmbientColor(Color(0.05f, 0.05f, 0.10f));

// Bright overcast exterior — shadows are barely visible
SetAmbientColor(Color(0.60f, 0.60f, 0.60f));

// Forest green ambient — dappled-light feel without a texture
SetAmbientColor(Color(0.15f, 0.25f, 0.10f));

// Default — restore to engine default
SetAmbientColor(Color(0.30f, 0.30f, 0.30f));

SetFogColor(color), SetFogRange(start, end), SetFogEnabled(bool)

Fog is linear: geometry at start metres is fully unfogged; geometry at end metres is fully replaced by the fog colour. Everything between the two distances is blended proportionally. Fog is disabled by default. The recommended pattern is to call SetFogColor and SetFogRange first, then call SetFogEnabled(true). You can call these methods in any order; the values are cached and applied together when fog is enabled.

For best results, match SetFogColor to SetClearColor and to the horizon colour in SetSkyGradient so that distant geometry fades into the sky seamlessly rather than into a visible coloured band. 

// Standard sky-blue horizon fog — set colour and range first
SetFogColor(Color(0.52f, 0.80f, 0.92f));
SetFogRange(80.0f, 250.0f);   // blends from 80 m; fully opaque at 250 m
SetFogEnabled(true);

// Dense underground mist — short range, dark colour
SetFogColor(Color(0.0f, 0.0f, 0.05f));
SetFogRange(10.0f, 40.0f);
SetFogEnabled(true);

// Remove fog for an outdoor space scene
SetFogEnabled(false);

// Range is safe to update while fog is disabled; it is applied on next enable
SetFogRange(60.0f, 200.0f);   // cached but not active yet
SetFogEnabled(true);          // now active with the cached range

entity->SetMaterialColor(color)

SetMaterialColor applies an unlit solid-colour material to the entity's StaticModel. "Unlit" means no lighting equation is computed — the colour is constant regardless of directional lights, point lights, shadows, or the ambient setting. This makes it ideal for UI elements placed in 3D space, debug markers, highlight overlays, or any object that should always appear at its full specified colour.

Because the material is programmatic, no material XML file is needed. The call is a no-op if the entity has no StaticModel (i.e. if you have not called AddModel first). 

// Bright unlit green — will look exactly this shade under any lighting
box->AddModel("Models/Box.mdl");
box->SetMaterialColor(Color(0.20f, 0.80f, 0.30f));

// Semitransparent red highlight — alpha < 1.0 enables transparency
marker->SetMaterialColor(Color(1.0f, 0.0f, 0.0f, 0.5f));

// Reuse the same entity and swap colour at runtime
indicator->SetMaterialColor(hp > 0.5f ? Color::GREEN : Color::RED);

entity->SetTileTexture(texPath, uOff, vOff, uScale, vScale)

SetTileTexture applies a lit diffuse material that shows a cropped rectangular region of a texture. This is the canonical way to use a UV texture atlas (a single image containing many tiles) without needing a separate texture file per tile. The five parameters specify the UV rectangle to sample:

  • texPath — path to the atlas texture.
  • uOff, vOff — UV offset of the top-left corner of the tile, in the range 0–1.
  • uScale, vScale — size of the tile in UV space. For an N×N atlas, each tile's scale is 1.0f / N.

Unlike SetMaterialColor, the resulting material is affected by lights and ambient, so tiles will appear darker in shadow as you would expect. The call is a no-op without a prior AddModel. 

// 4×4 atlas: 16 tiles, each occupying 1/4 of UV space in each axis
const float TILE = 1.0f / 4.0f;

// Grass tile — column 0, row 0
ground->SetTileTexture("Textures/terrain.png",
                         0 * TILE, 0 * TILE,   // uOff, vOff
                         TILE,     TILE);          // uScale, vScale

// Stone tile — column 1, row 0
wall->SetTileTexture("Textures/terrain.png",
                      1 * TILE, 0 * TILE,
                      TILE,     TILE);

// Lava tile — column 2, row 1
floor->SetTileTexture("Textures/terrain.png",
                       2 * TILE, 1 * TILE,
                       TILE,     TILE);

For an 8×8 atlas (64 tiles) use 1.0f / 8.0f as the tile size, and so on. The atlas does not need to be square, but a power-of-two texture size is recommended for best GPU compatibility.

SaveScreenshot(path)

SaveScreenshot captures the current rendered framebuffer and writes it to a PNG file at the given path. It returns true on success and false if the path is unwritable or another I/O error occurred. The screenshot includes everything visible in the viewport: the 3D scene, the sky dome, and the UI overlay.

The path is relative to the current working directory of the process, which is typically the directory containing your executable. Use std::to_string or a string formatter to include dynamic information like the current level or a timestamp in the filename. 

// Simple timestamped screenshot
bool ok = SaveScreenshot("screenshot.png");

// Include the world name in the filename
std::string path = "world" + std::to_string(currentWorld_) + ".png";
if (SaveScreenshot(path))
    label->SetText("Saved: " + path);
else
    label->SetText("Screenshot failed!");

Building per-world atmosphere presets

The most maintainable way to handle atmosphere in a multi-world game is a dedicated LoadWorld(int id) helper called from Start() and from whatever loads a new world at runtime. All atmosphere methods are safe to call at any time, so swapping worlds mid-session requires no teardown. 

struct WorldPreset
{
    Color  zenith, horizon;       // sky gradient
    bool   skyEnabled;
    Color  ambient;
    Color  fogColor;
    float  fogStart, fogEnd;
    bool   fogEnabled;
    std::string skyTex;           // empty = use gradient, not a file
};

const WorldPreset PRESETS[] = {
    // 0: Grassland
    { {0.10f,0.30f,0.80f}, {0.80f,0.60f,0.40f}, true,
      {0.25f,0.30f,0.20f}, {0.52f,0.80f,0.92f}, 80.f, 250.f, true,  "" },
    // 1: Cave
    { {0,0,0},            {0,0,0},            false,
      {0.05f,0.05f,0.10f}, {0,0,0.05f},         10.f, 40.f,  true,  "" },
    // 2: Volcanic
    { {0.10f,0.02f,0.00f}, {0.70f,0.20f,0.00f}, true,
      {0.40f,0.10f,0.05f}, {0.30f,0.05f,0.00f}, 20.f, 80.f,  true,  "" },
    // 3: Space
    { {0,0,0.02f},        {0,0,0.02f},        true,
      {0.05f,0.05f,0.10f}, {0,0,0},             0.f,  0.f,   false, "Textures/sky_stars.png" },
};

void ApplyPreset(int id)
{
    const auto& p = PRESETS[id];
    if (!p.skyTex.empty())
        SetSkyDome(p.skyTex);
    else
        SetSkyGradient(p.zenith, p.horizon);
    SetSkyDomeEnabled(p.skyEnabled);
    SetAmbientColor(p.ambient);
    SetFogColor(p.fogColor);
    if (p.fogEnabled)
        SetFogRange(p.fogStart, p.fogEnd);
    SetFogEnabled(p.fogEnabled);
}

Lua equivalent

All atmosphere and sky API calls are available as global functions inside any Lua script. The entity material methods are available on entity objects. SaveScreenshot returns a boolean result just as in C++. 

-- Scripts/atmosphere.lua
-- Attach to a persistent manager entity from C++.
-- Press 1–4 to switch worlds, F12 to screenshot.

local currentWorld   = 0
local screenshotTimer = 0.0

function Start()
    LoadWorld(0)
end

function LoadWorld(id)
    currentWorld = id

    if id == 0 then
        -- Grassland: gradient sky, moderate fog
        SetSkyGradient(0.10, 0.30, 0.80,   -- zenith
                       0.80, 0.60, 0.40)   -- horizon
        SetSkyDomeEnabled(true)
        SetClearColor(0.52, 0.80, 0.92)
        SetAmbientColor(0.25, 0.30, 0.20)
        SetFogColor(0.52, 0.80, 0.92)
        SetFogRange(80, 250)
        SetFogEnabled(true)

    elseif id == 1 then
        -- Cave: no sky, dense dark fog
        SetSkyDomeEnabled(false)
        SetClearColor(0, 0, 0)
        SetAmbientColor(0.05, 0.05, 0.10)
        SetFogColor(0, 0, 0.05)
        SetFogRange(10, 40)
        SetFogEnabled(true)

    elseif id == 2 then
        -- Volcanic: red gradient, hot ambient, ash fog
        SetSkyGradient(0.10, 0.02, 0.00,
                       0.70, 0.20, 0.00)
        SetSkyDomeEnabled(true)
        SetClearColor(0.15, 0.05, 0)
        SetAmbientColor(0.40, 0.10, 0.05)
        SetFogColor(0.30, 0.05, 0)
        SetFogRange(20, 80)
        SetFogEnabled(true)

    elseif id == 3 then
        -- Space: textured star-field dome, no fog
        SetSkyDome("Textures/sky_stars.png")
        SetSkyDomeEnabled(true)
        SetClearColor(0, 0, 0.02)
        SetAmbientColor(0.05, 0.05, 0.10)
        SetFogEnabled(false)
    end
end

function Update(dt)
    -- World switching
    if game:IsKeyPressed(Key.Num1) then LoadWorld(0) end
    if game:IsKeyPressed(Key.Num2) then LoadWorld(1) end
    if game:IsKeyPressed(Key.Num3) then LoadWorld(2) end
    if game:IsKeyPressed(Key.Num4) then LoadWorld(3) end

    -- Screenshot
    if game:IsKeyPressed(Key.F12) then
        local path = "screenshot_world" .. currentWorld .. ".png"
        local ok = SaveScreenshot(path)
        if ok then
            screenshotTimer = 3.0
        end
    end

    if screenshotTimer > 0 then
        screenshotTimer = screenshotTimer - dt
    end

    if game:IsKeyPressed(Key.Escape) then
        game:Quit()
    end
end

-- Apply a UV-atlas tile to an entity from Lua:
--   tileSize = 1/4 for a 4x4 atlas
--   entity:SetTileTexture(path, uOff, vOff, uScale, vScale)
function ApplyTile(entity, col, row)
    local t = 1.0 / 4.0
    entity:SetTileTexture("Textures/terrain-atlas.png",
                           col * t, row * t, t, t)
end

-- Apply a flat unlit colour to an entity from Lua:
--   entity:SetMaterialColor(r, g, b, a=1)
function MarkGreen(entity)
    entity:SetMaterialColor(0.2, 0.8, 0.3)
end

Quick reference

Method Returns Notes
SetSkyDome(texturePath) — Creates dome sphere on first call; swaps texture on subsequent calls. Empty path = untextured.
SetSkyDomeEnabled(bool) — Show or hide the dome without destroying it. Default: enabled after first SetSkyDome/SetSkyGradient call.
SetSkyGradient(zenith, horizon) — Generates a 1×16 gradient texture and applies it to the dome. No texture file required.
SetAmbientColor(color) — Minimum brightness on all surfaces. Default Color(0.3, 0.3, 0.3).
SetFogEnabled(bool) — Enable/disable linear fog. Default false. Enable after calling SetFogColor and SetFogRange.
SetFogColor(color) — Colour geometry fades into with distance. Match to clear colour for seamless sky blending.
SetFogRange(start, end) — Linear fog distances in world units. Safe to call while fog is disabled.
entity->SetMaterialColor(color) — Unlit solid colour. Not affected by lights or ambient. No-op without StaticModel.
entity->SetTileTexture(path, uOff, vOff, uScale, vScale) — Lit diffuse material using a UV-atlas crop. No-op without StaticModel.
SaveScreenshot(path) bool Writes current framebuffer to PNG. Returns false on I/O error.

Common mistakes

  • Sky dome clips against the far plane. The sky dome sphere has a radius of 450 world units. If you set cam->SetFarClip to less than 500, the back of the dome will be clipped and you will see a black ring around the horizon. Always set the far clip to at least 500 when using a sky dome.
  • Calling SetSkyGradient and then SetSkyDomeEnabled(false) expecting the gradient to reappear. SetSkyGradient and SetSkyDome always re-enable the dome. If you then call SetSkyDomeEnabled(false) in the same world setup function, the dome will be hidden. Call SetSkyDomeEnabled(false) only in presets where you genuinely want no sky visible.
  • Fog colour does not match the sky colour. When SetFogColor is different from SetClearColor and the sky horizon colour, a visible coloured band appears at the fog boundary. For outdoor scenes, all three should be set to the same or very similar value.
  • SetFogEnabled(true) before SetFogRange, getting zero-range fog. If you call SetFogEnabled(true) without first calling SetFogRange, the start and end distances default to 0 and 0, which makes everything immediately opaque. Always call SetFogRange before SetFogEnabled(true).
  • Using SetMaterialColor on an entity with no model. SetMaterialColor and SetTileTexture are both no-ops unless the entity already has a StaticModel added via AddModel. Call AddModel first.
  • Confusing SetMaterialColor (unlit) with a lit coloured material. SetMaterialColor produces an emissive, unlit surface. If you want a coloured surface that still responds to lights and shadows, use SetTileTexture with a small single-colour texture, or provide a custom material XML.
  • SetTileTexture UV coordinates outside [0,1] causing tiling artefacts. If uOff + uScale > 1.0 or vOff + vScale > 1.0, the UV rectangle wraps and samples from the wrong tile. For an N×N atlas always compute the tile size as 1.0f / N and check that col < N and row < N before multiplying.
  • SaveScreenshot path not writable. Relative paths are resolved from the working directory of the process, which may not be writable in installed or sandboxed builds. Check the return value and inform the user if false is returned. For portable paths prefer the save-data directory exposed by the OS (e.g. ~/.local/share/MyGame/).

What you learned

In this part you gained complete control over the visual atmosphere of a simple-3d scene:

  • Sky dome: SetSkyDome(path) wraps a texture around a large inside-out sphere that follows the camera. SetSkyDomeEnabled(false) hides it for indoor or underground scenes without destroying the node.
  • Procedural gradient sky: SetSkyGradient(zenith, horizon) generates a two-colour gradient texture in memory — no image file needed — and applies it to the dome. It is the fastest way to give a scene a distinctive sky.
  • Ambient light: SetAmbientColor sets the minimum brightness of every surface. Low values create dramatic shadows; high values produce the flat look of overcast daylight. Tinting the colour shifts the whole scene's mood.
  • Linear fog: SetFogColor, SetFogRange, and SetFogEnabled work together to blend distant geometry into a chosen colour. Matching the fog colour to the sky horizon produces seamless depth without visible edges.
  • Unlit material colour: entity->SetMaterialColor applies a programmatic emissive material. The colour is constant regardless of lights, shadows, or ambient — useful for debug markers, UI elements in 3D space, and objects that must always be fully visible.
  • UV-atlas tile texture: entity->SetTileTexture(path, uOff, vOff, uScale, vScale) crops a rectangular region from a texture atlas and applies it as a lit diffuse material. This avoids needing a separate image file per tile.
  • Screenshots: SaveScreenshot(path) captures the full rendered framebuffer to a PNG, including the sky, geometry, and UI overlay.
  • Per-world presets: combining all the above calls into a single LoadWorld(id) helper makes it trivial to switch between dramatically different visual environments at runtime with no teardown required.
