How the gamepad API works

simple-3d wraps SDL's GameController system behind a small set of free functions that live on the Game base class. Every function follows the same signature pattern: the last parameter is always an optional int index (default 0) that selects which controller slot to query. Index 0 is the first controller plugged in, index 1 is the second, and so on. SDL applies its standard button-mapping database automatically, so the same GamepadButton::A refers to the correct physical button on an Xbox pad, a PlayStation pad, and most generic USB controllers.

The API is polling-based: you call it from Update() each frame, exactly as you call IsKeyDown for keyboard input. There are no events to subscribe to for axis or button state (hot-plug connection/disconnection being the only exception). This makes it trivial to mix keyboard and gamepad input in the same code path.

Complete working example

The example below creates a third-person character that can be controlled by keyboard alone, gamepad alone, or both simultaneously. It also shows D-pad camera reset, trigger-based dash, rumble on landing, hot-plug hint labels, and multi-player index selection. 

// GamepadDemo.cpp
// Part 15 — Gamepad Input
// Build: link against simple-3d

#include <Simple3D/Simple3D.h>
#include <cmath>
using namespace Simple3D;

class GamepadDemo : public Game {
public:
    void Start() override
    {
        // ── Ground ────────────────────────────────────────────
        Entity* ground = CreateEntity("Ground");
        ground->AddModel("Models/Plane.mdl");
        ground->AddRigidBody(0.0f);
        ground->AddBoxCollider(Vector3(40, 0.2f, 40));
        ground->SetCollisionLayer(CollisionLayer::StaticGeometry);

        // ── Player ────────────────────────────────────────────
        player_ = CreateEntity("Player");
        player_->SetPosition(0, 1, 0);
        player_->AddModel("Models/Hero.mdl");

        cc_ = player_->AddCharacterController(0.35f, 1.8f);
        cc_->SetGravity(25.0f);
        player_->SetCollisionLayer(CollisionLayer::Actor);
        player_->SetCollisionMask(MakeCollisionMask({
            CollisionLayer::StaticGeometry
        }));

        // ── Camera ────────────────────────────────────────────
        cam_ = CreateCamera();
        cam_->Follow(player_, 7.0f, 2.5f);
        cam_->LookAt(player_);
        cam_->SetSmoothFollow(true, 5.0f);

        // ── Light ─────────────────────────────────────────────
        Entity* sun = CreateEntity("Sun");
        sun->SetRotation(Quaternion(50, 30, 0));
        sun->AddDirectionalLight(Color::WHITE, 1.1f);

        // ── Dead zone: tighten the default slightly ───────────
        // Default is 0.20; lower values make the stick more sensitive.
        SetGamepadDeadZone(0.15f);

        // ── Hot-plug callbacks ────────────────────────────────
        // Called whenever a controller is plugged in or removed.
        SetOnGamepadConnected([this](int index) {
            UpdateConnectionLabel();
            URHO3D_LOGINFO(("Gamepad connected: index " +
                            std::to_string(index)).c_str());
        });
        SetOnGamepadDisconnected([this](int /*index*/) {
            UpdateConnectionLabel();
            URHO3D_LOGINFO("Gamepad disconnected");
        });

        // ── HUD labels ────────────────────────────────────────
        connLabel_ = CreateLabel("");
        connLabel_->SetPosition(20, 20);
        connLabel_->SetFontSize(18);
        connLabel_->SetColor(Color::WHITE);

        axisLabel_ = CreateLabel("");
        axisLabel_->SetPosition(20, 48);
        axisLabel_->SetFontSize(15);
        axisLabel_->SetColor(Color(0.85f, 0.85f, 0.85f, 1));

        hintLabel_ = CreateLabel(
            "WASD/Arrows=move  Space/A=jump  Shift/LB=dash  Esc=quit");
        hintLabel_->SetPosition(20, 72);
        hintLabel_->SetFontSize(14);
        hintLabel_->SetColor(Color(0.7f, 0.7f, 0.7f, 1));

        // Initial label text based on whether a pad is already plugged in
        UpdateConnectionLabel();
    }

    void Update(float dt) override
    {
        // ── Quit ──────────────────────────────────────────────
        if (IsKeyPressed(Key::Escape) ||
            IsGamepadButtonPressed(GamepadButton::Start))
        {
            Quit();
            return;
        }

        // ── Keyboard movement ─────────────────────────────────
        float mx = 0.0f, mz = 0.0f;
        if (IsKeyDown(Key::A) || IsKeyDown(Key::Left))  mx -= 1.0f;
        if (IsKeyDown(Key::D) || IsKeyDown(Key::Right)) mx += 1.0f;
        if (IsKeyDown(Key::W) || IsKeyDown(Key::Up))    mz += 1.0f;
        if (IsKeyDown(Key::S) || IsKeyDown(Key::Down))  mz -= 1.0f;

        // ── Gamepad movement (left stick) ─────────────────────
        // Always guard with IsGamepadConnected() before reading axes.
        if (IsGamepadConnected()) {
            // LeftX: left = -1, right = +1
            // LeftY: up = -1, down = +1 (SDL convention — negate for forward)
            mx += GetGamepadAxis(GamepadAxis::LeftX);
            mz -= GetGamepadAxis(GamepadAxis::LeftY);
        }

        // Normalise so diagonal movement is not faster than straight
        float len = std::sqrt(mx * mx + mz * mz);
        if (len > 1.0f) { mx /= len; mz /= len; }

        float speed = 5.0f;

        // ── Dash (keyboard Shift OR gamepad LB) ───────────────
        if (IsKeyDown(Key::LShift) ||
            (IsGamepadConnected() &&
             IsGamepadButtonDown(GamepadButton::LB)))
        {
            speed = 10.0f;
        }

        cc_->Move(Vector3(mx, 0, mz) * speed, dt);

        // ── Jump (keyboard Space OR gamepad A) ────────────────
        // IsGamepadButtonPressed fires only on the FIRST frame of press.
        bool jumpKey = IsKeyPressed(Key::Space);
        bool jumpPad = IsGamepadConnected() &&
                       IsGamepadButtonPressed(GamepadButton::A);
        if (jumpKey || jumpPad) {
            bool wasGround = cc_->IsOnGround();
            cc_->TryJump(9.0f);
            // Rumble when leaving the ground (not when pressing in mid-air)
            if (wasGround && jumpPad) {
                // Short sharp buzz: 80% low-freq, 40% high-freq, 0.12 s
                SetGamepadRumble(0.8f, 0.4f, 0.12f);
            }
        }

        // ── Right stick: camera orbit ─────────────────────────
        // Right stick Y controls camera pitch (look up/down).
        // Here we just log the value; a real game would rotate the camera.
        if (IsGamepadConnected()) {
            rightStickX_ = GetGamepadAxis(GamepadAxis::RightX);
            rightStickY_ = GetGamepadAxis(GamepadAxis::RightY);
        }

        // ── Triggers: read acceleration / braking ─────────────
        // Triggers return 0.0 (released) to 1.0 (fully pressed).
        // No dead zone is applied to triggers.
        if (IsGamepadConnected()) {
            triggerL_ = GetGamepadAxis(GamepadAxis::LTrigger);
            triggerR_ = GetGamepadAxis(GamepadAxis::RTrigger);
        }

        // ── D-Pad: UI / special actions ───────────────────────
        if (IsGamepadConnected()) {
            if (IsGamepadButtonPressed(GamepadButton::DPadUp)) {
                // Example: reset camera zoom
                cam_->Follow(player_, 7.0f, 2.5f);
            }
            if (IsGamepadButtonPressed(GamepadButton::Select)) {
                // Example: toggle HUD (Back / Share button)
                hintLabel_->SetVisible(!hintLabel_->IsVisible());
            }
        }

        // ── Landing rumble ────────────────────────────────────
        // Detect the moment of landing (was airborne last frame).
        bool grounded = cc_->IsOnGround();
        if (grounded && !wasGroundedLastFrame_ && IsGamepadConnected()) {
            // Strong low-frequency rumble for landing impact
            SetGamepadRumble(1.0f, 0.2f, 0.25f);
        }
        wasGroundedLastFrame_ = grounded;

        // ── Axis debug display ────────────────────────────────
        if (IsGamepadConnected()) {
            char buf[128];
            std::snprintf(buf, sizeof(buf),
                "LX=%.2f LY=%.2f  RX=%.2f RY=%.2f  LT=%.2f RT=%.2f",
                GetGamepadAxis(GamepadAxis::LeftX),
                GetGamepadAxis(GamepadAxis::LeftY),
                rightStickX_, rightStickY_,
                triggerL_, triggerR_);
            axisLabel_->SetText(buf);
        } else {
            axisLabel_->SetText("");
        }
    }

private:
    void UpdateConnectionLabel()
    {
        if (IsGamepadConnected()) {
            connLabel_->SetText("Gamepad: connected (index 0)");
            connLabel_->SetColor(Color(0.4f, 1.0f, 0.4f, 1));
        } else {
            connLabel_->SetText("Gamepad: not connected");
            connLabel_->SetColor(Color(1.0f, 0.5f, 0.3f, 1));
        }
    }

    Entity*              player_              = nullptr;
    CharacterController* cc_                  = nullptr;
    Camera*              cam_                 = nullptr;
    Label*               connLabel_           = nullptr;
    Label*               axisLabel_           = nullptr;
    Label*               hintLabel_           = nullptr;
    bool                 wasGroundedLastFrame_ = true;
    float                rightStickX_         = 0.0f;
    float                rightStickY_         = 0.0f;
    float                triggerL_            = 0.0f;
    float                triggerR_            = 0.0f;
};

int main()
{
    return Simple3D::CreateGame<GamepadDemo>()->Run();
}

Step-by-step explanation

IsGamepadConnected(int index = 0)

Returns true if a controller is currently plugged in at index. You should call this at the start of any code block that reads gamepad axes or buttons, including every frame inside Update(). Controllers can be connected and disconnected at any moment at runtime, so a guard that was true on one frame may be false on the next. Calling the axis or button functions without checking first is safe (they return zero/false for missing pads), but guarding explicitly makes the intended logic clear. The most common pattern is: 

if (IsGamepadConnected()) {
    mx += GetGamepadAxis(GamepadAxis::LeftX);
}

For a split-screen two-player game, use index 1 for the second player: 

if (IsGamepadConnected(1)) {
    float p2x = GetGamepadAxis(GamepadAxis::LeftX, 1);
}

GetGamepadAxis(GamepadAxis axis, int index = 0)

Returns a float representing the current deflection of the named axis. Stick axes (LeftX, LeftY, RightX, RightY) return a value in [-1, +1] after applying the dead zone. Values whose magnitude falls inside the dead zone are returned as exactly 0.0f, and the remaining range is linearly remapped so that full deflection always returns exactly 1.0 or -1.0 — you will never see a jump from 0.0 to 0.2 when the stick leaves the dead zone.

Trigger axes (LTrigger, RTrigger) return a value in [0, +1] with 0 meaning fully released and 1 meaning fully depressed. No dead zone is applied to triggers: the raw SDL value is returned directly. Use a small threshold in your own code if you need to prevent accidental activation from resting trigger pressure.

Note the SDL convention for stick Y axes: pushing the stick up returns a negative value. In a standard third-person movement setup you will want to negate LeftY so that "push up = move forward": 

mz -= GetGamepadAxis(GamepadAxis::LeftY);  // push up (negative Y) adds positive Z

IsGamepadButtonDown(GamepadButton button, int index = 0)

Returns true on every frame while the button is held down. Use this for continuous actions that should remain active as long as the button is held, such as sprinting, charging a shot, or holding a block pose. It is the gamepad equivalent of IsKeyDown(). 

// Sprint while LB is held
float speed = IsGamepadButtonDown(GamepadButton::LB) ? 10.0f : 5.0f;

IsGamepadButtonPressed(GamepadButton button, int index = 0)

Returns true only on the first frame the button transitions from released to pressed, regardless of how long it is then held. This is the gamepad equivalent of IsKeyPressed() and is the correct function for discrete one-shot actions such as jumping, interacting, or opening a menu. Using IsGamepadButtonDown for jump would cause the character to jump repeatedly on every frame the button is held. 

// Jump fires once per button press, even if the button is held
if (IsGamepadButtonPressed(GamepadButton::A))
    cc_->TryJump(9.0f);

SetGamepadDeadZone(float threshold) and GetGamepadDeadZone()

Sets the dead zone radius applied to all stick axes globally (not per-axis or per-controller). The default is 0.20, which is appropriate for most modern controllers. Values are clamped to the range [0, 0.95]. Call this once in Start(); changing it during gameplay mid-frame is safe but unusual.

A smaller dead zone (e.g. 0.10f) makes sticks more responsive but may cause visible drift when a worn controller does not return to true center. A larger dead zone (e.g. 0.30f) eliminates drift but makes small precise movements impossible. The default of 0.20 works well as a starting point; consider exposing it as a player setting in your options menu for accessibility. 

SetGamepadDeadZone(0.15f);           // more sensitive than the default 0.20
float dz = GetGamepadDeadZone();      // read back the current value

SetGamepadRumble(float lowFreq, float highFreq, float durationSec, int index = 0)

Triggers the controller's haptic motors. lowFreq drives the large low-frequency motor (felt as a deep body rumble, like an explosion or landing impact) and highFreq drives the small high-frequency motor (felt as a sharp buzz, like a collision or UI confirmation). Both values are normalised intensities in [0, 1]. durationSec is how long the rumble plays in seconds; after that time the engine stops the motors automatically. You do not need to call any "stop rumble" function.

If the connected controller does not support SDL_GameControllerRumble, a one-time warning is logged to the console and the call is silently ignored — it will never crash or produce an error in your game logic. The same applies when no controller is connected at the given index. 

// Heavy landing: strong low-freq, light high-freq, 0.25 seconds
SetGamepadRumble(1.0f, 0.2f, 0.25f);

// UI confirm: no low-freq, strong high-freq buzz, 0.08 seconds
SetGamepadRumble(0.0f, 0.9f, 0.08f);

// Target player 2's controller specifically
SetGamepadRumble(0.5f, 0.5f, 0.3f, 1);

SetOnGamepadConnected(callback) and SetOnGamepadDisconnected(callback)

These register a callback that fires when a gamepad is plugged in or removed at runtime. The callbacks accept an int index identifying the slot. For the connect callback, index is the slot that just became active and can be used immediately for axis/button queries inside the callback. For the disconnect callback, index is a best-effort value (usually 0) because the controller has already been removed from SDL's list by the time the callback fires.

Register both callbacks in Start(). They replace any previously registered callbacks (there is only one slot per callback type). Inside each callback you can update UI labels, pause the game, reassign player indices, or do any other logic that makes sense for your game. 

SetOnGamepadConnected([this](int index) {
    hintLabel_->SetText("Gamepad ready! Press A to start.");
    URHO3D_LOGINFO(("Controller at index " +
                    std::to_string(index) + " connected").c_str());
});

SetOnGamepadDisconnected([this](int /*index*/) {
    hintLabel_->SetText("Controller disconnected! Plug in to continue.");
    // Consider pausing the game here if it's controller-only
});

Combining keyboard and gamepad input

Because both keyboard and gamepad produce the same type of float values (or boolean flags), you can accumulate them into a single movement vector. This lets a player use whichever device they prefer without any mode-switching code: 

float mx = 0.0f, mz = 0.0f;

// Keyboard contributes ±1 on each axis
if (IsKeyDown(Key::A)) mx -= 1.0f;
if (IsKeyDown(Key::D)) mx += 1.0f;
if (IsKeyDown(Key::W)) mz += 1.0f;
if (IsKeyDown(Key::S)) mz -= 1.0f;

// Gamepad stick adds on top (both can be active at once)
if (IsGamepadConnected()) {
    mx += GetGamepadAxis(GamepadAxis::LeftX);
    mz -= GetGamepadAxis(GamepadAxis::LeftY);  // negate: up = forward
}

// Clamp the combined vector to unit length
float len = std::sqrt(mx * mx + mz * mz);
if (len > 1.0f) { mx /= len; mz /= len; }

cc_->Move(Vector3(mx, 0, mz) * speed, dt);

The clamping step is important: if a player holds both the keyboard and the left stick simultaneously, the raw sum could exceed 1.0 and the character would move faster than intended. Normalising to unit length keeps speed consistent regardless of how many input sources are active.

Reading right stick and triggers

The right stick is typically used for camera control in third-person games, or for aiming in shooters. Triggers are commonly used for analogue acceleration and braking in racing games, or for variable zoom in action games. The following snippet shows how to read all four additional axes: 

if (IsGamepadConnected()) {
    // Right stick: camera orbit delta this frame
    float camYaw   =  GetGamepadAxis(GamepadAxis::RightX) * 120.0f * dt;
    float camPitch = -GetGamepadAxis(GamepadAxis::RightY) * 80.0f  * dt;
    // ... apply camYaw / camPitch to your camera orbit angles ...

    // Triggers: analogue accelerate / brake
    float accel = GetGamepadAxis(GamepadAxis::RTrigger);  // 0 to 1
    float brake = GetGamepadAxis(GamepadAxis::LTrigger);  // 0 to 1
    float throttle = accel - brake;                        // -1 to +1
}

D-Pad and face buttons for menus

The D-Pad values are exposed as buttons, not as axes. Each direction is a separate GamepadButton constant. This makes them ideal for grid navigation in menus, discrete item cycling, or any action that should fire once per press. Using IsGamepadButtonPressed (not Down) ensures the action fires once per tap and does not auto-repeat at the frame rate. 

if (IsGamepadConnected()) {
    if (IsGamepadButtonPressed(GamepadButton::DPadUp))    selectedRow_--;
    if (IsGamepadButtonPressed(GamepadButton::DPadDown))  selectedRow_++;
    if (IsGamepadButtonPressed(GamepadButton::DPadLeft))  selectedCol_--;
    if (IsGamepadButtonPressed(GamepadButton::DPadRight)) selectedCol_++;
    if (IsGamepadButtonPressed(GamepadButton::A))          ConfirmSelection();
    if (IsGamepadButtonPressed(GamepadButton::B))          CancelMenu();
}

For a more integrated approach to gamepad menu navigation — with automatic D-Pad cycling and focus highlighting — see SetFocusGroup() and EnableFocusNavigation(), which are covered in Part 17 (UI System).

Multi-player: querying a specific controller index

Every gamepad API call accepts an optional int index as its last argument. Passing 0 (the default) queries the first connected controller; 1 queries the second, and so on. The index is assigned by SDL in the order controllers were plugged in. When a controller disconnects and reconnects, SDL may assign it a new index. 

void UpdatePlayer(int padIndex, CharacterController* cc, float dt)
{
    if (!IsGamepadConnected(padIndex)) return;

    float mx = GetGamepadAxis(GamepadAxis::LeftX, padIndex);
    float mz = -GetGamepadAxis(GamepadAxis::LeftY, padIndex);
    cc->Move(Vector3(mx, 0, mz) * 5.0f, dt);

    if (IsGamepadButtonPressed(GamepadButton::A, padIndex))
        cc->TryJump(9.0f);
}

// In Update(dt):
UpdatePlayer(0, player1cc_, dt);
UpdatePlayer(1, player2cc_, dt);

Lua equivalent

The raw gamepad API is fully exposed to Lua as global functions. String names are used for axis and button values instead of C++ enum scoping. Integer enum values are accepted as a fallback but string names are preferred for readability. 

-- Scripts/gamepad_player.lua
-- Raw gamepad input from Lua (Part 15)

local cc      -- CharacterController
local player  -- Entity

function Start()
    player = GetEntity()
    cc     = player:AddCharacterController(0.35, 1.8)
    cc:SetGravity(25.0)

    -- Tighten the dead zone
    SetGamepadDeadZone(0.15)

    -- Hot-plug callbacks
    SetOnGamepadConnected(function(index)
        Log("Gamepad connected at index " .. index)
    end)

    SetOnGamepadDisconnected(function(index)
        Log("Gamepad disconnected")
    end)
end

function Update(dt)
    -- Quit
    if IsKeyPressed("Escape") or
       (IsGamepadConnected(0) and
        IsGamepadButtonPressed("Start", 0)) then
        Quit()
        return
    end

    -- Movement: combine keyboard and left stick
    local mx, mz = 0, 0

    if IsKeyDown("A") then mx = mx - 1 end
    if IsKeyDown("D") then mx = mx + 1 end
    if IsKeyDown("W") then mz = mz + 1 end
    if IsKeyDown("S") then mz = mz - 1 end

    if IsGamepadConnected(0) then
        -- Lua string names: "LeftX", "LeftY", "RightX", "RightY", "LTrigger", "RTrigger"
        mx = mx + GetGamepadAxis("LeftX", 0)
        mz = mz - GetGamepadAxis("LeftY", 0)
    end

    -- Normalise diagonal
    local len = math.sqrt(mx * mx + mz * mz)
    if len > 1.0 then
        mx = mx / len
        mz = mz / len
    end

    cc:Move(Vector3(mx, 0, mz) * 5.0, dt)

    -- Jump: keyboard Space OR gamepad A button
    local jumpGamepad = IsGamepadConnected(0) and
                        IsGamepadButtonPressed("A", 0)

    if IsKeyPressed("Space") or jumpGamepad then
        cc:TryJump(9.0)
        if jumpGamepad then
            SetGamepadRumble(0.8, 0.4, 0.12, 0)
        end
    end

    -- Read right stick and triggers for debug output
    if IsGamepadConnected(0) then
        local rx = GetGamepadAxis("RightX", 0)
        local ry = GetGamepadAxis("RightY", 0)
        local lt = GetGamepadAxis("LTrigger", 0)
        local rt = GetGamepadAxis("RTrigger", 0)
        -- Log(string.format("RS=%.2f,%.2f  LT=%.2f RT=%.2f", rx, ry, lt, rt))
    end
end

Raw gamepad vs Input Actions

The raw polling API covered in this part is the right tool for: checking device presence with IsGamepadConnected, triggering rumble effects, reading triggers for analogue values, and debug overlays that display raw axis numbers. For gameplay logic that needs to support both keyboard and gamepad from a single code path — and ideally allow the player to remap controls — the Input Actions system covered in Part 16 is a cleaner abstraction. BindAction / BindAxis2D wrap exactly the polling shown here behind named actions, and SaveInputBindings / LoadInputBindings handle persistence.

Common mistakes

Common mistakes to avoid

  • Reading axes without checking IsGamepadConnected() first. While it is technically safe (functions return 0 for a missing controller), skipping the check makes the code ambiguous: you cannot tell whether the axis returned 0 because the stick is centred or because no pad is connected. Always guard with IsGamepadConnected() before reading input.
  • Using IsGamepadButtonDown for one-shot actions like jump. IsGamepadButtonDown is true on every frame while the button is held. For a jump running at 60 fps, this would fire 60 jump attempts per second. Use IsGamepadButtonPressed for actions that should trigger once per press.
  • Forgetting that LeftY is inverted (SDL convention). Pushing the left stick up returns a negative Y value. If you use GetGamepadAxis(GamepadAxis::LeftY) directly as a forward movement value, pushing up will move the character backward. Always negate: mz -= GetGamepadAxis(GamepadAxis::LeftY).
  • Expecting the dead zone to apply to triggers. The dead zone is applied only to stick axes (LeftX/Y, RightX/Y). Triggers return raw SDL values in [0, 1]. If you need a dead zone on a trigger (to avoid accidental activation from resting pressure), apply a threshold in your own code: if (GetGamepadAxis(GamepadAxis::RTrigger) > 0.05f) { ... }
  • Calling SetGamepadRumble every frame for a continuous effect. Calling SetGamepadRumble(0.3f, 0.1f, 0.05f) every frame (i.e. with a duration equal to roughly one frame) is the correct way to simulate continuous rumble that scales with gameplay intensity. Do not call it with a long duration and rely on the motors to stop themselves while your game state has already changed — start a new short rumble each frame instead so you can update intensity dynamically.
  • Hardcoding index 0 in hot-plug callbacks when you support multiple players. The index parameter of SetOnGamepadConnected tells you which slot just became active. Store it so you know which slot to query in Update() for that player. Do not assume it is always 0.
  • Not normalising the combined keyboard + gamepad movement vector. If the player holds a keyboard key and pushes the stick in the same direction, the raw sum can exceed 1.0 and the character will move faster than the design speed. Always normalise to unit length after combining sources.
  • Registering hot-plug callbacks after the first frame. Callbacks registered after the game has started will miss any plug-in events that occurred before they were registered. Register both callbacks at the end of Start() before Run() enters its main loop.

What you learned

  • IsGamepadConnected(index) is the entry-point guard for all gamepad input. Controllers can arrive and depart at any time, so call it every frame before reading axes or buttons.
  • GetGamepadAxis(GamepadAxis, index) returns stick values in [-1, +1] with the dead zone already applied and linearly remapped. Trigger values come back in [0, +1] with no dead zone. Push-up on the left stick returns a negative Y value (SDL convention) — negate it when up should mean forward.
  • IsGamepadButtonDown is for continuous held actions; IsGamepadButtonPressed is for one-shot discrete actions. Using the wrong one is the most common gamepad input bug.
  • Keyboard and gamepad input can be accumulated into the same float vector. Normalise the combined result to unit length so maximum speed is consistent regardless of which devices are active simultaneously.
  • SetGamepadDeadZone(0.15f) tunes the global stick dead zone. The default of 0.20 suits most controllers; lower it for responsive precision controls, raise it to suppress drift on worn hardware. Triggers are exempt from the dead zone.
  • SetGamepadRumble(lowFreq, highFreq, duration) triggers haptic motors. Large low-frequency values feel like heavy impacts; large high-frequency values feel like sharp buzzes. The call is a safe no-op on controllers that do not support rumble.
  • SetOnGamepadConnected and SetOnGamepadDisconnected fire when controllers are hotplugged. Use them to update UI, reassign player slots, or pause the game. Register them in Start() before the first frame.
  • The complete GamepadButton set covers A, B, X, Y, LB, RB, Start, Select, and all four D-Pad directions. SDL's mapping database ensures these map to the correct physical button on Xbox, PlayStation, and most generic USB controllers.
  • All raw gamepad functions are available as Lua globals. Use string names for axes and buttons: GetGamepadAxis("LeftX", 0), IsGamepadButtonPressed("A", 0).
  • For production gameplay code that needs remappable controls, see Part 16 (Input Actions). BindAction and BindAxis2D wrap the raw polling shown in this part behind named abstractions that are easier to extend and persist.
