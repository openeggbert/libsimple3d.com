Part 13 of 302D Physics

New in this part

AddRigidBody2D(mass) — AddBoxCollider2D(halfX, halfY) — AddCircleCollider2D(radius) — SetLinearVelocity2D(Vector2) — GetLinearVelocity2D() — ApplyImpulse2D(Vector2) — IsOnGround2D() — 2D and 3D physics coexisting in the same scene — side-scroller platformer pattern

What you will learn

  • How simple-3d's 2D physics system (Box2D) differs from its 3D physics system (Bullet) and why both run simultaneously in every scene.
  • How to create static (mass 0) and dynamic (mass > 0) 2D rigid bodies with AddRigidBody2D.
  • The two standard 2D collision shapes: axis-aligned box (AddBoxCollider2D) and circle (AddCircleCollider2D), and when to use each.
  • How to drive smooth horizontal movement by replacing only the X component of a body's velocity with SetLinearVelocity2D while leaving the Y component (gravity) untouched.
  • How to launch a jump using ApplyImpulse2D, and how to gate the jump with IsOnGround2D to prevent double-jumping.
  • The complete side-scroller platformer pattern: ground, floating platforms, a player that runs and jumps, and collectible coins.
  • The Lua equivalents of every 2D physics call so you can script game behaviour without recompiling.

Box2D vs Bullet: two physics worlds in one scene

simple-3d ships with two independent physics engines that coexist in every scene:

Bullet (3D)

The 3D rigid-body solver. Powers AddRigidBody, AddBoxCollider, AddCapsuleCollider, AddSphereCollider, and the CharacterController.

Works in full X/Y/Z space. Gravity acts along the −Y axis.

Box2D (2D)

The 2D rigid-body solver. Powers AddRigidBody2D, AddBoxCollider2D, AddCircleCollider2D, and tile-map collision.

Works in the XY plane only. Z is used as draw-order depth and is ignored by physics.

No conflict

Bullet and Box2D operate on completely separate simulation islands. An entity can only belong to one world: either AddRigidBody (3D) or AddRigidBody2D (2D), never both.

You can, however, have some entities using 3D physics and others using 2D physics in the same scene — for example, a 2D platformer player next to a 3D background particle explosion.

PhysicsWorld2D is always available

PhysicsWorld2D (the Box2D scene node) is added to the scene automatically when Game::Run() starts. You never need to create it manually. Simply call AddRigidBody2D on any entity and it registers with the 2D world immediately. The same is true for the 3D Bullet world — both are always running.

How 2D physics works in simple-3d

The setup for 2D physics follows the same two-step pattern as 3D physics:

  1. Add a 2D rigid body — the mass, gravity, and velocity container. Created with AddRigidBody2D(mass).
  2. Add at least one 2D collision shape — the geometry Box2D uses for contact detection. Created with AddBoxCollider2D or AddCircleCollider2D.

Box2D gravity defaults to −10 m/s² downward along Y. Dynamic bodies fall and stack; static bodies (mass 0) block them. The simulation runs every frame in step with the engine's update loop, so you simply read and write velocity in Update(dt) and the physics handles collisions and integration automatically.

Because Box2D is a planar solver, the Z position of an entity has no effect on its physics. In a 2D game you set Z to control draw order (entities closer to the camera, i.e. higher Z, are drawn on top). The physics collision always projects onto the XY plane.

Choosing the right 2D collision shape

Box

Best for platforms, ground tiles, crates, and any rectangular object. The two parameters are the half-extents in metres: halfX is the half-width and halfY is the half-height. A AddBoxCollider2D(1.0f, 0.5f) produces a box 2 m wide and 1 m tall.

AddBoxCollider2D(halfX, halfY)

Circle

Best for coins, balls, bouncing projectiles, and any roughly round sprite. Circles are the cheapest 2D shape to test — contact between two circles is a single distance check. The single parameter is the radius in metres.

AddCircleCollider2D(radius)

Half-extents vs full size

Unlike the 3D AddBoxCollider(Vector3) which takes the full size, AddBoxCollider2D takes half-extents, following Box2D convention. So AddBoxCollider2D(0.3f, 0.5f) is a 0.6 m wide, 1.0 m tall bounding box.

Always halve your sprite's world dimensions before passing them here.

Complete example: side-scroller platformer

The game below implements a small side-scroller. The player runs left and right with the arrow keys and jumps with Space. Five coins are scattered across floating platforms; walking into one collects it. The demo exercises every new API introduced in this part: static ground and platforms with AddRigidBody2D(0) + AddBoxCollider2D, a dynamic player with AddRigidBody2D(mass) + AddBoxCollider2D, circular coin colliders via AddCircleCollider2D, horizontal movement via SetLinearVelocity2D, jumping via ApplyImpulse2D, and a ground check via IsOnGround2D. 

// Platformer2D.cpp — Part 13 of the simple-3d tutorial series
#include <Simple3D/Simple3D.h>
#include <string>
#include <array>

using namespace Simple3D;

class Platformer2D : public Game {
public:
    Entity*  player_     = nullptr;
    Label*   statusLbl_  = nullptr;
    int      coins_      = 0;
    static constexpr int kTotalCoins = 5;
    std::array<Entity*, kTotalCoins> coinEntities_ = {};

    void Start() override {
        SetWindowTitle("Part 13 — 2D Physics");
        SetClearColor(Color(0.05f, 0.10f, 0.22f));

        // ── 2D orthographic camera ──────────────────────────────────
        // Setup2D sets orthographic projection and positions the camera
        // on the Z axis so the XY world plane is fully visible.
        Camera* cam = CreateCamera();
        cam->Setup2D(100.0f);   // 100 pixels per world unit

        // A directional light is needed even in 2D to shade sprites
        // that use lit materials. Unlit sprites (SetMaterialColor) skip it.
        Entity* sun = CreateEntity("Sun");
        sun->SetRotation(Quaternion(45, 30, 0));
        sun->AddDirectionalLight(Color::WHITE, 1.2f);

        // ── GROUND — static 2D body ─────────────────────────────────
        // AddRigidBody2D(0.0f): mass 0 = static; immovable; blocks dynamics.
        Entity* ground = CreateEntity("Ground");
        ground->SetPosition(Vector3(0.0f, -3.5f, 0.0f));
        ground->SetMaterialColor(Color(0.30f, 0.55f, 0.25f));
        ground->AddModel("Models/Box.mdl");
        ground->SetScale(Vector3(16.0f, 1.0f, 0.2f));
        ground->AddRigidBody2D(0.0f);           // static ground
        // halfX = 8.0 m (full width 16 m), halfY = 0.5 m (full height 1 m)
        ground->AddBoxCollider2D(8.0f, 0.5f);

        // ── PLATFORMS — static 2D bodies ────────────────────────────
        // A lambda keeps the setup compact; each platform is an
        // independent static entity with its own box collider.
        auto makePlatform = [&](float x, float y, float halfW) {
            Entity* p = CreateEntity("Platform");
            p->SetPosition(Vector3(x, y, 0.0f));
            p->SetMaterialColor(Color(0.50f, 0.35f, 0.15f));
            p->AddModel("Models/Box.mdl");
            p->SetScale(Vector3(halfW * 2.0f, 0.5f, 0.2f));
            p->AddRigidBody2D(0.0f);
            p->AddBoxCollider2D(halfW, 0.25f);  // halfY = 0.25 (height 0.5 m)
        };
        makePlatform(-4.0f, -1.5f, 1.5f);   // left platform, 3 m wide
        makePlatform( 0.5f, -0.5f, 1.0f);   // centre platform, 2 m wide
        makePlatform( 4.5f,  1.0f, 1.2f);   // high right platform, 2.4 m wide

        // ── PLAYER — dynamic 2D body ─────────────────────────────────
        // AddRigidBody2D(1.0f): 1 kg dynamic body — falls under gravity,
        // responds to SetLinearVelocity2D and ApplyImpulse2D.
        player_ = CreateEntity("Player");
        player_->SetPosition(Vector3(-6.0f, 0.0f, 0.0f));
        player_->SetMaterialColor(Color(0.20f, 0.55f, 0.95f));
        player_->AddModel("Models/Box.mdl");
        player_->SetScale(Vector3(0.6f, 1.0f, 0.2f));
        player_->AddRigidBody2D(1.0f);          // 1 kg dynamic body
        // halfX = 0.3 m (width 0.6 m), halfY = 0.5 m (height 1.0 m)
        player_->AddBoxCollider2D(0.3f, 0.5f);

        // ── COINS — static 2D bodies with circle colliders ───────────
        // Coins are static (they should not fall). We use AddCircleCollider2D
        // because circles are the cheapest 2D shape and match coin sprites well.
        const float coinX[kTotalCoins] = { -4.0f, -4.5f,  0.5f,  4.5f,  5.0f };
        const float coinY[kTotalCoins] = { -0.7f, -0.7f,  0.3f,  1.8f,  1.8f };
        for (int i = 0; i < kTotalCoins; ++i) {
            Entity* coin = CreateEntity("Coin" + std::to_string(i));
            coin->SetPosition(Vector3(coinX[i], coinY[i], 0.1f)); // z=0.1 in front
            coin->SetMaterialColor(Color(1.0f, 0.85f, 0.10f));
            coin->AddModel("Models/Sphere.mdl");
            coin->SetScale(0.35f);
            coin->AddRigidBody2D(0.0f);         // static — coins float in place
            coin->AddCircleCollider2D(0.18f);   // radius 0.18 m
            coinEntities_[i] = coin;
        }

        // ── HUD ──────────────────────────────────────────────────────
        statusLbl_ = CreateLabel("Coins: 0 / 5  |  Arrows = move  Space = jump  ESC = quit");
        statusLbl_->SetPosition(12, 12);
        statusLbl_->SetFontSize(16);
        statusLbl_->SetColor(Color::WHITE);
    }

    void Update(float dt) override {
        if (IsKeyPressed(Key::Escape))
            Quit();

        // ── Horizontal movement via SetLinearVelocity2D ──────────────
        // We read the current 2D velocity first to preserve the Y component.
        // The Y component is driven by Box2D gravity and must not be zeroed out
        // or the player will hover and jumps will have no arc.
        Vector2 vel = player_->GetLinearVelocity2D();
        float moveX = 0.0f;
        if (IsKeyDown(Key::Left))  moveX = -5.0f;
        if (IsKeyDown(Key::Right)) moveX =  5.0f;

        // SetLinearVelocity2D replaces the body's velocity entirely.
        // We override X with our desired speed and keep Y from gravity.
        player_->SetLinearVelocity2D(Vector2(moveX, vel.y_));

        // Flip the sprite so the player faces the direction of travel.
        if (moveX < 0.0f) player_->SetFlipX2D(true);
        if (moveX > 0.0f) player_->SetFlipX2D(false);

        // ── Jump via ApplyImpulse2D ──────────────────────────────────
        // IsOnGround2D() casts a short ray downward and returns true only
        // when the player is resting on a static body. Gating the jump on
        // this prevents double-jumping in mid-air.
        if (IsKeyPressed(Key::Space) && player_->IsOnGround2D()) {
            // ApplyImpulse2D adds an instant velocity change, not replacing it.
            // For a 1 kg body Vector2(0, 8) gives an 8 m/s upward kick.
            player_->ApplyImpulse2D(Vector2(0.0f, 8.0f));
        }

        // ── Coin collection via proximity check ──────────────────────
        // We use a simple distance test against the player's world position.
        // When the player overlaps a coin we deactivate it (hide + suspend
        // physics) and increment the coin counter.
        Vector3 pPos = player_->GetPosition();
        for (int i = 0; i < kTotalCoins; ++i) {
            Entity* coin = coinEntities_[i];
            if (!coin || !coin->IsActive()) continue;
            Vector3 cPos = coin->GetPosition();
            float dist = (pPos - cPos).Length();
            if (dist < 0.6f) {
                coin->SetActive(false);   // hide + freeze physics
                ++coins_;
            }
        }

        // ── HUD update ────────────────────────────────────────────────
        std::string grounded = player_->IsOnGround2D() ? " [GROUNDED]" : " [AIRBORNE]";
        statusLbl_->SetText(
            "Coins: " + std::to_string(coins_) + " / " + std::to_string(kTotalCoins) +
            grounded + "  |  Arrows=move  Space=jump  ESC=quit"
        );
    }
};

int main() {
    return Simple3D::CreateGame<Platformer2D>()->Run();
}

Step-by-step explanation

AddRigidBody2D(mass)

This is the gateway call that opts an entity into the Box2D simulation. Internally it creates an Urho3D RigidBody2D component and registers it with the PhysicsWorld2D scene node. The single parameter is mass in kilograms.

  • 0.0f — static body. The body has infinite effective mass. Other bodies collide with it and bounce or slide, but the static body itself is never displaced by gravity or forces. Use 0.0f for all level geometry: the ground, walls, platforms, and any surface the player stands on. Static bodies are much cheaper to simulate than dynamic ones.
  • Any positive float — dynamic body. Box2D applies gravity each simulation step. The body accumulates velocity, responds to ApplyImpulse2D and SetLinearVelocity2D, and participates in collision resolution. The higher the mass value, the more force is required to accelerate the body. For a humanoid character 1 kg is common; the exact value is less important than tuning the impulse magnitudes to match.

Unlike the 3D AddRigidBody, the 2D version does not lock rotation axes by default. If your player sprite tilts when it slides against a wall, call body2D->SetFixedRotation(true) on the Urho3D RigidBody2D component directly, or simply keep the visual sprite rotation locked at zero in Update.

entity->AddRigidBody2D(float mass);

AddBoxCollider2D(halfX, halfY)

Attaches an axis-aligned rectangular collision shape to the entity. The two parameters are the half-extents in metres: halfX is half the total width and halfY is half the total height. This matches Box2D's native convention (b2PolygonShape::SetAsBox also takes half-extents).

Concretely: to cover a sprite that is 0.6 m wide and 1.0 m tall, pass AddBoxCollider2D(0.3f, 0.5f). If your collision box feels twice as wide as expected, you have probably passed the full size rather than half the size.

The shape is centered on the entity's local origin. If the sprite's visual center does not align with the physics center, offset the entity itself or use a child entity offset from the parent.

entity->AddBoxCollider2D(float halfX, float halfY);

AddCircleCollider2D(radius)

Attaches a circular collision shape with the given radius in metres. This is the cheapest shape Box2D can test — contact between two circles reduces to a single distance comparison, and circle-vs-box is also simpler than box-vs-box. Use circles for coins, projectiles, balls, and any sprite that is roughly round.

Like the box collider, the circle is centered on the entity's local origin. The radius does not have to match the sprite exactly; using a radius slightly smaller than the visual sprite is common practice to avoid the player feeling like they missed a coin they nearly touched.

entity->AddCircleCollider2D(float radius);

SetLinearVelocity2D(Vector2) and GetLinearVelocity2D()

SetLinearVelocity2D replaces the body's current velocity with the supplied Vector2. The X component is horizontal speed (positive = right) and the Y component is vertical speed (positive = up). Calling this every frame is the standard way to drive character movement in a 2D game — you set exactly how fast the player moves horizontally while Box2D handles vertical motion via gravity.

The critical side-scroller pattern is to never zero out Y: 

// CORRECT: read current Y (gravity-driven), only replace X
Vector2 vel = player_->GetLinearVelocity2D();
player_->SetLinearVelocity2D(Vector2(moveX, vel.y_));

// WRONG: this kills gravity and the player can never fall
player_->SetLinearVelocity2D(Vector2(moveX, 0.0f));

GetLinearVelocity2D returns the body's velocity as a Vector2. You can read it at any time, including before calling Set, to inspect the current speed for animation, sound, or HUD purposes.

entity->SetLinearVelocity2D(Vector2 velocity); Vector2 v = entity->GetLinearVelocity2D();

ApplyImpulse2D(Vector2)

Applies an instantaneous velocity change to the 2D rigid body. Unlike SetLinearVelocity2D, which replaces the velocity, ApplyImpulse2D adds to it. The unit is Newton-seconds (kg·m/s): for a 1 kg body, ApplyImpulse2D(Vector2(0, 8)) imparts exactly 8 m/s upward. For a 2 kg body, the same call gives only 4 m/s upward.

Impulses are the correct tool for discrete events such as jumps, knockback from a hit, and explosive blasts. Because they add to whatever velocity the body already has, you should check IsOnGround2D() before applying a jump impulse, otherwise repeated presses in the air would compound the upward speed indefinitely.

entity->ApplyImpulse2D(Vector2 impulse);

IsOnGround2D()

Casts a short ray (0.6 m) downward from the entity's 2D position and returns true if it hits any static 2D body. Use it to gate jumps, trigger landing dust particles, and switch between airborne and grounded animation states. The ray length is intentionally short so that a body falling at high speed still returns true only on the frame it actually makes contact, not while it is several metres above the floor.

bool grounded = entity->IsOnGround2D();

2D and 3D physics in the same scene

Because Bullet and Box2D operate on completely separate simulation islands you can freely mix them in a single scene. A common creative use is a 2.5D game: the player and all interactive objects use Box2D for physics and collision, while background decoration such as distant mountains, spinning gears, or fire particles use 3D rendering (and optionally 3D Bullet physics) to add depth.

The rule is: each entity may belong to at most one physics world. Call either AddRigidBody (3D) or AddRigidBody2D (2D) per entity, never both. The following short example demonstrates two entities in the same scene, each using a different physics world: 

// In Start() — mixing 2D and 3D physics in one scene

// This crate lives in the Box2D 2D world — falls under 2D gravity.
Entity* crate2D = CreateEntity("Crate2D");
crate2D->SetPosition(Vector3(0.0f, 4.0f, 0.0f));
crate2D->AddRigidBody2D(1.0f);
crate2D->AddBoxCollider2D(0.5f, 0.5f);  // 1m × 1m box, half-extents

// This spinning debris lives in the Bullet 3D world — tumbles freely.
Entity* debris3D = CreateEntity("Debris3D");
debris3D->SetPosition(Vector3(3.0f, 4.0f, -1.5f)); // behind the 2D plane
debris3D->AddRigidBody(0.5f);
debris3D->AddSphereCollider(0.3f);

// The two bodies NEVER interact — different simulation worlds.
// Visually they appear in the same scene; physically they are isolated.

Tip: Z-depth in 2D games

In 2D mode the camera looks along the −Z axis. Objects with a larger Z value appear in front of objects with a smaller Z value. This lets you layer background elements (Z = −5), gameplay entities (Z = 0), and foreground overlays (Z = 5) without any physics interaction between the layers. The Box2D solver only cares about X and Y; Z is purely visual.

The side-scroller pattern in detail

A side-scroller (horizontal platformer) built with simple-3d 2D physics follows a repeatable pattern. Here is the recipe extracted from the example above:

1. Orthographic camera

cam->Setup2D(100.0f);   // 100 pixels per world unit

Setup2D sets the camera to orthographic projection and positions it far enough along +Z to see the full XY plane without clipping. Pass the pixel-per-world-unit ratio that matches your sprite sheet. At 100 PPU, a sprite sheet tile that is 64 × 64 pixels occupies 0.64 m in world space.

2. Static ground and platforms

// Ground strip at the bottom of the level
Entity* ground = CreateEntity("Ground");
ground->SetPosition(Vector3(0, -3.5f, 0));
ground->AddRigidBody2D(0.0f);           // static
ground->AddBoxCollider2D(8.0f, 0.5f);   // 16m wide, 1m tall

// Floating platform
Entity* plat = CreateEntity("Platform");
plat->SetPosition(Vector3(2.0f, -0.5f, 0));
plat->AddRigidBody2D(0.0f);
plat->AddBoxCollider2D(1.0f, 0.25f);   // 2m wide, 0.5m tall

3. Dynamic player character

player_->AddRigidBody2D(1.0f);          // dynamic, falls under gravity
player_->AddBoxCollider2D(0.3f, 0.5f);   // 0.6m wide, 1m tall

4. Run: SetLinearVelocity2D, keep Y

Vector2 vel = player_->GetLinearVelocity2D();
float moveX = IsKeyDown(Key::Right) ? 5.0f : IsKeyDown(Key::Left) ? -5.0f : 0.0f;
player_->SetLinearVelocity2D(Vector2(moveX, vel.y_));

5. Jump: ApplyImpulse2D gated on IsOnGround2D

if (IsKeyPressed(Key::Space) && player_->IsOnGround2D())
    player_->ApplyImpulse2D(Vector2(0.0f, 8.0f));

6. Collectibles: static circle colliders, manual overlap check

coin->AddRigidBody2D(0.0f);         // static coin, will not fall
coin->AddCircleCollider2D(0.18f);   // circle shape, radius 0.18 m

// In Update — distance-based pickup
if ((player_->GetPosition() - coin->GetPosition()).Length() < 0.6f)
    coin->SetActive(false);

Camera follow in a side-scroller

The example above uses a fixed camera to keep the code short. In a real side-scroller you normally want the camera to follow the player horizontally. The simplest approach is to update the camera's X position each frame to track the player while keeping Y at a fixed comfortable height: 

// In Update(dt) — smooth horizontal camera follow
Vector3 camPos = cam->GetPosition();
Vector3 pPos   = player_->GetPosition();
camPos.x_ += (pPos.x_ - camPos.x_) * 5.0f * dt;  // smooth lerp
cam->SetPosition(camPos);

2D physics API quick reference

Method Notes
AddRigidBody2D(0.0f) Static 2D body. Immovable. Blocks and collides with dynamic bodies.
AddRigidBody2D(mass) Dynamic 2D body. Subject to Box2D gravity and forces. Mass in kg.
AddBoxCollider2D(halfX, halfY) Axis-aligned box shape. Parameters are half-extents in metres.
AddCircleCollider2D(radius) Circle shape. Cheapest shape. Ideal for coins, balls, projectiles.
SetLinearVelocity2D(Vector2) Replaces the body's velocity. Override only X; preserve Y for gravity.
GetLinearVelocity2D() Returns current 2D velocity as Vector2.
ApplyImpulse2D(Vector2) Adds an instant velocity change (kg·m/s). Ideal for jumps and knockback.
IsOnGround2D() Downward ray 0.6 m against static 2D bodies. Returns bool.

Lua equivalent

All 2D physics methods are available from Lua. Method names are identical. The snippet below is a complete Lua version of the platformer from the example above. Attach it to an entity with entity:AddScript("Scripts/Platformer2D.lua"), or load it as a standalone game script. 

-- Platformer2D.lua — Part 13 of the simple-3d tutorial series

local player_, statusLbl_
local coins_ = 0
local totalCoins = 5
local coinEntities = {}

function Start()
    SetWindowTitle("Part 13 — 2D Physics (Lua)")
    SetClearColor(Color(0.05, 0.10, 0.22))

    -- 2D orthographic camera
    local cam = CreateCamera()
    cam:Setup2D(100.0)

    local sun = CreateEntity("Sun")
    sun:SetRotation(Quaternion(45, 30, 0))
    sun:AddDirectionalLight(Color(1, 1, 1), 1.2)

    -- Ground — static 2D body
    local ground = CreateEntity("Ground")
    ground:SetPosition(Vector3(0, -3.5, 0))
    ground:SetMaterialColor(Color(0.30, 0.55, 0.25))
    ground:AddModel("Models/Box.mdl")
    ground:SetScale(Vector3(16, 1, 0.2))
    ground:AddRigidBody2D(0.0)
    ground:AddBoxCollider2D(8.0, 0.5)   -- halfX=8, halfY=0.5

    -- Platforms — static 2D bodies
    local function makePlatform(x, y, halfW)
        local p = CreateEntity("Platform")
        p:SetPosition(Vector3(x, y, 0))
        p:SetMaterialColor(Color(0.50, 0.35, 0.15))
        p:AddModel("Models/Box.mdl")
        p:SetScale(Vector3(halfW * 2, 0.5, 0.2))
        p:AddRigidBody2D(0.0)
        p:AddBoxCollider2D(halfW, 0.25)
    end
    makePlatform(-4.0, -1.5, 1.5)
    makePlatform( 0.5, -0.5, 1.0)
    makePlatform( 4.5,  1.0, 1.2)

    -- Player — dynamic 2D body
    player_ = CreateEntity("Player")
    player_:SetPosition(Vector3(-6, 0, 0))
    player_:SetMaterialColor(Color(0.20, 0.55, 0.95))
    player_:AddModel("Models/Box.mdl")
    player_:SetScale(Vector3(0.6, 1.0, 0.2))
    player_:AddRigidBody2D(1.0)           -- 1 kg dynamic body
    player_:AddBoxCollider2D(0.3, 0.5)    -- halfX=0.3, halfY=0.5

    -- Coins — static 2D bodies with circle colliders
    local coinX = { -4.0, -4.5,  0.5,  4.5,  5.0 }
    local coinY = { -0.7, -0.7,  0.3,  1.8,  1.8 }
    for i = 1, totalCoins do
        local coin = CreateEntity("Coin" .. i)
        coin:SetPosition(Vector3(coinX[i], coinY[i], 0.1))
        coin:SetMaterialColor(Color(1.0, 0.85, 0.10))
        coin:AddModel("Models/Sphere.mdl")
        coin:SetScale(0.35)
        coin:AddRigidBody2D(0.0)          -- static
        coin:AddCircleCollider2D(0.18)    -- radius 0.18 m
        coinEntities[i] = coin
    end

    -- HUD label
    statusLbl_ = CreateLabel("Coins: 0 / 5  |  Arrows = move  Space = jump")
    statusLbl_:SetPosition(12, 12)
    statusLbl_:SetFontSize(16)
    statusLbl_:SetColor(Color(1, 1, 1))
end

function Update(dt)
    if IsKeyPressed(Key.Escape) then Quit() end

    -- Horizontal movement: read Y, override X only
    local vel = player_:GetLinearVelocity2D()
    local moveX = 0
    if IsKeyDown(Key.Left)  then moveX = -5 end
    if IsKeyDown(Key.Right) then moveX =  5 end
    player_:SetLinearVelocity2D(Vector2(moveX, vel.y))

    if moveX < 0 then player_:SetFlipX2D(true)  end
    if moveX > 0 then player_:SetFlipX2D(false) end

    -- Jump: impulse upward when grounded
    if IsKeyPressed(Key.Space) and player_:IsOnGround2D() then
        player_:ApplyImpulse2D(Vector2(0, 8))
    end

    -- Coin collection
    local pPos = player_:GetPosition()
    for i = 1, totalCoins do
        local coin = coinEntities[i]
        if coin and coin:IsActive() then
            local cPos = coin:GetPosition()
            local dx = pPos.x - cPos.x
            local dy = pPos.y - cPos.y
            if (dx*dx + dy*dy) < 0.36 then   -- 0.6^2
                coin:SetActive(false)
                coins_ = coins_ + 1
            end
        end
    end

    local grounded = player_:IsOnGround2D() and " [GROUNDED]" or " [AIRBORNE]"
    statusLbl_:SetText("Coins: " .. coins_ .. " / " .. totalCoins .. grounded)
end

Lua note

The 2D physics methods (AddRigidBody2D, AddBoxCollider2D, AddCircleCollider2D, SetLinearVelocity2D, GetLinearVelocity2D, ApplyImpulse2D, IsOnGround2D) are available on every entity object in Lua using colon syntax. Vector2 is exposed as a global constructor; access its fields as .x and .y (note: in C++ they are x_ and y_; Lua strips the underscore).

Common mistakes

Common mistakes

  • Treating AddBoxCollider2D arguments as full size rather than half-extents. Unlike the 3D AddBoxCollider(Vector3), which takes the full width/height/depth, AddBoxCollider2D takes half-extents following Box2D convention. If your platform is twice as wide as expected or objects are floating above the ground, check that you are dividing the sprite dimensions by two before passing them here.
  • Zeroing out the Y component in SetLinearVelocity2D. A very common bug: writing SetLinearVelocity2D(Vector2(moveX, 0)) every frame cancels gravity entirely. The player will hover when walking off a ledge and jumps will have no arc. Always preserve the current Y with GetLinearVelocity2D().y_ and pass it back in.
  • Calling ApplyImpulse2D every frame instead of on key press. ApplyImpulse2D adds to the velocity each call. In Update, which runs 60 times per second, calling it unconditionally while a key is held will shoot the player into orbit. Use IsKeyPressed (fires once on the first frame) rather than IsKeyDown (fires every frame) to trigger jumps and one-shot events.
  • Mixing AddRigidBody (3D) and AddRigidBody2D (2D) on the same entity. Each entity may belong to exactly one physics world. Calling both on the same entity produces undefined behaviour: the entity will appear to work in one world while silently corrupting the other simulation. Keep them separate — one entity, one physics world.
  • Forgetting to set position before AddRigidBody2D. Once a Box2D body exists, the physics world records the entity's starting position. Call SetPosition before AddRigidBody2D. Moving an entity after its body exists requires SetLinearVelocity2D or ApplyImpulse2D; calling SetPosition on a live dynamic body can cause position discontinuities and collision tunnelling.
  • IsOnGround2D returning false even when standing on a static body. IsOnGround2D casts a ray 0.6 m downward. If the platform collider was created without a rigid body, or if the collider half-height is very large and the entity center is far above the surface, the ray may not reach it. Verify that every platform has both AddRigidBody2D(0.0f) and a collider, and that the collider half-extents are sensible relative to the entity's center.
  • Expecting 2D and 3D physics to interact. A 2D Box2D body and a 3D Bullet body will never generate a collision contact with each other, even if their visual models overlap in world space. The two simulation worlds are completely independent. Design your scene so either all gameplay-critical objects use 2D physics or all use 3D physics; reserve the other for pure visual decoration.

What you learned

In this part you built a working 2D physics-based platformer from scratch. Here is a concise summary of every concept covered:

  • simple-3d includes both Bullet (3D) and Box2D (2D) physics worlds in every scene. They run independently and never interact with each other; each entity can belong to at most one world.
  • AddRigidBody2D(0.0f) creates a static Box2D body (ground, walls, platforms). AddRigidBody2D(mass) with a positive mass creates a dynamic body that falls under Box2D gravity.
  • AddBoxCollider2D(halfX, halfY) attaches a rectangular shape using half-extents (not full size). A 2 m wide platform uses halfX = 1.0f.
  • AddCircleCollider2D(radius) attaches a circular shape — the cheapest 2D shape, ideal for coins, balls, and collectibles.
  • The side-scroller movement pattern reads the current velocity with GetLinearVelocity2D(), overrides only the X component with the desired horizontal speed, and writes it back with SetLinearVelocity2D — this preserves the Y component that Box2D calculates via gravity.
  • ApplyImpulse2D(Vector2) adds an instantaneous velocity change — the correct tool for jumps and knockback. Gate jumps with IsOnGround2D() to prevent double-jumping.
  • IsOnGround2D() casts a 0.6 m downward ray against static 2D bodies and returns true when the entity is resting on a surface.
  • The PhysicsWorld2D component is added automatically by Game::Run() — you never create it manually, just call the 2D API and it is ready immediately.
