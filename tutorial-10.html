Part 10 of 30Audio

New in this part

What you will learn

  • How to stream background music from an OGG file with automatic looping.
  • How to stop or switch the background music track at runtime.
  • How to trigger short, fire-and-forget WAV sound effects with a single call.
  • How to play a sound on a numbered channel so you can track and control it later.
  • How to stop a specific channel mid-playback with StopSound(channel).
  • How to poll whether a channel is still active with IsChannelPlaying(channel).
  • How to adjust the per-channel volume at runtime with SetChannelVolume(channel, v).
  • How to scale the overall audio output with SetMasterVolume(v).
  • Best-practice asset organisation: OGG for music, WAV for SFX.

Background: two kinds of audio

simple-3d separates audio into two clear categories, each with its own API surface:

Music (streaming)

A single background track decoded on the fly from disk. Only one music track plays at a time. Use OGG Vorbis (.ogg) — it compresses well and streams efficiently without holding the entire file in RAM.

Sound effects (channels)

Short clips loaded once and played immediately. Use WAV (.wav) — uncompressed means zero decode latency, which matters for responsive game feel. Channels let you reference a playing sound so you can stop or adjust it after the fact.

A good rule of thumb: anything longer than a few seconds belongs in OGG music; anything shorter belongs in WAV effects.

Recommended asset layout

Put your audio files under Data/ in your project tree so the resource cache can find them by relative path:

Data/
  Music/
    theme.ogg          // main menu / gameplay theme — OGG
    boss.ogg           // boss encounter track — OGG
  Sounds/
    jump.wav           // player jump — WAV
    land.wav           // landing impact — WAV
    coin.wav           // collectible pickup — WAV
    footstep.wav       // footstep — WAV
    engine_loop.wav    // vehicle engine (channel-managed) — WAV
    explosion.wav      // one-shot explosion — WAV

Complete working example

The example below builds a small interactive scene that demonstrates every audio API covered in this part. The player controls a character and the game reacts with music crossfades, footstep sounds, an engine loop on a dedicated channel, and a global volume slider driven by keyboard input. 

#include <Simple3D/Simple3D.h>

using namespace Simple3D;

// ── Channel indices ──────────────────────────────────────────────────
// Assigning named constants avoids magic numbers scattered across Update().
static constexpr int CH_ENGINE  = 0;   // looping engine rumble
static constexpr int CH_VOICE   = 1;   // character voice lines
static constexpr int CH_AMBIENT = 2;   // wind / rain loop

class AudioDemo : public Game
{
public:
    // ── Scene objects ────────────────────────────────────────────────
    Entity*  player_        = nullptr;
    Entity*  ground_        = nullptr;
    Camera*  cam_           = nullptr;
    Label*   statusLabel_   = nullptr;
    Label*   volumeLabel_   = nullptr;

    // ── Runtime state ────────────────────────────────────────────────
    float  masterVol_       = 0.8f;
    float  footstepTimer_   = 0.0f;
    bool   bossTrackActive_ = false;
    bool   engineRunning_   = false;

    // ── Footstep timing ──────────────────────────────────────────────
    static constexpr float FOOTSTEP_INTERVAL = 0.38f; // seconds between steps

    // ────────────────────────────────────────────────────────────────
    void Start() override
    {
        // ── Scene setup ─────────────────────────────────────────────
        ground_ = CreateEntity("Ground");
        ground_->AddModel("Models/Plane.mdl");
        ground_->SetScale(20.0f);
        ground_->AddRigidBody(0.0f);
        ground_->AddBoxCollider(Vector3(20.0f, 0.1f, 20.0f));

        player_ = CreateEntity("Player");
        player_->AddModel("Models/Box.mdl");
        player_->SetPosition(0.0f, 1.0f, 0.0f);
        CharacterController* cc = player_->AddCharacterController(0.35f, 1.8f);
        cc->SetGravity(20.0f);

        Entity* sun = CreateEntity("Sun");
        sun->AddDirectionalLight(Color::WHITE, 1.0f);
        sun->SetRotation(Quaternion(45.0f, 30.0f, 0.0f));

        cam_ = CreateCamera();
        cam_->Follow(player_, 7.0f, 3.0f);
        cam_->SetSmoothFollow(true, 5.0f);

        // ── HUD labels ───────────────────────────────────────────────
        statusLabel_ = CreateLabel("Press M = boss music | E = engine | B = voice | +/- = vol");
        statusLabel_->SetPosition(10, 10);
        statusLabel_->SetFontSize(14);
        statusLabel_->SetColor(Color::WHITE);

        volumeLabel_ = CreateLabel("Master volume: 80%");
        volumeLabel_->SetPosition(10, 34);
        volumeLabel_->SetFontSize(14);
        volumeLabel_->SetColor(Color(0.7f, 1.0f, 0.7f, 1.0f));

        // ── Audio setup ──────────────────────────────────────────────

        // Set master volume before playing anything.
        SetMasterVolume(masterVol_);

        // Start the main theme immediately; loop = true (default).
        PlayMusic("Music/theme.ogg");

        // Pre-warm the engine channel at low volume (silent until
        // the player presses E, but the buffer is ready).
        SetChannelVolume(CH_ENGINE, 0.0f);
        PlaySound("Sounds/engine_loop.wav", 0.0f, CH_ENGINE);
    }

    // ────────────────────────────────────────────────────────────────
    void Update(float dt) override
    {
        HandleMovementAndFootsteps(dt);
        HandleMusicKeys();
        HandleEngineKey();
        HandleVoiceKey();
        HandleVolumeKeys();
        HandleQuit();
    }

    // ────────────────────────────────────────────────────────────────
    void HandleMovementAndFootsteps(float dt)
    {
        CharacterController* cc = player_->GetComponent<CharacterController>();
        if (!cc) return;

        float mx = 0.0f, mz = 0.0f;
        if (IsKeyDown(Key::Left)  || IsKeyDown(Key::A)) mx -= 1.0f;
        if (IsKeyDown(Key::Right) || IsKeyDown(Key::D)) mx += 1.0f;
        if (IsKeyDown(Key::Up)    || IsKeyDown(Key::W)) mz -= 1.0f;
        if (IsKeyDown(Key::Down)  || IsKeyDown(Key::S)) mz += 1.0f;

        bool moving = (mx != 0.0f || mz != 0.0f);
        cc->Move(Vector3(mx, 0.0f, mz) * 5.0f, dt);

        if (IsKeyPressed(Key::Space) && cc->IsOnGround())
            cc->Jump(10.0f);

        // Footstep sounds — fire-and-forget (no channel needed).
        if (moving && cc->IsOnGround())
        {
            footstepTimer_ -= dt;
            if (footstepTimer_ <= 0.0f)
            {
                PlaySound("Sounds/footstep.wav", 0.55f);
                footstepTimer_ = FOOTSTEP_INTERVAL;
            }
        }
        else
        {
            footstepTimer_ = 0.0f; // reset so first step fires immediately
        }
    }

    // ────────────────────────────────────────────────────────────────
    void HandleMusicKeys()
    {
        // M toggles between the main theme and the boss track.
        if (IsKeyPressed(Key::M))
        {
            if (!bossTrackActive_)
            {
                // PlayMusic stops whatever is currently playing, then starts
                // the new track.  Passing loop=true (second arg, the default).
                PlayMusic("Music/boss.ogg", true);
                bossTrackActive_ = true;
            }
            else
            {
                PlayMusic("Music/theme.ogg", true);
                bossTrackActive_ = false;
            }
        }

        // N stops the music entirely (no music until PlayMusic is called again).
        if (IsKeyPressed(Key::N))
            StopMusic();
    }

    // ────────────────────────────────────────────────────────────────
    void HandleEngineKey()
    {
        // E toggles the engine channel on/off.
        // Because the sound was pre-loaded on CH_ENGINE in Start(), we
        // only need to adjust the volume — no disk I/O happens here.
        if (IsKeyPressed(Key::E))
        {
            if (!engineRunning_)
            {
                // Ramp the channel volume up to audible level.
                SetChannelVolume(CH_ENGINE, 0.75f);

                // If StopSound() was called previously, restart the loop.
                if (!IsChannelPlaying(CH_ENGINE))
                    PlaySound("Sounds/engine_loop.wav", 0.75f, CH_ENGINE);

                engineRunning_ = true;
            }
            else
            {
                // Hard stop — the channel index becomes free immediately.
                StopSound(CH_ENGINE);
                engineRunning_ = false;
            }
        }

        // While the engine is running, pressing R plays a rev sound
        // on the same channel, which cancels the loop and plays once.
        if (engineRunning_ && IsKeyPressed(Key::R))
        {
            // PlaySound with a channel stops whatever is on that channel first,
            // then plays the new clip — perfect for rev-over-loop.
            PlaySound("Sounds/engine_rev.wav", 1.0f, CH_ENGINE);
        }
    }

    // ────────────────────────────────────────────────────────────────
    void HandleVoiceKey()
    {
        // B plays a voice bark on CH_VOICE.  IsChannelPlaying prevents
        // overlapping the same bark before it finishes.
        if (IsKeyPressed(Key::B))
        {
            if (!IsChannelPlaying(CH_VOICE))
            {
                PlaySound("Sounds/voice_bark.wav", 0.9f, CH_VOICE);
            }
            // If the bark is still playing we silently ignore the key press —
            // no stutter, no abrupt restart.
        }
    }

    // ────────────────────────────────────────────────────────────────
    void HandleVolumeKeys()
    {
        const float STEP = 0.05f;
        bool changed = false;

        if (IsKeyPressed(Key::KPPlus))
        {
            masterVol_ = Min(1.0f, masterVol_ + STEP);
            changed = true;
        }
        if (IsKeyPressed(Key::KPMinus))
        {
            masterVol_ = Max(0.0f, masterVol_ - STEP);
            changed = true;
        }

        if (changed)
        {
            SetMasterVolume(masterVol_);
            int pct = static_cast<int>(masterVol_ * 100.0f);
            volumeLabel_->SetText("Master volume: " + String(pct) + "%");
        }
    }

    // ────────────────────────────────────────────────────────────────
    void HandleQuit()
    {
        if (IsKeyPressed(Key::Escape))
            Quit();
    }

    // ────────────────────────────────────────────────────────────────
    void Stop() override
    {
        // Good practice: silence everything before the engine shuts down.
        StopMusic();
        StopSound(CH_ENGINE);
        StopSound(CH_VOICE);
        StopSound(CH_AMBIENT);
    }
};

int main()
{
    return Simple3D::CreateGame<AudioDemo>()->Run();
}

Step-by-step explanation

PlayMusic(path, loop)

PlayMusic opens the given OGG file and begins streaming it from disk. The loop parameter defaults to true, which makes the track repeat seamlessly at the end. Pass false for a one-shot jingle that stops at the final sample. Calling PlayMusic a second time automatically stops the previous track before starting the new one — there is no separate crossfade call required. Always prefer OGG here: it reduces shipping size by 8–12× compared to WAV with minimal quality loss. 

// Looping background theme (default second arg)
PlayMusic("Music/theme.ogg");

// One-shot victory fanfare — stops when it reaches the end
PlayMusic("Music/victory.ogg", false);

// Switch to a different looping track (stops the current one first)
PlayMusic("Music/boss.ogg", true);

StopMusic()

Halts the current music stream immediately and releases the streaming buffer. The call is a no-op if no music is playing, so it is always safe to call unconditionally — for instance, in Stop() during shutdown, or when entering a cutscene that should play in silence. 

// Silence the music for a dramatic pause
StopMusic();

// Resume later by calling PlayMusic again
PlayMusic("Music/theme.ogg");

PlaySound(path, volume) — fire-and-forget

The two-argument form of PlaySound loads the WAV file, plays it at the requested volume (0.0 – 1.0), and returns immediately. You get no handle back. Use this for short, overlapping effects that you never need to stop early or query: footsteps, bullet impacts, GUI clicks, collectible pickups. The engine manages the lifetime; once the clip finishes the buffer is freed automatically. Multiple calls can overlap simultaneously without stepping on each other. 

// Full volume coin pickup
PlaySound("Sounds/coin.wav", 1.0f);

// Softer footstep (55% volume)
PlaySound("Sounds/footstep.wav", 0.55f);

// Distant explosion at 30% volume
PlaySound("Sounds/explosion.wav", 0.3f);

PlaySound(path, volume, channel) — indexed channels

Adding a third integer argument routes the sound through a named channel. If that channel already has a sound playing, it is stopped first and the new clip takes over immediately. This makes channels ideal for sounds that must not stack on top of themselves — engine hum, a talking character, an alarm loop. Channels are plain integers; by convention, define them as constants at the top of your class (as CH_ENGINE = 0, etc.) to keep code readable. 

// Engine loop on channel 0 — starts immediately
PlaySound("Sounds/engine_loop.wav", 0.75f, 0);

// Revving replaces the loop on the same channel without stacking
PlaySound("Sounds/engine_rev.wav",  1.0f,  0);

// Voice bark on a separate channel so it doesn't kill the engine
PlaySound("Sounds/voice_bark.wav", 0.9f, 1);

StopSound(channel)

Immediately halts playback on the given channel. The call is a no-op if the channel was never started or has already finished — it never throws or prints a warning. After StopSound returns, IsChannelPlaying will report false for that channel. Use this when you need clean, instant silence: shutting down an engine when the vehicle is destroyed, stopping an alarm when the puzzle is solved, or halting a reload loop when the gun is empty. 

if (vehicleDestroyed_)
    StopSound(CH_ENGINE);  // clean silence; no-op if already stopped

IsChannelPlaying(channel)

Returns true while the channel source is actively playing, and false once it has finished naturally or was stopped with StopSound. The main use cases are:

  • Guarding a PlaySound call so a short clip cannot be re-triggered while it is still in progress (prevents stuttering).
  • Detecting when a non-looped clip has ended so you can react — e.g., switch a state machine back to idle after a grunt animation finishes.
  • Deciding whether to restart a loop that ended due to StopSound.
// Only play if the previous bark has finished
if (!IsChannelPlaying(CH_VOICE))
    PlaySound("Sounds/voice_bark.wav", 0.9f, CH_VOICE);

// In Update: detect natural end-of-clip and loop manually
if (needsLoop_ && !IsChannelPlaying(CH_AMBIENT))
    PlaySound("Sounds/wind_loop.wav", 0.6f, CH_AMBIENT);

SetChannelVolume(channel, volume)

Adjusts the gain of a single channel without stopping or restarting the clip. The volume parameter is a 0.0 – 1.0 linear scale applied on top of the master volume. Typical uses include:

  • Fading an engine loop in or out as speed changes.
  • Ducking an ambient track under dialogue without stopping it.
  • Letting the player balance individual audio categories (SFX vs. voice).

You can call SetChannelVolume every frame — the change takes effect for the next audio buffer mix with no pop or click. 

// Fade engine volume with throttle (0..1)
float throttle = GetThrottle();   // 0 = idle, 1 = full
SetChannelVolume(CH_ENGINE, 0.3f + throttle * 0.7f);

// Duck ambient while dialogue is playing
if (IsChannelPlaying(CH_VOICE))
    SetChannelVolume(CH_AMBIENT, 0.15f);
else
    SetChannelVolume(CH_AMBIENT, 0.6f);

SetMasterVolume(volume)

Scales the output of all audio — music, channels, and ambient sources — by a single factor in the 0.0 – 1.0 range. This is the correct place to honour a "master volume" setting slider in your options menu. Calling it is cheap; you can safely call it every frame if you are animating a volume fade, or once when the player adjusts the slider. A value of 0.0f mutes everything without stopping any clips; 1.0f restores full output. 

// Options menu slider callback
volSlider_->SetOnChange([this](float v) {
    SetMasterVolume(v);           // v is already 0..1
    masterVol_ = v;
    SaveVolumePref(v);
});

// Hard mute on window minimise
SetMasterVolume(0.0f);

// Restore on window restore
SetMasterVolume(masterVol_);

Quick reference

Method Returns Notes
PlayMusic(path, loop=true) — Stream OGG; stops previous track automatically.
StopMusic() — Halt music stream; no-op if not playing.
PlaySound(path, volume=1.0f) — Fire-and-forget WAV; overlaps freely.
PlaySound(path, volume, channel) — Stops the channel first, then plays clip.
StopSound(channel) — Immediate silence on channel; no-op if idle.
IsChannelPlaying(channel) bool True while clip is active on channel.
SetChannelVolume(channel, 0–1) — Per-channel gain; applied on top of master.
SetMasterVolume(0–1) — Global scale for all audio output.

Lua equivalent

All audio methods are available on the game global inside any Lua script attached with entity:AddScript(). The signatures are identical to C++, channel indices and all. 

-- Scripts/audio_controller.lua
-- Attach this to any entity; it manages music and a footstep timer.

local CH_ENGINE  = 0
local CH_VOICE   = 1
local CH_AMBIENT = 2

local masterVol       = 0.8
local bossActive      = false
local engineRunning   = false
local footstepTimer   = 0.0
local FOOTSTEP_INTV   = 0.38

function Start()
    -- Master volume first
    game:SetMasterVolume(masterVol)

    -- Stream background music (looped by default)
    game:PlayMusic("Music/theme.ogg")

    -- Pre-warm engine channel silently
    game:SetChannelVolume(CH_ENGINE, 0.0)
    game:PlaySound("Sounds/engine_loop.wav", 0.0, CH_ENGINE)
end

function Update(dt)
    -- Toggle music with M
    if game:IsKeyPressed(Key.M) then
        if not bossActive then
            game:PlayMusic("Music/boss.ogg", true)
            bossActive = true
        else
            game:PlayMusic("Music/theme.ogg", true)
            bossActive = false
        end
    end

    -- N = stop music entirely
    if game:IsKeyPressed(Key.N) then
        game:StopMusic()
    end

    -- E toggles engine channel
    if game:IsKeyPressed(Key.E) then
        if not engineRunning then
            game:SetChannelVolume(CH_ENGINE, 0.75)
            if not game:IsChannelPlaying(CH_ENGINE) then
                game:PlaySound("Sounds/engine_loop.wav", 0.75, CH_ENGINE)
            end
            engineRunning = true
        else
            game:StopSound(CH_ENGINE)
            engineRunning = false
        end
    end

    -- B plays a voice bark if not already talking
    if game:IsKeyPressed(Key.B) then
        if not game:IsChannelPlaying(CH_VOICE) then
            game:PlaySound("Sounds/voice_bark.wav", 0.9, CH_VOICE)
        end
    end

    -- Numpad +/- adjusts master volume
    if game:IsKeyPressed(Key.KPPlus) then
        masterVol = math.min(1.0, masterVol + 0.05)
        game:SetMasterVolume(masterVol)
    end
    if game:IsKeyPressed(Key.KPMinus) then
        masterVol = math.max(0.0, masterVol - 0.05)
        game:SetMasterVolume(masterVol)
    end

    -- Footstep example — one-shot, fire-and-forget
    footstepTimer = footstepTimer - dt
    if footstepTimer <= 0.0 then
        game:PlaySound("Sounds/footstep.wav", 0.55)
        footstepTimer = FOOTSTEP_INTV
    end
end

Patterns and tips

Music crossfade pattern

simple-3d does not have a built-in crossfade, but you can fake a quick volume dip using a Tween on the master volume, then switch tracks at the trough: 

// Dip master to 0 over 0.5 s, swap track, ramp back up
CreateTween(masterVol_, 0.0f, 0.5f,
    [this](float v) { SetMasterVolume(v); },
    [this]() {
        PlayMusic("Music/boss.ogg");
        CreateTween(0.0f, masterVol_, 0.5f,
            [this](float v) { SetMasterVolume(v); });
    },
    TweenEase::EaseIn
);

Saving and restoring volume

Persist the master volume across sessions using SaveData from Part 19. Call SetMasterVolume in Start() after loading: 

void Start() override {
    auto& save = GetSaveData(0);
    save.Load(0);
    masterVol_ = save.GetFloat("masterVol", 0.8f);
    SetMasterVolume(masterVol_);
    PlayMusic("Music/theme.ogg");
    // ...
}

void Stop() override {
    auto& save = GetSaveData(0);
    save.Set("masterVol", masterVol_);
    save.Save(0);
    StopMusic();
}

Varying footstep sounds

Fire-and-forget is ideal for footsteps because each call plays an independent, overlapping buffer. To avoid repetitive sound, keep a small array of variants and cycle through them: 

const char* steps[] = {
    "Sounds/step1.wav",
    "Sounds/step2.wav",
    "Sounds/step3.wav",
    "Sounds/step4.wav"
};
int stepIdx_ = 0;

// In Update:
PlaySound(steps[stepIdx_], 0.55f);
stepIdx_ = (stepIdx_ + 1) % 4;

Common mistakes

  • Using WAV for music. A 3-minute stereo WAV at 44.1 kHz takes roughly 30 MB in RAM — an OGG of the same content is 3–5 MB. Always use OGG for anything that streams in the background.
  • Using OGG for sound effects. OGG requires decoding time. For sub-100 ms reaction sounds (shots, hits, button clicks) the decode latency adds an audible delay. Always use WAV for SFX.
  • Re-triggering a looped WAV on a channel every frame. PlaySound with a channel stops the current clip before starting the new one. If you call it every frame to "keep the loop playing" you will hear a constant stutter because each call restarts from the beginning. Play once, then use IsChannelPlaying to decide whether to restart after a natural end.
  • Using channel 0 for multiple unrelated sounds. Each channel index is a single slot. If you play a voice line on channel 0 and then play an engine sound also on channel 0, the engine sound cuts off the voice mid-sentence. Define distinct channel constants for each category.
  • Forgetting to call StopMusic() / StopSound() before scene transitions. Music keeps streaming after UnloadScene(). Call StopMusic() at the start of any transition to avoid the theme from the previous level leaking into the new one.
  • Setting SetMasterVolume to a value outside 0–1. Values above 1.0 can cause clipping distortion; values below 0.0 are clamped to silence. Always clamp your input before passing it to the function.
  • Assuming IsChannelPlaying detects fire-and-forget sounds. The no-channel form of PlaySound does not occupy a numbered slot. You cannot query its status. If you need to know when a clip ends, use the three-argument form with an explicit channel index.

What you learned

In this part you mastered the full flat-audio API in simple-3d:

  • Music: PlayMusic(path, loop) streams an OGG file and replaces any previous track. StopMusic() silences it.
  • Fire-and-forget SFX: PlaySound(path, volume) plays a WAV clip that manages its own lifetime — ideal for overlapping, short sounds that you never need to cancel.
  • Managed channels: PlaySound(path, volume, channel) routes audio through a numbered slot, stopping anything already on that slot first. Define channel constants to keep code readable.
  • Channel control: StopSound(channel) halts a channel; IsChannelPlaying(channel) polls its live state; SetChannelVolume(channel, v) adjusts gain without restarting the clip.
  • Global volume: SetMasterVolume(v) scales all audio output, making it the single point of control for an options menu slider.
  • Asset conventions: OGG for music (compressed streaming), WAV for SFX (uncompressed, zero-latency).
