Part 07 of 30Character Controller

What you will learn

  • How to drive movement each frame using cc->Move(velocity, dt) so the engine handles friction and collision response.
  • How to implement safe jumping with cc->TryJump(speed), which checks ground contact internally and prevents infinite mid-air jumps.
  • How to query ground contact yourself with cc->IsOnGround() and what the engine actually tests behind that call.
  • How to dial in a snappy platformer feel by overriding per-body gravity with cc->SetGravity().
  • How collision layers work and why the player must live on CollisionLayer::Actor for triggers to fire correctly.
  • How to limit which slopes the character can walk up using cc->SetMaxSlope().

Background: why not just use a RigidBody?

In Part 04 you gave entities a RigidBody and shaped colliders to participate in physics simulation. That is fine for crates and balls, but it causes two well-known problems for a player character:

  • Tipping over. An unconstrained rigid body falls sideways if it touches a wall at an angle. You need to lock the angular factor on all axes manually and keep re-locking it every frame.
  • Stairs and ramps. Physics solvers push the body smoothly up a slope only if the slope angle and simulation step are tuned carefully. In practice, a character walking into a low step launches into the air.

CharacterController wraps a capsule rigid body and handles both problems for you. Internally it locks angular rotation so the capsule never tips, fires a short downward ray each frame to detect ground, and filters out slopes steeper than your configured limit. The public API is intentionally minimal: Move(), TryJump(), and IsOnGround().

The root/visual entity split

A character controller entity is positioned at the centre of the capsule, not at the feet. If your character is 1.8 m tall and the capsule centre is at half that height (0.9 m), the entity origin floats 0.9 m above the ground. Placing a model on the same entity with no offset would have the mesh floating or sinking relative to its feet.

The solution used throughout simple-3d is called the body/visual split:

  • The root entity holds the physics (CharacterController) and moves under simulation.
  • A child entity created with root->CreateChild("PlayerVisual") holds the rendered model, set to SetLocalPosition(Vector3::ZERO). Because the child inherits the root's world position, it tracks the capsule centre exactly.

Rotations work cleanly too: when you rotate the root entity to turn the player, the visual child turns with it automatically because child transforms are parent-relative.

PlayerEntity (root) position = capsule centre (e.g. 0.9 m above floor) CharacterController <- physics drives this CollisionLayer::Actor └── "PlayerVisual" (child) SetLocalPosition(Vector3::ZERO) AddModel("Models/Character.mdl") <- model positioned at same world point as root

Complete example

The game below builds a small platformer arena: a large flat ground, two raised platforms, a player with a character controller, a third-person camera, and a HUD showing the ground state. Arrow keys or WASD steer the character; Space or Ctrl jumps. 

#include <Simple3D/Simple3D.h>
#include <cmath>
using namespace Simple3D;

class PlatformerGame : public Game
{
public:

    // ── scene objects ──────────────────────────────────────────────
    Entity*              player_  = nullptr;   // root physics node
    Entity*              visual_  = nullptr;   // child model node
    CharacterController* cc_      = nullptr;
    Camera*              cam_     = nullptr;
    Label*               hudLabel_= nullptr;

    // ── runtime state ──────────────────────────────────────────────
    float yaw_ = 0.0f;   // accumulated player rotation in degrees

    // ───────────────────────────────────────────────────────────────
    void Start() override
    {
        SetWindowTitle("Part 07 — Character Controller");
        SetClearColor(Color(0.35f, 0.55f, 0.85f));  // sky blue

        // ── Lighting ─────────────────────────────────────────────
        Entity* sun = CreateEntity("Sun");
        sun->AddDirectionalLight(Color(1.0f, 0.95f, 0.85f), 1.1f);
        sun->SetRotation(Quaternion(50.0f, -30.0f, 0.0f));

        Entity* fill = CreateEntity("Fill");
        fill->AddDirectionalLight(Color(0.35f, 0.4f, 0.6f), 0.35f);
        fill->SetRotation(Quaternion(-40.0f, 140.0f, 0.0f));

        // ── Ground platform ───────────────────────────────────────
        Entity* ground = CreateEntity("Ground");
        ground->AddModel("Models/Plane.mdl");
        ground->SetScale(16.0f);
        ground->SetMaterialColor(Color(0.25f, 0.55f, 0.25f));  // grass green
        ground->AddRigidBody(0.0f);                 // mass 0 = static
        ground->AddBoxCollider(Vector3(16.0f, 0.2f, 16.0f));
        ground->SetPosition(0.0f, -0.1f, 0.0f);  // just below Y=0
        ground->SetCollisionLayer(CollisionLayer::StaticGeometry);

        // ── Raised platforms ─────────────────────────────────────
        MakePlatform(Vector3( 5.0f, 1.5f,  3.0f), Vector3(3.0f, 0.4f, 2.0f));
        MakePlatform(Vector3(-4.0f, 3.0f, -2.0f), Vector3(2.5f, 0.4f, 2.5f));

        // ── Player ────────────────────────────────────────────────
        //
        // The player root entity is positioned at the *capsule centre*.
        // For a 1.8 m tall capsule the centre sits 0.9 m above the floor.
        // We add a tiny 0.05 m clearance so the capsule does not start
        // intersecting the ground collider.
        player_ = CreateEntity("Player");
        player_->SetPosition(0.0f, 0.95f, 0.0f);

        // AddCharacterController creates a capsule rigid body (radius=0.35 m,
        // height=1.8 m) with locked angular factor so the capsule never tips.
        // The call is idempotent — calling it again returns the same pointer.
        cc_ = player_->AddCharacterController(0.35f, 1.8f);

        // Override gravity to 25 m/s² downward for a snappier platformer feel.
        // The default physics world gravity (typically 9.81 m/s²) is too floaty
        // for tight jump arcs.  This override applies only to this body.
        cc_->SetGravity(25.0f);

        // Reject slopes steeper than 45°. The character will stop and
        // slide back down if it walks into a wall steeper than this.
        cc_->SetMaxSlope(45.0f);

        // Put the player on the Actor layer so trigger volumes can detect it.
        // The mask makes the player collide with StaticGeometry AND Trigger
        // volumes — both are required for full interaction.
        player_->SetCollisionLayer(CollisionLayer::Actor);
        player_->SetCollisionMask(MakeCollisionMask({
            CollisionLayer::StaticGeometry,
            CollisionLayer::Trigger
        }));

        // ── Visual child ─────────────────────────────────────────
        //
        // The model is attached to a child entity at local-space origin.
        // Because child transforms inherit the parent's world transform,
        // the mesh follows the capsule centre exactly.  Rotations applied
        // to the root also automatically rotate the visual child.
        visual_ = player_->CreateChild("PlayerVisual");
        visual_->AddModel("Models/Character.mdl");
        visual_->SetLocalPosition(Vector3::ZERO);

        // ── Camera ────────────────────────────────────────────────
        cam_ = CreateCamera();
        cam_->Follow(player_, 6.0f, 2.5f);   // distance, height
        cam_->LookAt(player_);
        cam_->SetSmoothFollow(true, 6.0f);

        // ── HUD ───────────────────────────────────────────────────
        hudLabel_ = CreateLabel("WASD / Arrows: move  |  Space / Ctrl: jump");
        hudLabel_->SetPosition(16, 16);
        hudLabel_->SetFontSize(15);
        hudLabel_->SetColor(Color(1.0f, 1.0f, 1.0f));
    }

    // ─────────────────────────────────────────────────────────────
    void Update(float dt) override
    {
        if (IsKeyDown(Key::Escape)) { Quit(); return; }

        // ── Turning ───────────────────────────────────────────────
        // Accumulate a yaw angle from Left/Right arrows (or A/D keys).
        float turn = 0.0f;
        if (IsKeyDown(Key::Right) || IsKeyDown(Key::D)) turn += 1.0f;
        if (IsKeyDown(Key::Left)  || IsKeyDown(Key::A)) turn -= 1.0f;
        yaw_ += turn * 120.0f * dt;

        // ── Forward / backward ───────────────────────────────────
        float fwd = 0.0f;
        if (IsKeyDown(Key::Up) || IsKeyDown(Key::W)) fwd += 1.0f;
        if (IsKeyDown(Key::Down) || IsKeyDown(Key::S)) fwd -= 1.0f;

        // Convert yaw to a world-space horizontal direction vector.
        float rad = yaw_ * (3.14159265f / 180.0f);
        Vector3 dir(std::sin(rad) * fwd, 0.0f, std::cos(rad) * fwd);

        // Move() sets the capsule's horizontal (X/Z) linear velocity each
        // frame.  The vertical (Y) component is left entirely to gravity
        // so jump arcs are not disrupted by movement input.
        cc_->Move(dir * 6.0f, dt);

        // ── Jump ─────────────────────────────────────────────────
        // TryJump() calls IsOnGround() internally before applying the
        // vertical impulse, so holding Space never triggers a second jump
        // while the player is already airborne.
        if (IsKeyPressed(Key::Space)
         || IsKeyPressed(Key::LCtrl)
         || IsKeyPressed(Key::RCtrl))
        {
            cc_->TryJump(12.0f);
        }

        // ── Rotate player to face movement direction ──────────────
        if (fwd != 0.0f || turn != 0.0f)
            player_->SetRotation(Quaternion(0.0f, yaw_, 0.0f));

        // ── HUD ───────────────────────────────────────────────────
        bool grounded = cc_->IsOnGround();
        std::string statusText = grounded
            ? "Status: ON GROUND  |  Space/Ctrl to jump"
            : "Status: AIRBORNE";
        hudLabel_->SetText(statusText);
    }

    // ── Helper: build a static coloured platform ─────────────────
    void MakePlatform(Vector3 pos, Vector3 size)
    {
        static int id = 0;
        Entity* p = CreateEntity("Platform" + std::to_string(id++));
        p->AddModel("Models/Box.mdl");
        p->SetMaterialColor(Color(0.6f, 0.45f, 0.3f));
        p->SetPosition(pos);
        p->SetScale(1.0f);  // scale via collider size, not entity scale
        p->AddRigidBody(0.0f);
        p->AddBoxCollider(size);
        p->SetCollisionLayer(CollisionLayer::StaticGeometry);
    }
};

int main()
{
    return CreateGame<PlatformerGame>()->Run();
}

Step-by-step API explanation

entity->AddCharacterController(radius, height)

This one call replaces the combination of AddRigidBody, AddCapsuleCollider, and manual angular-factor locking you would otherwise need. Internally it creates a capsule rigid body whose angular motion on all three axes is locked so the capsule never tips sideways.

The radius is the capsule's cross-section in metres (typically 0.3 to 0.5 for a humanoid). The height is the total capsule height including both hemispherical end-caps (so a 1.8 m character fits in AddCharacterController(0.35f, 1.8f)).

The call is idempotent — calling it a second time on the same entity returns the already-existing CharacterController without creating a duplicate, so you can safely call it from a helper function without tracking whether it was already called.

CharacterController* AddCharacterController(float radius, float height)

You can also retrieve an existing character controller later without re-adding it: 

CharacterController* cc = player_->GetCharacterController();  // nullptr if not added

cc->SetGravity(float g)

Overrides the global physics world gravity for this body only. The value is the downward acceleration in m/s². The global default is typically 9.81 m/s², which feels floaty for platformers with short jump arcs. A value of 25.0 gives tight, responsive jumps more akin to classic 2D platformers.

Call this immediately after AddCharacterController, before the first physics tick. You can change it at runtime to produce gravity-reversal effects, low-gravity zones, and similar.

void SetGravity(float gravityDownward)

cc->SetMaxSlope(float degrees)

Sets the steepest surface the character can walk on. Surfaces whose normal deviates more than degrees from the world UP vector are treated as walls: the character slides back down and IsOnGround() returns false while in contact with them. The default is 45°, which allows comfortable ramps while rejecting cliff faces.

void SetMaxSlope(float degrees)

CollisionLayer::Actor and MakeCollisionMask

simple-3d ships three named collision layers that form a conventional system:

LayerValueTypical use
CollisionLayer::Actor1The player, NPCs, projectiles — anything dynamic
CollisionLayer::StaticGeometry2Ground, platforms, walls, crates — everything static
CollisionLayer::Trigger4Trigger volumes added via AddTriggerBox etc.

SetCollisionLayer declares which layer this entity belongs to. SetCollisionMask declares which layers this entity is interested in hitting. The player must include StaticGeometry in its mask (to land on floors) and Trigger (so SetOnTriggerEnter callbacks reach the player).

MakeCollisionMask({...}) takes an initializer list of layers and ORs their values together, removing the need to remember the raw bit values yourself: 

player_->SetCollisionLayer(CollisionLayer::Actor);
player_->SetCollisionMask(MakeCollisionMask({
    CollisionLayer::StaticGeometry,
    CollisionLayer::Trigger
}));

The IsOnGround() raycast fires on layer 2 (StaticGeometry) only, which is why the player ground entity must be on StaticGeometry for jumping to work.

Visual child entity pattern

Create the visual child immediately after setting up the physics root. CreateChild(name) returns a new entity that is parented to the caller and whose local-space origin coincides with the parent's origin by default. 

// Physics root entity
player_ = CreateEntity("Player");
player_->SetPosition(0.0f, 0.95f, 0.0f);   // capsule centre height
cc_ = player_->AddCharacterController(0.35f, 1.8f);

// Visual child — model floats at the same world point as the capsule centre
visual_ = player_->CreateChild("PlayerVisual");
visual_->AddModel("Models/Character.mdl");
visual_->SetLocalPosition(Vector3::ZERO);

Key rule: always use SetLocalPosition (not SetPosition) to offset child entities. SetPosition on a child would set its world position, which gets immediately overridden by the parent transform every frame.

If you need to offset the mesh downward so the character's feet touch the ground, shift the visual child: visual_->SetLocalPosition(0, -0.9f, 0) moves the mesh 0.9 m below the capsule centre, placing the feet at Y=0.

cc->Move(velocity, dt)

Sets the capsule rigid body's horizontal linear velocity each frame. Critically, it only touches the X and Z components — the Y velocity (controlled by gravity and jump impulses) is left unchanged. This separation prevents movement input from interfering with airborne arcs.

void Move(Vector3 velocity, float dt)

Pass the desired world-space velocity vector, not a delta. If you want 6 m/s forward, pass Vector3(0, 0, 6) (or whatever the character's current facing direction). Setting it to Vector3::ZERO brings the character to a stop horizontally. The dt parameter is passed through for future interpolation; pass the frame delta time received in Update(float dt). 

// Compute a forward direction from accumulated yaw
float rad = yaw_ * (3.14159265f / 180.0f);
Vector3 dir(std::sin(rad) * fwd, 0.0f, std::cos(rad) * fwd);

// Set 6 m/s in that direction; Y is untouched
cc_->Move(dir * 6.0f, dt);

cc->TryJump(speed)

The safe way to initiate a jump. Before applying the vertical impulse, this method calls IsOnGround() internally. If the character is airborne, the call is a no-op. If it is grounded, the Y velocity is set to speed m/s upward and the method returns true.

bool TryJump(float speed)

Use TryJump whenever you want standard single-jump behaviour. You can use the return value to trigger a jump sound or animation: 

if (IsKeyPressed(Key::Space))
{
    if (cc_->TryJump(12.0f))
        PlaySound("Sounds/jump.wav");
}

The lower-level method cc->Jump(speed) exists too, but it does not check ground contact. Use it only when you need custom grounding logic, such as a coyote-time grace window or a double-jump mechanic: 

// Example: double-jump with a manual jump counter
if (IsKeyPressed(Key::Space) && jumpCount_ < 2)
{
    cc_->Jump(12.0f);   // unchecked — fires even in mid-air
    jumpCount_++;
}
if (cc_->IsOnGround()) jumpCount_ = 0;

cc->IsOnGround()

Returns true when all three of these conditions hold simultaneously:

  1. A downward ray of length radius + 0.2 m hits a surface on CollisionLayer::StaticGeometry (layer 2).
  2. The surface normal forms an angle less than maxSlope_ degrees with world UP.
  3. The character's current upward velocity is not greater than 0.5 m/s (prevents the jump impulse frame from immediately re-grounding).
bool IsOnGround()

You can also call entity->IsOnGround() directly on the entity. When a CharacterController is attached, the entity method delegates to the CC's grounding check automatically, so legacy code that calls the entity method does not need to change. 

bool grounded = cc_->IsOnGround();       // direct CC call
bool same     = player_->IsOnGround();  // identical result via entity delegate

Lua equivalent

The full Lua equivalent of the character controller player setup fits in a single script file. Attach it to the player entity in C++ with player->AddScript("Scripts/player_cc.lua"). The script expects the parent (C++) code to have already called AddCharacterController on the entity. 

-- Scripts/player_cc.lua
-- Expects the entity to already have a CharacterController attached in C++.

local cc        -- CharacterController pointer
local yaw  = 0  -- accumulated Y rotation in degrees
local SPEED   = 6
local TURN_SPEED = 120
local JUMP_SPEED = 12

function Start()
    entity = self.node
    cc = entity:GetCharacterController()
    if cc == nil then
        print("[player_cc.lua] ERROR: no CharacterController on entity '"
              .. entity:GetName() .. "'")
        return
    end
    cc:SetGravity(25.0)
    cc:SetMaxSlope(45.0)
end

function Update(dt)
    if cc == nil then return end

    -- Turning
    local turn = 0
    if Input:IsKeyDown("Right") or Input:IsKeyDown("D") then turn = turn + 1 end
    if Input:IsKeyDown("Left")  or Input:IsKeyDown("A") then turn = turn - 1 end
    yaw = yaw + turn * TURN_SPEED * dt

    -- Forward / backward
    local fwd = 0
    if Input:IsKeyDown("Up")   or Input:IsKeyDown("W") then fwd = fwd + 1 end
    if Input:IsKeyDown("Down") or Input:IsKeyDown("S") then fwd = fwd - 1 end

    -- Build world-space movement direction from yaw
    local rad = yaw * (math.pi / 180)
    local dir = Vector3(math.sin(rad) * fwd, 0, math.cos(rad) * fwd)

    -- Drive the controller; Y velocity is untouched
    cc:Move(dir * SPEED, dt)

    -- Jump (Space or LCtrl) — TryJump checks ground internally
    if Input:IsKeyPress("Space") or Input:IsKeyPress("LCtrl") then
        cc:TryJump(JUMP_SPEED)
    end

    -- Rotate player to face movement direction
    if fwd ~= 0 or turn ~= 0 then
        entity:SetRotation(Quaternion(0, yaw, 0))
    end

    -- Query ground state for animation or debugging
    local onGround = cc:IsOnGround()
    -- (In Part 09 you will connect this to an animation state machine)
end

Attaching the script in C++

The Lua script above calls entity:GetCharacterController() rather than AddCharacterController. Create the CC in C++ first during Start(), then attach the script. This keeps physics setup in C++ (where it is easier to set layer and mask) while moving per-frame movement logic into a hot-reloadable Lua file. 

// In Game::Start():
player_ = CreateEntity("Player");
player_->SetPosition(0.0f, 0.95f, 0.0f);
cc_ = player_->AddCharacterController(0.35f, 1.8f);
player_->SetCollisionLayer(CollisionLayer::Actor);
player_->SetCollisionMask(MakeCollisionMask({
    CollisionLayer::StaticGeometry,
    CollisionLayer::Trigger
}));
player_->AddScript("Scripts/player_cc.lua");

CharacterController API quick reference

Method Returns Description
AddCharacterController(radius, height) CharacterController* Creates capsule RigidBody with locked rotation; idempotent.
GetCharacterController() CharacterController* Returns existing CC or nullptr.
cc->Move(velocity, dt) void Sets X/Z linear velocity; Y component left to gravity.
cc->TryJump(speed) bool Checks IsOnGround() internally; sets Y velocity if grounded. Returns true if jump fired.
cc->Jump(speed) void Sets Y velocity unconditionally; caller must check ground if needed.
cc->IsOnGround() bool Slope-filtered downward ray on layer 2; also checks upward velocity.
cc->SetGravity(g) void Per-body gravity override in m/s² downward. Default: world gravity (~9.81).
cc->SetMaxSlope(degrees) void Maximum walkable slope angle. Default 45°.
cc->SetStepHeight(metres) void Step-up target for small obstacles. Default 0.4 m.
cc->SetEnabled(bool) void Freeze or unfreeze the controller and its rigid body.

Common mistakes

Common mistakes to avoid

Placing the player entity at the feet, not the capsule centre.
SetPosition(0, 0, 0) puts the capsule centre at Y=0, which means the lower hemisphere intersects the ground and physics pushes the character upward violently on the first frame. Always spawn at Y = height / 2 + small_clearance (e.g. Y = 0.9 m for a 1.8 m capsule, plus 0.05 m clearance).

Calling SetPosition on the visual child instead of SetLocalPosition.
SetPosition on a child entity sets its world-space position, but the parent transform overwrites it every frame. Always use SetLocalPosition to position children relative to their parent.

Forgetting CollisionLayer::StaticGeometry on the ground.
IsOnGround() fires a ray that only hits layer 2 (StaticGeometry). If you forget to call ground->SetCollisionLayer(CollisionLayer::StaticGeometry) the character never detects ground contact, TryJump does nothing, and the player falls through.

Using Jump instead of TryJump without a ground guard.
cc->Jump(speed) does not check IsOnGround(). Calling it every frame the key is held launches the player into space. Either use TryJump, or pair Jump with IsKeyPressed (single-frame detection) and your own IsOnGround() check.

Omitting CollisionLayer::Trigger from the player mask.
Trigger callbacks set via entity->SetOnTriggerEnter only fire when the player collides with something on a layer that is included in the player's collision mask. If you only put StaticGeometry in the mask, trigger volumes are invisible to the player and collectibles, checkpoints, and zone transitions never activate.

Calling AddCharacterController after the first physics tick.
The internal capsule rigid body must be registered with the physics world before simulation starts. Always call AddCharacterController inside Start(), not inside Update().

Passing a direction instead of a velocity to Move().
Move(dir, dt) expects a velocity in m/s, not a unit direction. If you pass a normalised vector without multiplying by speed, the character crawls at 1 m/s. Always scale: cc->Move(dir.Normalized() * speed, dt).

What you learned

In this part you replaced the raw rigid body and collider setup from Part 04 with a purpose-built CharacterController. The key takeaways are:

  • AddCharacterController(radius, height) creates a capsule body with locked rotation in a single idempotent call, returning a CharacterController* you use to drive movement and jumping.
  • cc->SetGravity(25.0f) overrides the global gravity for this body only, letting you tune platformer feel without touching the global physics world.
  • cc->SetMaxSlope(45.0f) rejects surfaces steeper than the given angle so the character cannot walk up sheer cliffs.
  • cc->Move(velocity, dt) sets only X/Z velocity each frame, leaving the Y component to gravity and jump impulses.
  • cc->TryJump(speed) is the correct way to jump: it checks ground contact internally and prevents double-jumping out of the box.
  • cc->IsOnGround() performs a slope-filtered, velocity-guarded downward ray against StaticGeometry (layer 2). The entity delegates this call too, so player->IsOnGround() returns the same result.
  • The visual child entity pattern (root = physics, child = model) keeps the rendered mesh exactly aligned with the capsule centre while allowing independent local-space positioning of the model.
  • CollisionLayer::Actor plus a mask of {StaticGeometry, Trigger} is the conventional layer setup for any player entity: it collides with floors and walls, and receives trigger callbacks from collectibles and zone volumes.
