  • How to use PickPosition(screenPos, mask) to get the exact world-space point where the ray hits a surface.
  • How to filter picks by CollisionLayer so clicks on the ground do not accidentally hit invisible trigger volumes, and vice versa.
  • How to show and hide the OS cursor with SetCursorVisible and query its state with IsCursorVisible.
  • How to lock the cursor into relative mode with SetCursorLocked(true) for FPS-style mouse look, and unlock it with SetCursorLocked(false).

  • How to implement middle-mouse camera panning using GetMouseDelta() without locking the cursor.
  • How to toggle between an RTS top-down camera and an FPS free-look camera at runtime.

Background: how ray picking works

Every time you call PickEntity or PickPosition, simple-3d performs the following steps invisibly:

  1. The screen pixel you pass in (e.g. the current mouse position) is converted to a normalised [0, 1] viewport coordinate.
  2. The active camera generates a world-space Ray that starts at the camera origin and passes through that viewport point.
  3. PhysicsWorld::RaycastSingle tests the ray against all physics bodies up to 1000 world units away that match the collision-layer mask.
  4. On a hit, PickEntity looks up the physics body's node in the entity registry and returns the matching Entity*; PickPosition returns the hit point as a Vector3.
  5. On a miss, both functions return nullptr or Vector3::ZERO respectively.

Because the ray is cast against the physics world, every entity you want to be pickable must have a RigidBody and at least one collider added to it (via AddBoxCollider, AddSphereCollider, AddCapsuleCollider, or AddMeshCollider). Entities with only a visual model and no physics shape are invisible to ray picks.

Collision layer mask reference

Both PickEntity and PickPosition accept an optional second argument: a CollisionLayer mask. The mask is bitwise-ANDed against each body's collision layer. Only bodies whose layer bit is set in the mask are tested by the ray. The default mask is CollisionLayer::All, which tests everything.

Mask constantNumeric valueWhat it hits
CollisionLayer::All0xFFFFEvery entity with a RigidBody
CollisionLayer::Actor1Player, enemies, selectable units
CollisionLayer::StaticGeometry2Ground, walls, platforms, crates
CollisionLayer::Trigger4Trigger volumes (usually excluded)

In a typical RTS, left-click should only pick actors and right-click should only pick the ground. Passing the appropriate mask to each call keeps triggers, decorative props without game logic, and other irrelevant bodies out of the results.

PickEntity — who did I click?

Use when you want to know which entity is under the cursor. Pass CollisionLayer::Actor to restrict hits to units, enemies, or NPCs only. Returns nullptr on a miss or if the mask excludes the entity under the cursor.

PickPosition — where did I click?

Use when you want the world-space coordinate of a surface point. Pass CollisionLayer::StaticGeometry to land picks on terrain and floors only, ignoring units, triggers, and other dynamic bodies. Returns Vector3::ZERO on a miss.

Complete working example

The example below implements a miniature RTS scene containing three selectable box units on a ground plane. Left-clicking a unit selects it (highlighted gold). Right-clicking the ground while a unit is selected issues a move order: the unit travels to that world position, guided by a green destination marker. Holding the middle mouse button and dragging pans the top-down camera using GetMouseDelta(). Pressing Tab toggles between the RTS view and a free FPS look mode in which the cursor is locked and mouse movement rotates the camera freely. The example demonstrates all six new API concepts in a single, self-contained file. 

#include <Simple3D/Simple3D.h>

using namespace Simple3D;

// ─────────────────────────────────────────────────────────────────────────────
// RTSGame — demonstrates ray picking, cursor control, and mouse-delta panning.
//
// Controls:
//   LMB              — pick a unit (PickEntity, CollisionLayer::Actor)
//   RMB              — move selected unit to ground point (PickPosition, StaticGeometry)
//   Middle drag      — pan camera (GetMouseDelta, cursor NOT locked)
//   Tab              — toggle FPS free-look (SetCursorLocked / SetCursorVisible)
//   Escape           — exit FPS mode or quit
// ─────────────────────────────────────────────────────────────────────────────
class RTSGame : public Game
{
public:
    // ── Scene objects ────────────────────────────────────────────────
    Entity*  ground_      = nullptr;
    Entity*  unit_[3]    = {};        // three selectable units
    Entity*  selected_    = nullptr;  // the currently highlighted unit
    Entity*  moveMarker_  = nullptr;  // green cone at move destination
    Camera*  cam_         = nullptr;

    // ── RTS camera state ─────────────────────────────────────────────
    Vector3  camPos_      = Vector3(0.0f, 16.0f, 9.0f);
    Vector3  camLookAt_   = Vector3::ZERO;

    // ── Unit movement ────────────────────────────────────────────────
    Vector3  unitTarget_  = Vector3::ZERO;
    bool     hasTarget_   = false;

    // ── Mode flag ────────────────────────────────────────────────────
    bool     fpsMode_     = false;

    // ── UI ───────────────────────────────────────────────────────────
    Label*   statusLbl_   = nullptr;
    Label*   controlsLbl_ = nullptr;

    // ── Tuning constants ─────────────────────────────────────────────
    static constexpr float UNIT_SPEED    = 5.0f;   // world units / second
    static constexpr float PAN_SCALE     = 0.07f;  // world units per cursor pixel
    static constexpr float ARRIVE_THRESH = 0.12f;  // stop when this close

    // ────────────────────────────────────────────────────────────────
    // Start() — build the scene, position camera, show cursor.
    // ────────────────────────────────────────────────────────────────
    void Start() override
    {
        SetClearColor(Color(0.1f, 0.14f, 0.1f));

        // ── Ground plane — StaticGeometry layer ──────────────────────
        // The layer assignment is CRITICAL for PickPosition.
        // Without it, PickPosition(mpos, CollisionLayer::StaticGeometry)
        // would miss the ground and always return Vector3::ZERO.
        ground_ = CreateEntity("Ground");
        ground_->AddModel("Models/Plane.mdl");
        ground_->SetScale(Vector3(24.0f, 1.0f, 24.0f));
        ground_->AddRigidBody(0.0f);                              // mass 0 = static
        ground_->AddBoxCollider(Vector3(24.0f, 0.2f, 24.0f));
        ground_->SetCollisionLayer(CollisionLayer::StaticGeometry);

        // ── Directional sunlight ─────────────────────────────────────
        Entity* sun = CreateEntity("Sun");
        sun->AddDirectionalLight(Color(1.0f, 0.95f, 0.8f), 1.0f);
        sun->SetRotation(Quaternion(50.0f, 30.0f, 0.0f));

        // ── Three selectable units — Actor layer ─────────────────────
        // SpawnUnit() puts each box on CollisionLayer::Actor so that
        // PickEntity(mpos, CollisionLayer::Actor) can find them while
        // PickPosition(mpos, CollisionLayer::StaticGeometry) ignores them.
        unit_[0] = SpawnUnit("Alpha",   Vector3(-4.0f, 0.4f,  0.0f));
        unit_[1] = SpawnUnit("Bravo",   Vector3( 0.0f, 0.4f,  0.0f));
        unit_[2] = SpawnUnit("Charlie", Vector3( 4.0f, 0.4f,  0.0f));

        // ── Move destination marker (starts hidden) ───────────────────
        moveMarker_ = CreateEntity("MoveMarker");
        moveMarker_->AddModel("Models/Cone.mdl");
        moveMarker_->SetScale(0.35f);
        moveMarker_->SetMaterialColor(Color(0.1f, 0.9f, 0.2f));
        moveMarker_->SetActive(false);     // hide until a move order is issued

        // ── Top-down RTS camera ───────────────────────────────────────
        // Detach() releases the camera from any Follow / FPS mode so
        // we can drive its position manually in Update().
        cam_ = CreateCamera();
        cam_->SetFOV(45.0f);
        cam_->Detach();
        cam_->SetPosition(camPos_);
        cam_->LookAt(camLookAt_);

        // ── Cursor — visible for RTS mode ────────────────────────────
        // In a typical game the cursor starts visible so the player can
        // see where they are clicking.  SetCursorVisible(true) is the
        // default, but being explicit makes intent clear.
        SetCursorVisible(true);

        // ── HUD ──────────────────────────────────────────────────────
        statusLbl_ = CreateLabel("Click a unit to select it");
        statusLbl_->SetPosition(10, 10);
        statusLbl_->SetFontSize(16);
        statusLbl_->SetColor(Color::WHITE);

        controlsLbl_ = CreateLabel(
            "LMB: select unit  |  RMB: move  |  MMB drag: pan  |  Tab: FPS mode");
        controlsLbl_->SetPosition(10, 36);
        controlsLbl_->SetFontSize(12);
        controlsLbl_->SetColor(Color(0.7f, 0.7f, 0.7f, 1.0f));
    }

    // ────────────────────────────────────────────────────────────────
    // SpawnUnit — creates a box entity on the Actor collision layer.
    // Only entities on CollisionLayer::Actor are returned by
    //   PickEntity(mpos, CollisionLayer::Actor).
    // ────────────────────────────────────────────────────────────────
    Entity* SpawnUnit(const std::string& name, Vector3 pos)
    {
        Entity* u = CreateEntity(name);
        u->AddModel("Models/Box.mdl");
        u->SetScale(0.8f);
        u->SetPosition(pos);

        // A RigidBody + collider are REQUIRED for the ray to hit this entity.
        // Mass 0 = kinematic; we will move the unit manually in MoveUnit().
        u->AddRigidBody(0.0f);
        u->AddBoxCollider(Vector3(0.8f, 0.8f, 0.8f));
        u->SetCollisionLayer(CollisionLayer::Actor);

        // Tag used later to confirm the hit entity is a selectable unit.
        u->AddTag("unit");
        u->SetMaterialColor(Color(0.25f, 0.45f, 0.85f));   // blue
        return u;
    }

    // ────────────────────────────────────────────────────────────────
    // Update() — dispatch to RTS or FPS handler based on current mode.
    // ────────────────────────────────────────────────────────────────
    void Update(float dt) override
    {
        if (IsKeyPressed(Key::Tab))
            ToggleFPSMode();

        if (IsKeyPressed(Key::Escape))
        {
            if (fpsMode_) ToggleFPSMode();   // exit FPS mode before quitting
            else          Quit();
        }

        if (!fpsMode_)
        {
            HandleRTSInput(dt);
            MoveUnit(dt);
            PanCamera();
        }
        // In FPS mode cam_->SetFPSMode() drives look automatically;
        // we only need to handle the Tab / Escape keys above.
    }

    // ────────────────────────────────────────────────────────────────
    // HandleRTSInput — ray picking on left and right mouse buttons.
    // ────────────────────────────────────────────────────────────────
    void HandleRTSInput(float dt)
    {
        Vector2 mpos = GetMousePosition();

        // ── Left click: select a unit ─────────────────────────────────
        // PickEntity casts a ray and returns the first Entity* whose
        // RigidBody is on CollisionLayer::Actor.  The ground is on
        // CollisionLayer::StaticGeometry (=2), so it is masked out here.
        if (IsMouseButtonDown(MouseButton::Left))
        {
            Entity* hit = PickEntity(mpos, CollisionLayer::Actor);

            if (hit && hit->HasTag("unit"))
            {
                SelectUnit(hit);
            }
            else
            {
                // Clicked empty space — deselect.
                Deselect();
            }
        }

        // ── Right click: issue a move order ──────────────────────────
        // PickPosition returns the WORLD-SPACE point where the ray hits
        // the first body on CollisionLayer::StaticGeometry.
        // The Actor-layer unit boxes are excluded, so the ray lands on
        // the ground plane even if a unit happens to be standing on that spot.
        if (IsMouseButtonDown(MouseButton::Right) && selected_)
        {
            Vector3 groundPos = PickPosition(mpos, CollisionLayer::StaticGeometry);

            if (groundPos != Vector3::ZERO)    // ZERO means the ray missed everything
            {
                // Keep the unit's current Y so it stays on the ground surface.
                unitTarget_ = Vector3(groundPos.x_,
                                      selected_->GetPosition().y_,
                                      groundPos.z_);
                hasTarget_ = true;

                // Place the green move-marker cone at the click point.
                moveMarker_->SetPosition(groundPos + Vector3(0.0f, 0.05f, 0.0f));
                moveMarker_->SetActive(true);

                statusLbl_->SetText("Moving: " + selected_->GetName());
            }
        }
    }

    // ────────────────────────────────────────────────────────────────
    // SelectUnit / Deselect — colour-tint the selection state.
    // ────────────────────────────────────────────────────────────────
    void SelectUnit(Entity* unit)
    {
        // Restore previous unit to its unselected colour.
        if (selected_)
        {
            selected_->SetMaterialColor(Color(0.25f, 0.45f, 0.85f));
            selected_->RemoveTag("selected");
        }

        selected_ = unit;
        selected_->AddTag("selected");
        selected_->SetMaterialColor(Color(1.0f, 0.75f, 0.1f));  // gold = selected

        hasTarget_  = false;
        moveMarker_->SetActive(false);
        statusLbl_->SetText("Selected: " + selected_->GetName() +
                             "  — RMB to move");
    }

    void Deselect()
    {
        if (selected_)
        {
            selected_->SetMaterialColor(Color(0.25f, 0.45f, 0.85f));
            selected_->RemoveTag("selected");
            selected_    = nullptr;
        }
        hasTarget_  = false;
        moveMarker_->SetActive(false);
        statusLbl_->SetText("Click a unit to select it");
    }

    // ────────────────────────────────────────────────────────────────
    // MoveUnit — slides the selected unit toward unitTarget_ each frame.
    // This is intentionally a simple lerp; for full pathfinding use the
    // NavMesh API from Part 14.
    // ────────────────────────────────────────────────────────────────
    void MoveUnit(float dt)
    {
        if (!selected_ || !hasTarget_) return;

        Vector3 pos = selected_->GetPosition();
        Vector3 dir = unitTarget_ - pos;
        float   dist = dir.Length();

        if (dist < ARRIVE_THRESH)
        {
            // Snap to target and hide the marker.
            selected_->SetPosition(unitTarget_);
            hasTarget_  = false;
            moveMarker_->SetActive(false);
            statusLbl_->SetText("Selected: " + selected_->GetName() +
                                 "  — RMB to move");
        }
        else
        {
            dir.Normalize();
            selected_->SetPosition(pos + dir * UNIT_SPEED * dt);
        }
    }

    // ────────────────────────────────────────────────────────────────
    // PanCamera — reads GetMouseDelta() while middle button is held.
    //
    // GetMouseDelta() reports how many pixels the mouse moved this frame.
    // This works WITHOUT locking the cursor: the cursor stays visible and
    // moveable, while the delta drives the camera pan.
    //
    // This is different from SetCursorLocked, which hides the cursor and
    // allows unlimited motion past screen edges (used for FPS look below).
    // ────────────────────────────────────────────────────────────────
    void PanCamera()
    {
        if (IsMouseButtonDown(MouseButton::Middle))
        {
            Vector2 delta = GetMouseDelta();    // pixels moved this frame

            // Map horizontal pixel movement to world-X, vertical to world-Z.
            // Negate so dragging right moves the camera right (feels natural).
            camPos_.x_    -= delta.x_ * PAN_SCALE;
            camPos_.z_    -= delta.y_ * PAN_SCALE;
            camLookAt_.x_ -= delta.x_ * PAN_SCALE;
            camLookAt_.z_ -= delta.y_ * PAN_SCALE;

            cam_->SetPosition(camPos_);
            cam_->LookAt(camLookAt_);
        }
    }

    // ────────────────────────────────────────────────────────────────
    // ToggleFPSMode — demonstrates SetCursorLocked / SetCursorVisible.
    //
    // RTS mode:  cursor VISIBLE, NOT locked — player clicks units and ground.
    // FPS mode:  cursor LOCKED (MM_RELATIVE) — mouse movement rotates camera;
    //            GetMouseDelta() returns raw unbounded motion each frame.
    //            SetCursorVisible() has NO EFFECT while the cursor is locked.
    // ────────────────────────────────────────────────────────────────
    void ToggleFPSMode()
    {
        fpsMode_ = !fpsMode_;

        if (fpsMode_)
        {
            // Hide cursor and enter MM_RELATIVE mode.
            // After this call IsCursorLocked() returns true,
            // and GetMouseDelta() reports raw unbounded mouse motion.
            SetCursorLocked(true);

            // Switch camera to free FPS look; mouse look is handled
            // automatically inside SetFPSMode() using GetMouseDelta().
            cam_->SetFPSMode();
            cam_->SetFPSSensitivity(0.18f);

            statusLbl_->SetText("FPS look mode — Tab to return to RTS");
            controlsLbl_->SetVisible(false);
        }
        else
        {
            // Release cursor lock — IsCursorLocked() returns false after this.
            // SetCursorLocked(false) automatically restores cursor visibility.
            SetCursorLocked(false);

            // Make the cursor explicitly visible again for RTS interaction.
            // (SetCursorLocked(false) already restores it, but being explicit
            //  makes the intent readable in code review.)
            SetCursorVisible(true);

            // Restore the top-down camera.
            cam_->Detach();
            cam_->SetPosition(camPos_);
            cam_->LookAt(camLookAt_);

            statusLbl_->SetText(selected_
                ? "Selected: " + selected_->GetName() + "  — RMB to move"
                : "Click a unit to select it");
            controlsLbl_->SetVisible(true);
        }
    }
};

int main()
{
    return Simple3D::CreateGame<RTSGame>()->Run();
}

Step-by-step explanation

PickEntity(screenPos, mask)

PickEntity casts a ray from the camera through the screen pixel at screenPos (pixel coordinates, top-left origin) and returns the Entity* of the first physics body whose collision layer passes the mask filter. Pass GetMousePosition() directly as the screen position argument.

The return value is nullptr when the ray hits nothing within the mask (including when it misses all geometry entirely). Always null-check before using the pointer. The function never returns a pointer to an entity that is on a layer excluded by the mask, so you do not need to manually test the layer after the call. 

// Pick the first Actor-layer entity under the cursor.
Entity* hit = PickEntity(GetMousePosition(), CollisionLayer::Actor);
if (hit)
{
    // The entity exists and is on CollisionLayer::Actor.
    hit->AddTag("selected");
}

// With the default All mask — hits everything with a RigidBody.
Entity* anything = PickEntity(GetMousePosition());

Using a mask is important in scenes that mix pickable units with trigger volumes, decorative props with mesh colliders, or other categories you do not want the player to accidentally select. Without a mask you might select an invisible trigger box sitting on the unit.

PickPosition(screenPos, mask)

PickPosition fires the same physics ray as PickEntity but instead of returning the entity, it returns the world-space Vector3 coordinate where the ray struck the surface. This is the exact 3D point under the mouse cursor in world coordinates, which makes it ideal for terrain placement, move-order targeting in RTS games, and object spawning at the cursor.

The function returns Vector3::ZERO on a miss. Check for this before using the result, because Vector3::ZERO is also a valid world position (the scene origin). Compare with != Vector3::ZERO as a fast miss guard; for scenes where the origin matters, add a hit-distance check instead. 

// Find the ground point under the cursor (ignores Actor-layer units).
Vector3 groundPos = PickPosition(GetMousePosition(),
                                   CollisionLayer::StaticGeometry);
if (groundPos != Vector3::ZERO)
{
    // Spawn a flag at the clicked ground location.
    Entity* flag = CreateEntity("Flag");
    flag->AddModel("Models/Flag.mdl");
    flag->SetPosition(groundPos);
}

// Pick with no mask — returns the closest hit of any type.
Vector3 anyHit = PickPosition(GetMousePosition());

Only the X and Z components of groundPos are useful when you want to move a unit to a flat terrain surface — preserve the unit's current Y so it does not sink into or float above the ground. This is the pattern used in the HandleRTSInput method in the example above.

SetCursorVisible(bool) and IsCursorVisible()

SetCursorVisible(false) hides the OS cursor. The mouse still moves and all mouse button events fire normally; only the visible pointer disappears. Use it during full-screen gameplay where you draw your own crosshair or cursor sprite. SetCursorVisible(true) restores the system cursor.

IsCursorVisible() returns the current visibility state as a bool. Note that SetCursorVisible has no effect while the cursor is locked (see below) — while locked, the cursor is always hidden regardless of the visibility flag. The visibility state is remembered and restored when the lock is released. 

// During gameplay: hide the OS cursor, draw a custom crosshair instead.
SetCursorVisible(false);

// In the pause menu: show the OS cursor so the player can click buttons.
if (IsKeyPressed(Key::Escape))
{
    paused_ = !paused_;
    SetCursorVisible(paused_);   // show cursor in pause menu
    pausePanel_->SetVisible(paused_);
}

// Query current state.
if (!IsCursorVisible())
{
    // Draw our custom crosshair UI image here.
}

SetCursorLocked(bool) and IsCursorLocked()

SetCursorLocked(true) engages Urho3D's MM_RELATIVE mouse mode: the OS cursor is hidden, the mouse pointer is confined inside the window, and raw per-frame mouse deltas become available via GetMouseDelta() without being bounded by the screen edges. This is the standard setup for FPS-style mouse look, where the player needs to be able to move the mouse indefinitely in any direction without the cursor hitting the screen border.

SetCursorLocked(false) exits relative mode, releases the cursor confinement, and restores cursor visibility. IsCursorLocked() returns the current lock state. 

// Enter FPS look mode — cursor hidden, mouse reports unbounded delta.
SetCursorLocked(true);
cam_->SetFPSMode();          // FPS camera uses GetMouseDelta() internally

// Exit FPS mode — cursor released and restored.
SetCursorLocked(false);
cam_->Detach();

// Query lock state.
bool locked = IsCursorLocked();   // true = MM_RELATIVE active

// Common guard: only issue ray picks when cursor is NOT locked,
// because PickEntity/PickPosition use the visible cursor position
// which is meaningless while the cursor is confined to the center.
if (!IsCursorLocked() && IsMouseButtonDown(MouseButton::Left))
{
    Entity* hit = PickEntity(GetMousePosition());
}

GetMouseDelta() and MM_RELATIVE mode

GetMouseDelta() returns a Vector2 containing how many pixels the mouse moved horizontally and vertically this frame. It works in two distinct modes depending on the cursor lock state:

Cursor NOT locked (default)

Delta is the frame-to-frame movement of the visible cursor position. When the cursor reaches the screen edge, motion stops and the delta is clamped. Useful for camera panning while holding a mouse button, drag operations, or tracking a dragged slider.

Cursor LOCKED (MM_RELATIVE)

Delta is raw unbounded hardware mouse motion. The cursor is invisibly pinned at the window center; the player can move the mouse freely in any direction for as long as they like. This is the correct mode for FPS camera look, orbit camera drag, and any interaction where you need infinite mouse travel. 

// ── Middle-mouse pan (cursor NOT locked) ─────────────────────────────────
if (IsMouseButtonDown(MouseButton::Middle))
{
    Vector2 delta = GetMouseDelta();     // cursor-position delta, bounded by screen
    camPos_.x_ -= delta.x_ * PAN_SCALE;
    camPos_.z_ -= delta.y_ * PAN_SCALE;
    cam_->SetPosition(camPos_);
}

// ── FPS rotation (cursor LOCKED — MM_RELATIVE mode) ──────────────────────
SetCursorLocked(true);   // call once to enter the mode

// In Update():
// cam_->SetFPSMode() handles this automatically, but you can also
// use GetMouseDelta() yourself for a custom orbit or look effect:
Vector2 look = GetMouseDelta();   // raw hardware delta; no screen-edge clamping
yaw_   += look.x_ * sensitivity;
pitch_ += look.y_ * sensitivity;
pitch_  = Clamp(pitch_, -80.0f, 80.0f);

The delta values are in pixels per frame. Multiply by a sensitivity constant (degrees per pixel or world-units per pixel) to convert them to a usable quantity. Typical FPS sensitivity values are in the range 0.1 to 0.3 degrees per pixel; typical pan values are 0.05 to 0.15 world units per pixel depending on zoom level.

RTS click-to-select and move-to-position pattern

The complete RTS pattern combines three picking techniques into a consistent control scheme. The key insight is that each mouse button sends the ray through a different collision layer mask:

  • Left click uses PickEntity(..., CollisionLayer::Actor) to select or deselect a unit. The ground and trigger volumes are masked out.
  • Right click uses PickPosition(..., CollisionLayer::StaticGeometry) to find the terrain point for a move order. Units, triggers, and other dynamic bodies are masked out, so the ray always reaches the ground beneath them.
  • Middle drag uses GetMouseDelta() without any picking to pan the camera in world space, keeping the cursor visible and functional. 
// Minimal self-contained RTS pick pattern:
void Update(float dt) override
{
    Vector2 mpos = GetMousePosition();

    // 1. Left click — select the unit under the cursor.
    if (IsMouseButtonDown(MouseButton::Left))
    {
        Entity* hit = PickEntity(mpos, CollisionLayer::Actor);
        if (selected_) selected_->RemoveTag("selected");
        selected_ = (hit && hit->HasTag("unit")) ? hit : nullptr;
        if (selected_) selected_->AddTag("selected");
    }

    // 2. Right click — move selected unit to the ground click position.
    if (IsMouseButtonDown(MouseButton::Right) && selected_)
    {
        Vector3 dest = PickPosition(mpos, CollisionLayer::StaticGeometry);
        if (dest != Vector3::ZERO)
            moveTarget_ = dest;
    }

    // 3. Slide unit toward move target each frame.
    if (selected_ && (moveTarget_ - selected_->GetPosition()).Length() > 0.1f)
    {
        Vector3 dir = (moveTarget_ - selected_->GetPosition()).Normalized();
        selected_->SetPosition(selected_->GetPosition() + dir * 5.0f * dt);
    }

    // 4. Middle-drag to pan the camera without locking the cursor.
    if (IsMouseButtonDown(MouseButton::Middle))
    {
        Vector2 d = GetMouseDelta();
        camOrigin_.x_ -= d.x_ * 0.07f;
        camOrigin_.z_ -= d.y_ * 0.07f;
        cam_->SetPosition(camOrigin_ + Vector3(0.0f, 16.0f, 9.0f));
        cam_->LookAt(camOrigin_);
    }
}

Lua equivalent

All picking and cursor API calls are available in Lua scripts. One difference from C++: the Lua PickEntity and PickPosition functions take the mouse X and Y pixel coordinates as two separate integer arguments rather than a single Vector2. Retrieve them from game:GetMousePosition(), which returns an IntVector2. 

-- Scripts/rts_controller.lua
-- Attach this script to a persistent entity created in C++ Start().
-- Implements the same RTS click-to-select + move-to-position logic.

local selected   = nil
local hasTarget  = false
local moveTarget = Vector3(0, 0, 0)
local UNIT_SPEED = 5.0

function Update(dt)
    -- GetMousePosition() returns an IntVector2 in Lua.
    local mpos = game:GetMousePosition()
    local mx, my = mpos.x, mpos.y

    -- ── Left click: select a unit ────────────────────────────────
    -- PickEntity takes x, y as separate integers in Lua.
    -- Third argument is the collision layer mask (1 = Actor).
    if game:IsMouseButtonDown(MouseButton.Left) then
        local hit = PickEntity(mx, my, 1)   -- mask 1 = CollisionLayer::Actor

        if selected then
            selected:RemoveTag("selected")
            selected = nil
        end

        if hit and hit:HasTag("unit") then
            selected = hit
            selected:AddTag("selected")
            print("Selected: " .. selected:GetName())
        end
    end

    -- ── Right click: move selected unit to ground position ────────
    -- PickPosition takes x, y, mask.
    -- Mask 2 = CollisionLayer::StaticGeometry (ground only).
    if game:IsMouseButtonDown(MouseButton.Right) and selected then
        local groundPos = PickPosition(mx, my, 2)

        if groundPos:Length() > 0 then   -- non-zero = something was hit
            local unitY = selected:GetPosition().y
            moveTarget  = Vector3(groundPos.x, unitY, groundPos.z)
            hasTarget   = true
        end
    end

    -- ── Move selected unit toward target ─────────────────────────
    if selected and hasTarget then
        local pos  = selected:GetPosition()
        local dir  = moveTarget - pos
        local dist = dir:Length()

        if dist < 0.12 then
            selected:SetPosition(moveTarget)
            hasTarget = false
        else
            dir:Normalize()
            selected:SetPosition(pos + dir * UNIT_SPEED * dt)
        end
    end

    -- ── Middle drag: pan camera ──────────────────────────────────
    -- GetMouseDelta() works the same as in C++.
    -- Cursor is NOT locked; delta is bounded by screen edges.
    if game:IsMouseButtonDown(MouseButton.Middle) then
        local delta = game:GetMouseDelta()
        -- pan logic handled by the C++ PanCamera helper via game event
    end
end

-- ── Cursor control from Lua ───────────────────────────────────────
function EnterFPSMode()
    game:SetCursorLocked(true)    -- hides cursor + enters MM_RELATIVE mode
    -- Camera FPS mode is set from C++; Lua scripts can still read the lock state:
    print("Cursor locked: " .. tostring(game:IsCursorLocked()))
end

function ExitFPSMode()
    game:SetCursorLocked(false)   -- restores cursor visibility automatically
    game:SetCursorVisible(true)   -- explicit restore for clarity
end

-- ── Pick with no mask (All layers) from Lua ───────────────────────
function PickAnything()
    local mpos = game:GetMousePosition()
    local hit  = PickEntity(mpos.x, mpos.y)           -- no mask = All
    local pos  = PickPosition(mpos.x, mpos.y)          -- world point
    if hit then
        print("Hit: " .. hit:GetName()
              .. " at (" .. pos.x .. ", " .. pos.y .. ", " .. pos.z .. ")")
    end
end

Common mistakes

  • Forgetting to add a RigidBody and collider. PickEntity and PickPosition test the physics world, not visual models. An entity with a mesh but no AddRigidBody and no AddBoxCollider (or equivalent) is completely invisible to ray picks. If clicking a unit does nothing, the first thing to check is whether it has a physics shape.
  • Setting the wrong collision layer. If you call PickEntity(mpos, CollisionLayer::Actor) but the unit's body is on CollisionLayer::StaticGeometry, the ray will miss it every time. Check that unit->SetCollisionLayer(CollisionLayer::Actor) is called on every selectable entity.
  • Not checking for nullptr from PickEntity. When the cursor is over empty space, PickEntity returns nullptr. Dereferencing it immediately without a null check causes a crash. Always write if (hit && hit->HasTag(...)).
  • Treating Vector3::ZERO as a valid move destination. PickPosition returns Vector3::ZERO on a miss. If your ground happens to pass through the world origin, a miss and a valid center-point hit are indistinguishable with just a != ZERO check. In that case, test the hit distance instead, or offset the ground plane slightly off the origin.
  • Calling PickEntity or PickPosition while the cursor is locked. In MM_RELATIVE mode the OS cursor is pinned at the window center, so GetMousePosition() always returns the center pixel. Ray picks will always hit whatever is at the screen center, not where the player last pointed before locking. Guard picking calls with if (!IsCursorLocked()).
  • Calling SetCursorVisible(false) while the cursor is locked and expecting it to do something. While IsCursorLocked() is true, the cursor is already forcibly hidden by the OS regardless of the visibility flag. SetCursorVisible has no effect in that state. Set cursor visibility after releasing the lock with SetCursorLocked(false).
  • Expecting SetCursorLocked(false) to keep the cursor hidden. Releasing the cursor lock automatically restores cursor visibility. If you want to keep the cursor hidden after exiting lock mode (e.g. you hid it before entering lock mode), call SetCursorVisible(false) again immediately after SetCursorLocked(false).
  • Confusing GetMouseDelta() with the mouse position. GetMouseDelta() returns the movement this frame in pixels, not the cursor's current position on screen. Use GetMousePosition() for the position and GetMouseDelta() only when you need the amount of movement. Applying the delta as a position will cause the camera or selected object to warp to bizarre locations.
  • Not multiplying GetMouseDelta() by dt for smooth movement. Mouse delta is already a per-frame quantity (pixels moved this frame), so applying it directly to a position or angle is the correct approach for feel-consistent input. Unlike velocity-based motion you should not multiply delta by dt — the delta already scales with frame time implicitly because it is the hardware movement that occurred during this frame. Do multiply by a sensitivity constant (e.g. degrees per pixel).
  • Forgetting that Lua PickEntity / PickPosition take (x, y, mask) not a Vector2. In Lua the picking functions accept separate integer screen coordinates, not a Vector2 object. Use mpos.x, mpos.y from game:GetMousePosition().

PickEntity(screenPos, mask=All) Entity* First entity hit by ray; nullptr on miss or mask exclusion.
PickPosition(screenPos, mask=All) Vector3 World-space hit point; Vector3::ZERO on miss.
GetMousePosition() Vector2 Current cursor pixel coordinate (top-left origin). Pass directly to PickEntity / PickPosition.

CollisionLayer::Actor 1 Player, enemies, selectable units.
CollisionLayer::StaticGeometry 2 Ground, walls, platforms.
CollisionLayer::Trigger 4 Trigger volumes (usually excluded from picks).

What you learned

In this part you gained a complete toolkit for mouse-driven 3D interaction:

  • Ray picking mechanics: simple-3d converts a screen pixel to a world-space ray via the active camera and tests it against the physics world up to 1000 units away. Every pickable entity must have a RigidBody and a collider shape.
  • PickEntity: returns the Entity* of the first body the ray hits, filtered by a CollisionLayer mask. Returns nullptr on a miss. Always null-check before use.
  • PickPosition: returns the world-space Vector3 hit point, ideal for terrain placement and RTS move orders. Returns Vector3::ZERO on a miss.
  • Collision layer masks: each pick call can target a specific category (Actor, StaticGeometry, etc.) so left-click selects units while right-click lands on the ground regardless of what is standing on it.
  • Cursor visibility: SetCursorVisible(false/true) shows or hides the OS cursor. Has no effect while the cursor is locked. IsCursorVisible() queries the current state.
  • Cursor locking: SetCursorLocked(true) enters Urho3D's MM_RELATIVE mode — the cursor is hidden and pinned inside the window, enabling unbounded mouse travel for FPS look. SetCursorLocked(false) restores the cursor. IsCursorLocked() queries the lock state.
  • Mouse delta: GetMouseDelta() returns the per-frame pixel movement of the mouse. In locked mode, the delta is raw hardware motion with no screen-edge clamping; unlocked, it tracks the visible cursor movement. Use it for camera panning, FPS rotation, and custom drag interactions.
  • RTS pattern: left-click with CollisionLayer::Actor selects units; right-click with CollisionLayer::StaticGeometry issues a move order to the ground; middle-drag reads GetMouseDelta() without locking the cursor to pan the camera. Tab toggles to FPS mode via SetCursorLocked and cam_→SetFPSMode().
