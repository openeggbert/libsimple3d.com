Phase 1 — Core

Foundation. Established the game loop, PIMPL architecture, public API boundary, and the zero-engine-coupling design principle that all later phases build on.

Game class

Subclass Game, override Start() and Update(dt). CreateGame<T>()->Run() is the entire entry point. No context wiring, no subsystem registration.

Entity system

CreateEntity(name) returns an opaque Entity*. All Urho3D node internals hidden behind PIMPL. Public headers contain zero engine includes.

Transform API

World and local SetPosition, SetRotation, SetScale, GetPosition, GetRotation. Deferred entity destroy prevents mid-frame crashes.

Type system

Math types (Vector2, Vector3, Color, Quaternion) re-exported as Simple3D:: aliases — ABI-identical, zero wrapping overhead.

  • PIMPL idiom enforced: every public class stores unique_ptr<Impl>; impl defined only in .cpp
  • Backend abstraction from day one: SIMPLE3D_ENGINE=U3D (default) or NOVA3D CMake flag
  • No Urho3D engine types in include/Simple3D/ — critical invariant held across all 96 tasks
  • CMake 3.21+ build system; C++23 standard throughout

Phase 2 — Physics

Bullet Physics (3D) and Box2D (2D). Rigid bodies, colliders, triggers, constraints, and a typed collision layer system that eliminated magic collision numbers forever.

3D rigid bodies (Bullet)

Dynamic and static rigid bodies via entity->AddRigidBody(mass). Box, capsule, sphere, and triangle-mesh colliders. Mass 0.0 = static geometry.

2D rigid bodies (Box2D)

AddRigidBody2D, box and circle colliders, linear velocity, impulse, and ground detection. Runs alongside the 3D physics world in the same scene.

Trigger volumes

Trigger colliders with OnTriggerEnter and OnTriggerExit callbacks. Deferred destroy ensures safe removal of entities contacted mid-frame.

Physics constraints

BallSocket, Hinge (with motor), Slider (with motor), and Fixed constraints between any two entities — all wrapped with the same PIMPL pattern.

  • CollisionLayer enum: Actor=1, StaticGeometry=2, Trigger=4 — raw numbers banned
  • ApplyImpulse, ApplyForce, SetLinearVelocity, GetLinearVelocity on Entity
  • Impulse / force / velocity control unified through the Entity interface

Phase 3 — 2D + Scripting

Sprite rendering, tile maps, and the Lua scripting foundation. Per-entity Lua scripts with the same Start()/Update(dt)/Stop() lifecycle as C++ — hot-reloadable at runtime.

2D sprites

AddStaticSprite2D(texPath) for still images. AddAnimatedSprite2D(scmlPath) for Spriter .scml skeletal animations. Horizontal flip control. Orthographic camera auto-configured.

Tiled tile maps

LoadTileMap(tmxPath) parses Tiled .tmx files and creates the scene geometry automatically. Tile layers rendered as static batched meshes.

Lua scripting (core)

entity->AddScript("Scripts/foo.lua") attaches a per-entity Lua script backed by Urho3D's LuaScriptInstance. Scripts are isolated and hot-reloadable.

Lua bindings (initial)

self.node alias, basic entity methods, Input: global, PlaySound / PlayMusic globals. AngelScript explicitly banned — Lua only.

Phase 4 — Navigation + Networking

Recast/Detour nav-mesh pathfinding and ENet client/server networking. Both systems wrapped so game code never touches engine-internal types.

Navigation (Recast/Detour)

BuildNavMesh() bakes from static scene geometry. entity->MoveToward(target, speed, dt) auto-steers along the nav-mesh. FindPath(from, to) returns a vector<Vector3> waypoint list.

Networking (ENet)

Client/server topology via NetworkManager. Scene entities auto-replicate. Custom messages with IDs 100–32767 carry raw byte payloads. Typed connection and disconnection callbacks.

  • StopMoving() cancels ongoing nav-mesh navigation
  • Navigation Lua bindings added in Task #82: BuildNavMesh(), entity:FindPath(target), entity:MoveToward(target, speed, dt)
  • ENet messages handled through NetworkManager — no direct Urho3D network headers in game code

Phase 5 — Animation

Skeletal animation for 3D models. Named clips, blend layers, weight control, speed multipliers, and fade in/out — all through a single AnimationController.

AnimationController

entity->AddAnimationController() wraps Urho3D's skeletal animation system. Play(clip, layer, loop, fadeTime), Stop(clip), SetWeight, SetSpeed, GetTime.

Blend layers

Multiple simultaneous animations on independent blend layers. Per-layer weight control for additive mixing — idle + upper-body shoot, run + wave, etc.

  • Accessed from Lua via entity:GetAnimationController() — same API surface as C++
  • Fade in/out via fadeTime parameter on Play — no separate tween required

Phase 6 — Character + AI

High-level character movement, finite state machines for animation, and a behaviour FSM for AI. All three systems fully Lua-accessible.

CharacterController

Capsule collider + slope-filtered raycast grounding + built-in gravity. IsOnGround(), MoveRelative(dir), SetMaxSlope, SetStepHeight, SetEnabled. No custom physics loop needed.

AnimStateMachine

String-keyed conditions drive animation transitions automatically. Condition types: bool, float, trigger, auto. asm:Update(dt) from Lua or C++.

BehaviorFSM

Custom states with enter/update/exit callbacks. Built-in NavMesh patrol and chase states. Condition transitions between states. fsm:Update(dt) drives the AI tick.

BehaviorFSM correctness

Task #84 fixed three root causes in BehaviorFSM::Update and IsInState. All 14 BehaviorFSM unit tests now pass. Zero regressions in the full suite.

  • CharacterController Lua proxy methods: SetEnabled/IsEnabled/SetMaxSlope/SetStepHeight added in Task #77
  • IsOnGround() uses slope-filtered raycast — reliable on ramps and stairs
  • BehaviorFSM integrates with NavMesh: built-in patrol/chase states call MoveToward internally

Phase 7 — Full UI

A complete retained-mode 2D UI system. Label, Image, Panel, Button, ProgressBar, and Slider — with anchor-based responsive layout, Z-ordering, and gamepad focus navigation. All widgets fully Lua-accessible.

Label

On-screen HUD text. Screen position, font size, RGBA color, visibility toggle. Custom font via SetFont. Scale control via SetScale.

Image

2D texture panel for HUD icons, overlays, and decorative elements. Full anchor and Z-order support.

Panel

Container widget. panel->AddChild(element) composes layouts. Background color, border, anchor-relative sizing.

Button

Clickable element with press/release/hover callbacks. Gamepad focus navigation. SetOnClick closure from Lua or C++.

ProgressBar

Horizontal or vertical fill bar. SetValue(0–1), SetVertical. Used for health bars, loading indicators, cooldown meters.

Slider

Value range slider. SetMin/SetMax, SetOnChange callback. Gamepad-navigable. Used for volume controls and settings panels.

  • SetAnchor: 9-point string anchoring ("TopLeft", "Center", "BottomRight", etc.) — Task #74
  • SetZOrder: explicit draw-order control for overlapping elements
  • Getter methods on all widgets: GetPosition, GetSize, GetText, GetMin, GetMax — Task #75
  • SetBackgroundColor, SetVertical on ProgressBar via Lua — Task #75
  • Gamepad focus navigation built in — D-pad moves focus between interactive widgets
  • UI parent-child: panel->AddChild(element) chains composable layouts

Phase 8 — Audio Expansion

Spatial 3D audio, per-channel routing, ambient playback, and full Lua coverage. Built on Urho3D's SoundSource3D component — attenuation handled automatically.

AudioSource component

entity->AddAudioSource() attaches a spatial 3D sound source. SetGain, SetPitch, SetRange, SetLooped, Play, Stop, PlayAmbient.

Per-channel audio

PlaySound(path, vol, channel) routes sounds to named channels. StopSound(channel), SetChannelVolume(channel, vol), IsChannelPlaying(channel) — Task #69.

Music streaming

PlayMusic(path) streams OGG/WAV with loop control. StopMusic(). Master volume via SetMasterVolume.

Audio listener

SetAudioListener(entity) Lua global designates which entity is the 3D listener — typically the camera or the player's head. Added in Task #80.

Phase 9 — Input Expansion

Input Actions API, binding persistence, gamepad raw API, and touch input. Raw keyboard/mouse/touch remain available; the Actions layer adds named, rebindable controls.

Input Actions API

BindAction("Jump", Key::Space) / BindAxis2D("Move", ...). Query with IsActionPressed("Jump") / GetAxis2D("Move"). Per-binding dead-zone configuration.

Binding persistence

SaveInputBindings(path), LoadInputBindings(path), ResetInputBindings(). Atomic load: parses into a temporary map first; swaps live only on full success — Task #72 / #92.

Gamepad raw API (Lua)

IsGamepadConnected(), GetGamepadAxis(name), IsGamepadButtonDown/Pressed(name), SetGamepadRumble(lo, hi). String axis/button names. Task #83.

Touch input (Lua)

Multi-touch globals: touch count, position, pressure. Accessed via Input: table in Lua scripts. Task #73.

  • Raw keyboard: IsKeyDown, IsKeyPress — available in both C++ and Lua
  • Mouse: position, delta, buttons — held and first-frame variants
  • Controller rumble: SetGamepadRumble(lowFreq, highFreq)
  • Axis2DBinding::DeadZone() clamped to [0.0f, 0.95f] — Task #92; prevents saturation lockout
  • LoadInputBindings validates array-length parity; malformed JSON no longer wipes live bindings

Phase 10 — Effects + Visual

Particle emitters, billboards, sky dome, debug drawing, ray picking, and the tween helper. Visual polish tools available from both C++ and Lua.

ParticleEmitter

entity->AddParticleEmitter(xmlPath). Play(), Stop(), Burst(count). Five built-in preset XMLs (sparks, smoke, fire, magic, dust).

Billboard

entity->AddBillboard(texPath, size). Always faces camera (FC_ROTATE_XYZ). SetBillboardUVRect for atlas sprites. Lua: entity:AddBillboard(texPath, size) — Task #79.

Sky dome

SetSkyDome(texPath), SetSkyDomeEnabled(bool), SetSkyGradient(zenith, horizon). Full Lua coverage. Task #68.

Debug draw

SetDebugDrawEnabled, DrawDebugLine, DrawDebugBox, DrawDebugSphere, DrawDebugText. Toggle at runtime. Lua globals — Task #76.

Ray picking

PickEntity(screenPos, mask) returns the first hit Entity*. PickPosition(screenPos, mask) returns the world-space hit point. Useful for mouse selection and placement.

Tween helper

CreateTween(...) returns a proxy with Cancel() and IsComplete(). 4 easing modes. Ownership changed to shared_ptr — safe from Lua. Task #78.

  • Screenshot: SaveScreenshot(path) — saves the current frame as PNG
  • Entity materials: SetTileTexture (UV atlas tiling), SetMaterialColor (flat unlit color)
  • Tween Lua binding: local t = CreateTween(...); t:Cancel(); t:IsComplete()

Phase 11 — Scene + Persistence

Level transitions, fog and atmosphere, time scaling, and a JSON save-data system. Scenes fade between levels; save slots persist across sessions.

Scene management

LoadScene(path), UnloadScene(), FadeOut(dur), FadeIn(dur), SetLevelList(list), LoadNextLevel(). UnloadScene calls FlushPendingDestroys() — fixed in Task #88.

SaveData

JSON key-value store. Multi-slot support. GetInt/SetInt, GetString/SetString, GetBool/SetBool, GetFloat/SetFloat. 14 unit tests. Full Lua binding.

Atmosphere

SetAmbientColor, SetFogEnabled(bool), SetFogColor, SetFogRange(start, end). Scene-wide environmental controls for mood and performance.

Time control

SetTimeScale(s) / GetTimeScale() scales the simulation tick globally. Use 0.0 to pause, 0.5 for slow-motion, 2.0 for fast-forward.

Phase 12 — Tools + Utilities

Resource diagnostics, runtime resource path registration, entity tagging, and activation toggles. Quality-of-life utilities that reduce debug time and support dynamic scene management.

Resource diagnostics

SetResourceDiagnosticsEnabled(bool), GetMissingResources(), ClearMissingResources(). Fallback model/texture/sound via SetFallback*. Task #96.

AddResourceDir

Game::AddResourceDir(path) registers additional resource search paths at runtime — useful for mod support, DLC, and test fixture isolation. Added in commit 9cd38f7.

Entity tagging

entity->AddTag("enemy"), HasTag, RemoveTag. Game::FindEntitiesByTag("enemy") returns all matching entities. Fully Lua-accessible.

Entity activation

entity->SetActive(bool) / IsActive() hides and disables entities without destroying them. Useful for pooling and off-screen objects.

Phase 14 — Mc3Bridge

MC3 scene format import, export, and live-reload. Mc3Bridge enables scene authoring tools to push incremental changes into a running game session.

Mc3Bridge

#include <Simple3D/Mc3/Mc3Bridge.h>. Import MC3 scene descriptions and materialise them as simple-3d entities. Export the current scene back to MC3 format for round-trip authoring.

Mc3Watcher

File-system watcher that monitors a .mc3 file for changes. On modification, triggers a live reload — updated entities appear in the running game without restarting. Accelerates iteration on level layouts.

  • Mc3 headers kept in a separate include path: <Simple3D/Mc3/Mc3Bridge.h>, <Simple3D/Mc3/Mc3Watcher.h>
  • Not included in the single-header umbrella Simple3D.h — opt-in for projects that need it
  • Designed to integrate with external scene tools targeting the MC3 interchange format
