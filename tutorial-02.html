Part 02 of 30Lights and Colors

New in this part

Color(r,g,b,a)  •  Color::WHITE / Color::BLACK  •  SetClearColor(Color)  •  SetAmbientColor(Color)  •  Entity::AddDirectionalLight(Color, intensity)  •  Entity::AddPointLight(Color, range)  •  Quaternion(yaw, pitch, roll) for light direction  •  Key + fill two-light rig

What you will learn

  • How the Color type works — RGBA normalized floats, and the built-in Color::WHITE / Color::BLACK constants.
  • How to set the window background colour with SetClearColor.
  • How to give the entire scene a base level of illumination using SetAmbientColor.
  • How to add a sun-like directional light with AddDirectionalLight(Color, intensity).
  • How to point a directional light in any direction using Quaternion(yaw, pitch, roll).
  • How to add a local radius-limited point light (like a torch or lamp) with AddPointLight(Color, range).
  • The classic key + fill two-light pattern that makes geometry look three-dimensional without advanced rendering.

Introduction

In Part 01 you created a window, an entity, and a camera. But with no lights in the scene the default rendering relies on a flat ambient pass — everything looks washed out or uniformly dark. Real-time lighting is what transforms flat geometry into convincing 3D shapes. Even a single well-placed directional light dramatically improves depth perception.

Before you can talk about lights you need a language for colour. In simple-3d every colour is expressed as four normalized floating-point components: red, green, blue, and alpha (opacity). All values live in the range 0.0 – 1.0. The same Color type is used everywhere: light tints, ambient settings, clear colour, UI widget tints, and label colours.

The Color type

Color(float r, float g, float b, float a = 1.0f)

Color is a plain value type defined in Simple3D/Simple3D.h. All four components are float values in the [0.0, 1.0] range; values outside this range are legal in some contexts (HDR lighting) but clamp to the visible range on standard displays. 

// Explicit RGBA construction
Color red   = Color(1.0f, 0.0f, 0.0f, 1.0f);  // fully opaque red
Color green  = Color(0.0f, 1.0f, 0.0f, 1.0f);  // fully opaque green
Color blue   = Color(0.0f, 0.0f, 1.0f, 1.0f);  // fully opaque blue

// Alpha channel
Color semiBlue = Color(0.0f, 0.3f, 1.0f, 0.5f);  // 50 % transparent blue

// Three-component form (alpha defaults to 1.0)
Color amber    = Color(1.0f, 0.75f, 0.0f);

// Predefined constants
Color::WHITE   // Color(1, 1, 1, 1) — pure white, fully opaque
Color::BLACK   // Color(0, 0, 0, 1) — pure black, fully opaque

0–1, not 0–255

A common mistake from CSS / SDL / GDI programming is to write Color(255, 128, 0). In simple-3d that produces a blindingly bright, clipped colour (255 times maximum red). Always use normalized floats: Color(1.0f, 0.5f, 0.0f).

The Color type inherits from Urho3D and exposes several additional predefined constants including Color::RED, Color::GREEN, Color::BLUE, Color::YELLOW, Color::CYAN, Color::MAGENTA, and Color::TRANSPARENT. Color::WHITE and Color::BLACK are the most commonly used for lighting.

SetClearColor — the window background

void SetClearColor(Color c)

Every frame the renderer first fills the entire framebuffer with a solid colour before drawing any 3D content. This is the clear colour, visible wherever no geometry or sky covers the screen. Call SetClearColor in Start() (and whenever you change scenes) to give your game the right background tone. 

// Dark navy sky — good for night / space scenes
SetClearColor(Color(0.05f, 0.08f, 0.18f));

// Midday blue sky
SetClearColor(Color(0.4f, 0.65f, 0.9f));

// Solid black (default if never called)
SetClearColor(Color::BLACK);

// Pure white — good for stylized/flat scenes
SetClearColor(Color::WHITE);

SetClearColor is a method on Game, so you call it without a receiver inside any Game subclass method. It is safe to call from both Start() and Update() — for example when transitioning between an outdoor (blue sky) and an indoor (black) scene.

SetAmbientColor — baseline illumination

void SetAmbientColor(Color c)

Ambient light is a constant, directionless illumination that reaches every surface in the scene equally, regardless of its orientation relative to light sources. It represents indirect light bouncing off walls, the sky, or the ground — light that comes from everywhere at once.

The default ambient is Color(0.3, 0.3, 0.3) — a moderate grey. Too high and the scene looks flat (all surfaces equally bright, no depth). Too low and shadows look completely black with no detail. A good ambient is subtle: around 10–25 % brightness. 

// Neutral grey ambient (default)
SetAmbientColor(Color(0.3f, 0.3f, 0.3f));

// Subtle cool ambient (moonlit outdoor scene)
SetAmbientColor(Color(0.10f, 0.12f, 0.20f));

// Warm amber ambient (interior lit by firelight)
SetAmbientColor(Color(0.20f, 0.12f, 0.06f));

// No ambient at all — pitch-black shadows (space, horror)
SetAmbientColor(Color::BLACK);

Ambient and the Zone component

Internally, SetAmbientColor sets the ambient light on Urho3D's Zone component that simple-3d creates for you. The same Zone also controls fog (covered in Part 29). You never need to touch the Zone directly.

AddDirectionalLight — sun-like lighting

void Entity::AddDirectionalLight(Color color, float intensity)

A directional light has no position — it sends parallel rays across the entire scene from a fixed direction, just like the sun. Every surface in the world is illuminated by the same light regardless of where it is. The brightness on each surface depends only on the angle between the surface normal and the light direction.

You call AddDirectionalLight on any entity. The entity's world-space rotation determines which way the light points. The color parameter tints the light, and intensity scales its overall brightness (1.0 is full power; 0.5 is half; values above 1.0 are valid for HDR-style overbright effects). 

// Create a dedicated entity to carry the sun light
Entity* sun = CreateEntity("Sun");

// Pure white, full brightness — neutral daytime sun
sun->AddDirectionalLight(Color::WHITE, 1.0f);

// Warm afternoon light (slightly orange-tinted, still bright)
sun->AddDirectionalLight(Color(1.0f, 0.92f, 0.75f), 1.0f);

// Dimmer fill light — cooler blue tone
Entity* fill = CreateEntity("FillLight");
fill->AddDirectionalLight(Color(0.40f, 0.55f, 0.80f), 0.35f);

Directional lights do not attenuate with distance — a surface 1 metre away is just as bright as one 1000 metres away. This is why they are used for global sources like the sun or moon.

Quaternion — pointing a directional light

Quaternion(float yaw, float pitch, float roll)

A Quaternion represents a 3D rotation. For directional lights, you set the entity's rotation and that rotation defines which direction the light shines. The constructor Quaternion(yaw, pitch, roll) takes angles in degrees.

  • Yaw — rotation around the world Y (up) axis. Positive yaw spins right.
  • Pitch — rotation around the local X axis (after yaw). Positive pitch tilts downward.
  • Roll — rotation around the local Z axis. For directional lights, roll has no visible effect.
// Sun from upper-left — tilted 45° down, 30° to the right
sun->SetRotation(Quaternion(45.0f, -30.0f, 0.0f));

// Sun directly overhead (noon) — straight down
sun->SetRotation(Quaternion(90.0f, 0.0f, 0.0f));

// Low sunrise from the left — shallow angle, wide shadow
sun->SetRotation(Quaternion(10.0f, 90.0f, 0.0f));

// Fill light from the opposite side as the sun above
fill->SetRotation(Quaternion(-45.0f, 150.0f, 0.0f));

Default direction

An entity with no rotation (Quaternion(0, 0, 0) or simply unset) has its forward direction pointing along +Z. A directional light attached to such an entity shines horizontally into the scene. Giving it a positive pitch tilts it downward, which is usually what you want.

For dynamic light rigs — for example a day/night cycle — you can update the rotation every frame inside Update(float dt): 

// In Update(dt): rotate the sun slowly (day/night cycle)
dayAngle_ += dt * 6.0f;  // 6°/s = one full day in 60 seconds
sun_->SetRotation(Quaternion(dayAngle_, -30.0f, 0.0f));

AddPointLight — local radius light

void Entity::AddPointLight(Color color, float range)

A point light radiates light outward equally in all directions from the entity's world position, like a light bulb or a torch. Unlike a directional light, it has a position and a finite range: surfaces farther than range world units from the light receive no illumination at all. Between 0 and range, brightness falls off smoothly following an inverse-square-ish curve. 

// A warm orange torch, reaches 8 metres
Entity* torch = CreateEntity("Torch");
torch->AddPointLight(Color(1.0f, 0.55f, 0.15f), 8.0f);
torch->SetPosition(-2.0f, 1.5f, 4.0f);

// A small cool LED, tight radius
Entity* lamp = CreateEntity("Lamp");
lamp->AddPointLight(Color(0.85f, 0.90f, 1.0f), 3.0f);
lamp->SetPosition(2.0f, 2.0f, 6.0f);

// A large area flood (white, neutral intensity)
Entity* flood = CreateEntity("Flood");
flood->AddPointLight(Color::WHITE, 20.0f);
flood->SetPosition(0.0f, 8.0f, 0.0f);

Point lights do not require the entity to have any particular rotation — direction is irrelevant for omnidirectional point lights. Moving the entity moves the light source in world space, so you can animate a torch bobbing, a magic projectile flying through the air, or a pick-up item pulsing on the ground.

SpotLight is also available

simple-3d also exposes AddSpotLight(Color, range, angle) for cone-shaped lights (flashlights, car headlights, stage spotlights). Spot lights use entity rotation to aim the cone, just like directional lights. They are covered in depth in a later part.

The key + fill two-light pattern

Placing just one directional light results in half of every object being fully lit and the other half receiving only ambient light. With a very low ambient that shadow side looks unnaturally dark. The key + fill rig is a cinematography technique that solves this:

  • The key light is the primary, dominant light source — bright, warm, and positioned from above-side (typically representing the sun). It defines the overall mood and the primary shadow direction.
  • The fill light is a secondary, dimmer light coming from roughly the opposite side or below — cooler in colour, 30–50 % the intensity of the key. It softens the shadows without eliminating them, keeping depth visible while revealing detail in dark areas. 
// Key light — warm sun, from upper-left
Entity* sun = CreateEntity("Sun");
sun->AddDirectionalLight(Color(1.0f, 0.92f, 0.75f), 1.0f);
sun->SetRotation(Quaternion(45.0f, -30.0f, 0.0f));

// Fill light — cool sky, from lower-right (opposite side)
Entity* fill = CreateEntity("SkyFill");
fill->AddDirectionalLight(Color(0.40f, 0.55f, 0.80f), 0.35f);
fill->SetRotation(Quaternion(-45.0f, 150.0f, 0.0f));

The contrast between the warm key and the cool fill is a powerful visual cue: the lit side looks sunny and the shadow side looks like it is receiving reflected skylight. This gives geometry a natural, photographic quality with minimal setup.

Quick recipe

Warm key at ~1.0 intensity + cool fill at ~0.3–0.4 intensity + dark ambient at Color(0.08, 0.10, 0.18) = a scene that reads clearly in all lighting conditions without looking flat or overly dark.

Complete C++ example

The program below demonstrates every concept from this part: the Color type, SetClearColor, SetAmbientColor, a directional key + fill rig, and a bobbing point light. The box rotates slowly so you can see how the lighting wraps around it. 

#include <Simple3D/Simple3D.h>
#include <cmath>      // std::sin
#include <string>
using namespace Simple3D;

class LightsDemo : public Game {
public:
    Entity* box_     = nullptr;  // the rotating crate
    Entity* torch_   = nullptr;  // the moving point light
    Label*  hud_     = nullptr;
    float   angle_   = 0.0f;    // box Y rotation in degrees
    float   bobTime_ = 0.0f;    // torch bobbing timer

    void Start() override {
        SetWindowTitle("Part 02 — Lights and Colors");

        // ── Background colour ─────────────────────────────────────────
        // Deep midnight-blue — visible wherever no geometry covers the screen.
        SetClearColor(Color(0.05f, 0.08f, 0.18f));

        // ── Ambient light ────────────────────────────────────────────
        // Subtle cool ambient so shadowed sides are dark but not pure black.
        SetAmbientColor(Color(0.10f, 0.12f, 0.20f));

        // ── Scene geometry ───────────────────────────────────────────
        // A crate in the centre of the view.
        box_ = CreateEntity("Crate");
        box_->AddModel("Models/Box.mdl");
        box_->SetPosition(0.0f, 0.0f, 5.0f);

        // A wide, flat floor to catch shadows and coloured spill.
        Entity* floor = CreateEntity("Floor");
        floor->AddModel("Models/Box.mdl");
        floor->SetPosition(0.0f, -1.0f, 5.0f);
        floor->SetScale(12.0f, 0.2f, 12.0f);

        // ── Key light: warm sun from upper-left ─────────────────────
        // Color(1, 0.92, 0.75) = slightly golden daylight.
        // Intensity 1.0 = full brightness.
        Entity* sun = CreateEntity("Sun");
        sun->AddDirectionalLight(Color(1.0f, 0.92f, 0.75f), 1.0f);
        // Quaternion(yaw, pitch, roll): pitch 45° down, yaw -30° (from left)
        sun->SetRotation(Quaternion(45.0f, -30.0f, 0.0f));

        // ── Fill light: cool sky from lower-right ───────────────────
        // Opposite side from the sun; 35 % intensity; cool blue-white tint.
        Entity* fill = CreateEntity("SkyFill");
        fill->AddDirectionalLight(Color(0.40f, 0.55f, 0.80f), 0.35f);
        fill->SetRotation(Quaternion(-45.0f, 150.0f, 0.0f));

        // ── Point light: an orange torch near the crate ─────────────
        // Color(1, 0.55, 0.15) = ember orange.  Range 8 metres.
        torch_ = CreateEntity("Torch");
        torch_->AddPointLight(Color(1.0f, 0.55f, 0.15f), 8.0f);
        torch_->SetPosition(-2.5f, 1.0f, 4.0f);

        // ── Camera ───────────────────────────────────────────────────
        CreateCamera();

        // ── HUD label ────────────────────────────────────────────────
        hud_ = CreateLabel("Part 02: Lights and Colors  |  ESC to quit");
        hud_->SetPosition(16, 16);
        hud_->SetFontSize(15);
        hud_->SetColor(Color(0.75f, 0.80f, 0.90f));
    }

    void Update(float dt) override {
        // Rotate the box so all faces catch light in turn.
        angle_ += dt * 25.0f;
        box_->SetRotation(Quaternion(0.0f, angle_, 0.0f));

        // Bob the torch up and down to simulate a flickering flame position.
        bobTime_ += dt;
        float bob = std::sin(bobTime_ * 3.5f) * 0.15f;
        torch_->SetPosition(-2.5f, 1.0f + bob, 4.0f);

        if (IsKeyDown(Key::Escape)) Quit();
    }
};

int main() {
    return CreateGame<LightsDemo>()->Run();
}

Step-by-step explanation

1

SetClearColor — paint the sky

SetClearColor(Color(0.05f, 0.08f, 0.18f)) sets the background to a deep midnight blue. The values are (red=0.05, green=0.08, blue=0.18); the alpha defaults to 1.0 (fully opaque). This colour becomes visible in any area of the screen not covered by geometry — including the corners of the window when the field of view is narrow.

2

SetAmbientColor — fill in the shadows

SetAmbientColor(Color(0.10f, 0.12f, 0.20f)) sets a very dark blue-grey ambient. This is the base brightness that every surface receives. Because it is so dark, shadow-side faces will be noticeably darker than lit faces, preserving depth. The slight blue bias complements the warm key light by making shadows feel "cooled" by reflected sky light.

3

AddDirectionalLight — the sun

sun->AddDirectionalLight(Color(1.0f, 0.92f, 0.75f), 1.0f) adds a parallel light that covers the entire scene. The colour is warm white with a slight golden tint (full red, slightly reduced green, reduced blue). The intensity of 1.0f is standard full brightness. Because directional lights use the entity's rotation for direction, calling AddDirectionalLight with no subsequent SetRotation would produce a light shining along the default forward axis — horizontally, which is rarely desirable.

4

SetRotation with Quaternion — aiming the sun

sun->SetRotation(Quaternion(45.0f, -30.0f, 0.0f)) tilts the sun entity 45° downward (pitch) and rotates it 30° to the left (negative yaw). Together these place the light source somewhere in the upper-left of the scene — a classic "early afternoon" angle. The 0.0f roll has no effect on a directional light.

5

Fill light — the other side

fill->AddDirectionalLight(Color(0.40f, 0.55f, 0.80f), 0.35f) adds a second directional light at 35 % of full brightness, tinted cool blue-white. Its rotation Quaternion(-45.0f, 150.0f, 0.0f) points it from the lower-right — the opposite hemisphere from the sun. Faces that are in shadow from the sun now receive this cooler light, so no surface goes completely dark. The intensity difference (1.0 vs. 0.35) preserves the sense of direction and drama.

6

AddPointLight — the torch

torch_->AddPointLight(Color(1.0f, 0.55f, 0.15f), 8.0f) creates an omnidirectional ember-orange light with a 8-metre radius. Surfaces more than 8 m away receive no illumination from it at all. Setting the entity's position with SetPosition controls where in the world the light is emitted from. Moving the entity in Update() moves the light source — this is how animated lights (flickering torches, moving headlights, magic projectiles) are implemented.

7

Bobbing animation — sinusoidal offset

Inside Update(float dt), bobTime_ accumulates elapsed time. Feeding it through std::sin(bobTime_ * 3.5f) produces a smooth oscillation between -1 and +1. Multiplying by 0.15f makes the range ±0.15 metres, which is subtle enough to look like a flame dancing rather than a platform elevator. This same pattern — sin(time * frequency) * amplitude — is fundamental to game animations and will appear again in later tutorials.

Lua equivalent

The lighting setup can be performed entirely from a Lua script that is attached to a dedicated manager entity, or done in C++ with only the per-frame animation in Lua. Below is a complete Lua implementation of the same scene: 

-- Data/Scripts/lights_demo.lua
-- Attach to a "SceneManager" entity created in C++ Start():
--   local mgr = CreateEntity("SceneManager")
--   mgr:AddScript("Scripts/lights_demo.lua")

local box
local torch
local angle    = 0
local bobTime  = 0

function Start()
    -- Background and ambient ─────────────────────────────────────────
    SetClearColor(Color(0.05, 0.08, 0.18))
    SetAmbientColor(Color(0.10, 0.12, 0.20))

    -- Scene geometry ─────────────────────────────────────────────────
    box = CreateEntity("Crate")
    box:AddModel("Models/Box.mdl")
    box:SetPosition(0, 0, 5)

    local floor = CreateEntity("Floor")
    floor:AddModel("Models/Box.mdl")
    floor:SetPosition(0, -1, 5)
    floor:SetScale(12, 0.2, 12)

    -- Key light: warm sun ────────────────────────────────────────────
    local sun = CreateEntity("Sun")
    sun:AddDirectionalLight(Color(1.0, 0.92, 0.75), 1.0)
    sun:SetRotation(Quaternion(45, -30, 0))

    -- Fill light: cool sky ───────────────────────────────────────────
    local fill = CreateEntity("SkyFill")
    fill:AddDirectionalLight(Color(0.40, 0.55, 0.80), 0.35)
    fill:SetRotation(Quaternion(-45, 150, 0))

    -- Point light: orange torch ──────────────────────────────────────
    torch = CreateEntity("Torch")
    torch:AddPointLight(Color(1.0, 0.55, 0.15), 8.0)
    torch:SetPosition(-2.5, 1.0, 4.0)

    -- Camera ─────────────────────────────────────────────────────────
    CreateCamera()

    -- HUD ─────────────────────────────────────────────────────────────
    local hud = CreateLabel("Part 02: Lights and Colors  |  ESC to quit")
    hud:SetPosition(16, 16)
    hud:SetFontSize(15)
    hud:SetColor(Color(0.75, 0.80, 0.90))
end

function Update(dt)
    -- Rotate the box
    angle = angle + dt * 25
    box:SetRotation(Quaternion(0, angle, 0))

    -- Bob the torch
    bobTime = bobTime + dt
    local bob = math.sin(bobTime * 3.5) * 0.15
    torch:SetPosition(-2.5, 1.0 + bob, 4.0)

    if Input:IsKeyDown(KEY_ESCAPE) then
        Quit()
    end
end

Lua vs C++ key differences

In Lua, SetClearColor and SetAmbientColor are global functions accessible in any script context. Method calls on entities use Lua's colon syntax entity:AddDirectionalLight(...) instead of C++'s arrow entity->AddDirectionalLight(...). Keyboard detection uses the Urho3D Lua constant KEY_ESCAPE via the global Input object rather than simple-3d's C++ enum Key::Escape.

Experiments to try

Once the example is running, try these modifications to build intuition for how lighting works:

  1. Change the ambient to black (Color::BLACK) — the shadow side of the box becomes completely invisible. This shows exactly what ambient light is doing.
  2. Remove the fill light — comment out the fill entity. Notice how the shadow side now only has ambient; it looks harsher and less natural.
  3. Make the key light blue (Color(0.5f, 0.7f, 1.0f)) and the fill light orange (Color(1.0f, 0.6f, 0.2f)) — a complementary-colour lighting setup popular in sci-fi and stylized games.
  4. Increase the torch range to 25.0f and move it to (0, 5, 5) (above the box) — it becomes the dominant light source, overriding the directional lights for nearby geometry.
  5. Try Color::WHITE everywhere — all lights pure white — to see how colour tinting changes the mood even when the scene structure is identical.

API quick reference

Call Where What it does
Color(r, g, b, a) Anywhere Constructs an RGBA colour from floats in [0,1]. Alpha defaults to 1.0.
Color::WHITE Anywhere Constant for Color(1,1,1,1).
Color::BLACK Anywhere Constant for Color(0,0,0,1).
SetClearColor(Color) Game method Background colour before 3D content is drawn. Call in Start().
SetAmbientColor(Color) Game method Scene-wide base illumination. Default is Color(0.3, 0.3, 0.3).
entity->AddDirectionalLight(Color, float intensity) Entity method Parallel-ray light (no position, only direction). Uses entity rotation. No distance falloff.
entity->AddPointLight(Color, float range) Entity method Omnidirectional local light. Uses entity world position. Zero illumination beyond range.
entity->SetRotation(Quaternion(yaw, pitch, roll)) Entity method Sets world-space rotation. For lights: controls the direction of directional/spot lights.

Common mistakes

  • Color values in 0–255 range. Writing Color(255, 128, 0) passes values far above 1.0 which saturates and clips. Always use floats in [0,1]: Color(1.0f, 0.5f, 0.0f).
  • Forgetting to rotate a directional light. A directional light on an entity with default rotation points along the +Z axis, shining horizontally. Add a pitch of at least 20–30 degrees to simulate any above-ground light source, otherwise surfaces will face the light only from the side, making the scene look strange.
  • Setting point light range to zero. AddPointLight(color, 0.0f) creates a light that illuminates nothing — zero range means zero radius. Always pass a positive range value.
  • Ambient too high. Setting SetAmbientColor(Color::WHITE) or any value near 1.0 makes every surface uniformly bright regardless of lights. The scene looks completely flat. Keep ambient in the 0.05–0.25 range.
  • Calling SetClearColor with Color::TRANSPARENT. This has alpha=0, which some drivers interpret correctly and some do not. Use Color(r, g, b) (alpha defaults to 1.0) for clear colours.
  • Placing both directional lights on the same side. The fill light only helps if it comes from a different direction than the key. Aim it into the shadow regions the key creates — roughly the opposite hemisphere.

What you learned

In this part you added real-time lighting to a simple-3d scene. Here is a summary of every new concept:

  • The Color(r, g, b, a) type uses normalized floats in [0,1] for all four components. Color::WHITE and Color::BLACK are convenient constants for pure white and pure black respectively.
  • SetClearColor(Color) controls the solid colour that fills the framebuffer before any 3D rendering takes place — the visible background / sky.
  • SetAmbientColor(Color) sets the scene-wide base illumination reaching every surface regardless of light source positions. Keep it dark to preserve depth cues.
  • entity->AddDirectionalLight(Color, intensity) adds a sun-like parallel light to an entity. The entity's world rotation determines the light direction; there is no position or distance falloff.
  • Quaternion(yaw, pitch, roll) with angles in degrees is the standard way to aim a directional light. Positive pitch tilts the light downward; yaw spins it around the vertical axis.
  • entity->AddPointLight(Color, range) creates an omnidirectional local light tied to the entity's position in world space. Moving the entity moves the light.
  • The key + fill rig — a bright warm directional light plus a dimmer cool directional light from the opposite side — is a minimal, high-quality starting setup that prevents flat-looking scenes without requiring any advanced rendering features.
