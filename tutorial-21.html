Background: particles in simple-3d

A particle system describes thousands of short-lived sprites — sparks, smoke puffs, snowflakes, glowing orbs — each with its own position, velocity, colour, and lifetime. Authoring them from scratch is tedious, so simple-3d uses Urho3D's XML-based ParticleEffect format: a plain text file that specifies emitter shape, particle count, velocity curves, colour gradients, texture, and lifetime. You hand the engine the path to one of these files and call Play(); the engine handles everything else.

Continuous emitters

Produce particles on every frame until you call Stop(). Best for effects that last as long as the underlying condition is true: a campfire, smoke rising from a chimney, snow falling from the sky, or a magic shield shimmering around the player.

One-shot bursts

Emit all particles in a single short window defined by the XML activetime field, then go silent while the particles coast through their lifetimes. Use Burst() for explosions, collectible pop effects, footstep dust puffs, and enemy death reactions.

Asset layout

Particle effect XML files and their textures live under Data/Particles/ in your project tree. Make sure that path is on your resource cache by calling game.AddResourcePath("Data") in Start() (or by shipping the Data/ directory alongside your executable, which is the default for projects created with the simple-3d project template). 

Data/
  Particles/
    Sparkle.xml         // short colourful burst — collectibles, level-up
    Smoke.xml           // continuous grey smoke column
    Explosion.xml       // fiery one-shot blast
    Dust.xml            // ground-level dust cloud
    Stars.xml           // drifting star particles — idle ambient
  Textures/
    particle_sparkle.png
    particle_smoke.png
    particle_fire.png
    particle_dust.png
    particle_star.png

The built-in preset XML files reference textures relative to the same Data/ root, so shipping that whole directory is sufficient. You can also load any custom Urho3D ParticleEffect XML by passing its path to AddParticleEmitter().

Complete working example

The scene below places a stone plinth in the centre of the world. Torchlight flickers above it continuously. Pressing F ignites the torch (starts the smoke column), pressing F again extinguishes it. Pressing X detonates a one-shot explosion at the plinth position. Pressing C scatters a collectible sparkle effect. Pressing D puffs ground dust at the player's feet. The ambient star field drifts across the whole scene from a world-space entity that is not attached to anything. 

#include <Simple3D/Simple3D.h>

using namespace Simple3D;

class ParticleDemo : public Game
{
public:
    // ── Scene entities ───────────────────────────────────────────────
    Entity*  ground_        = nullptr;
    Entity*  plinth_        = nullptr;
    Entity*  player_        = nullptr;
    Camera*  cam_           = nullptr;
    Label*   hintLabel_     = nullptr;
    Label*   stateLabel_    = nullptr;

    // ── Particle emitter handles ─────────────────────────────────────
    ParticleEmitter* smokeEmitter_   = nullptr;  // torch smoke (continuous)
    ParticleEmitter* explosionFx_    = nullptr;  // one-shot blast
    ParticleEmitter* sparkleFx_      = nullptr;  // collectible sparkle
    ParticleEmitter* dustFx_         = nullptr;  // player foot-dust
    ParticleEmitter* ambientStars_   = nullptr;  // world-space star field

    // ── State flags ──────────────────────────────────────────────────
    bool torchLit_    = false;

    // ── Child entities used for position offsets ─────────────────────
    Entity* smokeAnchor_    = nullptr;  // top of the plinth
    Entity* explosionAnchor_= nullptr;  // centre of the plinth
    Entity* sparkleAnchor_  = nullptr;  // above the plinth top
    Entity* dustAnchor_     = nullptr;  // at player's feet

    // ────────────────────────────────────────────────────────────────
    void Start() override
    {
        SetWindowTitle("simple-3d — Tutorial 21: Particle Systems");

        // ── Scene geometry ───────────────────────────────────────────
        ground_ = CreateEntity("Ground");
        ground_->AddModel("Models/Plane.mdl");
        ground_->SetScale(30.0f);
        ground_->AddRigidBody(0.0f);
        ground_->AddBoxCollider(Vector3(30.0f, 0.1f, 30.0f));
        ground_->SetCollisionLayer(CollisionLayer::StaticGeometry);

        plinth_ = CreateEntity("Plinth");
        plinth_->AddModel("Models/Box.mdl");
        plinth_->SetPosition(0.0f, 0.75f, 0.0f);
        plinth_->SetScale(Vector3(1.0f, 1.5f, 1.0f));
        plinth_->AddRigidBody(0.0f);
        plinth_->AddBoxCollider(Vector3(1.0f, 1.5f, 1.0f));
        plinth_->SetCollisionLayer(CollisionLayer::StaticGeometry);

        // ── Player ───────────────────────────────────────────────────
        player_ = CreateEntity("Player");
        player_->AddModel("Models/Box.mdl");
        player_->SetPosition(4.0f, 1.0f, 4.0f);
        CharacterController* cc = player_->AddCharacterController(0.35f, 1.8f);
        cc->SetGravity(20.0f);
        player_->SetCollisionLayer(CollisionLayer::Actor);
        player_->SetCollisionMask(MakeCollisionMask({
            CollisionLayer::StaticGeometry
        }));

        // ── Lighting ─────────────────────────────────────────────────
        Entity* sun = CreateEntity("Sun");
        sun->AddDirectionalLight(Color(1.0f, 0.9f, 0.8f), 0.9f);
        sun->SetRotation(Quaternion(50.0f, 25.0f, 0.0f));
        SetAmbientColor(Color(0.2f, 0.2f, 0.25f));

        // ── Camera ───────────────────────────────────────────────────
        cam_ = CreateCamera();
        cam_->Follow(player_, 8.0f, 3.5f);
        cam_->SetSmoothFollow(true, 6.0f);

        // ── HUD ──────────────────────────────────────────────────────
        hintLabel_ = CreateLabel(
            "WASD = move  |  F = torch smoke  |  X = explosion  |  C = sparkle  |  D = foot dust  |  Esc = quit");
        hintLabel_->SetPosition(10, 10);
        hintLabel_->SetFontSize(13);
        hintLabel_->SetColor(Color::WHITE);

        stateLabel_ = CreateLabel("Torch: OFF");
        stateLabel_->SetPosition(10, 32);
        stateLabel_->SetFontSize(15);
        stateLabel_->SetColor(Color(0.6f, 0.8f, 1.0f, 1.0f));

        // ────────────────────────────────────────────────────────────
        // PARTICLE SETUP
        // ────────────────────────────────────────────────────────────

        // ── 1. Torch smoke — continuous, mounted on a child entity ────
        //
        //   The plinth origin is at its geometric centre (y = 0.75).
        //   We want smoke to rise from the TOP of the plinth (y = 1.5).
        //   Solution: create a child entity offset 0.75 m upward in
        //   local space, then attach the emitter to that child.
        //
        //   This is the standard pattern for any emitter that needs to
        //   appear at a specific part of a model rather than its origin.

        smokeAnchor_ = plinth_->CreateChild("SmokeAnchor");
        smokeAnchor_->SetLocalPosition(0.0f, 0.75f, 0.0f);  // top of plinth

        smokeEmitter_ = smokeAnchor_->AddParticleEmitter("Particles/Smoke.xml");
        // Do NOT call Play() yet — torch starts extinguished.

        // ── 2. Explosion — one-shot, same position as torch ───────────
        //
        //   The explosion should appear at the plinth's geometric centre,
        //   not the top.  We use a separate child entity at the origin.
        //   Calling Burst() resets the emitter and runs one emission cycle.

        explosionAnchor_ = plinth_->CreateChild("ExplosionAnchor");
        explosionAnchor_->SetLocalPosition(0.0f, 0.0f, 0.0f);  // plinth centre

        explosionFx_ = explosionAnchor_->AddParticleEmitter("Particles/Explosion.xml");
        // No Play() — remains dormant until the player presses X.

        // ── 3. Sparkle — one-shot, above the plinth top ───────────────

        sparkleAnchor_ = plinth_->CreateChild("SparkleAnchor");
        sparkleAnchor_->SetLocalPosition(0.0f, 1.2f, 0.0f);  // well above top

        sparkleFx_ = sparkleAnchor_->AddParticleEmitter("Particles/Sparkle.xml");

        // ── 4. Foot dust — continuous while player moves ──────────────
        //
        //   This emitter moves WITH the player.  We attach it to a child
        //   entity at the player's feet (y = 0 in local space = ground level
        //   relative to the player's origin at y = 1.0).

        dustAnchor_ = player_->CreateChild("DustAnchor");
        dustAnchor_->SetLocalPosition(0.0f, -1.0f, 0.0f);  // at ground

        dustFx_ = dustAnchor_->AddParticleEmitter("Particles/Dust.xml");
        // Play() / Stop() controlled by movement in Update().

        // ── 5. Ambient star field — world-space, NOT attached to anything
        //
        //   We create a standalone entity at the scene's centre (x=0, y=5, z=0)
        //   and start the emitter immediately.  Because this entity has no
        //   parent and never moves, the stars drift through world-space
        //   coordinates, unaffected by player movement.

        Entity* starEntity = CreateEntity("AmbientStars");
        starEntity->SetPosition(0.0f, 5.0f, 0.0f);

        ambientStars_ = starEntity->AddParticleEmitter("Particles/Stars.xml");
        ambientStars_->Play();   // start immediately; will run for the whole session
    }

    // ────────────────────────────────────────────────────────────────
    void Update(float dt) override
    {
        HandleMovement(dt);
        HandleTorchToggle();
        HandleExplosion();
        HandleSparkle();
        HandleDustEmission();

        if (IsKeyPressed(Key::Escape))
            Quit();
    }

    // ────────────────────────────────────────────────────────────────
    void HandleMovement(float dt)
    {
        CharacterController* cc = player_->GetComponent<CharacterController>();
        if (!cc) return;

        float mx = 0.0f, mz = 0.0f;
        if (IsKeyDown(Key::A) || IsKeyDown(Key::Left))  mx -= 1.0f;
        if (IsKeyDown(Key::D) || IsKeyDown(Key::Right)) mx += 1.0f;
        if (IsKeyDown(Key::W) || IsKeyDown(Key::Up))    mz -= 1.0f;
        if (IsKeyDown(Key::S) || IsKeyDown(Key::Down))  mz += 1.0f;

        cc->Move(Vector3(mx, 0.0f, mz) * 5.0f, dt);

        if (IsKeyPressed(Key::Space))
            cc->TryJump(10.0f);
    }

    // ────────────────────────────────────────────────────────────────
    // F — toggle the torch smoke emitter on/off.
    // Demonstrates Play() / Stop() for a continuous emitter.
    void HandleTorchToggle()
    {
        if (!IsKeyPressed(Key::F)) return;

        if (!torchLit_)
        {
            // Light the torch: start continuous smoke emission.
            smokeEmitter_->Play();
            torchLit_ = true;
            stateLabel_->SetText("Torch: ON  (smoke rising)");
        }
        else
        {
            // Extinguish the torch: stop new particles being spawned.
            // Particles already in flight linger until their own lifetime
            // expires — this is intentional and looks natural.
            smokeEmitter_->Stop();
            torchLit_ = false;
            stateLabel_->SetText("Torch: OFF");
        }
    }

    // ────────────────────────────────────────────────────────────────
    // X — trigger a one-shot explosion burst at the plinth centre.
    // Demonstrates Burst() and IsEmitting() for a one-shot emitter.
    void HandleExplosion()
    {
        if (!IsKeyPressed(Key::X)) return;

        // Guard: don't trigger while the previous burst is still alive.
        // IsEmitting() returns true during the short emission window that
        // Burst() opens; it flips back to false once that window closes.
        if (explosionFx_->IsEmitting())
            return;

        // Burst() resets all live particles, then enables the emitter for
        // exactly one activetime cycle as defined in the XML.  After that
        // cycle the emitter stops on its own — you don't need to call Stop().
        explosionFx_->Burst();

        // Optional: add a screen shake to sell the impact.
        cam_->Shake(0.4f, 0.5f);

        stateLabel_->SetText("BOOM!");
    }

    // ────────────────────────────────────────────────────────────────
    // C — trigger a collectible sparkle burst above the plinth top.
    void HandleSparkle()
    {
        if (!IsKeyPressed(Key::C)) return;

        // We can call Burst() any number of times; each call restarts
        // the one-shot cycle from the beginning even if the previous
        // burst hasn't fully faded yet.
        sparkleFx_->Burst();
        stateLabel_->SetText("Sparkle!");
    }

    // ────────────────────────────────────────────────────────────────
    // Dust — continuous while the player is moving on the ground.
    // Demonstrates toggling an emitter based on game state without
    // using Burst(): Play() / Stop() on the same emitter per frame.
    void HandleDustEmission()
    {
        CharacterController* cc = player_->GetComponent<CharacterController>();
        if (!cc) return;

        bool shouldEmit = cc->IsOnGround() &&
                          (IsKeyDown(Key::A) || IsKeyDown(Key::D) ||
                           IsKeyDown(Key::W) || IsKeyDown(Key::S) ||
                           IsKeyDown(Key::Left) || IsKeyDown(Key::Right) ||
                           IsKeyDown(Key::Up)   || IsKeyDown(Key::Down));

        // Only call Play()/Stop() when the state changes — avoids resetting
        // the emitter every frame, which would stutter the particle output.
        if (shouldEmit && !dustFx_->IsEmitting())
            dustFx_->Play();
        else if (!shouldEmit && dustFx_->IsEmitting())
            dustFx_->Stop();
    }
};

int main()
{
    return Simple3D::CreateGame<ParticleDemo>()->Run();
}

Step-by-step explanation

entity->AddParticleEmitter(xmlPath)

This is the entry point for the whole particle system. Call it on any entity and pass the resource-relative path to a Urho3D ParticleEffect XML file. The engine loads the XML, creates a ParticleEmitter component on the entity's scene node, and returns a pointer to a thin C++ wrapper around it.

The emitter starts in the stopped state. It will not produce any particles until you call Play() or Burst(). This means it is safe to set up all your emitters unconditionally in Start() and activate them only when the game logic requires it. 

// Basic attachment — emitter is dormant until activated
ParticleEmitter* smoke = entity->AddParticleEmitter("Particles/Smoke.xml");

// Custom effect from your own Data/Effects/ subdirectory
ParticleEmitter* magic = entity->AddParticleEmitter("Effects/MagicAura.xml");

Each call to AddParticleEmitter adds a new emitter to the entity. An entity can carry multiple emitters simultaneously — for example, a vehicle could have separate exhaust smoke, tyre smoke, and spark emitters. Use GetParticleEmitter() to retrieve the most-recently-added one when you need it later.

emitter->Play() and emitter->Stop()

Play() enables continuous particle emission. New particles are spawned at the rate defined by the XML (typically a spawn-per-second or spawn-per-second range) until you call Stop(). Particles that are already in flight when you call Stop() do not vanish instantly; they continue coasting through their own remaining lifetime. This looks natural for smoke, fire, and dust: the trail tapers off rather than cutting abruptly. 

// Start a continuous smoke plume when the torch is lit
smokeEmitter_->Play();

// Later, when the torch is doused, stop spawning new particles.
// Existing smoke puffs drift away naturally over their lifetimes.
smokeEmitter_->Stop();

The recommended pattern for conditional continuous effects (like foot dust that emits only while moving) is to check IsEmitting() before calling Play() or Stop(), so you do not reset the emitter's internal state on every frame. 

bool moving = /* ... */;

if (moving && !dustFx_->IsEmitting())
    dustFx_->Play();
else if (!moving && dustFx_->IsEmitting())
    dustFx_->Stop();

emitter->Burst()

Burst() is the right call for any effect that should play once and then go silent: an explosion, a collectible pickup shimmer, an impact puff, or an ability cast. Internally it does three things in order:

  1. Removes all currently live particles from the emitter (resets to clean state).
  2. Sets the emitter's internal timer back to zero.
  3. Enables the emitter for exactly one activetime cycle as specified in the XML — typically a fraction of a second. After that window closes, the emitter stops on its own.

You do not need to call Stop() after a burst. Because the emitter is self-limiting you can call Burst() again at any time to restart the one-shot cycle from scratch, even while the previous burst's particles are still decaying. 

// Enemy dies — detonate an explosion at its position
ParticleEmitter* fx = enemy->GetParticleEmitter();
if (fx) fx->Burst();

// Player collects a coin — sparkle pop above the coin entity
coinEmitter_->Burst();

// Landing dust — burst when the character touches the ground
if (IsKeyPressed(Key::Space) && cc->IsOnGround()) {
    cc->TryJump(10.0f);
    dustFx_->Burst();   // jump-off dust puff
}

emitter->IsEmitting() and emitter->SetEmitting(bool)

IsEmitting() returns true during the window in which the emitter is actively producing new particles. For continuous emitters started with Play() this stays true until Stop() is called. For burst emitters it is true only during the short emission window opened by Burst().

SetEmitting(bool) is the low-level toggle that both Play() and Stop() delegate to internally. Use it directly only when you need to synchronise the emitter state with an external flag without the overhead of the play/stop guard logic. 

// Poll-guard to avoid re-triggering an explosion mid-burst
if (IsKeyPressed(Key::X) && !explosionFx_->IsEmitting())
    explosionFx_->Burst();

// Raw toggle — equivalent to Play() / Stop() without reset logic
emitter->SetEmitting(true);
emitter->SetEmitting(false);

entity->GetParticleEmitter()

GetParticleEmitter() retrieves the emitter component that was most recently added via AddParticleEmitter(). It returns nullptr if no emitter has been added. This is useful when the entity is created in a prefab loader or a Lua script and your C++ code receives the entity by name: 

// Entity was created somewhere else; retrieve and fire its emitter
Entity* boom = FindEntity("ExplosionPoint");
if (boom) {
    ParticleEmitter* fx = boom->GetParticleEmitter();
    if (fx) fx->Burst();
}

Attaching emitters to child entities for position offsets

An emitter added to an entity always spawns particles at that entity's world-space origin. When you want the effect to appear at a specific part of a model — the tip of a gun barrel, the top of a chimney, or a character's hands — use a child entity positioned at the right local-space offset, then add the emitter to the child. Because the child inherits its parent's transform, moving or rotating the parent automatically repositions the effect. 

// Emitter at the top of a pillar model (pillar origin at centre, height 3 m)
Entity* pillar = CreateEntity("Pillar");
pillar->AddModel("Models/Pillar.mdl");
pillar->SetPosition(0.0f, 1.5f, 0.0f);   // pillar sits on the ground

Entity* flameAnchor = pillar->CreateChild("FlameAnchor");
flameAnchor->SetLocalPosition(0.0f, 1.5f, 0.0f);  // top of 3 m pillar

ParticleEmitter* flame = flameAnchor->AddParticleEmitter("Particles/Smoke.xml");
flame->Play();

// Emitter above an enemy's head — floats with the enemy as it moves
Entity* statusAnchor = enemy_->CreateChild("StatusFX");
statusAnchor->SetLocalPosition(0.0f, 2.5f, 0.0f);
ParticleEmitter* stars = statusAnchor->AddParticleEmitter("Particles/Stars.xml");
stars->Play();   // play while stunned

World-space particle effects

For ambient effects that fill the scene rather than following any specific object — falling rain, drifting snow, floating embers — create a standalone entity, place it at a fixed world position, and never parent it to anything. Its transform stays constant in world space regardless of where the player or camera moves. 

// Ambient rain at height 10 m, centred on the level origin
Entity* rain = CreateEntity("AmbientRain");
rain->SetPosition(0.0f, 10.0f, 0.0f);

ParticleEmitter* rainFx = rain->AddParticleEmitter("Particles/Dust.xml");
rainFx->Play();

// Ambient star-field at a comfortable height above the play area
Entity* stars = CreateEntity("StarField");
stars->SetPosition(0.0f, 5.0f, 0.0f);
stars->AddParticleEmitter("Particles/Stars.xml")->Play();

// Kill world-space effects before a scene transition so they don't
// carry over into the next level.
rain->SetActive(false);    // hides + suspends rendering; reactivate later
// OR destroy the entity permanently:
// DestroyEntity(rain);  rain = nullptr;

ParticleEmitter quick reference

Method Returns Notes
entity->AddParticleEmitter(xmlPath) ParticleEmitter* Attaches emitter from XML. Starts stopped. Idempotent for unique paths.
entity->GetParticleEmitter() ParticleEmitter* Returns most-recently-added emitter, or nullptr.
emitter->Play() — Start continuous emission. Stays on until Stop().
emitter->Stop() — Stop spawning. Live particles linger until their lifetime ends.
emitter->Burst() — Reset + enable for one activetime cycle. Self-stops.
emitter->IsEmitting() bool True while actively spawning. Use to guard re-trigger logic.
emitter->SetEmitting(bool) — Low-level toggle; skips the reset logic inside Burst().

Lua equivalent

In Lua, you set up emitters from C++ (or a scene loader) and retrieve them with entity:GetParticleEmitter(). All playback methods are available on the returned handle. The example below shows a self-contained Lua script that manages a torch effect and a death-burst on an entity: 

-- Scripts/torch_fx.lua
-- Attach this script to the torch entity.
-- Assumes the emitter was already added in C++ Start() as:
--   torchEntity->AddParticleEmitter("Particles/Smoke.xml");

local smokeEmitter = nil
local torchLit     = false

function Start()
    -- Retrieve the emitter component already added in C++.
    smokeEmitter = entity:GetParticleEmitter()
    if smokeEmitter == nil then
        print("[torch_fx] No particle emitter found on entity!")
    end
end

function Update(dt)
    if smokeEmitter == nil then return end

    -- F key toggles the torch on and off
    if game:IsKeyPressed(Key.F) then
        if not torchLit then
            smokeEmitter:Play()
            torchLit = true
        else
            smokeEmitter:Stop()
            torchLit = false
        end
    end
end
-- Scripts/enemy_death.lua
-- Trigger an explosion burst when the enemy dies.
-- The explosion emitter was added in C++:
--   enemy->AddParticleEmitter("Particles/Explosion.xml");

local explodeFx = nil
local hp        = 100

function Start()
    explodeFx = entity:GetParticleEmitter()
end

function TakeDamage(amount)
    hp = hp - amount
    if hp <= 0 then
        if explodeFx and not explodeFx:IsEmitting() then
            explodeFx:Burst()   -- one-shot death blast
        end
        entity:SetActive(false)   -- hide the enemy model
    end
end
-- Scripts/ambient_stars.lua
-- World-space ambient emitter — attach to a dedicated entity at
-- a fixed position.  The emitter was added in C++:
--   starEntity->AddParticleEmitter("Particles/Stars.xml");

local starFx = nil

function Start()
    starFx = entity:GetParticleEmitter()
    if starFx then
        starFx:Play()   -- begin continuous ambient emission
    end
end

-- SetEmitting example: pause stars during cutscene
function PauseAmbient()
    if starFx then starFx:SetEmitting(false) end
end

function ResumeAmbient()
    if starFx then starFx:SetEmitting(true) end
end

Patterns and tips

Death effect + entity removal

A common problem is that you want an explosion to play when an enemy dies, but if you call DestroyEntity() immediately the emitter is also destroyed and no particles appear. The cleanest solution is to hide the enemy model at once but defer the actual entity destruction until the burst has had time to play: 

void KillEnemy(Entity* e)
{
    // Stop the model from rendering immediately
    e->SetActive(false);

    // Detach the explosion anchor from the enemy so it stays
    // in world space even after the parent is deactivated.
    Entity* fxAnchor = e->GetChild(0);  // or store the pointer
    if (fxAnchor) {
        // Reparent to scene root: keep world position, cut parent link
        Vector3 worldPos = fxAnchor->GetPosition();
        fxAnchor->SetPosition(worldPos);
        // Now trigger the burst on the now-independent node
        ParticleEmitter* fx = fxAnchor->GetParticleEmitter();
        if (fx) fx->Burst();
    }

    // Schedule destruction for later (e.g. after 2 seconds using a tween counter)
    deathTimer_ = 2.0f;
}

Combining emitters on one entity

A single entity can carry multiple particle emitters — each AddParticleEmitter call appends a new one. This is useful for layered effects, such as a campfire that has both a flame emitter and a smoke emitter, or a magic weapon with a sparkle overlay on top of a base glow: 

// Campfire with both flame and smoke
Entity* fire = CreateEntity("Campfire");
fire->SetPosition(3.0f, 0.0f, -2.0f);

// Add point light at the same spot for warm glow
fire->AddPointLight(Color(1.0f, 0.5f, 0.1f), 6.0f);

// Emitter 1 — flame sparks (Sparkle preset)
// Child anchor at origin of the fire entity
Entity* flameAnchor = fire->CreateChild("FlameLayer");
flameAnchor->SetLocalPosition(0.0f, 0.0f, 0.0f);
ParticleEmitter* flameFx = flameAnchor->AddParticleEmitter("Particles/Sparkle.xml");
flameFx->Play();

// Emitter 2 — smoke above the flames
Entity* smokeAnchor = fire->CreateChild("SmokeLayer");
smokeAnchor->SetLocalPosition(0.0f, 0.5f, 0.0f);
ParticleEmitter* smokeFx = smokeAnchor->AddParticleEmitter("Particles/Smoke.xml");
smokeFx->Play();

Pausing world-space effects during cutscenes

Use SetEmitting(false) rather than Stop() when you want to freeze an ambient effect exactly as it is (particles currently in flight stop updating), then resume later with SetEmitting(true). Alternatively, use SetTimeScale(0.0f) to freeze everything including particles. 

// Freeze all particles without destroying them
SetTimeScale(0.0f);   // pauses Update(), physics, and particle simulation

// Restore
SetTimeScale(1.0f);

Common mistakes

  • Calling Play() every frame instead of once. Calling Play() on an already-playing emitter has no visible effect most of the time, but it does reset some internal bookkeeping. Always guard the call with if (!emitter->IsEmitting()) emitter->Play(); when the desired state changes from the current state, not on every tick.
  • Expecting Burst() to add to an ongoing emission. Burst() first clears all live particles, then starts a fresh one-shot cycle. If you call Burst() while a smoke emitter is in the middle of a puff cloud, you will see the cloud vanish and a new one appear. For additive one-shots (without clearing), use a separate emitter on a child entity instead.
  • Adding the emitter directly to a moving entity when an offset is needed. If the emitter origin and the entity origin don't coincide (barrel tip, chimney top, hand socket), always use a child entity with a local-space offset. Without this the effect appears at the model's pivot, which is usually buried in the geometry.
  • Destroying the entity immediately after Burst(). The emitter and all its live particles are destroyed along with the entity. Either delay destruction with a timer (a simple float countdown in Update), or detach the effect node to the scene root before calling DestroyEntity().
  • Forgetting to add the Data/Particles/ path to the resource cache. If you move your executable or ship without the Data/ directory at the expected relative path, AddParticleEmitter() will log a "Resource not found" warning and return a null-emitter that no-ops silently. Enable resource diagnostics in debug builds with SetResourceDiagnosticsEnabled(true) to catch this early.
  • Using a world-space emitter entity as a parent for game objects. An ambient star field entity is fine as a standalone root entity, but do not parent your player or enemies to it — those children would need to counter-transform to stay in place, which is both fragile and confusing. World-space emitter entities should have no children other than effect anchors.
  • Trying to use IsEmitting() to detect whether coasting particles are still visible. IsEmitting() only reports whether the emitter is spawning new particles, not whether any previously spawned particles are still alive. After Stop() or after a burst's emission window closes, IsEmitting() returns false even if hundreds of particles are still decaying across the screen.

What you learned

In this part you gained full control over simple-3d's particle emitter API:

  • Attachment: AddParticleEmitter(xmlPath) attaches a Urho3D ParticleEffect XML to any entity and returns a handle. The emitter starts dormant.
  • Continuous effects: Play() starts spawning particles indefinitely; Stop() halts new spawns while existing particles coast out naturally. Guard both calls with IsEmitting() to avoid resetting state on every frame.
  • One-shot bursts: Burst() clears live particles, resets the internal timer, and opens a single emission window defined by the XML's activetime. The emitter self-stops after that window — no manual Stop() required.
  • State queries: IsEmitting() tells you whether the emitter is in its spawn window; SetEmitting(bool) is the raw toggle behind Play() and Stop().
  • Retrieval: GetParticleEmitter() fetches an emitter that was set up elsewhere (a prefab, a Lua script, or an earlier C++ call). Returns nullptr if none was added.
  • Built-in presets: Five ready-to-use effects ship with simple-3d under Data/Particles/: Sparkle (collectibles), Smoke (continuous columns), Explosion (one-shot blasts), Dust (ground impacts), and Stars (ambient drift).
  • Position offsets via child entities: Create a child entity with CreateChild(), set its SetLocalPosition() to the desired offset, and add the emitter there. The effect then inherits and follows the parent's world transform automatically.
  • World-space effects: Create a standalone entity at a fixed world position (no parent) and call Play() for ambient effects like rain, snow, or a star field. Disable with SetActive(false) or SetEmitting(false) during transitions or cutscenes.
