Overview

This tutorial builds the simplest meaningful simple-3d application: a single box that spins continuously in the centre of the screen, lit by a directional light, viewed through a stationary perspective camera, with a short HUD text label in the top-left corner. Pressing Escape exits the program.

By the end you will have working code that exercises all twelve foundational API calls. Every subsequent tutorial in this series builds directly on the patterns established here, so it is worth reading this one carefully even if the result looks simple.

Complete source

The listing below is the complete, self-contained program. Copy it into main.cpp, adjust the resource path in main(), and build. The step-by-step explanation that follows breaks every section down in detail. 

// main.cpp — Part 01: Hello World
#include <Simple3D/Core/Game.h>
#include <Simple3D/Entity/Entity.h>
#include <Simple3D/Camera/Camera.h>
#include <Simple3D/UI/Label.h>

using namespace Simple3D;

// ─────────────────────────────────────────────────────────────────────────────
// HelloWorld — the application class
// Every simple-3d program is a class that inherits from Simple3D::Game.
// ─────────────────────────────────────────────────────────────────────────────
class HelloWorld : public Game {
public:

    // ── Start() ──────────────────────────────────────────────────────────────
    // Called once by the engine after the window and renderer are ready.
    // Use it to build your scene: create entities, set up cameras, load assets.
    // ─────────────────────────────────────────────────────────────────────────
    void Start() override {

        // ── 1. Window and viewport ───────────────────────────────────────────
        SetWindowTitle("Hello World — simple-3d");
        SetClearColor(Color(0.08f, 0.08f, 0.15f, 1.0f));   // deep navy background

        // ── 2. Directional light ("the sun") ─────────────────────────────────
        // Lights are entities too. Rotation determines ray direction.
        // A Quaternion(pitch, yaw, roll) with pitch=45, yaw=45 points light
        // downward and to the right — standard three-quarter overhead sun angle.
        Entity* sun = CreateEntity("Sun");
        sun->AddDirectionalLight(Color(1.0f, 0.95f, 0.8f), 1.2f);
        sun->SetRotation(Quaternion(45.0f, 45.0f, 0.0f));

        // ── 3. Rotating box ──────────────────────────────────────────────────
        // CreateEntity() registers the object with the scene and returns a pointer.
        // AddModel() loads a .mdl file relative to a Data/ directory.
        // Box.mdl is bundled with every Urho3D / U3D installation.
        box_ = CreateEntity("Box");
        box_->AddModel("Models/Box.mdl");
        box_->SetScale(1.5f);                  // make it a bit larger than default
        box_->SetPosition(0.0f, 0.0f, 0.0f);   // place at world origin

        // ── 4. Camera ────────────────────────────────────────────────────────
        // CreateCamera() creates the primary viewport camera.
        // We position it behind and slightly above the box, then aim it at (0,0,0).
        cam_ = CreateCamera();
        cam_->SetPosition(0.0f, 2.0f, -6.0f);
        cam_->LookAt(Vector3::ZERO);
        cam_->SetFOV(60.0f);

        // ── 5. HUD label ─────────────────────────────────────────────────────
        // CreateLabel() creates an on-screen text element in pixel coordinates.
        // Position (16, 16) places it 16 px from the top-left corner.
        label_ = CreateLabel("Hello, simple-3d!");
        label_->SetPosition(16, 16);
        label_->SetFontSize(22);
        label_->SetColor(Color::WHITE);
    }

    // ── Update(dt) ───────────────────────────────────────────────────────────
    // Called every frame. dt is the elapsed time in seconds since the last frame.
    // Always multiply motion by dt to keep animation framerate-independent.
    // ─────────────────────────────────────────────────────────────────────────
    void Update(float dt) override {

        // Accumulate rotation angles (degrees per second × elapsed seconds)
        yaw_   += 40.0f * dt;
        pitch_ += 25.0f * dt;
        roll_  += 15.0f * dt;

        // Apply the accumulated rotation to the box each frame.
        // Quaternion(pitch, yaw, roll) converts Euler angles to a rotation.
        box_->SetRotation(Quaternion(pitch_, yaw_, roll_));

        // IsKeyPressed() is true only on the FIRST frame the key goes down.
        // Use it for one-shot actions (jump, shoot, quit).
        // Use IsKeyDown() when you want a held-key behaviour (movement).
        if (IsKeyPressed(Key::Escape)) Quit();
    }

private:
    Entity* box_   = nullptr;   // the spinning box
    Camera* cam_   = nullptr;   // primary viewport camera
    Label*  label_ = nullptr;   // HUD text (not animated in this part)

    float yaw_   = 0.0f;
    float pitch_ = 0.0f;
    float roll_  = 0.0f;
};

// ── main() ────────────────────────────────────────────────────────────────────
// CreateGame<T>() instantiates your game class and hands it to the engine.
// SetResourcePrefixPaths() tells the engine where to find Data/ and CoreData/.
// Run() starts the game loop and returns an exit code when the loop ends.
// ─────────────────────────────────────────────────────────────────────────────
int main(int, char**) {
    auto game = Simple3D::CreateGame<HelloWorld>();
    game->SetResourcePrefixPaths("/path/to/u3d/bin");  // adjust to your install
    return game->Run();
}

Step-by-step explanation

1. Subclassing Game

Every simple-3d program starts with a class that inherits publicly from Simple3D::Game: 

class HelloWorld : public Game { ... };

Game is the top-level object. It owns the engine context, window, renderer, scene, physics world, audio system, and input manager. You never create any of those directly — the base class manages all of it. Your subclass only overrides the lifecycle callbacks to inject game-specific behaviour.

You launch your game by calling Simple3D::CreateGame<HelloWorld>() in main(). This template function allocates your subclass and returns a raw pointer to its Game base. Calling Run() on that pointer initialises the engine, opens the window, and enters the main loop. When Run() returns, the loop has ended (either via Quit() or the user closing the window) and the return value is the process exit code.

2. The Start() and Update() lifecycle

Simple3D::Game has three overridable lifecycle callbacks:

MethodCalled whenTypical use
void Start() Once, after the engine and window are fully ready, before the first rendered frame Create entities, load models, set up camera, configure HUD
void Update(float dt) Once per rendered frame, in the game logic thread Move entities, read input, update HUD text, run AI
void Stop() Once, just before the engine shuts down Save game data, release any external resources

The dt parameter in Update is the delta time: the number of seconds that elapsed since the previous call. On a 60 fps monitor dt is approximately 0.01667; on a 30 fps machine it is approximately 0.03333. Multiplying any speed or rate by dt makes your animation framerate-independent — a box that rotates 40.0f * dt degrees per call turns at exactly 40 degrees per second regardless of how fast or slow the hardware is.

You never call these methods yourself; the engine calls them automatically at the right moment.

3. SetWindowTitle() and SetClearColor()

SetWindowTitle("Hello World — simple-3d");
SetClearColor(Color(0.08f, 0.08f, 0.15f, 1.0f));

SetWindowTitle() sets the text shown in the OS title bar. It is safe to call both in Start() and from within Update() at runtime — for example to display the current score or frame rate during development.

SetClearColor() sets the background colour that fills the viewport on each frame before anything is drawn. It takes a Simple3D::Color value whose four components (red, green, blue, alpha) each range from 0.0f to 1.0f. The values (0.08, 0.08, 0.15, 1.0) produce a deep dark navy that makes the box stand out clearly. The alpha component must be 1.0f for an opaque background.

Both calls belong at the top of Start() so the correct colour appears on the very first frame.

4. CreateEntity()

Entity* sun = CreateEntity("Sun");
Entity* box = CreateEntity("Box");

CreateEntity(name) is the primary factory method for scene objects. It allocates a new Entity, registers it with the active scene, and returns a raw (non-owning) pointer. The engine owns the entity's lifetime; you should never call delete on the returned pointer. To remove an entity from the scene use DestroyEntity(ptr) instead (introduced in a later tutorial).

The name string is for debugging and for looking up entities by name later with FindEntity("Sun"). Names do not have to be unique, although giving them unique names makes debugging easier.

A fresh entity starts at the world origin with no rotation, unit scale, and no components attached. You add behaviour by calling methods on the returned pointer.

5. AddModel()

box_->AddModel("Models/Box.mdl");

AddModel(path) loads a 3D model from a resource file and attaches it to the entity as a StaticModel component. The path is relative to a Data/ directory that the engine locates beneath the resource prefix paths you configure (see the note on SetResourcePrefixPaths() below).

Box.mdl is a built-in unit cube that ships with every Urho3D / U3D installation. It lives at <install>/bin/Data/Models/Box.mdl. The model loads with the default material (white unlit). Later tutorials show how to apply textures and custom materials.

Models can also be loaded from other common formats (FBX, OBJ) depending on your build configuration. Always store models under a Models/ subfolder inside Data/ to keep resources organised.

6. SetPosition() and SetRotation()

box_->SetPosition(0.0f, 0.0f, 0.0f);
box_->SetRotation(Quaternion(pitch_, yaw_, roll_));
sun->SetRotation(Quaternion(45.0f, 45.0f, 0.0f));

Both methods operate in world space — absolute coordinates relative to the scene origin, not relative to any parent object.

SetPosition(x, y, z) accepts three floats or a Vector3. In simple-3d's coordinate system Y is up, X is right, and Z is into the screen (right-handed, Y-up). Placing the box at (0, 0, 0) and the camera at (0, 2, -6) puts the camera 2 units above and 6 units in front of the box.

SetRotation() takes a Quaternion. The most convenient constructor is Quaternion(pitch, yaw, roll), which converts Euler angles (in degrees) to the internal quaternion representation. In this tutorial pitch is rotation around X (nose up/down), yaw is rotation around Y (left/right), and roll is rotation around Z (banking). Each angle is accumulated over time in Update() and re-applied every frame to create the spinning animation.

For the directional light, the rotation determines which direction the parallel rays travel. Quaternion(45, 45, 0) tilts the light 45 degrees downward and 45 degrees to the right, producing a classic top-right three-quarter sun angle that lights the front and top faces of the box.

7. CreateCamera()

cam_ = CreateCamera();
cam_->SetPosition(0.0f, 2.0f, -6.0f);
cam_->LookAt(Vector3::ZERO);
cam_->SetFOV(60.0f);

CreateCamera() creates and returns a Camera object that represents the player's viewpoint. simple-3d automatically assigns this camera to the full-screen viewport. There can be more than one camera in advanced setups (split-screen, mirrors), but for now one is enough.

After creation the camera is positioned at the world origin with no rotation. We move it to (0, 2, -6) — 2 units up and 6 units in front of the box (remember: Z negative is toward the camera when using a right-handed Y-up system). LookAt(Vector3::ZERO) rotates the camera so it faces the world origin where the box sits.

SetFOV(60.0f) sets the vertical field of view to 60 degrees. This is a comfortable value for desktop games. First-person shooters typically use 90 degrees or more; cinematic cutscenes may go as low as 30 degrees.

Because we do not call cam_->Follow(), cam_->SetFPSMode(), or cam_->SetOrbitMode(), the camera stays exactly where we put it. Later tutorials add follow and orbit behaviour.

8. CreateLabel()

label_ = CreateLabel("Hello, simple-3d!");
label_->SetPosition(16, 16);
label_->SetFontSize(22);
label_->SetColor(Color::WHITE);

CreateLabel(text) creates a 2D text element in screen space (the HUD layer) and returns a Label*. Unlike entities, labels live in the UI layer and are always drawn on top of the 3D scene.

Position coordinates for labels are in pixels, measured from the top-left corner of the window. SetPosition(16, 16) places the label's top-left anchor 16 pixels from each edge.

SetFontSize(22) sets the text size in points. The default font is Fonts/Anonymous Pro.ttf, which ships with the engine. You can call label_->SetFont("Fonts/MyFont.ttf") to use a different font file.

SetColor(Color::WHITE) sets the text colour. Color::WHITE is the named constant Color(1, 1, 1, 1). You can use any RGBA colour. Labels support full opacity control through the alpha component.

To update the label text at runtime (for a score display, for example) call label_->SetText("Score: " + std::to_string(score_)) from within Update().

9. IsKeyPressed() and Quit()

if (IsKeyPressed(Key::Escape)) Quit();

simple-3d provides two ways to check keyboard state:

MethodReturns true whenTypical use
IsKeyPressed(Key k) Only on the first frame the key transitions from up to down One-shot actions: jump, shoot, open menu, quit
IsKeyDown(Key k) Every frame while the key is held down Continuous actions: movement, camera pan, charge-up

For quitting we use IsKeyPressed — we want the quit to fire exactly once the moment Escape is pressed, not every frame it remains held. The Key enum lists all keyboard keys: Key::Escape, Key::Space, Key::W, Key::Left, Key::F1, and so on (see the C++ API reference for the full list).

Quit() signals the engine to exit the main loop cleanly. The loop finishes the current frame, calls Stop() on your game class, shuts down all subsystems, and then returns from Run().

10. Resource paths

game->SetResourcePrefixPaths("/path/to/u3d/bin");

The engine searches for assets (models, textures, fonts, shaders) inside Data/ and CoreData/ subdirectories. SetResourcePrefixPaths() tells the engine where to look for those subdirectories.

In a U3D installation these directories live inside the bin/ folder of the build tree, for example /home/user/u3d/build/bin/Data/ and /home/user/u3d/build/bin/CoreData/. Pass the bin/ path as the prefix and the engine discovers both automatically.

You can pass multiple semicolon-separated paths if assets are spread across several directories. During development it is common to add your own project's GameData/ folder alongside the engine's built-in data directories.

Tip — relative paths in shipped games

When packaging a release build, place Data/ and CoreData/ next to your executable and call game->SetResourcePrefixPaths(".") so the engine always resolves paths relative to the executable's directory. This makes the package relocatable.

Delta time in depth

The spinning animation in Update() deserves a closer look: 

yaw_   += 40.0f * dt;   // 40 degrees per second around Y
pitch_ += 25.0f * dt;   // 25 degrees per second around X
roll_  += 15.0f * dt;   // 15 degrees per second around Z
box_->SetRotation(Quaternion(pitch_, yaw_, roll_));

The three angle accumulators are private member variables initialised to zero. Each frame the engine calls Update(dt) and we add a small increment proportional to the time that passed since the last frame. After 1 second the yaw accumulates to exactly 40 degrees. After 9 full seconds it completes exactly one yaw revolution (360 degrees).

If you had written yaw_ += 40.0f without the * dt the animation speed would be tied directly to the frame rate. On a 60 fps display the box would rotate 40 * 60 = 2400 degrees per second. On a 30 fps display it would rotate only 1200 degrees per second — half as fast. Multiplying by dt eliminates this dependency.

Different rotation speeds for yaw, pitch, and roll (40, 25, 15 degrees per second) create a pleasing tumbling motion that never looks exactly periodic. You can choose any combination of values you like.

Lua equivalent

The same behaviour can be written entirely in Lua. simple-3d supports per-entity Lua scripts (covered in depth in Part 06), but it also supports a game-level Lua script that maps directly to Start() and Update(). The script below is the Lua equivalent of the C++ program above.

Lua note

In Lua scripts the Game object is accessed through the global game reference. Entity and camera creation methods are called on game. Input methods are called on the global Input object. The Color, Vector3, and Quaternion types are available as global constructors. 

-- game.lua — Lua equivalent of the Hello World C++ program

local box
local cam
local label
local yaw   = 0
local pitch = 0
local roll  = 0

function Start()
    -- Window and background
    game:SetWindowTitle("Hello World — simple-3d")
    game:SetClearColor(Color(0.08, 0.08, 0.15, 1.0))

    -- Directional light
    local sun = game:CreateEntity("Sun")
    sun:AddDirectionalLight(Color(1.0, 0.95, 0.8), 1.2)
    sun:SetRotation(Quaternion(45, 45, 0))

    -- Rotating box
    box = game:CreateEntity("Box")
    box:AddModel("Models/Box.mdl")
    box:SetScale(1.5)
    box:SetPosition(0, 0, 0)

    -- Camera
    cam = game:CreateCamera()
    cam:SetPosition(0, 2, -6)
    cam:LookAt(Vector3(0, 0, 0))
    cam:SetFOV(60)

    -- HUD label
    label = game:CreateLabel("Hello, simple-3d!")
    label:SetPosition(16, 16)
    label:SetFontSize(22)
    label:SetColor(Color.WHITE)
end

function Update(dt)
    yaw   = yaw   + 40 * dt
    pitch = pitch + 25 * dt
    roll  = roll  + 15 * dt
    box:SetRotation(Quaternion(pitch, yaw, roll))

    if Input:IsKeyPressed("Escape") then
        game:Quit()
    end
end

Key differences from C++: in Lua, key names are passed as strings ("Escape") rather than enum values (Key::Escape). Also, variables are module-level locals rather than class members, and you use the colon operator (:) to call methods on objects.

Common mistakes

Common mistakes — Part 01

  • Forgetting override on Start() or Update(). Without override the compiler will not warn you if you misspell a method name (e.g. start() with a lowercase s). The base class version is called silently, your scene never builds, and the window appears black. Always add override.
  • Not multiplying speed by dt. Writing yaw_ += 40.0f instead of yaw_ += 40.0f * dt ties the animation speed to the frame rate. On a 144 Hz monitor the box spins 3.6x faster than on a 40 Hz monitor. Always multiply any per-frame increment by dt.
  • Storing entity names instead of pointers. It is tempting to look up entities by name in Update() using FindEntity("Box"). Do not do this — the lookup is O(n) in the number of entities. Store the Entity* returned by CreateEntity() in a member variable instead.
  • Using IsKeyDown instead of IsKeyPressed for one-shot actions. If you write if (IsKeyDown(Key::Escape)) Quit(), the quit fires every frame while Escape is held. Although the outcome is the same in this case (the game quits), for actions such as toggling a menu or firing a single bullet IsKeyDown would repeat dozens of times per second. Use IsKeyPressed for one-shot events.
  • Missing or incorrect SetResourcePrefixPaths(). If the engine cannot find Data/ and CoreData/, models and shaders fail to load and you see either a black screen or the engine exits immediately with an error message. Double-check that the path you pass ends at the bin/ directory (the parent of Data/), not inside it.
  • Calling delete on an Entity*. The engine owns all entities created through CreateEntity(). Calling delete box_ causes a double-free crash. Use DestroyEntity(box_) if you need to remove an entity from the scene.
  • Placing the camera inside the model. If the box appears fully dark or invisible, the camera may be inside or very close to the box. Check that the camera Z offset is large enough to be in front of (or behind) the model. With the box at origin and scale 1.5, any camera Z more negative than -0.75 is safely outside the box.
  • Treating label pixel positions as 3D world units. label->SetPosition(16, 16) means 16 screen pixels, not 16 world-space metres. These are completely separate coordinate systems. If your label is nowhere to be found, check that the position values are reasonable pixel values for your window size.

Try it yourself

Once the example is running, experiment with these small changes to deepen your understanding:

  1. Change the box colour by calling box_->SetMaterialColor(Color(0.2f, 0.6f, 1.0f)) in Start(). This sets a solid unlit blue colour. Notice that without a light the colour is uniform with no shading. Now add a directional light and observe the difference.
  2. Change the three rotation speeds (currently 40, 25, 15 degrees per second) to different values, including negative numbers. Try making one axis negative to spin in the opposite direction.
  3. In Update(), update the label text each frame to display the current rotation angle: label_->SetText("Yaw: " + std::to_string(static_cast<int>(yaw_) % 360)). This makes the label a live readout.
  4. Move the camera to several different positions (try (3, 5, -8) or (-4, 1, -5)) and observe how perspective changes. Try different FOV values (30, 60, 90, 120) to see the distortion effect.
  5. Replace IsKeyPressed(Key::Escape) with IsKeyDown(Key::Escape) and hold the key down for a moment. Observe that the behaviour is the same for quitting but understand why it would matter for a toggled action.

What you learned

This part established the complete minimal pattern that every simple-3d application follows:

  • Subclassing Game is the entry point for all simple-3d programs. CreateGame<T>()->Run() launches the application.
  • Start() is called once when the engine is ready. Use it for one-time scene setup: creating entities, attaching models, building cameras, and configuring the HUD.
  • Update(float dt) is called every frame. Multiply all per-frame increments by dt to keep behaviour framerate-independent.
  • CreateEntity() allocates a new scene object. The engine owns its lifetime. Store the returned pointer in a member variable for fast access in Update().
  • AddModel(path) loads a 3D model from a resource file and attaches it to the entity. Paths are relative to a Data/ directory under the resource prefix.
  • SetPosition() and SetRotation() place an entity in the scene. Both operate in world space. Use Quaternion(pitch, yaw, roll) to convert Euler angles to the required rotation type.
  • CreateCamera() creates the primary viewport camera. Use LookAt() and SetFOV() to configure its view.
  • CreateLabel(text) creates a screen-space HUD text element. Position is in pixels from the top-left of the window.
  • SetClearColor() sets the viewport background colour. SetWindowTitle() sets the OS title bar text.
  • IsKeyPressed(Key k) is true only on the first frame a key is pressed. IsKeyDown(Key k) is true every frame the key is held. Quit() exits the main loop cleanly.
