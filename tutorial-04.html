Part 04 of 303D Physics

New in this part

AddRigidBody(mass) — AddBoxCollider(Vector3) — AddCapsuleCollider(radius, height) — AddSphereCollider(radius) — CollisionLayer enum (Actor, StaticGeometry, Trigger) — SetCollisionLayer — MakeCollisionMask — ApplyImpulse — SetLinearVelocity — IsOnGround

What you will learn

  • How physics bodies work in simple-3d: the difference between static (mass 0) and dynamic (mass > 0) bodies.
  • How to attach the three standard 3D collision shapes: box, sphere, and capsule.
  • Why the shape and the rigid body are two separate components and must both be present.
  • How the CollisionLayer enum controls which layer an entity lives on and which layers it interacts with via MakeCollisionMask.
  • How to apply instant velocity changes (ApplyImpulse) and drive movement through direct velocity assignment (SetLinearVelocity).
  • How to test whether a dynamic body is resting on the ground with IsOnGround.
  • The equivalent Lua API so you can prototype physics scenes without recompiling.

How physics works in simple-3d

simple-3d wraps Bullet physics (via the Urho3D engine) for all 3D simulation. Every physics-enabled entity needs exactly two things added to it:

  1. A rigid body — describes mass, gravity, and velocity. Created with AddRigidBody(mass).
  2. At least one collision shape — the actual geometry Bullet uses for contact detection. Created with AddBoxCollider, AddSphereCollider, or AddCapsuleCollider.

You can add multiple collision shapes to the same entity; Bullet treats them as a compound shape. The rigid body and each shape must be added to the same entity — shapes on child entities do not automatically register with the parent's rigid body.

Static vs Dynamic

Passing 0.0f to AddRigidBody creates a static body: it generates collision responses for other objects but never moves itself and is not affected by gravity or forces. This is the correct choice for ground planes, walls, platforms, and level geometry. Passing any positive value sets the mass in kilograms and creates a dynamic body that falls, bounces, and reacts to impulses.

Collision layers and masks

Bullet (and therefore simple-3d) filters collisions using a layer/mask pair. An entity only registers a physical contact with another entity when:

  • The source's mask includes the target's layer, and
  • The target's mask includes the source's layer.

simple-3d ships the CollisionLayer enum (in <Simple3D/Core/Collision.h>, automatically pulled in by <Simple3D/Simple3D.h>) with three named layers that cover the vast majority of games:

Name Raw value Use for
CollisionLayer::Actor 1 The player, NPCs, dynamic enemies — anything that moves and collides with the world.
CollisionLayer::StaticGeometry 2 Ground, walls, platforms, crates — static mass-0 surfaces the world is built from.
CollisionLayer::Trigger 4 Trigger volumes. AddTriggerBox/Sphere/Capsule set this automatically.
CollisionLayer::None 0 Collides with nothing. Useful to temporarily disable an entity.
CollisionLayer::All 0xFFFF Collides with every layer. The default mask when you call AddRigidBody without setting anything.

MakeCollisionMask takes an initializer list of CollisionLayer values and ORs their bits together into an unsigned. This lets you express a multi-layer mask without remembering raw bit values: 

// Player should collide with geometry and trigger volumes, but NOT with other actors.
player->SetCollisionLayer(CollisionLayer::Actor);
player->SetCollisionMask(MakeCollisionMask({
    CollisionLayer::StaticGeometry,
    CollisionLayer::Trigger
}));

Choosing the right collision shape

Box

Best for crates, platforms, and rectangular level pieces. The Vector3 argument is the full size (width, height, depth) — not half-extents.

AddBoxCollider(Vector3(2, 1, 2))

Sphere

Best for balls, projectiles, and any roughly round object. Cheapest shape to compute contacts for.

AddSphereCollider(0.5f)

Capsule

The preferred shape for player characters and humanoid NPCs. Handles steps and slopes smoothly without catching on edges.

AddCapsuleCollider(0.35f, 1.8f)

Complete example

The scene below creates a static ground plane, drops three differently-shaped dynamic bodies onto it, and lets the player press Space to shoot a sphere with a sideways impulse. Press Escape to quit. The output demonstrates static bodies, three collider shapes, collision-layer assignment, ApplyImpulse, SetLinearVelocity, and the IsOnGround ground check. 

// PhysicsDemo.cpp — Part 04 of the simple-3d tutorial series
#include <Simple3D/Simple3D.h>
#include <string>

using namespace Simple3D;

class PhysicsDemo : public Game {
public:
    // ── Pointers we want to keep across frames ─────────────────────
    Entity* ground_  = nullptr;
    Entity* sphere_  = nullptr;
    Entity* box_     = nullptr;
    Entity* capsule_ = nullptr;
    Label*  label_   = nullptr;
    int     shotsFired_ = 0;

    void Start() override {
        SetWindowTitle("Part 04 — 3D Physics");
        SetClearColor(Color(0.08f, 0.10f, 0.14f));

        // ── Camera ────────────────────────────────────────────────────
        Camera* cam = CreateCamera();
        cam->SetPosition(Vector3(0, 8, -14));
        cam->LookAt(Vector3(0, 2, 0));

        // ── Directional light ─────────────────────────────────────────
        Entity* sun = CreateEntity("Sun");
        sun->SetRotation(Quaternion(45, 30, 0));
        sun->AddDirectionalLight(Color::WHITE, 1.0f);

        // ── GROUND ────────────────────────────────────────────────────
        // mass = 0.0f → static body: never moves, never falls, blocks dynamics.
        ground_ = CreateEntity("Ground");
        ground_->SetPosition(Vector3(0, 0, 0));
        ground_->SetScale(1.0f);
        ground_->AddModel("Models/Box.mdl");
        ground_->SetMaterialColor(Color(0.25f, 0.55f, 0.25f));

        // Step 1: Add a rigid body — mass 0 = static.
        ground_->AddRigidBody(0.0f);

        // Step 2: Add a box collider — argument is FULL size (width, height, depth).
        // The ground visual is a 1×1×1 unit model scaled to 20×0.5×20 m.
        ground_->SetScale(1.0f);  // reset visual scale before transform
        ground_->AddBoxCollider(Vector3(20.0f, 0.5f, 20.0f));

        // Step 3: Assign to the StaticGeometry layer so dynamic actors know what to stand on.
        ground_->SetCollisionLayer(CollisionLayer::StaticGeometry);

        // Stretch the model to match the collider visually.
        ground_->SetScale(Vector3(20.0f, 0.5f, 20.0f));

        // ── DYNAMIC SPHERE ────────────────────────────────────────────
        // mass = 1.0f → dynamic body, pulled down by gravity each frame.
        sphere_ = CreateEntity("Sphere");
        sphere_->SetPosition(Vector3(-3.0f, 6.0f, 0.0f));
        sphere_->AddModel("Models/Sphere.mdl");
        sphere_->SetMaterialColor(Color(0.85f, 0.25f, 0.25f));
        sphere_->AddRigidBody(1.0f);          // 1 kg dynamic body
        sphere_->AddSphereCollider(0.5f);      // radius 0.5 m
        sphere_->SetCollisionLayer(CollisionLayer::Actor);
        sphere_->SetCollisionMask(MakeCollisionMask({
            CollisionLayer::StaticGeometry
        }));

        // ── DYNAMIC BOX ───────────────────────────────────────────────
        box_ = CreateEntity("Box");
        box_->SetPosition(Vector3(0.0f, 8.0f, 0.0f));
        box_->AddModel("Models/Box.mdl");
        box_->SetMaterialColor(Color(0.25f, 0.50f, 0.85f));
        box_->AddRigidBody(2.0f);               // 2 kg dynamic body
        // Full size of the box: 1 m × 1 m × 1 m (matches the default Model/Box.mdl scale).
        box_->AddBoxCollider(Vector3(1.0f, 1.0f, 1.0f));
        box_->SetCollisionLayer(CollisionLayer::Actor);
        box_->SetCollisionMask(MakeCollisionMask({
            CollisionLayer::StaticGeometry
        }));

        // ── DYNAMIC CAPSULE (character stand-in) ──────────────────────
        capsule_ = CreateEntity("Capsule");
        capsule_->SetPosition(Vector3(3.0f, 7.0f, 0.0f));
        capsule_->AddModel("Models/Cylinder.mdl");
        capsule_->SetMaterialColor(Color(0.85f, 0.70f, 0.20f));
        capsule_->SetScale(Vector3(0.7f, 1.8f, 0.7f));
        capsule_->AddRigidBody(75.0f);             // 75 kg — typical character mass
        // Capsule collider: radius 0.35 m, total height 1.8 m.
        // Great for characters: smooth on slopes, no edge-catching on steps.
        capsule_->AddCapsuleCollider(0.35f, 1.8f);
        capsule_->SetCollisionLayer(CollisionLayer::Actor);
        capsule_->SetCollisionMask(MakeCollisionMask({
            CollisionLayer::StaticGeometry
        }));

        // ── HUD label ─────────────────────────────────────────────────
        label_ = CreateLabel("Press SPACE to fire a sphere  |  ESC = quit");
        label_->SetPosition(12, 12);
        label_->SetFontSize(16);
        label_->SetColor(Color(0.85f, 0.85f, 0.85f));
    }

    void Update(float dt) override {
        if (IsKeyPressed(Key::Escape))
            Quit();

        // ── Fired-sphere keyboard action ──────────────────────────────
        if (IsKeyPressed(Key::Space)) {
            // Create a new sphere slightly in front of the camera.
            shotsFired_++;
            auto name = "Shot" + std::to_string(shotsFired_);
            Entity* shot = CreateEntity(name);
            shot->SetPosition(Vector3(-8.0f, 3.0f, 0.0f));
            shot->AddModel("Models/Sphere.mdl");
            shot->SetMaterialColor(Color(1.0f, 0.5f, 0.0f));
            shot->SetScale(0.4f);
            shot->AddRigidBody(0.5f);
            shot->AddSphereCollider(0.2f);          // radius matches the 0.4× scale
            shot->SetCollisionLayer(CollisionLayer::Actor);
            shot->SetCollisionMask(MakeCollisionMask({
                CollisionLayer::StaticGeometry,
                CollisionLayer::Actor
            }));

            // ApplyImpulse: an instant velocity change (kg·m/s impulse).
            // Positive X moves right; we fire toward the pile of objects.
            shot->ApplyImpulse(Vector3(8.0f, 2.0f, 0.0f));
        }

        // ── Drive the capsule left/right with arrow keys ───────────────
        // SetLinearVelocity overrides the body's current velocity directly.
        float mx = 0.0f;
        if (IsKeyDown(Key::Left))  mx = -4.0f;
        if (IsKeyDown(Key::Right)) mx =  4.0f;

        // Preserve the current Y velocity (gravity) while overriding only X.
        Vector3 vel = capsule_->GetLinearVelocity();
        capsule_->SetLinearVelocity(Vector3(mx, vel.y_, 0.0f));

        // ── IsOnGround demo: tint the capsule when it lands ───────────
        if (capsule_->IsOnGround()) {
            capsule_->SetMaterialColor(Color(0.2f, 0.9f, 0.4f));  // green when grounded
        } else {
            capsule_->SetMaterialColor(Color(0.85f, 0.70f, 0.20f));  // gold in the air
        }

        // Update HUD with live velocity info.
        Vector3 sv = sphere_->GetLinearVelocity();
        std::string info = "Sphere velocity: (" +
            std::to_string((int)sv.x_) + ", " +
            std::to_string((int)sv.y_) + ", " +
            std::to_string((int)sv.z_) + ")";
        label_->SetText(info + "  |  Shots: " + std::to_string(shotsFired_)
                       + "  |  Arrows=move capsule  Space=fire  ESC=quit");
    }
};

int main() {
    return Simple3D::CreateGame<PhysicsDemo>()->Run();
}

Step-by-step explanation

AddRigidBody(mass)

This is the first call you make on any entity that needs to participate in physics. Internally it creates an Urho3D RigidBody component and registers it with Bullet. The single parameter is mass in kilograms.

  • 0.0f — static. The body has infinite effective mass. Other bodies collide with it and bounce off, but the body itself is never moved by forces or gravity. Use this for all level geometry: ground, walls, platforms.
  • Any positive float — dynamic. Gravity is applied each frame. The body accumulates velocity and responds to impulses and forces. The higher the value, the more force is needed to accelerate the body.

One important built-in: AddRigidBody automatically sets the body's angular factor to zero, which prevents characters from tumbling over when they receive off-center impacts. This saves you from having to lock rotation axes manually.

entity->AddRigidBody(float mass);

AddBoxCollider(Vector3 size)

Attaches a rectangular box collision shape to the entity. The Vector3 argument is the full size in metres (width, height, depth), not half-extents. A Vector3(2, 1, 2) produces a 2 m wide, 1 m tall, 2 m deep box.

The shape is centered on the entity's local origin. If you want it offset (for example, a collision box that sits on the ground rather than straddling it), you can place the entity accordingly or use a child entity with a rigid body of its own. Multiple calls to AddBoxCollider on the same entity are allowed; Bullet treats them as a compound shape.

entity->AddBoxCollider(Vector3 fullSize);

AddSphereCollider(radius)

Attaches a sphere collision shape with the given radius in metres. Spheres are the cheapest shape for Bullet to test — contact between two spheres reduces to a single distance check. Use them for balls, cannonballs, projectiles, and any object that is roughly round. Like the other shapes, the sphere is centered on the entity's local origin.

entity->AddSphereCollider(float radius);

AddCapsuleCollider(radius, height)

Attaches a capsule shape: a cylinder with hemispherical end caps. radius is the radius of the cylinder and caps; height is the total height including both caps. A (0.35f, 1.8f) capsule is 0.35 m wide and 1.8 m tall.

Capsules are the recommended shape for humanoid characters. Because they have no sharp edges or corners, they slide up steps naturally and do not snag on small geometry imperfections the way a box would. The CharacterController (Part 07) also uses a capsule internally.

entity->AddCapsuleCollider(float radius, float height);

SetCollisionLayer and the CollisionLayer enum

SetCollisionLayer declares which layer this entity lives on. Other entities use this value when deciding whether to generate a collision contact with it. The CollisionLayer enum provides three named constants that are sufficient for most games: Actor (1), StaticGeometry (2), and Trigger (4). The values are powers of two so they can be combined into bitmasks.

If you need a custom layer (for example, a "projectile" layer that does not interact with other actors), you can pass a raw unsigned value to the overload that accepts one. Keep in mind that the three named layers are conventions: the engine does not enforce any special meaning on them beyond what you configure via masks.

entity->SetCollisionLayer(CollisionLayer layer);  // preferred entity->SetCollisionLayer(unsigned rawBit);      // raw overload for custom layers

MakeCollisionMask

SetCollisionMask accepts a single unsigned bitmask that describes which layers this entity will respond to. MakeCollisionMask is a convenience helper that ORs together an initializer list of CollisionLayer values so you never have to write raw bit arithmetic: 

// These two lines are equivalent:
player->SetCollisionMask(MakeCollisionMask({
    CollisionLayer::StaticGeometry,
    CollisionLayer::Trigger
}));
player->SetCollisionMask(2 | 4);   // less readable raw version

If you omit SetCollisionMask entirely, the default is CollisionLayer::All (0xFFFF), meaning the entity interacts with every layer. This is fine for quick prototyping but can cause unexpected contacts in larger projects where you want actors to ignore one another.

unsigned MakeCollisionMask(std::initializer_list<CollisionLayer> layers);

ApplyImpulse(Vector3)

Applies an instantaneous velocity change to the rigid body. The unit is Newton-seconds (kg·m/s); for a 1 kg body, ApplyImpulse(Vector3(0, 10, 0)) imparts 10 m/s upward. For a 2 kg body the same call only produces 5 m/s, because impulse is divided by mass. Use this for jump launches, explosions, projectile kicks, and any discrete collision response you want to drive manually.

entity->ApplyImpulse(Vector3 impulse);

SetLinearVelocity(Vector3) and GetLinearVelocity()

Unlike ApplyImpulse, which adds to the existing velocity, SetLinearVelocity replaces it completely. This makes it ideal for direct character movement where you want absolute control over speed. A common pattern is to read the current velocity first with GetLinearVelocity(), modify only the axes you care about (typically X and Z for horizontal movement), and write the result back while leaving the Y component (gravity-driven vertical) untouched — exactly as the example above does for the capsule.

entity->SetLinearVelocity(Vector3 velocity); Vector3 v = entity->GetLinearVelocity();

IsOnGround()

Casts a short ray downward from the entity's position and returns true if it hits any body on CollisionLayer::StaticGeometry (layer 2) within 0.7 m. Use it to gate jumps so the player cannot double-jump, or to trigger landing animations. When a CharacterController is attached (see Part 07), the call is delegated to the controller's own slope-filtered grounding test instead of a raw ray.

bool grounded = entity->IsOnGround();

Physics API quick reference

Method Notes
AddRigidBody(0.0f) Static body. Immovable. Generates contacts for dynamic bodies.
AddRigidBody(mass) Dynamic body. Gravity and forces applied. Mass in kg.
AddBoxCollider(Vector3) Full width×height×depth in metres. Not half-extents.
AddSphereCollider(radius) Cheapest shape. Great for balls and projectiles.
AddCapsuleCollider(radius, height) Total height includes end caps. Best for characters.
SetCollisionLayer(CollisionLayer) Which layer this entity is on.
SetCollisionMask(unsigned) Which layers this entity detects. Default: All.
MakeCollisionMask({...}) ORs a list of CollisionLayer values into a mask.
ApplyImpulse(Vector3) Adds an instant velocity change (kg·m/s). Good for jumps and explosions.
ApplyForce(Vector3) Continuous force (Newtons per second). Accumulates over time.
SetLinearVelocity(Vector3) Replaces the body's velocity. Good for direct character movement.
GetLinearVelocity() Returns current velocity as Vector3.
IsOnGround() Downward ray 0.7 m against StaticGeometry layer. Returns bool.

Lua equivalent

The entire physics API is available from Lua scripts (attached via entity:AddScript("...") or run standalone). Method names are identical; CollisionLayer constants are exposed as a global table. 

-- PhysicsDemo.lua
-- Attach to an entity with:  entity:AddScript("Scripts/PhysicsDemo.lua")

local ground, sphere_, box_, capsule_, label_

function Start()
    -- ── Ground ────────────────────────────────────────────────────
    ground = CreateEntity("Ground")
    ground:SetPosition(Vector3(0, 0, 0))
    ground:AddModel("Models/Box.mdl")
    ground:SetMaterialColor(Color(0.25, 0.55, 0.25))
    ground:SetScale(Vector3(20, 0.5, 20))
    ground:AddRigidBody(0.0)   -- static
    ground:AddBoxCollider(Vector3(20, 0.5, 20))
    ground:SetCollisionLayer(CollisionLayer.StaticGeometry)

    -- ── Sphere ────────────────────────────────────────────────────
    sphere_ = CreateEntity("Sphere")
    sphere_:SetPosition(Vector3(-3, 6, 0))
    sphere_:AddModel("Models/Sphere.mdl")
    sphere_:SetMaterialColor(Color(0.85, 0.25, 0.25))
    sphere_:AddRigidBody(1.0)
    sphere_:AddSphereCollider(0.5)
    sphere_:SetCollisionLayer(CollisionLayer.Actor)
    sphere_:SetCollisionMask(MakeCollisionMask({CollisionLayer.StaticGeometry}))

    -- ── Box ───────────────────────────────────────────────────────
    box_ = CreateEntity("Box")
    box_:SetPosition(Vector3(0, 8, 0))
    box_:AddModel("Models/Box.mdl")
    box_:SetMaterialColor(Color(0.25, 0.50, 0.85))
    box_:AddRigidBody(2.0)
    box_:AddBoxCollider(Vector3(1, 1, 1))
    box_:SetCollisionLayer(CollisionLayer.Actor)
    box_:SetCollisionMask(MakeCollisionMask({CollisionLayer.StaticGeometry}))

    -- ── Capsule ───────────────────────────────────────────────────
    capsule_ = CreateEntity("Capsule")
    capsule_:SetPosition(Vector3(3, 7, 0))
    capsule_:AddModel("Models/Cylinder.mdl")
    capsule_:SetMaterialColor(Color(0.85, 0.70, 0.20))
    capsule_:SetScale(Vector3(0.7, 1.8, 0.7))
    capsule_:AddRigidBody(75.0)
    capsule_:AddCapsuleCollider(0.35, 1.8)   -- radius, total height
    capsule_:SetCollisionLayer(CollisionLayer.Actor)
    capsule_:SetCollisionMask(MakeCollisionMask({CollisionLayer.StaticGeometry}))

    -- ── HUD ───────────────────────────────────────────────────────
    label_ = CreateLabel("Space = fire  |  ESC = quit")
    label_:SetPosition(12, 12)
    label_:SetFontSize(16)
end

local shots = 0

function Update(dt)
    if IsKeyPressed(Key.Escape) then Quit() end

    -- Fire a shot
    if IsKeyPressed(Key.Space) then
        shots = shots + 1
        local shot = CreateEntity("Shot" .. shots)
        shot:SetPosition(Vector3(-8, 3, 0))
        shot:AddModel("Models/Sphere.mdl")
        shot:SetMaterialColor(Color(1, 0.5, 0))
        shot:SetScale(0.4)
        shot:AddRigidBody(0.5)
        shot:AddSphereCollider(0.2)
        shot:SetCollisionLayer(CollisionLayer.Actor)
        shot:SetCollisionMask(MakeCollisionMask({
            CollisionLayer.StaticGeometry,
            CollisionLayer.Actor
        }))
        shot:ApplyImpulse(Vector3(8, 2, 0))
    end

    -- Move capsule
    local mx = 0
    if IsKeyDown(Key.Left)  then mx = -4 end
    if IsKeyDown(Key.Right) then mx =  4 end
    local vel = capsule_:GetLinearVelocity()
    capsule_:SetLinearVelocity(Vector3(mx, vel.y, 0))

    -- Ground check tint
    if capsule_:IsOnGround() then
        capsule_:SetMaterialColor(Color(0.2, 0.9, 0.4))
    else
        capsule_:SetMaterialColor(Color(0.85, 0.70, 0.20))
    end
end

Lua note

CollisionLayer.Actor, CollisionLayer.StaticGeometry, and CollisionLayer.Trigger are exposed as fields of the global CollisionLayer table. MakeCollisionMask accepts a Lua table (array) of CollisionLayer values and returns the combined unsigned bitmask. The raw integer overloads of SetCollisionLayer and SetCollisionMask also work if you prefer to pass the values 1, 2, and 4 directly.

Visualizing layer interactions

The table below shows which pairs of layers can generate physics contacts in a typical setup (assuming you configure masks as shown in the examples above).

StaticGeometry (2) Actor (1) Trigger (4)
StaticGeometry (2) No Yes No
Actor (1) Yes No* Yes (callback only)
Trigger (4) No Yes (callback only) No

*Actor-to-Actor contacts are disabled in the example above because each actor's mask only includes StaticGeometry. If you want actors to push each other, add CollisionLayer::Actor to their masks. The fired spheres in the example do include the Actor layer in their mask so they knock the standing shapes over.

Triangle mesh colliders (static only)

For complex level geometry you can use a triangle mesh collider instead of a box: 

level->AddRigidBody(0.0f);   // MUST be static — mesh colliders are not dynamic-safe
level->AddMeshCollider("Models/Level.mdl");

Triangle mesh colliders are only valid on static bodies. Bullet does not support dynamic rigid bodies with concave triangle meshes because the contact-point algorithm would be undefined. If you need a concave dynamic object, decompose it into convex hulls. For level terrain, walls, and floors, AddMeshCollider is the most accurate option.

Common mistakes

Common mistakes

  • Adding a collider without a rigid body (or vice versa). A collision shape alone does nothing — Bullet needs a rigid body to own it. Conversely a rigid body with no shape passes through everything. Always call AddRigidBody before or alongside a shape call.
  • Treating the AddBoxCollider argument as half-extents. simple-3d's AddBoxCollider takes the full size (width, height, depth). If your box is twice as large as expected or objects are floating in mid-air, this is almost certainly the cause. The raw Bullet API uses half-extents; simple-3d doubles them for you.
  • Setting position after AddRigidBody. Once a rigid body exists, Bullet tracks the entity's transform internally. Call SetPosition before AddRigidBody, or use SetLinearVelocity to move a dynamic body after simulation starts. Calling SetPosition on a live dynamic body can cause tunnelling and velocity discontinuities.
  • Using AddMeshCollider on a dynamic body. Triangle mesh colliders require mass 0 (static). Attaching one to a dynamic body will either crash Bullet or produce completely incorrect contacts at runtime.
  • Forgetting to call SetCollisionLayer. If you call IsOnGround() and always get false, check that the ground entity's layer is CollisionLayer::StaticGeometry. The ray cast inside IsOnGround only tests against layer 2.
  • Mixing 3D and 2D physics on the same entity. Calling both AddRigidBody and AddRigidBody2D on the same entity leads to undefined behaviour. Pick one physics world per entity and stick to it.
  • Expecting ApplyImpulse and SetLinearVelocity to behave identically. ApplyImpulse adds to the existing velocity; if the body is already moving you will compound the speed. SetLinearVelocity replaces the velocity; it ignores whatever speed the body had before. Use impulses for discrete events (jumps, hits) and direct velocity for continuous driven movement.

What you learned

By the end of this part you know how to build a fully working 3D physics scene from scratch:

  • Every physics-enabled entity needs both a rigid body (AddRigidBody) and at least one collision shape.
  • AddRigidBody(0.0f) creates a static body; any positive mass creates a dynamic one.
  • AddBoxCollider(Vector3) takes the full size (not half-extents).
  • AddSphereCollider(radius) is the cheapest and easiest shape for round objects.
  • AddCapsuleCollider(radius, height) is the best shape for humanoid characters.
  • The CollisionLayer enum provides Actor, StaticGeometry, and Trigger as named layer constants with raw values 1, 2, and 4.
  • SetCollisionLayer declares which layer an entity is on; SetCollisionMask / MakeCollisionMask controls which layers it detects collisions against.
  • ApplyImpulse adds a one-frame velocity change; SetLinearVelocity replaces the body's velocity entirely.
  • IsOnGround() does a downward ray cast against StaticGeometry to test whether the entity is standing on solid ground.
