Camera modes overview

simple-3d provides three built-in camera personalities that cover most game genres. Each is set by calling a single method on the Camera* object returned by CreateCamera(). You can switch between modes at runtime — for example, when a player presses a button to toggle perspective.

Follow / LookAt

Camera stays at a fixed world offset behind the entity. Call Follow() to set distance and height, then optionally LookAt() to aim. Best for fixed-angle platformers.

Orbit (third-person)

Camera orbits a target at a configurable radius. Right-mouse-button drag rotates freely. This is the most common mode for action and platformer games.

FPS (first-person)

Camera is attached to the entity at head level. Mouse-look is captured. Best for shooters, exploration games, or any perspective requiring immersion.

Complete working example

The example below builds a small 3D platformer world — a ground plane plus four staggered platforms — with a physics-driven player character. The default view is an orbit camera you can rotate with the right mouse button. Press F to toggle to first-person and back. This is a self-contained file; you can drop it into any simple-3d project as src/main.cpp. 

#include <Simple3D/Simple3D.h>
#include <string>
using namespace Simple3D;

// ── Tuning constants ──────────────────────────────────────────────
static constexpr float MOVE_SPEED     = 6.0f;   // m/s horizontal
static constexpr float JUMP_VEL       = 8.5f;   // m/s upward impulse
static constexpr float ORBIT_RADIUS   = 7.0f;   // metres from player
static constexpr float ORBIT_SMOOTH   = 8.0f;   // lerp speed (higher = snappier)
static constexpr float FPS_HEAD_OFFSET = 1.4f;  // metres above entity origin

class PlatformerCamGame : public Game {
public:
    Entity* player_  = nullptr;
    Camera* cam_     = nullptr;
    Label*  hud_     = nullptr;
    bool    fpsMode_ = false;

    void Start() override {
        SetWindowTitle("Part 05 — Third-Person Camera");
        SetClearColor(Color(0.12f, 0.16f, 0.22f));

        // Fog so distant geometry fades gracefully
        SetFogEnabled(true);
        SetFogColor(Color(0.12f, 0.16f, 0.22f));
        SetFogRange(35.0f, 90.0f);

        // ── Lighting ─────────────────────────────────────────────
        auto* sun = CreateEntity("Sun");
        sun->AddDirectionalLight(Color(1.0f, 0.95f, 0.85f), 1.2f);
        sun->SetRotation(Quaternion(50.0f, -30.0f, 0.0f));

        auto* fill = CreateEntity("Fill");
        fill->AddDirectionalLight(Color(0.3f, 0.4f, 0.6f), 0.35f);
        fill->SetRotation(Quaternion(-40.0f, 150.0f, 0.0f));

        // ── Ground plane (static rigid body) ─────────────────────
        auto* ground = CreateEntity("Ground");
        ground->AddModel("Models/Plane.mdl");
        ground->SetPosition(0.0f, 0.0f, 0.0f);
        ground->SetScale(30.0f);
        ground->SetMaterialColor(Color(0.22f, 0.55f, 0.22f));
        ground->AddRigidBody(0.0f);
        ground->AddBoxCollider(Vector3(30.0f, 0.5f, 30.0f));
        ground->SetCollisionLayer(CollisionLayer::StaticGeometry);

        // ── Staggered platforms ───────────────────────────────────
        const Vector3 platPositions[] = {
            Vector3(-5.0f, 1.5f,  6.0f),
            Vector3( 0.0f, 2.8f,  10.0f),
            Vector3( 4.5f, 4.0f,  8.0f),
            Vector3(-2.0f, 5.5f,  14.0f),
        };
        for (int i = 0; i < 4; ++i) {
            std::string name = "Platform" + std::to_string(i);
            auto* p = CreateEntity(name);
            p->AddModel("Models/Box.mdl");
            p->SetPosition(platPositions[i]);
            p->SetScale(Vector3(3.0f, 0.4f, 3.0f));
            p->SetMaterialColor(Color(0.55f, 0.4f, 0.2f));
            p->AddRigidBody(0.0f);
            p->AddBoxCollider(Vector3(3.0f, 0.4f, 3.0f));
            p->SetCollisionLayer(CollisionLayer::StaticGeometry);
        }

        // ── Player character (dynamic rigid body + capsule) ───────
        player_ = CreateEntity("Player");
        player_->AddModel("Models/Box.mdl");
        player_->SetPosition(0.0f, 1.5f, 0.0f);
        player_->SetScale(Vector3(0.7f, 1.5f, 0.7f));
        player_->SetMaterialColor(Color(0.25f, 0.45f, 0.9f));
        player_->AddRigidBody(70.0f);
        player_->AddCapsuleCollider(0.35f, 1.6f);
        player_->SetCollisionLayer(CollisionLayer::Actor);
        player_->SetCollisionMask(MakeCollisionMask({
            CollisionLayer::StaticGeometry
        }));

        // ── Camera — orbit mode (the focus of this tutorial) ──────
        cam_ = CreateCamera();

        // Orbit around the player at 7 m radius, starting at 20° pitch
        cam_->SetOrbitMode(player_, ORBIT_RADIUS);
        cam_->SetOrbitAngles(0.0f, 20.0f);

        // Clamp vertical look: never below -10° (through floor) or above 65°
        cam_->SetOrbitPitchLimits(-10.0f, 65.0f);

        // Mouse drag sensitivity for RMB orbit (degrees moved per pixel dragged)
        cam_->SetOrbitSensitivity(0.3f);

        // Lerp-smooth camera trailing so sudden player turns feel natural
        cam_->SetSmoothFollow(true, ORBIT_SMOOTH);

        // Pull camera in automatically when a wall or platform is in the way
        cam_->SetCollisionEnabled(true);

        // Slightly wider FOV gives a better sense of speed in a platformer
        cam_->SetFOV(70.0f);

        // ── HUD ──────────────────────────────────────────────────
        hud_ = CreateLabel("Mode: Orbit  |  WASD=move  Space=jump  RMB-drag=orbit  F=FPS  ESC=quit");
        hud_->SetPosition(12, 12);
        hud_->SetFontSize(14);
        hud_->SetColor(Color(0.85f, 0.85f, 0.85f));
    }

    void Update(float dt) override {
        if (IsKeyPressed(Key::Escape)) Quit();

        // ── Player movement (velocity-driven, same pattern as Part 04) ──
        Vector3 vel = player_->GetLinearVelocity();
        float mx = 0.0f, mz = 0.0f;
        if (IsKeyDown(Key::A)) mx = -1.0f;
        if (IsKeyDown(Key::D)) mx =  1.0f;
        if (IsKeyDown(Key::W)) mz =  1.0f;
        if (IsKeyDown(Key::S)) mz = -1.0f;
        vel.x_ = mx * MOVE_SPEED;
        vel.z_ = mz * MOVE_SPEED;

        if (IsKeyPressed(Key::Space) && player_->IsOnGround())
            vel.y_ = JUMP_VEL;

        player_->SetLinearVelocity(vel);

        // ── F key: toggle between orbit and first-person ─────────
        if (IsKeyPressed(Key::F)) {
            fpsMode_ = !fpsMode_;

            if (fpsMode_) {
                // Switch to FPS: attach to player at head height, lock cursor
                cam_->SetFPSMode(player_);
                cam_->SetFPSHeadOffset(FPS_HEAD_OFFSET);
                cam_->SetFPSSensitivity(0.18f);
                cam_->SetFOV(90.0f);  // typical FPS FOV is wider
                SetCursorLocked(true);
            } else {
                // Switch back to orbit: release cursor, re-apply orbit settings
                cam_->SetOrbitMode(player_, ORBIT_RADIUS);
                cam_->SetOrbitAngles(0.0f, 20.0f);
                cam_->SetOrbitPitchLimits(-10.0f, 65.0f);
                cam_->SetSmoothFollow(true, ORBIT_SMOOTH);
                cam_->SetCollisionEnabled(true);
                cam_->SetFOV(70.0f);
                SetCursorLocked(false);
            }
        }

        // ── Update HUD ────────────────────────────────────────────
        std::string mode = fpsMode_ ? "FPS (press F to orbit)"
                                      : "Orbit (RMB-drag to rotate)";
        hud_->SetText("Mode: " + mode +
                       "  |  WASD=move  Space=jump  F=toggle  ESC=quit");
    }
};

int main() {
    return CreateGame<PlatformerCamGame>()->Run();
}

Step-by-step explanation

Fixed offset with Follow() and LookAt()

The simplest way to attach a camera to an entity is Follow(). It positions the camera in world space at a fixed distance behind and above the entity each frame, without any rotation controls: 

// Position camera 6 m behind and 2 m above the player every frame
cam_->Follow(player_, 6.0f, 2.0f);   // entity, distance, height

// Aim the camera at the entity (or at a fixed world point)
cam_->LookAt(player_);              // track the entity
cam_->LookAt(Vector3(0.0f, 1.0f, 0.0f));  // aim at a fixed world point

Follow() gives you an isometric-style or fixed-angle action camera — think classic top-down or side-scrolling perspectives. However it provides no mouse-look control. For games where the player controls the camera angle, prefer SetOrbitMode().

LookAt(entity) re-aims the camera toward the entity every frame even if you move the camera manually. LookAt(Vector3) is useful when the camera should always look at the center of an arena rather than the player specifically.

Motion smoothing with SetSmoothFollow()

Without smoothing, the camera teleports to its exact target position every frame. This feels mechanical and makes sudden changes in player direction visually jarring. Calling SetSmoothFollow(true, speed) enables linear interpolation (lerp) between the camera's current position and its target position: 

// Enable smooth following at lerp speed 8.0 (higher = snappier, lower = floatier)
cam_->SetSmoothFollow(true, 8.0f);

// Disable smooth following (camera snaps immediately)
cam_->SetSmoothFollow(false, 0.0f);

The speed value is the lerp coefficient applied per second. A value of 8.0 means the camera closes about 99.97% of the remaining gap in one second. For a relaxed trailing camera use values around 4.0–6.0; for a tight responsive follow use 10.0–15.0. Smooth follow works with both Follow() and SetOrbitMode().

Orbit camera with SetOrbitMode()

Orbit mode is the most popular third-person camera style. The camera circles around a target entity at a configurable radius. The player rotates the view by holding the right mouse button and dragging: 

// Target the player entity; keep the camera 7 m away
cam_->SetOrbitMode(player_, 7.0f);

// Starting angle: 0° yaw (behind player), 20° pitch (slightly above)
cam_->SetOrbitAngles(0.0f, 20.0f);

// Restrict vertical rotation so the player can't look through the floor
// or flip the camera upside-down over the top
cam_->SetOrbitPitchLimits(-10.0f, 65.0f);

// Mouse drag sensitivity: degrees of orbit angle per pixel dragged
cam_->SetOrbitSensitivity(0.3f);

The first argument to SetOrbitMode() is the entity to orbit around; the second is the nominal orbit radius in metres. The actual distance may be shorter when collision avoidance pulls the camera in (see below).

SetOrbitAngles(yaw, pitch) sets the starting orientation of the orbit sphere. Yaw is measured in degrees around the Y axis (0 = behind the entity in world space); pitch is the vertical angle above the horizon. Both are updated automatically as the player drags the mouse, so you only need to set them once in Start().

SetOrbitPitchLimits(min, max) prevents the camera from going below the floor (negative pitch) or flipping upside-down (pitch above 90°). A range of -10° to 65° works well for most third-person games.

Collision avoidance with SetCollisionEnabled()

Without collision avoidance, the orbit camera clips straight through walls and platforms whenever the player stands close to them. Enabling collision avoidance fires a ray from the target entity toward the camera's ideal position. If the ray hits static geometry (layer 2), the camera is moved to the hit point instead: 

// Enable collision avoidance — raycasts on StaticGeometry layer only
cam_->SetCollisionEnabled(true);

// Disable (default) — camera passes through geometry
cam_->SetCollisionEnabled(false);

This single call is often the difference between a camera that feels professional and one that constantly breaks immersion. It only tests against CollisionLayer::StaticGeometry (layer 2) so dynamic objects like crates or enemies do not push the camera around.

Collision avoidance works in both orbit mode and plain Follow() mode. It has no effect in FPS mode because the camera is at head level and cannot pass through geometry the player is also colliding with.

Field of view with SetFOV()

Field of view (FOV) controls how wide the camera's viewing cone is. A narrower FOV makes objects look larger and the scene feel more compressed; a wider FOV gives a stronger sense of speed and space but can cause distortion near the edges: 

// Default is 45°. 60°–75° is typical for third-person games.
cam_->SetFOV(70.0f);

// FPS games conventionally use 90°–100° for a wider sense of immersion.
cam_->SetFOV(90.0f);

// Cinematic, telephoto look (zoom effect):
cam_->SetFOV(30.0f);

You can call SetFOV() at any time, including inside Update(), to animate zoom effects. For example, zooming in when the player aims a weapon: 

// Smooth zoom-in while holding right mouse button in a shooter
float targetFOV = IsMouseButtonDown(MouseButton::Right) ? 35.0f : 70.0f;
float currentFOV = /* store in member variable */;
currentFOV += (targetFOV - currentFOV) * dt * 10.0f;
cam_->SetFOV(currentFOV);

First-person mode with SetFPSMode()

SetFPSMode(entity) attaches the camera to the entity at a configurable head height and enables full mouse-look. The OS cursor is hidden and its motion is captured; GetMouseDelta() still works and is used internally to update the look angles: 

// Attach to the player entity at head level
cam_->SetFPSMode(player_);

// Height above the entity's origin (player capsule is 1.6 m tall; eyes near top)
cam_->SetFPSHeadOffset(1.4f);

// Mouse-look sensitivity in degrees per pixel
cam_->SetFPSSensitivity(0.18f);

// Lock and hide the cursor so the mouse controls the view (not a visible pointer)
SetCursorLocked(true);

// Optional: set initial look direction
cam_->SetFPSAngles(0.0f, 0.0f);  // yaw°, pitch°

SetFPSHeadOffset() is the height in metres above the entity's origin where the camera will be placed. For a player capsule with height 1.6 and origin at the feet, 1.4 puts the camera near the top of the capsule, just below the apex, which looks natural.

When you switch away from FPS mode (for example back to orbit) always call SetCursorLocked(false) to restore the visible cursor, and re-apply the orbit settings with SetOrbitMode(). The camera automatically stops consuming mouse delta once you switch modes.

You can also use SetFPSMode() with no argument for a free-floating FPS camera unattached to any entity — useful for dev fly-cam or spectator mode: 

// Free-floating fly camera (no entity attached)
cam_->SetFPSMode();
SetCursorLocked(true);

Releasing a follow target with Detach()

Detach() stops any follow, orbit, or FPS behaviour and leaves the camera at its current world position. The camera position and orientation can then be set manually via the underlying entity transform. This is useful for cutscenes, death sequences, or menu cameras: 

// Camera stops following; remains at its current world position
cam_->Detach();

// After detaching you can position the camera manually
// (Camera* extends Entity* in simple-3d; transform methods work directly)
cam_->LookAt(Vector3(0.0f, 0.0f, 0.0f));

FPS mode vs. third-person orbit: comparison

Choosing the right camera mode depends on game genre and the kind of spatial awareness you want to give the player. Here is a quick reference:

Feature Third-person orbit First-person FPS
Setup call SetOrbitMode(entity, radius) SetFPSMode(entity)
Camera position Behind and above the character at configurable distance At head level, inside (or coincident with) the character model
Mouse-look activation Hold right mouse button and drag Always active; cursor must be locked
Cursor visibility Cursor remains visible; call SetCursorLocked() only if desired Must call SetCursorLocked(true) to hide and capture cursor
Character visibility Character model is visible from behind Character model is not visible (you are inside it)
Spatial awareness High — player can see obstacles around character Low — limited to what the character is looking at
Typical genres Platformers, action-adventure, RPGs, TPS shooters FPS shooters, horror, exploration, VR-style games
Collision avoidance SetCollisionEnabled(true) prevents clipping Not needed (camera is inside the character)
Recommended FOV 60°–75° 85°–100°
Smoothing SetSmoothFollow(true, 6-10) recommended No additional smoothing needed (camera follows exactly)

Lua equivalent

Camera setup belongs in C++ because the Camera* pointer lives in the Game subclass. However, you can expose camera switches to Lua by caching a reference to the camera in a global and calling its methods from scripts. The pattern below shows how to configure an orbit camera and perform the FPS toggle entirely from Lua: 

-- Scripts/CameraController.lua
-- Attach to an invisible "GameManager" entity in Start().

local cam     = nil
local player  = nil
local fpsMode = false

local ORBIT_RADIUS = 7.0
local ORBIT_SMOOTH = 8.0
local FPS_OFFSET   = 1.4

function Start()
    -- Retrieve entities that were already created in C++ Start()
    player = scene:GetChild("Player", true)
    cam    = GetCamera()   -- simple-3d Lua global: returns the active Camera

    -- Orbit camera setup
    cam:SetOrbitMode(player, ORBIT_RADIUS)
    cam:SetOrbitAngles(0, 20)
    cam:SetOrbitPitchLimits(-10, 65)
    cam:SetOrbitSensitivity(0.3)
    cam:SetSmoothFollow(true, ORBIT_SMOOTH)
    cam:SetCollisionEnabled(true)
    cam:SetFOV(70)
end

function Update(dt)
    if Input:IsKeyPress("F") then
        fpsMode = not fpsMode

        if fpsMode then
            cam:SetFPSMode(player)
            cam:SetFPSHeadOffset(FPS_OFFSET)
            cam:SetFPSSensitivity(0.18)
            cam:SetFOV(90)
            SetCursorLocked(true)
        else
            cam:SetOrbitMode(player, ORBIT_RADIUS)
            cam:SetOrbitAngles(0, 20)
            cam:SetOrbitPitchLimits(-10, 65)
            cam:SetSmoothFollow(true, ORBIT_SMOOTH)
            cam:SetCollisionEnabled(true)
            cam:SetFOV(70)
            SetCursorLocked(false)
        end
    end
end

Lua note — GetCamera()

The global GetCamera() in simple-3d Lua returns the Camera object created by CreateCamera() in C++. If you create the camera in Lua instead, use game:CreateCamera() and store the result in a module-level variable. All Camera methods available in C++ are bound 1:1 in Lua.

Practical tips

Choosing an orbit radius

The orbit radius should scale with the size of your world and the player character's height. A good starting point is 3× the character height. For a 1.6 m tall capsule, try 5–8 m. Too small and the character occludes too much of the scene; too large and the character feels disconnected from the world.

Preventing the camera from going underground

Set the minimum pitch to a small positive value or exactly 0.0° if your world is flat. Negative pitch values (looking upward from below) only make sense if the level has enough vertical room: 

// For a flat game: never allow the camera below the horizon
cam_->SetOrbitPitchLimits(5.0f, 70.0f);

// For a game with open vertical space (cliffs, tall interiors):
cam_->SetOrbitPitchLimits(-15.0f, 80.0f);

Adding a landing impact shake

Camera::Shake() adds a linearly-decaying random XY jitter in camera-local space, which pairs perfectly with a jump-and-land mechanic: 

bool wasGrounded_ = true;   // member variable, initialized to true

// In Update():
bool grounded = player_->IsOnGround();
if (grounded && !wasGrounded_) {
    // Player just landed — shake the camera proportional to fall height
    cam_->Shake(0.25f, 0.35f);   // intensity (world units), duration (seconds)
}
wasGrounded_ = grounded;

Animating the FOV for a speed boost

Briefly widening the FOV gives players a visceral sense of acceleration. Lerp between a normal and boosted FOV in Update(): 

float targetFOV_ = 70.0f;   // member variable
float currentFOV_ = 70.0f;  // member variable

// In Update(), while the player is sprinting:
targetFOV_ = isSprinting ? 85.0f : 70.0f;
currentFOV_ += (targetFOV_ - currentFOV_) * dt * 6.0f;
cam_->SetFOV(currentFOV_);

Common mistakes

  • Forgetting SetCursorLocked(true) in FPS mode. Without it the cursor remains visible and mouse motion moves the OS pointer instead of rotating the view. Always pair SetFPSMode() with SetCursorLocked(true), and pair SetOrbitMode() / Detach() with SetCursorLocked(false).
  • Not calling SetCollisionEnabled(true) in orbit mode. The camera will clip through walls. This is off by default and must be explicitly enabled. Add it right after SetOrbitMode().
  • Calling camera methods before CreateCamera(). CreateCamera() must be called first; all camera configuration methods go after it. Never call camera methods in the constructor — use Start().
  • Mixing SetOrbitMode() and Follow() on the same camera. Each call replaces the previous behaviour. Only one camera personality is active at a time. To switch modes, simply call the new mode's setup method and the previous one is automatically deactivated.
  • Setting pitch limits with min > max. SetOrbitPitchLimits(-30, 70) is correct; SetOrbitPitchLimits(70, -30) silently results in undefined clamping behaviour. Always pass the smaller value first.
  • Expecting SetSmoothFollow() to work in FPS mode. Smooth follow is only meaningful for orbit and Follow() modes. In FPS mode the camera follows the entity exactly at head height — adding a lag would make it feel like the camera is detached from the body.
  • Using a very low smooth follow speed (e.g. 0.5). Extremely low values make the camera trail far behind during rapid movement, and can cause nausea. Values below 3.0 are rarely desirable. Keep the range 5.0–12.0 for gameplay cameras.

What you learned

In this part you learned how to make a camera that feels like it belongs in a real game:

  • Follow(entity, dist, height) attaches the camera at a fixed world offset behind and above an entity. Combine it with LookAt() to keep the gaze on the target.
  • SetSmoothFollow(true, speed) lerps the camera toward its target position each frame, eliminating the mechanical "teleport" feel. A speed of 6–10 covers most use cases.
  • SetOrbitMode(entity, radius) enables a full right-mouse-button orbit camera. Pair it with SetOrbitAngles(), SetOrbitPitchLimits(), and SetOrbitSensitivity() for fine control.
  • SetCollisionEnabled(true) fires a raycast from the target entity toward the camera and pulls it in when static geometry is in the way — no camera clipping through walls.
  • SetFOV(degrees) adjusts the viewing cone. Wider values (90°) suit FPS games; narrower values (60–70°) suit third-person action games. You can animate it in Update() for zoom or speed effects.
  • SetFPSMode(entity) switches to first-person view at configurable head height with captured mouse-look. Pair with SetCursorLocked(true).
  • The difference between the three camera personalities: Follow/LookAt (fixed offset), Orbit (user-rotatable third-person), and FPS (immersive first-person), and how to switch between them at runtime.
