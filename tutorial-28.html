What is a tween?

A tween (from in-between) is a time-limited float interpolator. You give it a start value, an end value, a duration in seconds, and a callback. The engine calls your callback every frame with the current interpolated value until the duration expires. You never write the timer or the lerp math yourself.

In simple-3d, tweens are created with CreateTween() and are owned and ticked by the Game. You do not need to store them to keep them alive, and you do not call anything in Update() to advance them — the engine does that for you. Tweens tick using the scaled frame time, so they automatically slow down or speed up when you call SetTimeScale().

Common uses: health bar fills, score number pop-ups, menu slide-ins, colour flashes, entity fly-up animations, fade-in / fade-out transitions, and any other smooth value change that you want to fire-and-forget.

Easing modes at a glance

Easing controls the rate of change across the tween duration. All four modes cover the same total range in the same total time; they only differ in which part of the motion is fast or slow.

Enum value Formula Feel Best for…
TweenEase::Linear (default) constant rate mechanical, robotic progress bars, timers
TweenEase::EaseIn t² — slow start, fast end gathering momentum objects falling, coin fly-away
TweenEase::EaseOut t(2−t) — fast start, gentle landing natural deceleration UI sliding in, health bar draining
TweenEase::EaseInOut smoothstep — slow at both ends polished, organic camera moves, menu transitions

Complete working example — coin collect fly-up

The example below places five coins in a small arena. Walking into a coin triggers three simultaneous tween effects:

  1. The coin entity flies upward using a position Lerp tween (EaseIn) and is hidden when it arrives.
  2. A “+1” score label pops in by scaling from 0 to 1.2 (EaseOut), then immediately chains into a scale-out tween so it disappears.
  3. Pressing H simulates taking damage: the health bar drains with EaseOut, any previous health tween is cancelled first, and a red damage-flash overlay fades in then chains to a fade-out.
// CoinCollectDemo.cpp
// Part 28 — Tweens
// Build: link against simple-3d. Place model files under Data/Models/.

#include <Simple3D/Simple3D.h>
#include <string>
using namespace Simple3D;

static constexpr int   COIN_COUNT  = 5;
static constexpr float MAX_HEALTH  = 100.0f;
static constexpr float MOVE_SPEED  = 4.0f;

class CoinCollectDemo : public Game {
public:
    void Start() override
    {
        // ── Ground ────────────────────────────────────────────
        Entity* ground = CreateEntity("Ground");
        ground->AddModel("Models/Plane.mdl");
        ground->SetScale(5.0f);
        ground->AddRigidBody(0.0f);
        ground->AddBoxCollider(Vector3(25, 0.2f, 25));
        ground->SetCollisionLayer(CollisionLayer::StaticGeometry);

        // ── Player ────────────────────────────────────────────
        player_ = CreateEntity("Player");
        player_->SetPosition(0, 1, 0);
        player_->AddModel("Models/Capsule.mdl");
        player_->SetMaterialColor(Color(0.3f, 0.5f, 0.9f));

        cc_ = player_->AddCharacterController(0.35f, 1.8f);
        cc_->SetGravity(20.0f);
        player_->SetCollisionLayer(CollisionLayer::Actor);
        player_->SetCollisionMask(MakeCollisionMask({
            CollisionLayer::StaticGeometry,
            CollisionLayer::Trigger
        }));

        // ── Coins ─────────────────────────────────────────────
        Vector3 positions[COIN_COUNT] = {
            Vector3( 3.0f, 0.5f,  0.0f),
            Vector3(-3.0f, 0.5f,  2.0f),
            Vector3( 0.0f, 0.5f,  4.0f),
            Vector3( 4.0f, 0.5f, -3.0f),
            Vector3(-2.0f, 0.5f, -4.0f),
        };

        for (int i = 0; i < COIN_COUNT; ++i) {
            coins_[i] = CreateEntity("Coin" + std::to_string(i));
            coins_[i]->SetPosition(positions[i]);
            coins_[i]->AddModel("Models/Sphere.mdl");
            coins_[i]->SetScale(0.25f);
            coins_[i]->SetMaterialColor(Color(1.0f, 0.85f, 0.0f));

            // Trigger sphere: fired when the player walks close
            coins_[i]->AddTriggerSphere(0.5f);
            coins_[i]->SetCollisionMask(CollisionLayer::Actor);

            // Capture i by value so each lambda refers to its own coin index
            coins_[i]->SetOnTriggerEnter([this, i](Entity* other) {
                if (other == player_ && coins_[i]->IsActive())
                    CollectCoin(i);
            });
        }

        // ── Light ─────────────────────────────────────────────
        Entity* sun = CreateEntity("Sun");
        sun->SetRotation(Quaternion(45, 30, 0));
        sun->AddDirectionalLight(Color::WHITE, 1.2f);

        // ── Camera ────────────────────────────────────────────
        Camera* cam = CreateCamera();
        cam->Follow(player_, 7.0f, 3.0f);
        cam->LookAt(player_);
        cam->SetSmoothFollow(true, 5.0f);

        // ── HUD — health bar ──────────────────────────────────
        healthBar_ = CreateProgressBar();
        healthBar_->SetSize(200, 16);
        healthBar_->SetPosition(10, 10);
        healthBar_->SetColor(Color(0.2f, 0.8f, 0.2f, 1));
        healthBar_->SetBackgroundColor(Color(0.1f, 0.1f, 0.1f, 0.8f));
        healthBar_->SetValue(1.0f);  // full health

        // ── HUD — score text ──────────────────────────────────
        scoreLabel_ = CreateLabel("Coins: 0 / 5");
        scoreLabel_->SetPosition(10, 34);
        scoreLabel_->SetFontSize(18);
        scoreLabel_->SetColor(Color::WHITE);

        // ── HUD — "+1" pop-up (initially hidden, scale=0) ────
        popLabel_ = CreateLabel("+1");
        popLabel_->SetPosition(220, 10);
        popLabel_->SetFontSize(28);
        popLabel_->SetColor(Color(1.0f, 0.9f, 0.1f, 1));
        popLabel_->SetScale(0.0f);
        popLabel_->SetVisible(false);

        // ── HUD — damage flash overlay (full-screen panel) ────
        flashPanel_ = CreatePanel();
        flashPanel_->SetSize(4096, 4096);
        flashPanel_->SetPosition(0, 0);
        flashPanel_->SetColor(Color(0.8f, 0.0f, 0.0f, 1));
        flashPanel_->SetOpacity(0.0f);
        flashPanel_->SetVisible(false);
        flashPanel_->SetZOrder(100);  // above all other UI

        // ── Hint ──────────────────────────────────────────────
        hintLabel_ = CreateLabel("Arrows = move   Space = jump   H = take damage   Esc = quit");
        hintLabel_->SetPosition(10, 58);
        hintLabel_->SetFontSize(14);
        hintLabel_->SetColor(Color(0.65f, 0.65f, 0.65f, 1));

        // ── Input bindings ────────────────────────────────────
        BindAxis2D("Move")
            .Keys(Key::Left, Key::Right, Key::Down, Key::Up)
            .GamepadStick(GamepadAxis::LeftX, GamepadAxis::LeftY)
            .InvertY(true);

        BindAction("Jump").Key(Key::Space).GamepadButton(GamepadButton::A);
        BindAction("Hit").Key(Key::H);
        BindAction("Quit").Key(Key::Escape);
    }

    void Update(float dt) override
    {
        if (IsActionPressed("Quit")) { Quit(); return; }

        Vector2 input = GetAxis2D("Move");
        cc_->Move(Vector3(input.x_, 0, input.y_) * MOVE_SPEED, dt);
        if (IsActionPressed("Jump")) cc_->TryJump(10.0f);

        // Simulate player taking 25 points of damage
        if (IsActionPressed("Hit") && health_ > 0.0f)
            TakeDamage(25.0f);
    }

private:

    // ─────────────────────────────────────────────────────
    // CollectCoin: called once when the player enters a coin trigger.
    // Demonstrates: position Lerp tween, score label pop chain.
    // ─────────────────────────────────────────────────────
    void CollectCoin(int index)
    {
        // Snapshot start and end positions (by value into the lambda)
        const Vector3 startPos = coins_[index]->GetPosition();
        const Vector3 endPos   = startPos + Vector3(0, 5.0f, 0);

        // ── Tween 1: fly the coin upward (EaseIn = slow start, fast end) ──
        // We use a 0→1 range so that t maps directly to Lerp's parameter.
        // Store the pointer so we can cancel it later if the scene unloads.
        coinTweens_[index] = CreateTween(
            0.0f, 1.0f, 0.55f,
            [this, index, startPos, endPos](float t) {
                // Lerp between the two world positions using the current t
                coins_[index]->SetPosition(startPos.Lerp(endPos, t));
            },
            [this, index] {
                // onComplete: hide coin and update score
                coinTweens_[index] = nullptr;  // pointer invalid after this callback
                coins_[index]->SetActive(false);

                score_++;
                scoreLabel_->SetText("Coins: " + std::to_string(score_) +
                                      " / " + std::to_string(COIN_COUNT));
                PlaySound("Sounds/Coin.wav");
            },
            TweenEase::EaseIn);

        // ── Tween 2: score pop-up (EaseOut = fast start, soft landing) ──
        // Scale the "+1" label from 0 → 1.2 so it visually "pops".
        popLabel_->SetVisible(true);
        popLabel_->SetScale(0.0f);

        CreateTween(0.0f, 1.2f, 0.20f,
            [this](float v) { popLabel_->SetScale(v); },
            [this] {
                // Chain: settle back to 1.0 immediately after the overshoot
                CreateTween(1.2f, 1.0f, 0.08f,
                    [this](float v) { popLabel_->SetScale(v); },
                    [this] {
                        // Chain: scale down to 0 over 0.5 s (EaseIn = accelerate out)
                        CreateTween(1.0f, 0.0f, 0.5f,
                            [this](float v) { popLabel_->SetScale(v); },
                            [this] { popLabel_->SetVisible(false); },
                            TweenEase::EaseIn);
                    });
            },
            TweenEase::EaseOut);
    }

    // ─────────────────────────────────────────────────────
    // TakeDamage: drain health bar and flash the screen red.
    // Demonstrates: Cancel(), IsComplete(), EaseOut tween.
    // ─────────────────────────────────────────────────────
    void TakeDamage(float amount)
    {
        health_ -= amount;
        if (health_ < 0.0f) health_ = 0.0f;

        // Cancel any health tween that is still running so we don't
        // have two tweens fighting over the same progress bar.
        if (healthTween_ && !healthTween_->IsComplete()) {
            healthTween_->Cancel();
            healthTween_ = nullptr;  // pointer is invalid at end of frame
        }

        // Read the current bar value so the drain starts from wherever
        // the previous tween stopped, giving a smooth continuation.
        float fromValue = healthBar_->GetValue();
        float toValue   = health_ / MAX_HEALTH;

        // EaseOut: bar drains quickly at first then slows as it settles
        healthTween_ = CreateTween(fromValue, toValue, 0.45f,
            [this](float v) { healthBar_->SetValue(v); },
            [this] {
                healthTween_ = nullptr;  // null out once complete
                if (health_ <= 0.0f)
                    scoreLabel_->SetText("You died!");
            },
            TweenEase::EaseOut);

        // Turn the bar red when health is critically low
        if (health_ < MAX_HEALTH * 0.3f)
            healthBar_->SetColor(Color(0.85f, 0.15f, 0.15f, 1));

        FlashDamageOverlay();
    }

    // ─────────────────────────────────────────────────────
    // FlashDamageOverlay: fade a red panel in, then chain a fade-out.
    // Demonstrates: UI opacity tween, chaining via onComplete.
    // ─────────────────────────────────────────────────────
    void FlashDamageOverlay()
    {
        flashPanel_->SetVisible(true);
        flashPanel_->SetOpacity(0.0f);

        // Fade in very fast (0.08 s) — Linear feels punchy for a hit flash
        CreateTween(0.0f, 0.55f, 0.08f,
            [this](float v) { flashPanel_->SetOpacity(v); },
            [this] {
                // onComplete: immediately chain a slower fade-out
                CreateTween(0.55f, 0.0f, 0.40f,
                    [this](float v) { flashPanel_->SetOpacity(v); },
                    [this] { flashPanel_->SetVisible(false); },
                    TweenEase::EaseOut);
            });
    }

    // Entities
    Entity*              player_             = nullptr;
    CharacterController* cc_                 = nullptr;
    Entity*              coins_[COIN_COUNT]  = {};

    // Game state
    int   score_  = 0;
    float health_ = MAX_HEALTH;

    // UI elements
    UI::ProgressBar* healthBar_  = nullptr;
    Label*           scoreLabel_ = nullptr;
    Label*           popLabel_   = nullptr;
    UI::Panel*       flashPanel_ = nullptr;
    Label*           hintLabel_  = nullptr;

    // Tween handles — store pointers so we can Cancel() if needed.
    // Always null out after Cancel() or in the onComplete callback.
    Tween* coinTweens_[COIN_COUNT] = {};
    Tween* healthTween_             = nullptr;
};

int main()
{
    return Simple3D::CreateGame<CoinCollectDemo>()->Run();
}

Step-by-step explanation

CreateTween(from, to, duration, onUpdate, onComplete, ease)

This is the single entry point for all tweens. The full signature is: 

Tween* CreateTween(
    float from,           // starting value
    float to,             // ending value
    float duration,       // total time in seconds
    std::function<void(float)> onUpdate,    // called every frame with interpolated value
    std::function<void()>    onComplete = {}, // called once when done (optional)
    TweenEase               ease = TweenEase::Linear  // easing curve (optional)
);

Lifetime: the returned Tween* pointer is valid from the moment of creation until the tween completes (or is cancelled). At that point the engine destroys the internal tween object; the raw pointer becomes a dangling pointer. You do not need to delete it. If you need to cancel or check completion, store the pointer immediately and null it out inside onComplete (or right after Cancel()).

Tick source: tweens advance using the same scaled frame time that is passed to Update(dt). If you call SetTimeScale(0.5f), all tweens run at half speed — they take twice as long to complete. This is usually what you want for slow-motion effects, but be aware of it if you are using tweens for UI animations that should be independent of game speed.

The onUpdate callback

The lambda you pass as onUpdate receives the current interpolated value every frame — not a normalized 0–1 parameter, but the actual value somewhere between from and to. So:

  • CreateTween(100.0f, 0.0f, 1.0f, [&](float v){ bar->SetValue(v/100.0f); }) — v counts down from 100 to 0.
  • CreateTween(0.0f, 1.0f, 0.3f, [&](float v){ label->SetScale(v); }) — v climbs from 0 to 1.

When you want a normalized 0–1 parameter to use with Lerp or other math functions, simply pass from=0.0f, to=1.0f and the received value is the t parameter.

Easing modes — TweenEase::Linear / EaseIn / EaseOut / EaseInOut

Pass the desired easing as the sixth argument. The default when you omit it is TweenEase::Linear. Here are examples with the choice of easing motivated: 

// Health bar drains: EaseOut feels natural (fast at first, slows as it settles)
CreateTween(1.0f, 0.2f, 0.45f,
    [this](float v) { healthBar_->SetValue(v); },
    {}, TweenEase::EaseOut);

// Coin fly-up: EaseIn = slow start, object accelerates away (like gravity)
CreateTween(0.0f, 1.0f, 0.55f,
    [=](float t) { coin->SetPosition(start.Lerp(end, t)); },
    {}, TweenEase::EaseIn);

// Score pop-up label: EaseOut = springs into view with a gentle stop
CreateTween(0.0f, 1.0f, 0.2f,
    [this](float v) { popLabel_->SetScale(v); },
    {}, TweenEase::EaseOut);

// Menu panel slides from off-screen: EaseInOut = smooth start and end
CreateTween(-400.0f, 0.0f, 0.3f,
    [this](float x) { menuPanel_->SetPosition((int)x, 100); },
    {}, TweenEase::EaseInOut);

The onComplete callback

The optional fifth argument is called exactly once, after the tween delivers the final to value to onUpdate. It receives no arguments. Common uses: hide a UI element after a fade-out, trigger the next event in a sequence, play a sound when a bar fills up, or destroy an entity that has finished its exit animation.

If you pass {} (an empty std::function) or simply omit the argument, no completion logic runs. Either form is valid: 

// With completion callback
CreateTween(0.0f, 1.0f, 0.3f,
    [this](float v) { overlay_->SetOpacity(v); },
    [this] { StartNextWave(); });

// Without completion callback (omit the argument entirely)
CreateTween(0.0f, 1.0f, 0.3f,
    [this](float v) { overlay_->SetOpacity(v); });

// Without completion callback (explicit empty function)
CreateTween(0.0f, 1.0f, 0.3f,
    [this](float v) { overlay_->SetOpacity(v); }, {});

Tweening entity position with Vector3::Lerp()

CreateTween interpolates a single float, but you can use it to animate any value type by passing from=0.0f, to=1.0f and doing the multi-dimensional interpolation yourself in the callback. Vector3::Lerp(rhs, t) linearly interpolates between the calling vector and rhs at parameter t (0 = this vector, 1 = rhs). 

// Snapshot start and end before the tween runs
const Vector3 start = enemy->GetPosition();
const Vector3 end   = Vector3(10, 0, 5);

// Capture start and end BY VALUE (copy into lambda) — never capture by reference
// if the variables live on the stack and the tween outlives the current scope.
CreateTween(0.0f, 1.0f, 1.5f,
    [enemy, start, end](float t) {
        enemy->SetPosition(start.Lerp(end, t));
    },
    {}, TweenEase::EaseInOut);

The same pattern works for any other type that provides a Lerp-like function, such as Color (built-in RGBA interpolation) or a custom struct where you write the math inline. 

// Tween a point light from dim orange to bright white
const Color warmOrange(1.0f, 0.5f, 0.1f);
const Color brightWhite(1.0f, 1.0f, 1.0f);
CreateTween(0.0f, 1.0f, 2.0f,
    [lamp, warmOrange, brightWhite](float t) {
        Color c = warmOrange.Lerp(brightWhite, t);
        lamp->SetMaterialColor(c);
    });

Tweening UI alpha — Panel::SetOpacity()

UI::Panel exposes SetOpacity(float v) where 0.0 is fully transparent and 1.0 is fully opaque. This is the canonical way to fade any panel-based overlay in or out. UI::Button also has SetOpacity. 

// Fade a HUD panel in from invisible to fully opaque
panel_->SetVisible(true);
panel_->SetOpacity(0.0f);
CreateTween(0.0f, 1.0f, 0.25f,
    [this](float v) { panel_->SetOpacity(v); },
    {}, TweenEase::EaseOut);

// Fade a panel out and hide it when fully transparent
CreateTween(1.0f, 0.0f, 0.3f,
    [this](float v) { panel_->SetOpacity(v); },
    [this] { panel_->SetVisible(false); },
    TweenEase::EaseIn);

For Label, which does not have SetOpacity, drive the alpha through the color instead: label->SetColor(Color(r, g, b, alpha)).

Tweening UI size — Element::SetSize()

All UI elements accept integer dimensions from SetSize(int w, int h). You can animate size by tweening a float and casting to int on each frame. This is useful for expanding bars, progress indicators that grow outward, or panels that slide open like a drawer. 

// Expand a panel from 0 px wide to 250 px wide (EaseOut for a snappy feel)
infoPanel_->SetVisible(true);
CreateTween(0.0f, 250.0f, 0.22f,
    [this](float v) { infoPanel_->SetSize((int)v, 64); },
    {}, TweenEase::EaseOut);

// Collapse the panel closed, then hide it
CreateTween(250.0f, 0.0f, 0.18f,
    [this](float v) { infoPanel_->SetSize((int)v, 64); },
    [this] { infoPanel_->SetVisible(false); },
    TweenEase::EaseIn);

// Grow a score label from small to normal (Label::SetScale is smoother for text)
scoreLabel_->SetScale(0.0f);
scoreLabel_->SetVisible(true);
CreateTween(0.0f, 1.0f, 0.25f,
    [this](float v) { scoreLabel_->SetScale(v); },
    {}, TweenEase::EaseOut);

Chaining tweens in onComplete

Because onComplete is a plain callback, you can call CreateTween() inside it to start a follow-up tween the moment the current one finishes. This is how multi-stage animations are built without timers or coroutines. 

// Three-stage flash: fade in → hold at opacity 1 → fade out
void FlashOverlay()
{
    overlay_->SetVisible(true);
    overlay_->SetOpacity(0.0f);

    // Stage 1: fade in quickly
    CreateTween(0.0f, 1.0f, 0.12f,
        [this](float v) { overlay_->SetOpacity(v); },
        [this] {
            // Stage 2: hold at full opacity for 0.2 s
            // (use a trivial tween from 1→1 as a delay)
            CreateTween(1.0f, 1.0f, 0.2f,
                [this](float v) { overlay_->SetOpacity(v); },
                [this] {
                    // Stage 3: fade out slowly
                    CreateTween(1.0f, 0.0f, 0.4f,
                        [this](float v) { overlay_->SetOpacity(v); },
                        [this] { overlay_->SetVisible(false); },
                        TweenEase::EaseOut);
                });
        });
}

Chains can be as long as needed. Each level of nesting adds one stage with its own easing and duration. If chains grow deep, consider extracting each stage into a named helper function and calling it from the previous stage’s onComplete.

Tween::Cancel()

Calling Cancel() on a live tween stops it immediately. The onUpdate callback stops receiving values, and the onComplete callback is not called — the tween simply disappears. After calling Cancel(), the pointer becomes invalid at the end of the current frame. Always null the stored pointer immediately after cancelling: 

// Guard: only cancel if the pointer is non-null and not yet done
if (healthTween_ && !healthTween_->IsComplete()) {
    healthTween_->Cancel();
    healthTween_ = nullptr;
}

// If IsComplete() is true, the pointer is already invalid — do not cancel again.
// Always null-check before accessing any Tween pointer.

A common pattern is to cancel all tweens associated with a scene object when that object is destroyed or when the scene changes, to prevent dangling callbacks: 

void CancelAllCoinTweens()
{
    for (int i = 0; i < COIN_COUNT; ++i) {
        if (coinTweens_[i] && !coinTweens_[i]->IsComplete()) {
            coinTweens_[i]->Cancel();
        }
        coinTweens_[i] = nullptr;
    }
}

Tween::IsComplete()

Returns true when the tween has either finished its full duration or been cancelled. After IsComplete() returns true, the pointer is no longer valid and must not be used again. The primary use cases are:

  1. Guard before Cancel(): only cancel if the tween is still running. Calling Cancel() on a completed tween would crash (dangling pointer).
  2. Sequential logic without chaining: poll IsComplete() in Update() to know when a tween has finished so you can start the next step of a sequence. Using the onComplete chain is cleaner, but polling is sometimes necessary when the next step depends on other conditions. 
// Polling pattern in Update(dt)
if (openingTween_ && openingTween_->IsComplete()) {
    openingTween_ = nullptr;    // clear the stale handle
    EnablePlayerInput(true);    // safe to allow input now
}

Additional patterns

Health bar fill on level start

Use EaseOut to fill a health bar from 0 to full when a level loads. The rapid initial fill communicates “you are fully healed” clearly, and the soft landing prevents the animation from feeling abrupt. 

// Called from Start() after building the HUD
healthBar_->SetValue(0.0f);
CreateTween(0.0f, 1.0f, 0.6f,
    [this](float v) { healthBar_->SetValue(v); },
    {}, TweenEase::EaseOut);

Menu panel slide-in

Slide a pause menu panel in from the left by tweening its X position. Use EaseOut so the panel decelerates as it arrives in place — snapping to a hard stop feels wrong; easing makes it feel like the panel has weight. 

// Panel starts at x = −300 (off-screen left) and slides to x = 0
pausePanel_->SetPosition(-300, 120);
pausePanel_->SetVisible(true);

CreateTween(-300.0f, 0.0f, 0.25f,
    [this](float x) { pausePanel_->SetPosition((int)x, 120); },
    {}, TweenEase::EaseOut);

// To close: slide back and hide
CreateTween(0.0f, -300.0f, 0.2f,
    [this](float x) { pausePanel_->SetPosition((int)x, 120); },
    [this] { pausePanel_->SetVisible(false); },
    TweenEase::EaseIn);

Score number count-up

Tween a float from the old score to the new score and format it as an integer on each frame. This gives the classic arcade “counting up” effect without any per-frame timer code. 

int oldScore = score_;
score_ += pointsEarned;

CreateTween((float)oldScore, (float)score_, 0.8f,
    [this](float v) {
        scoreLabel_->SetText("Score: " + std::to_string((int)v));
    },
    {}, TweenEase::EaseOut);

Lua equivalent

The Lua API mirrors C++ one-to-one. Easing is specified as a lowercase string instead of an enum: "linear", "easein", "easeout", or "easeinout". 

-- Basic health drain over 1 s (Linear, no completion callback)
CreateTween(100, 0, 1.0, function(v)
    healthLabel:SetText(string.format("HP: %d", math.floor(v)))
end)

-- Score pop-up: scale 0 → 1, then hide (EaseOut + completion callback)
popLabel:SetVisible(true)
popLabel:SetScale(0)
CreateTween(0, 1, 0.25,
    function(v) popLabel:SetScale(v) end,
    function()
        -- Chain: fade out
        CreateTween(1, 0, 0.4,
            function(v) popLabel:SetScale(v) end,
            function() popLabel:SetVisible(false) end,
            "easein")
    end,
    "easeout")

-- Coin fly-up using Vector3 Lerp
local startPos = coin:GetPosition()
local endPos   = Vector3(startPos.x, startPos.y + 5, startPos.z)

local tw = CreateTween(0, 1, 0.55,
    function(t)
        coin:SetPosition(startPos:Lerp(endPos, t))
    end,
    function()
        coin:SetActive(false)
        PlaySound("Sounds/Coin.wav")
    end,
    "easein")

-- Cancel the tween if the scene ends before the coin arrives
if tw and not tw:IsComplete() then
    tw:Cancel()
    tw = nil
end

-- Panel opacity fade-in with completion callback
panel:SetVisible(true)
panel:SetOpacity(0)
CreateTween(0, 1, 0.3,
    function(v) panel:SetOpacity(v) end,
    function() print("panel fully visible") end,
    "easeout")

Lua note: tween string vs enum

In Lua, pass the easing as a lowercase string: "linear", "easein", "easeout", or "easeinout". The C++ TweenEase:: enum is not accessible from Lua scripts. An unrecognised string silently falls back to Linear.

Common mistakes

Common mistakes to avoid

  • Using the Tween* pointer after it completes. The engine destroys the tween object once it finishes or is cancelled. Any pointer you stored before that point is now a dangling pointer. Always null stored handles inside onComplete and right after Cancel().
  • Calling Cancel() without checking IsComplete() first. If the tween has already finished (pointer is stale), calling Cancel() crashes. Guard with if (ptr && !ptr->IsComplete()) ptr->Cancel();.
  • Capturing stack variables by reference in the onUpdate lambda. A tween outlives the current function call. If you capture a local variable by &, the tween will reference deallocated memory after the function returns. Always capture local variables that are needed inside the lambda by value ([=] or explicit [var]).
  • Starting multiple tweens on the same property without cancelling the previous one. Two tweens both writing healthBar_->SetValue() every frame will race against each other, producing jittery or incorrect animation. Cancel the old tween before starting a new one on the same target.
  • Forgetting to null the handle inside onComplete. When onComplete fires, the tween is about to be destroyed. If your stored pointer still points at it and you check IsComplete() on the next frame before nulling it, you are reading freed memory. Set the stored pointer to nullptr as the first line of every onComplete lambda.
  • Calling SetOpacity() on a UI::Image. Image does not have SetOpacity. Use UI::Panel for overlays that need opacity control, or use SetColor(Color(r, g, b, alpha)) to drive the alpha channel of an Image manually.
  • Expecting tween callbacks to ignore SetTimeScale(0.0f). Tweens tick with scaled time. At time scale 0 all tweens are frozen, including UI animations. If you pause with SetTimeScale(0.0f) while a tween is running, the tween stops advancing until you restore the time scale. This is usually the correct behavior, but keep it in mind for opening or closing menus during a pause state.
  • Using very short durations (below one frame). If you pass a duration shorter than a single frame’s delta time, the tween will complete in one frame: onUpdate is called once with the to value and then onComplete fires. This is safe but effectively instant — choose a minimum of 0.05 s if you want the animation to be visible.

Tween::IsComplete() () → bool true once finished or cancelled
TweenEase::Linear — Constant rate (default)
TweenEase::EaseIn t² Slow start, fast end

What you learned

  • CreateTween(from, to, duration, onUpdate, onComplete, ease) creates a one-shot float interpolator owned by the Game. No Update() wiring is needed — the engine ticks it automatically every frame.
  • The onUpdate lambda receives the actual interpolated value between from and to on each frame. Pass from=0, to=1 when you need a normalized 0–1 parameter (e.g. for Vector3::Lerp).
  • The four easing modes — Linear, EaseIn, EaseOut, EaseInOut — alter the rate of change across the duration without changing the total time. Choosing the right easing is what separates a mechanical UI from one that feels responsive and polished.
  • Entity position can be tweened by combining a 0→1 tween with Vector3::Lerp(target, t) inside the onUpdate callback. Always capture start and end positions by value in the lambda.
  • UI alpha is tweened via Panel::SetOpacity(float) (or Button::SetOpacity). UI label size is best tweened via Label::SetScale(float); integer panel size via SetSize((int)v, h).
  • Tweens are chained by calling CreateTween() inside the onComplete callback of the previous tween. Each stage has its own easing and duration; chains can be arbitrarily long.
  • Store a Tween* handle when you need to cancel a tween early. Guard with if (ptr && !ptr->IsComplete()) before calling Cancel(). Null the pointer inside onComplete and right after Cancel() to prevent accidental access to freed memory.
  • Tweens tick with scaled frame time: SetTimeScale(0.5f) halves the speed of all running tweens. At SetTimeScale(0.0f) tweens freeze completely.
  • In Lua, pass the easing mode as a lowercase string: "linear", "easein", "easeout", or "easeinout". The rest of the API is identical to C++.
